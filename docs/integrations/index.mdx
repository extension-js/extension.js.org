Connect Extension.js with formatting, linting, CSS tooling, and UI component workflows used in modern web projects.

## Choose by need

| Need                             | Start here                        |
| -------------------------------- | --------------------------------- |
| Add custom transforms beyond SWC | [Babel](/docs/integrations/babel)              |
| Enforce code-quality rules       | [ESLint](/docs/integrations/eslint)            |
| Keep formatting consistent       | [Prettier](/docs/integrations/prettier)        |
| Transform CSS with plugins       | [PostCSS](/docs/integrations/postcss)          |
| Lint stylesheets in CI           | [Stylelint](/docs/integrations/stylelint)      |
| Build composable UI primitives   | [Shadcn/ui](/docs/integrations/shadcnui)       |
| Utility-first styling system     | [Tailwind CSS](/docs/integrations/tailwindcss) |

## Practical integration order

1. Add baseline quality tools: ESLint + Prettier.
2. Add style tooling: PostCSS + Stylelint (and Tailwind if needed).
3. Add framework-specific tooling only when required.

## Next steps

- Align project defaults in [Extension config](/docs/features/extension-configuration).
- Validate changes with [Workflows](/docs/workflows/index).
