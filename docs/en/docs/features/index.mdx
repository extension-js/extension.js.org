# Extension.js features overview

Understand the core platform capabilities that make Extension.js practical for real-world browser extension development.

## Feature chooser

| Goal                                         | Start here                                                        |
| -------------------------------------------- | ----------------------------------------------------------------- |
| Ship one codebase to multiple browsers       | [Cross-browser compatibility](./cross-browser-compatibility.mdx)  |
| Handle browser-specific manifest differences | [Browser-specific manifest fields](./browser-specific-fields.mdx) |
| Keep non-manifest assets organized           | [Special folders](./special-folders.mdx)                          |
| Manage environment-specific behavior         | [Environment variables](./environment-variables.mdx)              |
| Centralize build/runtime defaults            | [Extension configuration](./extension-configuration.mdx)          |
| Extend bundler behavior safely               | [Rspack configuration](./rspack-configuration.mdx)                |
| Understand update behavior in dev mode       | [Page reload and HMR](./reload-and-hmr.mdx)                       |
| Avoid path drift between source and output   | [Path resolution](./path-resolution.mdx)                          |
| Build and package for target matrices        | [Multi-platform builds](./multi-platform-builds.mdx)              |
| Review the telemetry privacy contract        | [Telemetry and privacy](./telemetry-and-privacy.mdx)              |

## Recommended progression

1. Validate browser output guarantees (`cross-browser-compatibility`).
2. Lock project defaults (`extension-configuration`).
3. Add advanced behavior (`rspack-configuration`, `environment-variables`).
4. Optimize release flows (`multi-platform-builds`).
