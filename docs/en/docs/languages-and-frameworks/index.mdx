# Languages and frameworks guide

Choose the right language and UI framework strategy for your extension surfaces, then use Extension.js defaults to keep setup minimal.

## Choose by goal

| Goal                                      | Start here                                     |
| ----------------------------------------- | ---------------------------------------------- |
| Strong typing and safer refactors         | [TypeScript](./typescript.mdx)                 |
| Rich UI components with large ecosystem   | [React](./react.mdx)                           |
| Smaller React-like runtime                | [Preact](./preact.mdx)                         |
| Vue single-file component workflow        | [Vue.js](./vue.mdx)                            |
| Modern module syntax                      | [ECMAScript modules](./ecmascript-modules.mdx) |
| Scoped styling by default                 | [CSS modules](./css-modules.mdx)               |
| Sass or Less styling workflows            | [Sass](./sass.mdx), [LESS](./less.mdx)         |
| Browser-safe Node compatibility decisions | [Node APIs in browser extensions](./node.mdx)  |
| Performance-heavy wasm workloads          | [WebAssembly](./webassembly.mdx)               |

## Suggested progression

1. Pick language/runtime (`TypeScript`, `React`, `Vue`, or `Preact`).
2. Choose styling model (`CSS modules`, `Sass`, or `LESS`).
3. Add build/runtime constraints (`Node APIs`, `WebAssembly`) only when needed.
