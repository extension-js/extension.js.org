      <Accordion title="什麼是瀏覽器擴充功能？">
        瀏覽器擴充功能是為 Chrome、Firefox、Edge 或其他瀏覽器新增功能的軟體。擴充功能會注入 UI、回應瀏覽器事件，並呼叫網頁本身無法存取的特權 API。完整介紹請見[什麼是瀏覽器擴充功能？](/zh-Hant/docs/concepts/what-is-a-browser-extension)。
      </Accordion>
      <Accordion title="Extension.js 是 Chrome 擴充功能框架嗎？">
        是的。Extension.js 可作為 Chrome 擴充功能框架，同時也是 Firefox 與 Edge 擴充功能框架，全部在同一份專案中運作。使用 `--browser=chrome`、`--browser=firefox` 或 `--browser=edge` 選擇目標。請見[可用的瀏覽器](/zh-Hant/docs/browsers/browsers-available)。
      </Accordion>
      <Accordion title="開發時 Extension.js 如何處理熱模組替換？">
        執行 `npx extension dev`，CLI 會監看你的專案變更。對 popup、content script、service worker 與 options 頁的編輯會觸發最快的安全更新路徑。CLI 在支援時使用熱模組替換（HMR），其他情況下進行針對性的重新載入，並自動處理像是過期 service worker 與遺漏的 content script 更新等常見痛點。
      </Accordion>
      <Accordion title="如何為 Chrome、Edge 與 Firefox 一次建置一份擴充功能？">
        Extension.js 把同一份程式碼編譯成各瀏覽器專屬的輸出。一組指令就能為 Chrome、Edge、Firefox 與其他 Chromium 系瀏覽器產出正確封裝的建置。每個建置都有對應的 manifest 格式、路徑與平台調整。你的持續整合（CI）流程使用相同的建置步驟，只是切換瀏覽器旗標，能減少各家商店送審之間的差異。
      </Accordion>
      <Accordion title="可以使用 React、Preact、Vue、Svelte 或 TypeScript 嗎？">
        可以。Extension.js 內附 React、Preact、Vue、Svelte、TypeScript 與純 JavaScript 的可上線範本。撰寫元件的方式與任何前端專案無異，框架的選擇不會影響核心工作流。
      </Accordion>
      <Accordion title="Extension.js 支援 Manifest V3 嗎？">
        支援。工具鏈圍繞 Manifest V3 與目前瀏覽器商店的限制設計：service worker、宣告式 API、更新後的權限以及更嚴格的封裝。Manifest V3（MV3）是預設，而不是補丁。常見問題請見 [Manifest V3 疑難排解](/zh-Hant/docs/concepts/manifest-v3)。
      </Accordion>
      <Accordion title="如何排查 Manifest V3 的 service worker 與權限問題？">
        大部分 Manifest V3 的疑難排解可分為幾類：service worker 註冊與 `type: "module"`、新的 `web_accessible_resources` 結構、`permissions` 與 `host_permissions` 的拆分，以及 Firefox 的 `declarative_net_request` 差異。修復方式請見 [Manifest V3 疑難排解](/zh-Hant/docs/concepts/manifest-v3)。
      </Accordion>
      <Accordion title="如何從自訂的 webpack 或 Vite 設定遷移？">
        Extension.js 取代了手寫打包工具設定中與擴充功能相關的黏合層，處理 manifest 版本、多瀏覽器輸出、content script 注入與重新載入協調。把你的原始檔與 manifest 移到 Extension.js 專案中，CLI 會處理其餘的部分。你的 React/Vue/Svelte/TS 程式碼無需重寫即可運作。
      </Accordion>
      <Accordion title="Extension.js 支援 monorepo 與環境變數嗎？">
        Extension.js 支援 pnpm、npm 與 Yarn workspaces。路徑解析與共用相依套件可跨 workspace 邊界運作。Extension.js 依瀏覽器與建置模式注入環境變數，並在 JS、JSON 與 HTML 中替換值。開發階段的密鑰絕不會外洩到正式版打包檔。
      </Accordion>
      <Accordion title="Extension.js 可以免費使用嗎？">
        可以。Extension.js 以 [MIT 授權](https://github.com/extension-js/extension.js/blob/main/LICENSE)開源，沒有付費方案、用量上限或強制遙測模式。框架由贊助者與貢獻者支持。實際資料形態請參閱[遙測契約](/zh-Hant/docs/features/telemetry-and-privacy)。
      </Accordion>
      <Accordion title="Extension.js 支援 Safari 嗎？">
        尚未支援。目前 `--browser=safari` 不是 CLI 可用的目標。你仍可使用共用的 Web 程式碼撰寫 Safari 擴充功能，並在此工作流之外加上 Safari 專屬的建置步驟。目前的範疇請見[可用的瀏覽器](/zh-Hant/docs/browsers/browsers-available#safari-and-other-webkit-targets)。
      </Accordion>
      <Accordion title="Extension.js 會在我的打包檔加入哪些東西？">
        正式版建置（`extension build`）只會輸出擴充功能實際引用的程式碼，加上一個很小的重新載入執行階段；在正式版中會被移除。開發版建置會包含額外的 HMR 與重新載入黏合層。輸出位於 `dist/<browser>` 之下，你可以比較不同瀏覽器之間的大小。Content script 的打包大小會接近你的原始碼加上所選 UI 框架的產出。
      </Accordion>
      <Accordion title="如何從 CRXJS、WXT 或 Plasmo 遷移？">
        大多數 React、Vue 與 Svelte 原始檔可以直接複製過來而不必重寫。要動手的有三件事：替換打包工具設定、把產生的 manifest 改回手寫的 `manifest.json` 並使用[帶瀏覽器前綴的鍵](/zh-Hant/docs/features/browser-specific-fields)，以及更新 `package.json` 的腳本。逐步流程請見[從 CRXJS 遷移](/zh-Hant/docs/migrate/from-crxjs)。
