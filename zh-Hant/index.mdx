      <Accordion title="什麼是瀏覽器擴充功能？">
        瀏覽器擴充功能是為 Chrome、Firefox、Edge 或其他瀏覽器新增功能的軟體。擴充功能可注入 UI、回應瀏覽器事件，以及呼叫頁面本身無法使用的特權 API。完整入門請見[什麼是瀏覽器擴充功能？](/docs/concepts/what-is-a-browser-extension)。
      </Accordion>
      <Accordion title="Extension.js 是 Chrome 擴充功能框架嗎？">
        是的。Extension.js 可以作為 Chrome 擴充功能框架使用，也能在同一個專案中作為 Firefox 與 Edge 的擴充功能框架。使用 `--browser=chrome`、`--browser=firefox` 或 `--browser=edge` 來指定目標。請參閱[可用瀏覽器](/docs/browsers/browsers-available)。
      </Accordion>
      <Accordion title="Extension.js 在開發期間如何處理 hot module replacement？">
        執行 `npx extension dev`，CLI 會監看你的專案變更。對彈出視窗、內容腳本、service worker 與設定頁的修改，會觸發最快且安全的更新路徑。CLI 在支援時會使用 hot module replacement（HMR），否則使用針對性的重新載入。它會自動處理常見痛點，例如過期的 service worker 與遺漏的內容腳本更新。
      </Accordion>
      <Accordion title="我要如何為 Chrome、Edge 與 Firefox 建構同一個擴充功能？">
        Extension.js 會將單一程式碼基底編譯為各瀏覽器專用的輸出。同一組命令可為 Chrome、Edge、Firefox 與其他基於 Chromium 的瀏覽器產生正確封裝的建構結果。每個建構都會得到對應的 manifest 格式、路徑與平台調整。你的持續整合（CI）流程使用相同的建構步驟，只需傳入不同的瀏覽器旗標。這能降低各商店送審之間的版本漂移。
      </Accordion>
      <Accordion title="我可以使用 React、Preact、Vue、Svelte 或 TypeScript 嗎？">
        可以。Extension.js 提供 React、Preact、Vue、Svelte、TypeScript 與純 JavaScript 的正式可用範本。你可以像在任何前端專案中一樣撰寫元件。框架的選擇並不影響核心工作流程。
      </Accordion>
      <Accordion title="Extension.js 支援 Manifest V3 嗎？">
        是的。整套工具鏈是以 Manifest V3 以及當今瀏覽器商店所強制執行的限制為核心打造：service worker、宣告式 API、更新後的權限與更嚴格的封裝。Manifest V3（MV3）是預設目標，而非事後追加。常見陷阱請參閱 [Manifest V3 疑難排解](/docs/concepts/manifest-v3)。
      </Accordion>
      <Accordion title="我要如何排解 Manifest V3 的 service worker 與權限問題？">
        大多數 Manifest V3 疑難排解都落在幾個類別：service worker 的註冊與 `type: "module"`、新的 `web_accessible_resources` 結構、`permissions` 與 `host_permissions` 的拆分，以及 Firefox 中 `declarative_net_request` 的差異。修正方式請參閱 [Manifest V3 疑難排解](/docs/concepts/manifest-v3)。
      </Accordion>
      <Accordion title="我要如何從自訂的 webpack 或 Vite 設定遷移？">
        Extension.js 取代了手寫打包工具設定中與擴充功能相關的黏合層程式碼。它會處理 manifest 版本、多瀏覽器輸出、內容腳本注入與重新載入協調。把原始碼檔案與 manifest 移到 Extension.js 專案內，剩下的就交給 CLI。你的 React/Vue/Svelte/TS 程式碼無須改寫即可運作。
      </Accordion>
      <Accordion title="Extension.js 支援 monorepo 與環境變數嗎？">
        Extension.js 支援 pnpm、npm 與 Yarn 的 workspaces。路徑解析與共用相依套件在 workspace 之間都能正常運作。Extension.js 會依瀏覽器與建構模式注入環境變數，並跨 JS、JSON 與 HTML 替換值。開發用的機密資訊絕不會外洩到正式環境的打包檔。
      </Accordion>
      <Accordion title="Extension.js 是免費的嗎？">
        是的。Extension.js 採用
        [MIT 授權](https://github.com/extension-js/extension.js/blob/main/LICENSE)的開源軟體。沒有付費方案、使用量上限，也沒有強制遙測模式。本框架由贊助者與貢獻者資助。實際資料格式請參閱[遙測契約](/docs/features/telemetry-and-privacy)。
      </Accordion>
      <Accordion title="Extension.js 支援 Safari 嗎？">
        目前尚未支援。`--browser=safari` 目前還不是可用的 CLI 目標。你仍可使用共享的網頁程式碼來撰寫 Safari 擴充功能，並在此工作流程之外加入 Safari 專屬的建構步驟。目前的支援範圍請參閱[可用瀏覽器](/docs/browsers/browsers-available#safari-and-other-webkit-targets)。
      </Accordion>
      <Accordion title="Extension.js 會在我的打包檔中加入什麼？">
        正式環境建構（`extension build`）只會輸出你的擴充功能實際引用的程式碼，加上一個小型的重新載入執行階段（會在正式建構中移除）。開發建構則包含額外的 HMR 與重新載入黏合程式。輸出位於 `dist/<browser>` 之下，方便你比較不同瀏覽器間的體積。內容腳本的打包大小，會接近於你的原始碼加上所選 UI 框架的產出。
      </Accordion>
      <Accordion title="我要如何從 CRXJS、WXT 或 Plasmo 遷移？">
