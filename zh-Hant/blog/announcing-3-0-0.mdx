Extension.js 3.0.0 現已發布。

本次發布聚焦於一件事：協助團隊以更少的設定、更清晰的工作流程與更好的預設值，在 Chrome、Edge 與 Firefox 上交付生產級的瀏覽器擴充功能。

## v3 帶來了什麼

**版本 3.0.0**

- 透過統一的命令介面，強化了在 Chrome、Edge 與 Firefox 上的跨瀏覽器工作流程。
- 擴充的範本體驗，涵蓋現代技術堆疊，包括 TypeScript、React、Vue、Svelte 與 Preact。
- 更完善的開發期更新行為，讓熱模組替換（HMR）、硬重新載入，以及需要重新啟動的邊界更加清晰。
- 透過疑難排解、安全檢查、效能檢查、持續整合（CI）範本，以及 Playwright 端對端（E2E）測試，提供更可靠的生產信心工作流程。
- 透過 `extension.config.js` 以及具瀏覽器意識的建置輸出，讓專案設定更可預測。
- 更清爽的文件與首頁體驗，專為更快的上手與更低的認知負擔而設計。

## 第一天的上手流程更簡單

從建立專案到產出生產產物，使用同一條路徑：

1. `create` 用於建立鷹架。
2. `dev` 用於快速迭代。
3. `build` 用於產生生產產物。
4. `start` 或 `preview` 用於驗證生產行為。

```bash
npx extension@latest create my-extension --template=new-typescript
cd my-extension
pnpm dev
pnpm build
```

## 為團隊設計，而不僅是示範

v3 專為維護真實擴充功能程式碼庫的真實團隊而設計：

- 型別化範本與以框架為先的基線，減少了設定漂移。
- 依瀏覽器目標的建置與旗標，讓相容性檢查更加明確。
- 工作流程文件現在直接對應到發布信心相關的任務。
- 標準交付流程現在包含安全與效能指引。

## 這次發布適合誰

- 你正在從自訂的擴充功能建置腳本遷移過來。
- 你正從應用程式開發轉向擴充功能開發。
- 你正在跨多個儲存庫與貢獻者之間，標準化擴充功能開發體驗。

## 升級說明

如果你已經在使用 Extension.js：

- 將套件升級到最新版本，
- 優先使用標準的範本別名（`new-react`、`new-typescript`、`new-vue` 等），
- 保持指令流程與 `create` → `dev` → `build` 一致。
