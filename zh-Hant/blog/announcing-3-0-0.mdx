<Update label="v3.0.0" description="February 18, 2026" tags={["release", "major"]}>

Extension.js 3.0.0 現已推出。

這個版本聚焦於一件事：幫助團隊以更少的設定、更清晰的工作流程，以及在 Chrome、Edge 與 Firefox 上更佳的預設值，發佈正式環境的瀏覽器擴充功能。

## v3 帶來什麼

**3.0.0 版本**

- 在 Chrome、Edge 與 Firefox 上提供更強大的跨瀏覽器工作流程，並使用同一套命令介面。
- 為現代技術棧擴充範本體驗，包括 TypeScript、React、Vue、Svelte 與 Preact。
- 改善開發更新行為，hot module replacement（HMR）、hard-reload 與需重新啟動的界線更為清晰。
- 在疑難排解、安全檢查、效能檢查、持續整合（CI）範本，以及 Playwright 端到端（E2E）測試方面，提供更佳的正式環境信心工作流程。
- 透過 `extension.config.js` 與瀏覽器感知的建構輸出，提供更可預測的專案設定。
- 更清爽的文件與首頁體驗，為更快的上手與更低的認知負擔而設計。

## 第一天的流程更簡單

從建立專案到正式輸出，使用一條路徑：

1. `create` 來建立專案。
2. `dev` 用於快速迭代。
3. `build` 用於產生正式環境產物。
4. `start` 或 `preview` 用於驗證正式環境行為。

```bash
npx extension@latest create my-extension --template=new-typescript
cd my-extension
pnpm dev
pnpm build
```

## 更適合團隊，不只是 demo

v3 是為維護真實擴充功能程式碼基底的真實團隊所設計：

- 帶型別的範本與以框架優先的基線可減少設定差異。
- 以瀏覽器為目標的建構與旗標，讓相容性檢查變得明確。
- 工作流程文件現在能直接對應到發行信心相關的任務。
- 標準的交付流程現在包含安全與效能指引。

## 這個版本適合誰

- 你正從自訂的擴充功能建構腳本遷移過來。
- 你正從應用程式開發轉向擴充功能開發。
- 你正在跨多個程式碼倉庫與貢獻者間標準化擴充功能的開發者體驗。

## 升級注意事項

如果你已經在使用 Extension.js：

- 將套件更新到最新版本，
- 優先使用標準的範本識別碼（`new-react`、`new-typescript`、`new-vue` 等），
- 讓你的命令流程維持在 `create` → `dev` → `build`。
