<Update label="v3.0.0" description="February 18, 2026" tags={["release", "major"]}>

Extension.js 3.0.0 正式推出。

這個版本只專注一件事：讓團隊以更少的設定、更清楚的工作流，以及在 Chrome、Edge 與 Firefox 都更好的預設值,發佈正式版瀏覽器擴充功能。

## v3 帶來什麼

**Version 3.0.0**

- 強化跨瀏覽器工作流，在 Chrome、Edge 與 Firefox 之間使用同一套指令介面。
- 擴大範本體驗,涵蓋 TypeScript、React、Vue、Svelte 與 Preact 等現代技術堆疊。
- 改善開發更新行為，提供更清楚的熱模組替換（HMR）、hard-reload 與需要重啟的邊界。
- 在疑難排解、安全檢查、效能檢查、持續整合（CI）範本，以及 Playwright 端對端（E2E）測試上,提供更好的正式版信心工作流。
- 透過 `extension.config.js` 與瀏覽器感知的建置輸出，讓專案設定更可預期。
- 重新設計、更乾淨的文件與首頁體驗，讓上手更快、認知負擔更低。

## 第一天的流程更簡單了

從建立專案到產出正式版產物，只需要一條路徑：

1. `create` 建立鷹架。
2. `dev` 快速迭代。
3. `build` 產生正式版產物。
4. `start` 或 `preview` 驗證正式版行為。

```bash
npx extension@latest create my-extension --template=new-typescript
cd my-extension
pnpm dev
pnpm build
```

## 為團隊設計，不只是 demo

v3 是為了維護真實擴充功能程式碼庫的真實團隊而設計：

- 帶型別的範本與框架優先的基線，降低設定的漂移。
- 鎖定瀏覽器目標的建置與旗標，讓相容性檢查更明確。
- 工作流文件現在直接對應到發佈信心相關的任務。
- 標準交付流程現在包含安全性與效能的指引。

## 這個版本適合誰

- 你正從自訂的擴充功能建置腳本遷移過來。
- 你正從應用程式開發轉向擴充功能開發。
- 你正在多個程式碼倉庫與多位貢獻者之間，標準化擴充功能的開發者體驗。

## 升級提示

如果你已經在使用 Extension.js：

- 將套件更新到最新版本，
- 優先使用規範的範本代稱（`new-react`、`new-typescript`、`new-vue` 等），
- 讓指令流程維持 `create` → `dev` → `build` 的順序。
