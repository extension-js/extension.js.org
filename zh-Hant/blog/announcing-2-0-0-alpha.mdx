<Update label="v2.0.0-alpha" description="September 9, 2024" tags={["release", "alpha"]}>

![Extension.js 2.0.0-alpha banner](/images/blog/extension-2-0-0-alpha.png)

Extension.js 的目標是讓跨瀏覽器擴充功能開發盡可能容易。

自從在 [HackerNews 上獲得關注](https://hnrankings.info/40210627/) 後，我們做了許多內部改進，讓套件更小、更快，也對開發者更友善。

對我來說，「非常容易」意味著非常低的認知負擔。它意味著事情是熟悉的。開發者熟悉複製貼上。身為一位開發者，如果我能複製貼上程式碼，並立刻得到視覺上的回應、了解它的意義，而且無須任何設定，我會覺得這非常容易。即使你一開始不理解這段程式碼，快速的視覺回應就是「事情應該如何運作」的明確指標。這正是 Extension.js v2 想要達到的期望：非常容易上手。

**2.0.0-alpha 版本**

- 為熱門 JavaScript 框架、CSS 預處理器以及擴充功能 API 提供 30 多個範本。[在 GitHub 上瀏覽](https://github.com/extension-js/examples/tree/main/examples)。
- 支援在 manifest.json 中加入僅套用於目標瀏覽器的瀏覽器專屬欄位，例如 `{chrome:service_worker: "sw.js"}`
- `extension.config.js`——適用於需要存取編譯流程的進階使用情境。[參閱設定文件](/docs/features/extension-configuration)。
- 由 [SWC（Speedy Web Compiler）](https://swc.rs/) 帶來的更快編譯。
- 重新打造的文件網站（就是你看到的這個！）。
- 新的範本網站（即將推出）。

alpha 版本聚焦於錯誤修正，並在我們完成文件與範本網站的同時，測試所有既有功能。後續更新請持續關注本部落格，或在 X 上追蹤我們。

下一個版本，我們將聚焦於把所有既有功能做到足以撰寫文件並進行測試的穩定程度。

感謝你閱讀。再聊。
