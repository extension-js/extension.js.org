![Extension.js 2.0.0-alpha 橫幅](/images/blog/extension-2-0-0-alpha.png)

Extension.js 的目標，是讓跨瀏覽器擴充功能開發盡可能簡單。

自從在 [HackerNews 上獲得關注](https://hnrankings.info/40210627/)以來，我們做了大量內部改進，讓這個套件更小、更快，對開發者更友善。

對我而言，「非常簡單」意味著非常低的認知負擔，意味著熟悉感。開發者熟悉複製／貼上。作為一名開發者，如果我能複製／貼上程式碼，並在沒有任何設定的情況下立即得到視覺化回饋，體驗就會非常輕鬆。即使你一開始不理解程式碼，快速的視覺化回饋也清楚地表明事情**應該**如何運作。這就是 Extension.js v2 的期望：上手非常簡單。

**版本 2.0.0-alpha**

- 30 多個面向熱門 JavaScript 框架、CSS 預處理器及擴充功能 API 的範本。[在 GitHub 上查看](https://github.com/extension-js/examples/tree/main/examples)。
- 支援在 manifest.json 中加入僅對目標瀏覽器生效的依瀏覽器區分欄位，例如 `{chrome:service_worker: "sw.js"}`
- `extension.config.js` —— 用於需要存取編譯流程的進階情境。[查看設定文件](/docs/features/extension-configuration)。
- 得益於 [SWC (Speedy Web Compiler)](https://swc.rs/)，編譯速度更快。
- 全新改版的文件網站（就是這個！）。
- 全新的範本網站（即將推出）。

alpha 版本在我們完成文件與範本網站的同時，專注於修復 bug 並測試所有現有功能。請持續關注本部落格，或在 X 上追蹤我們以取得後續更新。

下一個版本，我們將專注於讓所有現有功能穩定到足以撰寫文件並進行測試。

感謝閱讀，回頭見。
