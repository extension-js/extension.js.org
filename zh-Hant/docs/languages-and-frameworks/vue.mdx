Extension.js 會從你的相依套件偵測 Vue，並自動設定單一檔案元件（SFC）編譯、框架別名與開發階段的更新行為。Scoped styles、Composition API 與 `<script setup>` 都無需額外 bundler 設定即可運作。

## 何時適合使用 Vue

- 你已經出貨 Vue 正式環境應用。
- 你偏好以 Vue 單一檔案元件搭配 scoped styles 與 Composition API 撰寫程式。
- 你希望擴充功能 UI 介面能反映既有的 Vue 架構。

## 在既有擴充功能中使用

依下列步驟，將 Vue 加入既有擴充功能。

### 安裝

安裝必要相依套件：

<PackageManagerTabs command="install vue" />

若要在既有專案中明確設定，也請安裝 Vue SFC 工具鏈：

<PackageManagerTabs command="install -D vue-loader @vue/compiler-sfc" />

### 設定

Extension.js 預期 Vue 元件位於 `.vue` 檔案中。它會在 Rspack 流水線中設定 `vue-loader`、`VueLoaderPlugin` 與 Vue 相關的 define 旗標。

## 開發行為

當 Extension.js 偵測到 Vue 時，它會：

- 在框架外掛中啟用 `.vue` 編譯。
- 套用 Vue 執行階段別名（以維持一致的執行階段解析）。
- 在相關 Vue SFC 變更後重新掛載，支援內容腳本更新。

若你的專案缺乏選用 Vue 工具，Extension.js 會安裝並要求你重新啟動。

### 疑難排解

- **缺少 Vue 工具警告：** 安裝 `vue-loader` 與 `@vue/compiler-sfc`。
- **安裝後提示重新啟動：** 停止並重新執行 `extension dev`，讓 Extension.js 能載入新裝的 loader／plugin。
- **`.vue` 處理異常：** 確認 `vue` 出現在專案相依套件中，讓 Extension.js 能偵測 Vue 整合。

## 使用範例

### 在新分頁擴充功能中

要在新分頁擴充功能中使用 Vue，請在 HTML 中引入進入檔案：

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>New Extension</title>
  </head>
  <body>
    <noscript>You need to enable JavaScript to run this extension.</noscript>
    <div id="app"></div>
  </body>
  <script src="./main.ts"></script>
</html>
```

```ts
import { createApp } from "vue";
import App from "./App.vue";

createApp(App).mount("#app");
```

```vue
<!-- App.vue -->
<template>
  <h1>Hello, Vue.js Extension!</h1>
</template>

<script setup lang="ts">
// Component logic goes here.
</script>

<style scoped>
h1 {
  color: #42b983;
}
</style>
```

### 在 `content_script` 檔案中

對於內容腳本，將 Vue 應用掛載到注入的根節點：

```ts
import { createApp } from "vue";
import App from "./App.vue";
import "./content.css";

const rootDiv = document.createElement("div");
rootDiv.id = "extension-root";
document.body.appendChild(rootDiv);

createApp(App).mount("#extension-root");
```

## 最佳實踐

- 讓 UI 進入點以框架為主（`main.ts`、`App.vue`），將擴充功能 API 放在專用模組中。
- 在 SFC 中盡可能使用 scoped styles，以減少擴充功能頁面的樣式外洩。
- 對於大型 UI，分割元件與共用 composables，使內容腳本維持精簡。
