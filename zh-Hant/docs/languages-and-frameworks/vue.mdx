Extension.js 會從你的相依套件中偵測到 Vue,並自動設定單一檔案元件(SFC)編譯、框架別名以及開發時的更新行為。Scoped 樣式、Composition API 與 `<script setup>` 皆可使用,不需額外打包器設定。

## 何時適合使用 Vue

- 你已在正式環境中發布 Vue 應用程式。
- 你偏好使用 scoped 樣式與 Composition API 的 Vue 單一檔案元件寫法。
- 你希望擴充功能 UI 介面與既有的 Vue 架構保持一致。

## 在現有擴充功能中使用

依照以下步驟將 Vue 加入既有的擴充功能。

### 安裝

安裝所需相依套件:

<PackageManagerTabs command="install vue" />

如需在既有專案中明確設定,請一併安裝 Vue SFC 工具鏈:

<PackageManagerTabs command="install -D vue-loader @vue/compiler-sfc" />

### 設定

Extension.js 期望 Vue 元件位於 `.vue` 檔案中。它會在 Rspack 流程中設定 `vue-loader`、`VueLoaderPlugin` 以及 Vue 相關的 define 旗標。

## 開發行為

當 Extension.js 偵測到 Vue 時,它會:

- 在框架外掛中啟用 `.vue` 編譯。
- 套用 Vue 執行時別名(以維持執行時解析的一致性)。
- 在相關 Vue SFC 變更後重新掛載,以支援內容腳本更新。

若你的專案缺少選用的 Vue 工具,Extension.js 會自動安裝並要求重新啟動。

### 疑難排解

- **缺少 Vue 工具警告:** 安裝 `vue-loader` 與 `@vue/compiler-sfc`。
- **安裝後提示重新啟動:** 停止並重新執行 `extension dev`,以便 Extension.js 載入新安裝的 loader/外掛。
- **意外的 `.vue` 處理問題:** 確認 `vue` 已出現在專案相依套件中,讓 Extension.js 能偵測到 Vue 整合。

## 使用範例

### 在新分頁擴充功能中

若要在新分頁擴充功能中使用 Vue,在 HTML 中引入進入點檔案:

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>New Extension</title>
  </head>
  <body>
    <noscript>You need to enable JavaScript to run this extension.</noscript>
    <div id="app"></div>
  </body>
  <script src="./main.ts"></script>
</html>
```

```ts
import { createApp } from "vue";
import App from "./App.vue";

createApp(App).mount("#app");
```

```vue
<!-- App.vue -->
<template>
  <h1>Hello, Vue.js Extension!</h1>
</template>

<script setup lang="ts">
// Component logic goes here.
</script>

<style scoped>
h1 {
  color: #42b983;
}
</style>
```

### 在 `content_script` 檔案中

針對內容腳本,將 Vue 應用掛載到注入的根節點:

```ts
import { createApp } from "vue";
import App from "./App.vue";
import "./content.css";

const rootDiv = document.createElement("div");
rootDiv.id = "extension-root";
document.body.appendChild(rootDiv);

createApp(App).mount("#extension-root");
```

## 最佳實踐

- 讓 UI 進入點保持以框架為主(`main.ts`、`App.vue`),並將擴充功能 API 放在專門的模組中。
- 在 SFC 中盡量使用 scoped 樣式,減少擴充功能頁面中的樣式外洩。
- 對於大型 UI,拆分元件並抽出共用 composable,以維持內容腳本的輕量。
