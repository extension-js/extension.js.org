使用 Sass,以變數、巢狀語法與 SCSS modules 擴展瀏覽器擴充功能的樣式,同時維持單一的建置流程。

Extension.js 透過基於 Rspack 的樣式管線,支援 `.scss`、`.sass`,以及 SCSS module 變體(`.module.scss`、`.module.sass`)。

## 何時適合使用 Sass

- 你的設計系統已仰賴 Sass 變數與 mixin。
- 你需要適合元件導向擴充功能 UI 的模組化樣式。
- 你希望與 Web 應用程式中既有的 Sass 工作流程保持一致。

## Sass modules

與 CSS modules 一樣,使用 Sass modules 可將樣式範圍區域化。使用 `.module.scss` 或 `.module.sass` 啟用範圍隔離,避免樣式表中的命名衝突。

## 在既有擴充功能中使用 Sass modules

要啟用 Sass modules,將你的 Sass 檔重新命名,加上 `.module.scss` 或 `.module.sass`。這會自動將樣式範圍限定在元件中。

```diff
- button.scss
+ button.module.scss
```

### 使用範例

重新命名 Sass 檔後,於你的腳本中匯入它:

```ts
import styles from "./styles/button.module.scss";

const element = document.createElement("h1");
element.className = styles.primaryColor;
element.innerText = "Hello, Extension!";
document.body.appendChild(element);
```

對應的 Sass 檔:

```scss
/* styles/button.module.scss */

$primary-color: #64ff00;

:export {
  primaryColor: $primary-color;
}
```

## React/Preact 使用範例

將 Sass module 匯入到你的 React 或 Preact 元件,並使用範圍隔離的類別名稱:

```jsx
import styles from "./styles/button.module.scss";

export default function MyComponent() {
  return <h1 className={styles.primaryColor}>Hello, Extension!</h1>;
}
```

## Vue 使用範例

在 Vue SFC 中,優先使用 `lang="scss"` 的 module 樣式:

```vue
<template>
  <h1 :class="$style.primaryColor">Hello, Extension!</h1>
</template>

<style lang="scss" module>
@import "./styles/button.module.scss";
</style>
```

## Svelte 使用範例

在 Svelte 中,匯入 Sass module 對應並套用類別:

```svelte
<script>
  import styles from "./styles/button.module.scss";
</script>

<h1 class={styles.primaryColor}>Hello, Extension!</h1>
```

## 行為注意事項

- 在擴充功能頁面中,`.module.scss` 與 `.module.sass` 會匯出 JS/TS 可匯入的類別對應 map。
- 在 content script 中,Extension.js 會將 Sass 輸出為 CSS 資源,而不附帶 JavaScript 的類別名稱對應。
- 若你的專案缺少 Sass 工具,Extension.js 會安裝選用的相依套件並要求重新啟動。
