Extension.js 預設以瀏覽器執行階段為目標。它不會自動為 Node 核心模組加上 polyfill，這可讓套件保持精簡。如果相依套件需要 `buffer`、`stream` 或 `crypto`，你會在建置時看到解析錯誤。以下章節說明何時應加入 polyfill，何時應改用瀏覽器原生替代方案。

## 何時適合使用 Node polyfill

- 必要的相依套件在瀏覽器執行階段需要 Node 核心模組。
- 你正逐步遷移以 Node 為核心的套件程式碼。
- 你可接受為特定執行階段能力承擔較大套件大小。

## 預設行為

- 建置目標以瀏覽器為優先（`web`）。
- 解析優先採用瀏覽器匯出（`browser`、`module`、`main`）。
- Extension.js 預設停用 Node 核心 fallback（例如 `crypto`、`path` 與 `fs`）。

也就是說，除非你明確新增 fallback，否則匯入 Node API 可能會失敗。

## 設定 Node polyfill

使用 `extension.config.js`（或 `.mjs` ／ `.cjs`）來擴充 Rspack 設定，並定義安全的 fallback。

```js
import NodePolyfillPlugin from "node-polyfill-webpack-plugin";
import { createRequire } from "node:module";

const require = createRequire(import.meta.url);

export default {
  config: (config) => {
    config.resolve = config.resolve || {};
    config.resolve.fallback = {
      ...(config.resolve.fallback || {}),
      crypto: require.resolve("crypto-browserify"),
      path: require.resolve("path-browserify"),
      fs: false,
    };
    config.plugins = config.plugins || [];
    config.plugins.push(new NodePolyfillPlugin());
    return config;
  },
};
```

## 擴充功能 API 與 Node API 比較

在 CLI／設定中設定 `polyfill: true` 會啟用 `webextension-polyfill`，這是一層相容性層，讓 Chromium 系瀏覽器能存取 `browser.*` API 命名空間。它並不會啟用 Node 核心模組 polyfill。

僅在無法以標準 Web API 執行的程式庫上使用 Node polyfill。

## 注意事項

- 不要假設擴充功能執行階段中有檔案系統存取；除非你有特定的瀏覽器安全策略，否則請保留 `fs: false`。
- 在可行時，優先使用 Web Crypto（`crypto.subtle`）而非大型的 Node crypto shim。
- Polyfill（Node API 的瀏覽器相容替代品）會增加套件大小，並可能影響擴充功能頁面與內容腳本的啟動時間。
- 部分使用 `node:` 指定符的套件，可能需要明確的 fallback 處理。
