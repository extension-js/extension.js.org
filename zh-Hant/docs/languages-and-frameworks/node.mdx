Extension.js 預設以瀏覽器為目標執行階段。它不會自動為 Node 核心模組加上 polyfill,以維持較小的打包體積。若某個相依套件需要 `buffer`、`stream` 或 `crypto`,你會在建置階段看到解析錯誤。下面各節說明何時應加入 polyfill,以及何時應改用瀏覽器原生的替代方案。

## 何時適合使用 Node polyfill

- 必要的相依套件在瀏覽器執行階段中需要 Node 核心模組。
- 你正在逐步從以 Node 為主的套件遷移程式碼。
- 為了某些特定的執行階段能力,你可以接受較大的打包體積。

## 預設行為

- 建置目標以瀏覽器優先(`web`)。
- 模組解析優先採用瀏覽器導出(`browser`、`module`、`main`)。
- Extension.js 預設停用 Node 核心模組的 fallback(例如 `crypto`、`path` 與 `fs`)。

這表示除非你明確加入 fallback,否則匯入 Node API 可能會失敗。

## 設定 Node polyfill

使用 `extension.config.js`(或 `.mjs` / `.cjs`)擴充 Rspack 設定,並定義安全的 fallback。

```js
import NodePolyfillPlugin from "node-polyfill-webpack-plugin";
import { createRequire } from "node:module";

const require = createRequire(import.meta.url);

export default {
  config: (config) => {
    config.resolve = config.resolve || {};
    config.resolve.fallback = {
      ...(config.resolve.fallback || {}),
      crypto: require.resolve("crypto-browserify"),
      path: require.resolve("path-browserify"),
      fs: false,
    };
    config.plugins = config.plugins || [];
    config.plugins.push(new NodePolyfillPlugin());
    return config;
  },
};
```

## 擴充功能 API 與 Node API

在 CLI 或設定中將 `polyfill` 設為 `true` 會啟用 `webextension-polyfill`,這是一層相容性套件,讓 Chromium 系列瀏覽器能存取 `browser.*` API 命名空間。它並不會啟用 Node 核心模組的 polyfill。

只有當某些函式庫無法使用標準 Web API 執行時,才使用 Node polyfill。

## 注意事項

- 不要假設擴充功能執行階段可以存取檔案系統;除非你有專為瀏覽器設計的安全策略,否則請保持 `fs: false`。
- 在可行的情況下,優先使用 Web Crypto(`crypto.subtle`),而不是龐大的 Node crypto shim。
- Polyfill(瀏覽器相容的 Node API 替代品)會增加打包體積,並可能影響擴充功能頁面與 content script 的啟動時間。
- 部分使用 `node:` 前綴的套件,可能需要明確的 fallback 處理。
