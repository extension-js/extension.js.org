Extension.js 在所有擴充功能環境中執行 TypeScript:背景、content script、popup、options 與側邊欄。它預設使用 Rspack + SWC 管線。你不需要額外的 `tsc` 步驟、ts-loader,或額外的打包工具規則。

## 何時適合使用 TypeScript

- 你希望在各個擴充功能環境之間進行更安全的重構,並擁有更清晰的契約。
- 你會在背景腳本、content script 與 UI 介面之間共用邏輯。
- 在處理 AI 產生的程式碼時,你需要可預期的 API 用法。

## 自動產生的環境型別

對於 TypeScript 專案,Extension.js 會在專案根目錄產生 `extension-env.d.ts`,裡面包含環境型別宣告(瀏覽器與執行階段的全域物件、`EXTENSION_PUBLIC_*` 環境變數,以及打包工具的型別)。執行 [`dev`](/zh-Hant/docs/commands/dev) 與 [`build`](/zh-Hant/docs/commands/build) 時都會重新產生這個檔案,因此編輯器型別與 CI 中的 `tsc --noEmit` 會保持同步。你可以選擇將它納入版控,也可以加入 git ignore,兩種做法都行;下次執行時就會重新建立。

## 在既有擴充功能中使用

依照下方步驟,將 TypeScript 加入既有的擴充功能。

### 安裝

1. 將 TypeScript 安裝為開發相依套件:

<PackageManagerTabs command="install -D typescript" />

2. 初始化 TypeScript 設定檔 `tsconfig.json`:

<PackageManagerTabs command="tsc --init" />

### 設定

#### TypeScript 偵測與需求

Extension.js 會在 `package.json` 旁尋找 `tsconfig.json`。

- 若存在 TypeScript 原始檔但缺少 `tsconfig.json`,Extension.js 會丟出錯誤,並要求你建立一份。
- 若 Extension.js 透過相依套件或設定偵測到 TypeScript,但沒有原始檔,它可以為你產生一份基礎的 `tsconfig.json`。

基礎設定範例:

```json5
{
  compilerOptions: {
    allowJs: true,
    allowSyntheticDefaultImports: true,
    esModuleInterop: true,
    forceConsistentCasingInFileNames: true,
    isolatedModules: false,
    jsx: "react-jsx",
    lib: ["dom", "dom.iterable", "esnext"],
    moduleResolution: "node",
    module: "esnext",
    noEmit: true,
    resolveJsonModule: true,
    strict: true,
    target: "esnext",
    skipLibCheck: true,
  },
  exclude: ["node_modules", "dist"],
}
```

#### 自動型別

Extension.js 會在開發期間在你的專案根目錄產生 `extension-env.d.ts`。這個檔案包含 Extension.js API 的型別宣告與瀏覽器 polyfill 型別。

```ts
// Required Extension.js types for TypeScript projects.
// This file is auto-generated and should not be excluded.
// If you need additional types, consider creating a new *.d.ts file and
// referencing it in the "include" array of your tsconfig.json file.
// See https://www.typescriptlang.org/tsconfig#include for more information.
/// <reference types="extension/types" />

// Polyfill types for browser.* APIs.
/// <reference types="extension/types/polyfill" />
```

#### 編譯與型別檢查

預設的打包管線會編譯 TypeScript,但不會在打包過程中執行完整的 `tsc` 型別檢查。

建議的 scripts 設定:

```json
{
  "scripts": {
    "dev": "extension dev",
    "typecheck": "tsc --noEmit"
  }
}
```
