Extension.js 在每個擴充功能情境中執行 TypeScript：background、content scripts、popup、options 與 sidebar。它預設使用 Rspack + SWC 流水線。你不需要額外的 `tsc` 步驟、ts-loader，或額外的 bundler 規則。

## 何時適合使用 TypeScript

- 你希望在擴充功能各情境之間進行更安全的重構並擁有更清楚的契約。
- 你在背景腳本、內容腳本與 UI 介面之間共用邏輯。
- 當與 AI 產生的程式碼互動時，你需要可預期的 API 使用方式。

## 自動產生的 ambient 型別

對於 TypeScript 專案，Extension.js 會在專案根目錄產生 `extension-env.d.ts` 檔，內含 ambient 宣告（browser／runtime 全域變數、`EXTENSION_PUBLIC_*` 環境變數，以及 bundler 型別）。[`dev`](/docs/commands/dev) 與 [`build`](/docs/commands/build) 都會重新產生它，讓編輯器型別與 CI 的 `tsc --noEmit` 保持同步。是否提交它（或加入 git-ignore）皆可，因為下一次執行時都會重新產生。

## 在既有擴充功能中使用

依下列步驟，將 TypeScript 加入既有擴充功能。

### 安裝

1. 將 TypeScript 安裝為開發相依套件：

<PackageManagerTabs command="install -D typescript" />

2. 初始化 TypeScript 設定檔 `tsconfig.json`：

<PackageManagerTabs command="tsc --init" />

### 設定

#### TypeScript 偵測與需求

Extension.js 會在你的 `package.json` 旁邊尋找 `tsconfig.json`。

- 若已有 TypeScript 來源檔案但缺少 `tsconfig.json`，Extension.js 會丟出錯誤並要求你建立。
- 若 Extension.js 從相依套件或設定中偵測到 TypeScript，但沒有來源檔案，它可以協助產生基線 `tsconfig.json`。

範例基線設定：

```json5
{
  compilerOptions: {
    allowJs: true,
    allowSyntheticDefaultImports: true,
    esModuleInterop: true,
    forceConsistentCasingInFileNames: true,
    isolatedModules: false,
    jsx: "react-jsx",
    lib: ["dom", "dom.iterable", "esnext"],
    moduleResolution: "node",
    module: "esnext",
    noEmit: true,
    resolveJsonModule: true,
    strict: true,
    target: "esnext",
    skipLibCheck: true,
  },
  exclude: ["node_modules", "dist"],
}
```

#### 自動產生型別

Extension.js 在開發期間會於專案根目錄產生 `extension-env.d.ts`。此檔案包含 Extension.js API 與瀏覽器 polyfill 型別的型別宣告。

```ts
// Required Extension.js types for TypeScript projects.
// This file is auto-generated and should not be excluded.
// If you need additional types, consider creating a new *.d.ts file and
// referencing it in the "include" array of your tsconfig.json file.
// See https://www.typescriptlang.org/tsconfig#include for more information.
/// <reference types="extension/types" />

// Polyfill types for browser.* APIs.
/// <reference types="extension/types/polyfill" />
```

#### 轉譯與型別檢查

預設 bundler 流水線會編譯 TypeScript，但不會在打包過程中執行完整的 `tsc` 型別檢查。

建議的腳本：

```json
{
  "scripts": {
    "dev": "extension dev",
    "typecheck": "tsc --noEmit"
  }
}
```
