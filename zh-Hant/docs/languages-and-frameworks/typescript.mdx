Extension.js 在每一個擴充功能情境中皆執行 TypeScript:背景、內容腳本、彈出視窗、選項頁與側邊欄。它預設使用 Rspack + SWC 流程。你不需要額外的 `tsc` 步驟、ts-loader,也不需要額外的打包器規則。

## 何時適合使用 TypeScript

- 你希望在不同擴充功能情境之間獲得更安全的重構與更清晰的契約。
- 你在背景腳本、內容腳本與 UI 介面之間共用邏輯。
- 你在使用 AI 產生的程式碼時需要可預期的 API 使用方式。

## 自動產生的環境型別

對於 TypeScript 專案,Extension.js 會在專案根目錄產生一個 `extension-env.d.ts` 檔案,其中包含環境宣告(瀏覽器/執行時全域變數、`EXTENSION_PUBLIC_*` 環境變數以及打包器相關型別)。[`dev`](/docs/commands/dev) 與 [`build`](/docs/commands/build) 都會重新產生此檔案,因此編輯器型別與 CI 中的 `tsc --noEmit` 會保持同步。提交它或將其加入 git 忽略名單皆可 —— 任一方式都沒問題,因為下一次執行時會重新建立。

## 在現有擴充功能中使用

依照以下步驟將 TypeScript 加入既有的擴充功能。

### 安裝

1. 將 TypeScript 安裝為開發相依套件:

<PackageManagerTabs command="install -D typescript" />

2. 初始化 TypeScript 設定檔 `tsconfig.json`:

<PackageManagerTabs command="tsc --init" />

### 設定

#### TypeScript 偵測與要求

Extension.js 會在 `package.json` 旁邊尋找 `tsconfig.json`。

- 若存在 TypeScript 原始檔但缺少 `tsconfig.json`,Extension.js 會丟出錯誤並要求你建立一個。
- 若 Extension.js 透過相依套件或設定偵測到 TypeScript 但沒有原始檔,可以為你建立一個基線的 `tsconfig.json`。

基線設定範例:

```json5
{
  compilerOptions: {
    allowJs: true,
    allowSyntheticDefaultImports: true,
    esModuleInterop: true,
    forceConsistentCasingInFileNames: true,
    isolatedModules: false,
    jsx: "react-jsx",
    lib: ["dom", "dom.iterable", "esnext"],
    moduleResolution: "node",
    module: "esnext",
    noEmit: true,
    resolveJsonModule: true,
    strict: true,
    target: "esnext",
    skipLibCheck: true,
  },
  exclude: ["node_modules", "dist"],
}
```

#### 自動型別

Extension.js 會在開發過程中於專案根目錄產生 `extension-env.d.ts`。此檔案包含 Extension.js API 的型別宣告與瀏覽器 polyfill 的型別。

```ts
// Required Extension.js types for TypeScript projects.
// This file is auto-generated and should not be excluded.
// If you need additional types, consider creating a new *.d.ts file and
// referencing it in the "include" array of your tsconfig.json file.
// See https://www.typescriptlang.org/tsconfig#include for more information.
/// <reference types="extension/types" />

// Polyfill types for browser.* APIs.
/// <reference types="extension/types/polyfill" />
```

#### 轉譯 vs 型別檢查

預設的打包器流程會編譯 TypeScript,但不會在打包時執行完整的 `tsc` 型別檢查。

建議的腳本設定:

```json
{
  "scripts": {
    "dev": "extension dev",
    "typecheck": "tsc --noEmit"
  }
}
```
