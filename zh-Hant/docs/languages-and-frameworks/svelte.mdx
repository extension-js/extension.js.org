使用 Svelte 元件打造擴充功能 UI,框架的串接由 Extension.js 負責。

Extension.js 會從你的專案相依套件中偵測到 Svelte,並自動設定 `.svelte` 解析、loader 設定與 Svelte 用戶端別名。

## 何時適合使用 Svelte

- 你希望彈出視窗、選項頁、側邊欄或新分頁的 UI 擁有較小的執行時。
- 你偏好以元件為主的撰寫方式,並使用簡潔的語法。
- 你需要一個能同時支援擴充功能頁面與內容腳本 UI 掛載的框架方案。

## 在現有擴充功能中使用

安裝 Svelte:

<PackageManagerTabs command="install svelte" />

若需要在既有專案中明確設定,請一併安裝 Svelte loader:

<PackageManagerTabs command="install -D svelte-loader typescript" />

## 開發行為

當 Extension.js 偵測到 Svelte 時,它會:

- 啟用 `.svelte` 檔案解析。
- 在 JS 框架流程中設定 Svelte loader。
- 套用 Svelte 用戶端別名(例如 `svelte`、`svelte/store` 與 `svelte/reactivity`)。

若你的專案缺少選用工具,Extension.js 會安裝所需項目並要求重新啟動。

## 使用範例

### 在擴充功能頁面中

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>Svelte Extension</title>
  </head>
  <body>
    <div id="app"></div>
  </body>
  <script src="./main.ts"></script>
</html>
```

```ts
import App from "./App.svelte";

new App({
  target: document.getElementById("app")!,
});
```

### 在 `content_script` 檔案中

針對內容腳本,建立一個掛載節點,並從預設匯出回傳一個清理函式。這讓 Extension.js 能在開發時安全地重新掛載:

```ts
import App from "./App.svelte";

export default function main() {
  const target = document.createElement("div");
  target.id = "extension-root";
  document.body.appendChild(target);

  const app = new App({ target });

  return () => {
    app.$destroy();
    target.remove();
  };
}
```

## 最佳實踐

- 讓 Svelte 元件專注於 UI,將瀏覽器 API 的協調工作移到專門的模組中。
- 在內容腳本中,始終回傳清理函式,讓重新掛載的行為保持可預期。
- 先在擴充功能頁面中使用 Svelte,待內容腳本的功能邊界清楚之後再擴展到 UI。
