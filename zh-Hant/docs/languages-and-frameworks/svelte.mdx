用 Svelte 元件打造擴充功能 UI,並讓 Extension.js 負責框架串接。

Extension.js 會從你的專案相依套件中偵測 Svelte,並自動配置 `.svelte` 解析、loader 設定與 Svelte 客戶端別名。

## 何時適合使用 Svelte

- 你希望 popup、options、側邊欄或新分頁的 UI 使用更輕量的執行階段。
- 你偏好以元件為主、語法精簡的撰寫風格。
- 你希望選用的框架既能用於擴充功能頁面,也能用於 content script UI 掛載點。

## 在既有擴充功能中使用

安裝 Svelte:

<PackageManagerTabs command="install svelte" />

若要在既有專案中進行明確設定,請一併安裝 Svelte loader:

<PackageManagerTabs command="install -D svelte-loader typescript" />

## 開發階段行為

當 Extension.js 偵測到 Svelte 時,它會:

- 啟用 `.svelte` 檔案解析。
- 在 JS 框架管線中配置 Svelte loader。
- 套用 Svelte 客戶端別名(例如 `svelte`、`svelte/store` 與 `svelte/reactivity`)。

若你的專案缺少選用的工具,Extension.js 會自動安裝所需內容並要求重新啟動。

## 使用範例

### 在擴充功能頁面中

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>Svelte Extension</title>
  </head>
  <body>
    <div id="app"></div>
  </body>
  <script src="./main.ts"></script>
</html>
```

```ts
import App from "./App.svelte";

new App({
  target: document.getElementById("app")!,
});
```

### 在 `content_script` 檔案中

對於 content script,建立掛載節點,並在預設匯出的函式中回傳一個清理函式。如此一來,Extension.js 在開發階段能安全地重新掛載:

```ts
import App from "./App.svelte";

export default function main() {
  const target = document.createElement("div");
  target.id = "extension-root";
  document.body.appendChild(target);

  const app = new App({ target });

  return () => {
    app.$destroy();
    target.remove();
  };
}
```

## 最佳實踐

- 讓 Svelte 元件專注於 UI,將瀏覽器 API 的協作邏輯移至專屬模組。
- 在 content script 中務必回傳清理函式,讓重新掛載行為保持可預期。
- 先在擴充功能頁面中使用 Svelte,確認功能邊界後再擴展到 content script UI。
