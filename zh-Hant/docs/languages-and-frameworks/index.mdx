為你的擴充功能介面挑選合適的語言與 UI 框架策略,並善用 Extension.js 的預設值,讓設定維持在最低限度。

## 依目標選擇

| 目標                                       | 從這裡開始                                                                                |
| ------------------------------------------ | ----------------------------------------------------------------------------------------- |
| 強型別與更安全的重構                       | [TypeScript](/zh-Hant/docs/languages-and-frameworks/typescript)                            |
| 擁有豐富生態系的 UI 元件                   | [React](/zh-Hant/docs/languages-and-frameworks/react)                                      |
| 體積更小的類 React 執行階段                | [Preact](/zh-Hant/docs/languages-and-frameworks/preact)                                    |
| Vue 單一檔案元件工作流程                   | [Vue.js](/zh-Hant/docs/languages-and-frameworks/vue)                                       |
| 現代模組語法                               | [ECMAScript modules](/zh-Hant/docs/languages-and-frameworks/ecmascript-modules)            |
| 預設樣式範圍隔離                           | [CSS modules](/zh-Hant/docs/languages-and-frameworks/css-modules)                          |
| Sass 或 Less 樣式工作流程                  | [Sass](/zh-Hant/docs/languages-and-frameworks/sass)、[Less](/zh-Hant/docs/languages-and-frameworks/less) |
| 瀏覽器安全的 Node 相容性決策               | [瀏覽器擴充功能中的 Node API](/zh-Hant/docs/languages-and-frameworks/node)                  |
| 重度效能需求的 Wasm 工作負載               | [WebAssembly](/zh-Hant/docs/languages-and-frameworks/webassembly)                          |

## 建議的學習順序

1. 選擇語言或執行階段(`TypeScript`、`React`、`Vue` 或 `Preact`)。
2. 選擇樣式模型(`CSS modules`、`Sass` 或 `Less`)。
3. 僅在需要時再加入建置或執行階段限制(`Node APIs`、`WebAssembly`)。

## 後續步驟

- 在[整合](/zh-Hant/docs/integrations/index)中加入生態系工具。
- 透過[指令參考](/zh-Hant/docs/commands/index)讓指令用法清晰明瞭。
