為你的擴充功能介面選擇合適的語言與 UI 框架策略，然後利用 Extension.js 預設值，將設定降到最低。

## 依目標選擇

| 目標                                     | 從這裡開始                                                                               |
| ---------------------------------------- | ---------------------------------------------------------------------------------------- |
| 強型別與更安全的重構                     | [TypeScript](/docs/languages-and-frameworks/typescript)                                  |
| 擁有龐大生態系的豐富 UI 元件              | [React](/docs/languages-and-frameworks/react)                                            |
| 較小的類 React 執行階段                  | [Preact](/docs/languages-and-frameworks/preact)                                          |
| Vue 單一檔案元件工作流程                  | [Vue.js](/docs/languages-and-frameworks/vue)                                             |
| 現代模組語法                             | [ECMAScript modules](/docs/languages-and-frameworks/ecmascript-modules)                  |
| 預設樣式作用域                           | [CSS modules](/docs/languages-and-frameworks/css-modules)                                |
| Sass 或 Less 樣式工作流程                | [Sass](/docs/languages-and-frameworks/sass), [Less](/docs/languages-and-frameworks/less) |
| 適用瀏覽器的 Node 相容性決策              | [Node APIs in browser extensions](/docs/languages-and-frameworks/node)                   |
| 高效能 Wasm 工作負載                     | [WebAssembly](/docs/languages-and-frameworks/webassembly)                                |

## 建議的進階順序

1. 選擇語言／執行階段（`TypeScript`、`React`、`Vue` 或 `Preact`）。
2. 選擇樣式模型（`CSS modules`、`Sass` 或 `Less`）。
3. 僅在必要時加入建置／執行階段限制（`Node APIs`、`WebAssembly`）。

- 透過 [指令參考](/docs/commands/index) 維持指令使用清晰。
