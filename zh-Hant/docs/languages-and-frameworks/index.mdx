為你的擴充功能介面選擇合適的語言與 UI 框架策略,然後使用 Extension.js 的預設值,將設定流程降到最低。

## 依目標選擇

| 目標                                   | 從這裡開始                                                                              |
| -------------------------------------- | --------------------------------------------------------------------------------------- |
| 強型別與更安全的重構                   | [TypeScript](/docs/languages-and-frameworks/typescript)                                 |
| 豐富的 UI 元件與龐大的生態             | [React](/docs/languages-and-frameworks/react)                                           |
| 更小的 React 風格執行時                | [Preact](/docs/languages-and-frameworks/preact)                                         |
| Vue 單一檔案元件工作流程               | [Vue.js](/docs/languages-and-frameworks/vue)                                            |
| 現代模組語法                           | [ECMAScript modules](/docs/languages-and-frameworks/ecmascript-modules)                 |
| 預設範圍化的樣式                       | [CSS modules](/docs/languages-and-frameworks/css-modules)                               |
| Sass 或 Less 樣式工作流程              | [Sass](/docs/languages-and-frameworks/sass)、[Less](/docs/languages-and-frameworks/less) |
| 瀏覽器安全的 Node 相容性決策           | [瀏覽器擴充功能中的 Node API](/docs/languages-and-frameworks/node)                      |
| 效能密集的 Wasm 工作負載               | [WebAssembly](/docs/languages-and-frameworks/webassembly)                               |

## 建議的循序步驟

1. 選擇語言/執行時(`TypeScript`、`React`、`Vue` 或 `Preact`)。
2. 選擇樣式方案(`CSS modules`、`Sass` 或 `Less`)。
3. 僅在需要時才加上建置/執行時的限制(`Node APIs`、`WebAssembly`)。

## 後續步驟

- 在[整合](/docs/integrations/index)中加入生態系工具。
- 透過[指令參考](/docs/commands/index)讓指令使用更清楚。
