打造更小的擴充功能 UI 產物,同時保留類似 React 的開發者體驗與快速的本地迭代。

Extension.js 會從你的相依套件中偵測到 Preact,並自動設定 JSX/TSX 轉換、React 相容別名以及 fast-refresh 支援。你的 React 匯入會透過這些別名對應到 Preact。

## 何時適合使用 Preact

- 你希望彈出視窗/側邊欄/新分頁等介面擁有較小的 UI 產物大小。
- 你喜歡 React 風格的元件,但偏好更輕量的執行時。
- 你正在最佳化擴充功能在低階裝置上的啟動速度與 UI 反應性。

## 在現有擴充功能中使用

依照以下步驟將 Preact 加入既有的擴充功能。

### 安裝

安裝所需相依套件:

<PackageManagerTabs command="install preact" />

Preact 自帶 TypeScript 型別,因此不需要另外安裝 `@types/preact` 套件。

### 設定

Extension.js 期望 Preact 檔案使用以下副檔名:

- 若未啟用 TypeScript:`*.jsx`
- 若啟用 TypeScript:`*.tsx`

## 開發行為

當 Extension.js 偵測到 Preact 時,會進行以下設定:

- 相容別名(例如 `react` → `preact/compat`)。
- 針對 Preact 調整過的 JSX 處理。
- 開發時透過 `@rspack/plugin-preact-refresh` 提供的 refresh 整合。

如果你的專案缺少選用的 refresh 相依套件,Extension.js 會自動安裝並要求你重新啟動。

### 疑難排解

- **缺少 refresh 相依套件警告:** 安裝 `@rspack/plugin-preact-refresh`(以及若有提示的相關套件)。
- **安裝後提示重新啟動:** 停止並重新執行 `extension dev`,讓開發伺服器能乾淨地重新載入 refresh 設定。
- **未偵測到 Preact 整合:** 確認 `preact` 出現在 `dependencies` 或 `devDependencies` 中。

## 使用範例

### 在新分頁擴充功能中

若要在新分頁擴充功能中使用 Preact,在 HTML 檔案中以 `<script>` 引入:

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>New Extension</title>
  </head>
  <body>
    <noscript>You need to enable JavaScript to run this extension.</noscript>
    <div id="root"></div>
  </body>
  <script src="./index.tsx"></script>
</html>
```

```tsx
import { h, render } from "preact";
import App from "./App";

const root = document.getElementById("root");

render(<App />, root);
```

```tsx
export default function App() {
  return <h1>Hello, Preact Extension!</h1>;
}
```

### 在 `content_script` 檔案中

針對內容腳本,透過建立 HTML 元素並在其中渲染來將 Preact 注入頁面:

```tsx
import { h, render } from "preact";
import App from "./App";
import "./content.css";

const rootDiv = document.createElement("div");
rootDiv.id = "extension-root";
document.body.appendChild(rootDiv);

render(<App />, rootDiv);
```
