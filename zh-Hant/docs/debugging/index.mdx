瀏覽器擴充功能很難除錯,因為有趣的狀態存在你不易觸及的地方:會進入閒置的 MV3 service worker、隔離的內容腳本世界,以及一失去焦點就關閉的 popup。Extension.js 在你執行中的開發階段中開啟一個小巧的 **本機控制通道**,讓你(或一個 AI 代理,或一個 CI 工作)可以直接抵達這些情境 —— 讀取它們記錄的內容、檢視它們渲染的內容、呼叫它們,並觸發使用者會觸發的事件。

它是本機的、不需要帳號,觀察永遠免費。任何 _變更_ 狀態的事都是按工作階段選擇加入的。

## 開啟

啟動一個帶有你需要的控制旗標的開發工作階段:

```bash
# 唯讀永遠開啟。加上 control 才能對擴充功能執行動作:
pnpm extension dev --allow-control

# 加上 eval 才能在情境中執行任意運算式:
pnpm extension dev --allow-control --allow-eval
```

下面的每個操作都針對該工作階段,無論你是在讀取還是在執行動作,定址情境的方式都一致。

## 你可以做什麼

| 指令                                       | 作用                                                                                                                | 門檻              |
| ------------------------------------------ | ------------------------------------------------------------------------------------------------------------------- | ----------------- |
| `extension logs`                           | 在單一有序時間線上串流每個情境(SW、content、popup、options、sidebar、devtools)的記錄                              | —                 |
| `extension inspect`                        | 讀取某個介面的即時(注入後)DOM                                                                                     | —                 |
| `extension storage get \| set`             | 讀取或寫入你擴充功能的 `chrome.storage`                                                                             | `--allow-control` |
| `extension reload`                         | 重新載入擴充功能或某個分頁                                                                                          | `--allow-control` |
| `extension open <popup\|options\|sidebar>` | 開啟一個擴充功能介面                                                                                                | `--allow-control` |
| `extension open action`                    | [觸發工具列 action](/docs/debugging/trigger-actions-and-commands) —— 開啟其 popup,或重播 `onClicked`                | `--allow-control` |
| `extension open command --name <cmd>`      | [重播某個鍵盤捷徑指令](/docs/debugging/trigger-actions-and-commands)                                               | `--allow-control` |
| `extension eval "<expr>" --context <c>`    | 在情境中對運算式求值並回傳值                                                                                        | `--allow-eval`    |

## 定址情境

讀取與執行動作共用同一套詞彙 —— 你指定介面名稱,Extension.js 會在它已追蹤的工作階段中解析。

| 定址                                                | 意義                                              |
| --------------------------------------------------- | ------------------------------------------------- |
| `--context background`                              | service worker(或 MV2 背景頁面)                 |
| `--context popup` / `options` / `sidebar`           | 該擴充功能介面(若已開啟)                        |
| `--context content --url "https://shop.example/*"`  | 符合頁面上的隔離內容世界                          |
| `--context content --tab 7`                         | 特定分頁中的內容世界                              |

## 範例:我的背景處理器到底有沒有跑?

```bash
pnpm extension eval "globalThis.__lastSyncAt" --context background --allow-eval
```

```json
{ "ok": true, "value": 1718000000000 }
```

你的運算式所觸發的任何 `console.log` 都會同時透過 `extension logs` 流出,並依序列相關聯 —— 你可以同時看到回傳值 _以及_ 副作用。

## 跨瀏覽器支援

Extension.js 透過 **瀏覽器內部夥伴** 進行除錯,而不是透過 Chrome DevTools Protocol,因此核心迴圈能觸及 **Chrome 與 Firefox** 上你自己的介面 —— 舊的「Firefox 用 RDP,不支援」的高牆在這些工具上已經消失。

| 能力                                                                              | Chromium | Firefox      |
| --------------------------------------------------------------------------------- | -------- | ------------ |
| `logs`、`inspect`、`eval`、`storage`、`reload`、`open`(含 `action`/`command`)   | ✓        | ✓(已驗證)  |
| `inspect --deep-dom`(closed shadow roots)                                        | ✓        | —(使用 CDP)|
| `extension_list_extensions`(MCP 工具)                                           | ✓        | —(使用 CDP)|

(`extension_list_extensions` 是 MCP 工具,而不是 CLI 動詞 —— 它透過 DevTools Protocol 連線,因此僅限 Chromium。)

## 安全性

這些門檻是刻意設計,而不是繁文縟節:

- **觀察不需要任何東西。** 讀取記錄與 DOM 永遠可用。
- **有界操作需要 `--allow-control`。** `storage`、`reload` 與 `open` 會改變狀態,所以按工作階段選擇加入。
- **`eval` 需要 `--allow-eval` _以及_ 一個按工作階段的權杖。** 該權杖寫入 `dist/` 之外的 `0600` 權限檔案,因此絕不會混入建構成品 —— 隨機本機程序無法默默驅動你的 service worker。
- **不會觸及正式環境。** 控制通道僅在 `dev`/`preview` 期間存在;它由一個不會出現在已建構套件中的連接埠控制。

## 與 AI 代理搭配

相同的操作以 MCP 工具的形式透過 [`@extension.dev/mcp`](/docs/integrations/chrome-devtools-mcp) 公開(`extension_logs`、`extension_eval`、`extension_storage`、`extension_reload`、`extension_open`、`extension_list_extensions`)。門檻完全相同 —— 助理可以自由觀察,但只有在你按工作階段啟用時才會執行動作。
