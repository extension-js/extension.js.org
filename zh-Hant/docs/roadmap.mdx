Extension.js 的開發是公開進行的。本頁告訴你該到哪裡看正在交付、待排程,以及正在討論的內容。

## 追蹤工作的地方

| 介面                                                                            | 它告訴你什麼                                                                    |
| ------------------------------------------------------------------------------- | ------------------------------------------------------------------------------- |
| [GitHub Releases](https://github.com/extension-js/extension.js/releases)        | 每個已發布版本與備註。是判斷「何時交付了什麼」的唯一來源。                      |
| [GitHub Milestones](https://github.com/extension-js/extension.js/milestones)    | 依即將釋出的版本分組的 issue。是判斷接下來會處理什麼的最佳訊號。                |

| [npm 套件頁](https://www.npmjs.com/package/extension)                           | 最新發佈的版本。                                                                |

## 發行如何運作

Extension.js 遵循語意化版本:

- **Patch 釋出**只修 bug,不改變行為。
- **Minor 釋出**會新增 flag、功能或範本選項,但不破壞現有專案。
- **Major 釋出**會調整預設值或移除已棄用的行為。這類釋出會在 release 條目中附上遷移備註。

請在 `devDependencies` 中釘住特定版本,或使用 `extension@latest` 取得最新發佈。[npm 套件](https://www.npmjs.com/package/extension)頁列出每個版本。

## 提出新需求

如果你想要的功能還不存在:

1. 搜尋[開啟的 issue](https://github.com/extension-js/extension.js/issues) 與[討論](https://github.com/extension-js/extension.js/discussions)。可能有人已在追蹤。
2. 如果沒有,請先開一則討論說明用例,再開 issue。具體用例會被優先處理;抽象的請求很少會。
3. 用所觸及的介面標籤分類討論(CLI、manifest、reload、browsers、templates、docs)。

## 近期方向

最新的方向請閱讀[部落格](/blog)的最新文章,以及 [GitHub 上最新的 release 條目](https://github.com/extension-js/extension.js/releases)。部落格說明重大版本背後的原因;GitHub releases 則涵蓋實際內容。
