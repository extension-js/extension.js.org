Extension.js 的開發在公開場域進行。本頁告訴你該到哪裡查看目前正在發佈的內容、正在排程的項目，以及正在討論的方向。

## 在哪裡追蹤進度

| 介面                                                                            | 你能從中得知什麼                                                              |
| ------------------------------------------------------------------------------ | ------------------------------------------------------------------------------ |
| [GitHub Releases](https://github.com/extension-js/extension.js/releases)       | 每個已發佈版本與更新說明。何時發佈了什麼的權威來源。                              |
| [GitHub Milestones](https://github.com/extension-js/extension.js/milestones)   | 依預定版本分組的 issues。最能反映下一波即將推出內容的訊號。                       |
| [GitHub Discussions](https://github.com/extension-js/extension.js/discussions) | 提案、RFC 與比較長篇的方向討論。                                                |

| [Discord](https://discord.gg/v9h2RgeTSN)                                       | 日常協作、提問與非正式的預覽。                                                  |
| [npm package page](https://www.npmjs.com/package/extension)                    | 最新已發佈版本。                                                                |

## 發佈是怎麼運作的

Extension.js 遵循語意化版本：

- **Patch 版本**修正錯誤，但不變更行為。
- **Minor 版本**新增旗標、能力或範本選項，且不會破壞既有專案。
- **Major 版本**會變更預設值或移除被棄用的行為，並會在 release 條目中附上遷移說明。

在你的 `devDependencies` 釘住特定版本，或使用 `extension@latest` 取得最新發佈。所有版本可在 [npm 套件頁面](https://www.npmjs.com/package/extension)查看。

## 提出新需求

如果你想要的功能還不存在：

1. 先搜尋 [open issues](https://github.com/extension-js/extension.js/issues) 與 [discussions](https://github.com/extension-js/extension.js/discussions)，可能已經有人在追蹤。
2. 若沒有，先在 discussion 描述你的使用情境再開 issue。具體使用情境會被優先考慮;抽象的請求很少被處理。
3. 為討論加上對應的範疇標籤（CLI、manifest、reload、browsers、templates、docs）。

## 最近的方向

最新方向請閱讀[部落格](/zh-Hant/blog)上最近的文章，以及 [GitHub 上最新的 release 條目](https://github.com/extension-js/extension.js/releases)。部落格說明 major 版本背後的「為什麼」;GitHub releases 說明「做了什麼」。
