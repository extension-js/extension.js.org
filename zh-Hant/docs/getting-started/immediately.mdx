## 30 秒內執行任何 GitHub 擴充功能

下方章節涵蓋兩條路徑。你會了解 Extension.js 將什麼視為專案根，以及它如何解析像「裸範例名稱」這種模糊輸入。

## `dev` 如何解析專案

`extension dev` 接受可選的第一個引數。解析方式如下：

- **沒有引數：** 使用目前的工作資料夾作為擴充功能專案。
- **本機路徑：** Extension.js 將其視為相對於目前工作資料夾的路徑（例如，複製儲存庫後的 `./my-extension` 或 `../samples/page-redder`）。
- **`https://github.com/...` URL：** Extension.js 將儲存庫（或 tree 路徑）擷取到你的工作資料夾，然後從解壓後的資料夾進行建構。詳情請參閱 [Project detection and inputs](https://github.com/extension-js/extension.js/blob/main/programs/develop/README.md#project-detection-and-inputs)。
- **其他 `http(s)` URL：** Extension.js 將 URL 視為 ZIP 封存、下載並在本機解壓。

像 `sample.page-redder` 這樣的裸名稱 **並非** 遠端擷取，它只是你目前資料夾下的資料夾名稱。請使用完整的 GitHub URL，或先複製再將 `dev` 指向該路徑。

## 快速執行 Chrome 擴充功能範例

使用 Chrome Extension Samples 儲存庫中的範例來驗證你的設定並快速學習工作流程。

### 選項 A：完整 GitHub tree URL（推薦）

1. 開啟終端機。
2. `cd` 到要放置複製/解壓專案的資料夾（通常是空資料夾）。
3. 執行：

<CodeGroup>

```bash npm
npx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash pnpm
pnpx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash yarn
yarn dlx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash bun
bunx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash bun
bunx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

</CodeGroup>

Extension.js 會擷取該範例、建構它，並啟動載入該擴充功能的 Chrome。瀏覽 [Chrome Extension Samples](https://github.com/GoogleChrome/chrome-extensions-samples) 嘗試更多 URL。要鎖定不同的瀏覽器，可附加 `--browser=firefox` 或 `--browser=edge`。

## 先複製，再對本機路徑執行 `dev`

當你需要穩定的資料夾佈局（而非一次性嘗試）時，請複製儲存庫並將 `dev` 指向本機路徑：

```bash
git clone https://github.com/GoogleChrome/chrome-extensions-samples.git
cd chrome-extensions-samples/functional-samples/sample.page-redder
npx extension@latest dev .
```

## 用 polyfill 在 Edge 中執行 Mozilla 擴充功能

你可以在 Edge 中執行 MDN WebExtensions 範例。傳入 GitHub tree URL 並啟用 polyfill（相容層）。polyfill 會將 Firefox 風格的 `browser.*` 呼叫對應到 Chrome 風格的 `chrome.*` 呼叫，讓你以 Firefox 為導向的擴充功能能在 Chromium 系列瀏覽器中運作。

<CodeGroup>

```bash npm
npx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash pnpm
pnpx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash yarn
yarn dlx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash bun
bunx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash bun
bunx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

</CodeGroup>

### 先複製，再 polyfill + Edge

```bash
git clone https://github.com/mdn/webextensions-examples.git
cd webextensions-examples/apply-css
npx extension@latest dev . --browser=edge --polyfill=true
```

本範例使用 MDN WebExtensions Examples 的 [Apply CSS](https://github.com/mdn/webextensions-examples/tree/main/apply-css)。

## 快速提示

- **使用 TypeScript：** 在專案根目錄加上 `tsconfig.json` 檔案。
- **使用 React：** 在你的 `package.json` 中加入 `react` 與 `react-dom`。
- 為 React 設定的 `tsconfig.json` 可啟用 TypeScript + React 撰寫。
- 如果你需要處理 manifest 中未宣告的資產，請進一步了解 [特殊資料夾](/docs/features/special-folders)。

## 最佳實踐

- **使用 `extension` 套件** 從單一工具鏈建構、執行與打包擴充功能。
- **使用 `--browser`** 在開發時鎖定特定瀏覽器。
- **使用 `--polyfill`** 在將 Mozilla 為導向的範例調整為 Chromium 系列瀏覽器時使用。
- **優先使用完整 GitHub URL** 進行一次性嘗試；當你需要穩定的資料夾佈局時，**複製 + 本機路徑**。
