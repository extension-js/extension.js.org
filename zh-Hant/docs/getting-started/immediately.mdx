## 30 秒內執行任何 GitHub 上的擴充功能

下方章節涵蓋兩條路徑。你會了解 Extension.js 如何認定專案根目錄，以及它如何解析像是裸樣本名稱這類模糊輸入。

## `dev` 如何解析專案

`extension dev` 接受一個可選的第一個參數。解析方式如下：

- **無參數:** 將目前工作資料夾當作擴充功能專案。
- **本機路徑:** Extension.js 會把它視為相對於目前工作資料夾的路徑（例如,clone 後的 `./my-extension` 或 `../samples/page-redder`）。
- **`https://github.com/...` URL:** Extension.js 會把 repository（或 tree 路徑）擷取到你的工作資料夾,然後從解壓的資料夾建置。詳情請見[專案偵測與輸入](https://github.com/extension-js/extension.js/blob/main/programs/develop/README.md#project-detection-and-inputs)。
- **其他 `http(s)` URL:** Extension.js 會將該 URL 視為 ZIP 壓縮檔下載並在本地解壓。

像 `sample.page-redder` 這種裸名稱**不會**被遠端抓取;它只是你目前資料夾底下的資料夾名稱。請使用完整的 GitHub URL，或先 clone 再讓 `dev` 指向該路徑。

## 快速執行 Chrome 擴充功能範例

使用 Chrome Extension Samples repository 的範例來驗證你的設定，並快速熟悉工作流。

### 方案 A:完整 GitHub tree URL（推薦）

1. 開啟你的終端機。
2. `cd` 到希望放置 clone/解壓專案的資料夾（通常是一個空資料夾）。
3. 執行：

<CodeGroup>

```bash npm
npx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash pnpm
pnpx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash yarn
yarn dlx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash bun
bunx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash bun
bunx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

</CodeGroup>

Extension.js 會抓取範例、進行建置，並啟動已載入該擴充功能的 Chrome。更多可嘗試的 URL,請瀏覽 [Chrome Extension Samples](https://github.com/GoogleChrome/chrome-extensions-samples)。若要鎖定不同瀏覽器，加上 `--browser=firefox` 或 `--browser=edge`。

## 先 clone,再以本機路徑執行 `dev`

當你想要一個穩定的資料夾配置（而非單次嘗試），先 clone repository 並讓 `dev` 指向本機路徑：

```bash
git clone https://github.com/GoogleChrome/chrome-extensions-samples.git
cd chrome-extensions-samples/functional-samples/sample.page-redder
npx extension@latest dev .
```

## 用 polyfill 在 Edge 中執行 Mozilla 擴充功能

你可以在 Edge 中執行 MDN WebExtensions 範例。傳入 GitHub tree URL 並啟用 polyfill（相容層）。Polyfill 會把 Firefox 風格的 `browser.*` 呼叫對應為 Chrome 風格的 `chrome.*` 呼叫，讓你以 Firefox 為導向的擴充功能能在 Chromium 系瀏覽器上運作。

<CodeGroup>

```bash npm
npx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash pnpm
pnpx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash yarn
yarn dlx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash bun
bunx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash bun
bunx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

</CodeGroup>

### 先 clone,再加上 polyfill + Edge

```bash
git clone https://github.com/mdn/webextensions-examples.git
cd webextensions-examples/apply-css
npx extension@latest dev . --browser=edge --polyfill=true
```

此範例採用 MDN WebExtensions Examples 中的 [Apply CSS](https://github.com/mdn/webextensions-examples/tree/main/apply-css)。

## 小撇步

- **使用 TypeScript:** 在你的專案根目錄加上 `tsconfig.json`。
- **使用 React:** 把 `react` 與 `react-dom` 加入 `package.json`。
- 設定好支援 React 的 `tsconfig.json` 即可進行 TypeScript + React 的開發。
- 如果你需要處理未在 manifest 宣告的資源，了解更多請見[特殊資料夾](/zh-Hant/docs/features/special-folders)。

## 最佳實踐

- **使用 `extension` 套件**從同一條工具鏈建置、執行與打包擴充功能。
- **使用 `--browser`** 在開發時鎖定特定瀏覽器。
- **使用 `--polyfill`** 把以 Mozilla 為導向的範例改為適用於 Chromium 系瀏覽器。
- **單次嘗試時優先使用完整的 GitHub URL;若想要穩定的資料夾配置,則先 clone + 本機路徑。**
