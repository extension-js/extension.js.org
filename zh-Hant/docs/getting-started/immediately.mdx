## 30 秒內執行任何 GitHub 擴充功能

下面的章節涵蓋兩種路徑。你會了解 Extension.js 把什麼視為專案根目錄,以及它如何解析像「裸範例名稱」這類模糊的輸入。

## `dev` 如何解析專案

`extension dev` 接受一個可選的第一個參數。解析方式如下:

- **沒有參數:** 將目前工作目錄作為擴充功能專案使用。
- **本地路徑:** Extension.js 會將其視為相對於目前工作目錄的路徑(例如複製儲存庫後使用 `./my-extension` 或 `../samples/page-redder`)。
- **`https://github.com/...` URL:** Extension.js 會將該儲存庫(或 tree 路徑)抓取到你的工作目錄,然後從解壓縮後的資料夾進行建置。詳情參見[專案偵測與輸入](https://github.com/extension-js/extension.js/blob/main/programs/develop/README.md#project-detection-and-inputs)。
- **其他 `http(s)` URL:** Extension.js 會將該 URL 視為一個 ZIP 壓縮檔,下載並於本地解壓縮。

像 `sample.page-redder` 這類「裸名稱」**不是**遠端抓取;它只是目前目錄下的一個資料夾名稱。請使用完整的 GitHub URL,或先複製再讓 `dev` 指向該路徑。

## 快速執行 Chrome 擴充功能範例

使用 Chrome Extension Samples 儲存庫中的範例來驗證你的環境,並快速熟悉工作流。

### 方式 A:完整的 GitHub tree URL(推薦)

1. 開啟終端機。
2. `cd` 到要放置複製/解壓縮後專案的資料夾(通常是空資料夾)。
3. 執行:

<CodeGroup>

```bash npm
npx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash pnpm
pnpx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash yarn
yarn dlx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash bun
bunx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash bun
bunx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

</CodeGroup>

Extension.js 會抓取該範例、進行建置,並啟動 Chrome 載入該擴充功能。瀏覽 [Chrome Extension Samples](https://github.com/GoogleChrome/chrome-extensions-samples) 取得更多可嘗試的 URL。若要指定其他瀏覽器,可附加 `--browser=firefox` 或 `--browser=edge`。

## 先複製,再讓 `dev` 指向本地路徑

當你想要穩定的目錄結構(而非一次性嘗試)時,可以複製儲存庫並讓 `dev` 指向本地路徑:

```bash
git clone https://github.com/GoogleChrome/chrome-extensions-samples.git
cd chrome-extensions-samples/functional-samples/sample.page-redder
npx extension@latest dev .
```

## 透過 polyfill 在 Edge 中執行 Mozilla 擴充功能

你可以在 Edge 中執行 MDN WebExtensions 範例。傳入 GitHub tree URL 並啟用 polyfill(一個相容層)。polyfill 會把 Firefox 風格的 `browser.*` 呼叫對應到 Chrome 風格的 `chrome.*` 呼叫,讓面向 Firefox 的擴充功能也能在基於 Chromium 的瀏覽器中運作。

<CodeGroup>

```bash npm
npx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash pnpm
pnpx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash yarn
yarn dlx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash bun
bunx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash bun
bunx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

</CodeGroup>

### 先複製,然後在 Edge 中搭配 polyfill

```bash
git clone https://github.com/mdn/webextensions-examples.git
cd webextensions-examples/apply-css
npx extension@latest dev . --browser=edge --polyfill=true
```

此範例參考 MDN WebExtensions Examples 中的 [Apply CSS](https://github.com/mdn/webextensions-examples/tree/main/apply-css) 範例。

## 快速小撇步

- **使用 TypeScript:** 在你的專案根目錄加入一個 `tsconfig.json` 檔案。
- **使用 React:** 在 `package.json` 中加入 `react` 和 `react-dom`。
- 一份針對 React 設定好的 `tsconfig.json` 即可啟用 TypeScript + React 開發。
- 如果你需要處理 manifest 中未宣告的資源,可了解更多關於[特殊資料夾](/docs/features/special-folders)的內容。

## 最佳實踐

- **使用 `extension` 套件**,以一套工具鏈建置、執行並打包你的擴充功能。
- **使用 `--browser`** 在開發時指定目標瀏覽器。
- **使用 `--polyfill`** 將面向 Mozilla 的範例改造後用於基於 Chromium 的瀏覽器。
- **一次性嘗試時優先使用完整的 GitHub URL**;**需要穩定目錄結構時,先複製再使用本地路徑**。
