你將會建立一個 Omnibox(網址列)捷徑。在網址列輸入 `gh`、輸入查詢內容,然後跳轉到 GitHub 搜尋結果頁面。過程中你會撰寫 `manifest.json`,並在背景 service worker 中處理輸入。你也會練習每個專案都會採用的開發循環(`create` → `dev` → `build`)。

## 你將會建立什麼

| 能力                | 你會得到                                            |
| ------------------- | --------------------------------------------------- |
| Omnibox 關鍵字流程  | 從瀏覽器網址列使用 `gh` 觸發擴充功能行為            |
| 背景事件處理        | 透過 service worker 處理使用者輸入                  |
| 本地開發循環        | 快速執行、載入並驗證擴充功能行為                    |
| 漸進式增強          | 在基本流程跑通後,加入即時的 GitHub 建議            |

## 計畫

讓 GitHub 搜尋像瀏覽器原生捷徑一樣快。這個擴充功能會佔用關鍵字 `gh`;當你輸入 `gh` 加查詢內容後,它會開啟 GitHub 搜尋結果。

## 步驟 1:建立擴充功能

使用 Extension.js 的 `create` 指令搭建一個名為 `github-search` 的最小化擴充功能。

<CodeGroup>

```bash npm
npx extension@latest create github-search
```

```bash pnpm
pnpx extension@latest create github-search
```

```bash yarn
yarn dlx extension@latest create github-search
```

```bash bun
bunx extension@latest create github-search
```

```bash bun
bunx extension@latest create github-search
```

</CodeGroup>

每個擴充功能都從一個 manifest 檔案開始。它定義中繼資料、權限以及執行階段檔案。根據[上面的計畫](#計畫),設定 `gh` 捷徑並加入一個用於處理使用者事件的 service worker。

```json
{
  "manifest_version": 3,
  "name": "GitHub Search",
  "version": "1.0",
  "omnibox": { "keyword": "gh" },
  "background": {
    "service_worker": "service_worker.js"
  }
}
```

- `omnibox.keyword`:當你輸入 `gh` 時,瀏覽器會觸發一個事件。
- `background.service_worker`:監聽你觸發的事件。

## 步驟 3:建立背景 service worker

在瀏覽器擴充功能中,背景 service worker(獨立於任何可見頁面執行的腳本)負責處理瀏覽器事件。

在這個範例中,加入一個監聽 Omnibox 輸入,並將查詢路由到 GitHub 搜尋的腳本。

建立 `service_worker.js`:

```js
// When the user has accepted what is typed into the omnibox.
chrome.omnibox.onInputEntered.addListener((text) => {
  // Convert any special character (spaces, &, ?, etc)
  // into a valid character for the URL format.
  const encodedSearchText = encodeURIComponent(text);
  const url = `https://github.com/search?q=${encodedSearchText}&type=issues`;

  chrome.tabs.create({ url });
});
```

上面的腳本會在你於網址列的 "gh" 之後輸入任何內容時,開啟一個新分頁顯示 GitHub 搜尋結果。

## 步驟 4:載入你的擴充功能

你的 `package.json` 檔案現在看起來像這樣:

```json
{
  "scripts": {
    "dev": "extension dev",
    "start": "extension start",
    "build": "extension build"
  },
  "devDependencies": {
    "extension": "latest"
  }
}
```

這些腳本是預設的 Extension.js 指令。首次執行擴充功能:

```bash npm
npm run dev
```

```bash pnpm
pnpm run dev
```

```bash yarn
yarn run dev
```

```bash bun
bun run dev
```

```bash bun
bun run dev
```

如果設定正確,Extension.js 會以全新的 profile 啟動 Chrome,將 `github-search` 作為未封裝擴充功能載入,並在終端機中印出就緒橫幅。Chrome 網址列現在會將 `gh` 識別為關鍵字。

輸入 `gh` 加一個空格,接著輸入 `extension.js`,按下 Enter。會開啟一個新分頁前往 `https://github.com/search?q=extension.js&type=issues`。

## 步驟 5:讓它變得更好

透過加入一個 Omnibox 輸入監聽器,直接在網址列中提供建議來改善搜尋體驗。

更新 `service_worker.js`,在輸入時抓取 GitHub 建議並即時顯示。

```js title="service_worker.js"
// Create a debounce function to avoid excessive
// calls to the GitHub API while the user is still
// typing the search query.
function debounce(fn, delay) {
  let timeoutID;
  return function (...args) {
    if (timeoutID) clearTimeout(timeoutID);
    timeoutID = setTimeout(() => fn(...args), delay);
  };
}

// When the user has changed what is typed into the omnibox.
chrome.omnibox.onInputChanged.addListener(
  debounce(async (text, suggest) => {
    const response = await fetch(
      `https://api.github.com/search/issues?q=${text}`,
    );
    const data = await response.json();
    const suggestions = data.items.map((issue) => ({
      content: issue.html_url,
      description: issue.title,
    }));

    suggest(suggestions);
  }, 250),
);

// When the user has accepted what is typed into the omnibox.
chrome.omnibox.onInputEntered.addListener((text) => {
  // Convert any special character (spaces, &, ?, etc)
  // into a valid character for the URL format.
  const encodedSearchText = encodeURIComponent(text);
  const url = `https://github.com/search?q=${encodedSearchText}&type=issues`;

  chrome.tabs.create({ url });
});
```

這段程式碼直接在網址列中加入來自 GitHub 的即時建議。
