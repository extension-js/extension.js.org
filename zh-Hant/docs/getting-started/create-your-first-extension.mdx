你將打造一個 Omnibox（網址列）快捷功能。在網址列輸入 `gh`、再輸入查詢字串，就會落到 GitHub 搜尋結果頁。過程中你會接好一份 `manifest.json`，並在背景 service worker 處理輸入。同時你也會練習每個專案都會用到的開發迴圈（`create` → `dev` → `build`）。

## 你會建立什麼

| 能力                       | 你會得到什麼                                              |
| -------------------------- | --------------------------------------------------------- |
| Omnibox keyword 流程        | 從瀏覽器網址列輸入 `gh` 就能觸發擴充功能行為                  |
| 背景事件處理                | 透過 service worker 處理使用者輸入                          |
| 本機開發迴圈                | 快速執行、載入並驗證擴充功能行為                            |
| 漸進增強                    | 基本流程完成後,再加上 GitHub 即時建議                       |

## 計畫

讓 GitHub 搜尋像原生瀏覽器快捷功能一樣快。擴充功能會保留關鍵字 `gh`;輸入 `gh` 加上查詢字串後,就會開啟 GitHub 搜尋結果。

## 步驟 1:建立擴充功能

使用 Extension.js 的 `create` 指令建立一個名為 `github-search` 的最小擴充功能鷹架。

<CodeGroup>

```bash npm
npx extension@latest create github-search
```

```bash pnpm
pnpx extension@latest create github-search
```

```bash yarn
yarn dlx extension@latest create github-search
```

```bash bun
bunx extension@latest create github-search
```

```bash bun
bunx extension@latest create github-search
```

</CodeGroup>

每個擴充功能都從一份 manifest 檔開始,它定義中繼資料、權限與執行階段檔案。依據[上方的計畫](#the-plan),設定 `gh` 快捷關鍵字,並為使用者事件加上一個 service worker。

```json
{
  "manifest_version": 3,
  "name": "GitHub Search",
  "version": "1.0",
  "omnibox": { "keyword": "gh" },
  "background": {
    "service_worker": "service_worker.js"
  }
}
```

- `omnibox.keyword`:輸入 `gh` 時,瀏覽器會觸發事件。
- `background.service_worker`:監聽你觸發的事件。

## 步驟 3:建立背景 service worker

在瀏覽器擴充功能中,背景 service worker（獨立於任何可見頁面執行的指令稿）負責處理瀏覽器事件。

在這個範例,加入一段指令稿來監聽 Omnibox 輸入,並把查詢字串導向 GitHub 搜尋。

建立 `service_worker.js`：

```js
// When the user has accepted what is typed into the omnibox.
chrome.omnibox.onInputEntered.addListener((text) => {
  // Convert any special character (spaces, &, ?, etc)
  // into a valid character for the URL format.
  const encodedSearchText = encodeURIComponent(text);
  const url = `https://github.com/search?q=${encodedSearchText}&type=issues`;

  chrome.tabs.create({ url });
});
```

上述指令稿會在你在網址列輸入「gh」後再輸入任何文字時,開啟新分頁顯示 GitHub 搜尋結果。

## 步驟 4:載入你的擴充功能

你的 `package.json` 現在會長這樣：

```json
{
  "scripts": {
    "dev": "extension dev",
    "start": "extension start",
    "build": "extension build"
  },
  "devDependencies": {
    "extension": "latest"
  }
}
```

這些腳本是 Extension.js 的預設指令。第一次執行擴充功能：

<CodeGroup>

```bash npm
npm run dev
```

```bash pnpm
pnpm run dev
```

```bash yarn
yarn run dev
```

```bash bun
bun run dev
```

```bash bun
bun run dev
```

如果你的設定正確,Extension.js 會以全新設定檔啟動 Chrome、將 `github-search` 載入為未封裝的擴充功能,並在你的終端機顯示就緒橫幅。Chrome 的網址列現在會把 `gh` 認得為關鍵字。

輸入 `gh` 加上空白,接著輸入 `extension.js`,按下 Enter。會開啟新分頁前往 `https://github.com/search?q=extension.js&type=issues`。

## 步驟 5:讓它更好用

在網址列輸入時直接顯示建議,改善搜尋體驗。用 Omnibox input 監聽器來實作。

更新 `service_worker.js`,在輸入時抓取 GitHub 建議並顯示出來。

```js title="service_worker.js"
// Create a debounce function to avoid excessive
// calls to the GitHub API while the user is still
// typing the search query.
function debounce(fn, delay) {
  let timeoutID;
  return function (...args) {
    if (timeoutID) clearTimeout(timeoutID);
    timeoutID = setTimeout(() => fn(...args), delay);
  };
}

// When the user has changed what is typed into the omnibox.
chrome.omnibox.onInputChanged.addListener(
  debounce(async (text, suggest) => {
    const response = await fetch(
      `https://api.github.com/search/issues?q=${text}`,
    );
    const data = await response.json();
    const suggestions = data.items.map((issue) => ({
      content: issue.html_url,
      description: issue.title,
    }));

    suggest(suggestions);
  }, 250),
);

// When the user has accepted what is typed into the omnibox.
chrome.omnibox.onInputEntered.addListener((text) => {
  // Convert any special character (spaces, &, ?, etc)
  // into a valid character for the URL format.
  const encodedSearchText = encodeURIComponent(text);
  const url = `https://github.com/search?q=${encodedSearchText}&type=issues`;

  chrome.tabs.create({ url });
});
```

這段程式碼會在網址列直接加上即時的 GitHub 建議。
