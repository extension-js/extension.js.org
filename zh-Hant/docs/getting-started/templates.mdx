從官方範本建立擴充功能，省去自己搭建資料夾結構與設定的時間，讓你能更快開始。

## 範本能做什麼

- **現成的專案結構：** 一開始就有可運作的 manifest、腳本與原始碼配置。
- **針對特定技術棧：** 依據框架與執行環境的需求選擇範本。
- **更快上手：** 在數分鐘內從點子推進到可執行的擴充功能。
- **一致的預設值：** 降低不同專案與成員之間的設定差異。

## 快速使用

<CodeGroup>

```bash npm
npx extension@latest create my-extension --template=<slug>
```

```bash pnpm
pnpx extension@latest create my-extension --template=<slug>
```


```bash bun

```

## 運作方式

執行 `create --template=<slug>` 時,Extension.js 會抓取所選範本、建立專案檔案,並可選擇安裝相依套件。

## 更多範本

可在範例儲存庫中找到官方範本原始碼:

- [extension-js/examples](https://github.com/extension-js/examples/tree/main/examples)

範本資料夾名稱直接對應 `--template=<slug>`。

## 最佳實務

- 透過範本快速建立你的專案。
- 從你熟悉的技術棧範本開始。
- 選擇最符合專案需求的瀏覽器擴充功能 context。
