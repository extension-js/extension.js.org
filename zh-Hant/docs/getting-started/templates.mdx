從官方範本建立擴充功能，而不是從零開始建構資料夾結構與設定，讓起步更快。

## 範本能力

- **就緒的專案結構：** 從可運作的 manifest、腳本與來源佈局開始。
- **特定技術堆疊的設定：** 依框架與執行階段需求選擇範本。
- **更快入門：** 在幾分鐘內從構想轉變為可執行的擴充功能。
- **一致的預設值：** 減少跨專案與團隊成員之間的設定差異。

## 快速使用

<CodeGroup>

```bash npm
npx extension@latest create my-extension --template=<slug>
```

```bash pnpm
pnpx extension@latest create my-extension --template=<slug>
```

```bash yarn
yarn dlx extension@latest create my-extension --template=<slug>
```

```bash bun
bunx extension@latest create my-extension --template=<slug>
```

```bash bun
bunx extension@latest create my-extension --template=<slug>
```

</CodeGroup>

將 `<slug>` 替換為你要開始的範本。

## 運作方式

當你執行 `create --template=<slug>` 時，Extension.js 會擷取所選範本、建立專案檔案，並可選擇性地安裝相依。
