瀏覽器擴充功能是一種為瀏覽器增加功能的軟體。擴充功能可以改變頁面的呈現方式,也可以在瀏覽器 chrome 中加入新的 UI:工具列、popup、側邊欄與新分頁。它們也可以回應網路與分頁事件,並呼叫一般網頁無法使用的特權瀏覽器 API。

Extension.js 是一個 JavaScript 框架,可從單一專案跨 Chrome、Firefox 與 Edge 建構瀏覽器擴充功能。

## 瀏覽器擴充功能能做什麼

確切的 API 範圍因瀏覽器而異,但大多數擴充功能會做以下其中一項或多項:

- **變更頁面內容。** 將 CSS 或腳本注入到特定 URL(內容腳本)。
- **加入瀏覽器 UI。** 從工具列 action 渲染 popup、側邊欄、選項頁或自訂新分頁。
- **回應瀏覽器事件。** 監聽分頁更新、導覽、下載、鬧鐘,或來自頁面的訊息。
- **儲存資料。** 讀寫橫跨工作階段的擴充功能範圍儲存。
- **呼叫特權 API。** 管理 cookie、宣告式網路請求、書籤、歷史記錄、下載或通知,受 `permissions` 控管。

擴充功能不能做使用者未授權的事情,瀏覽器也會將擴充功能沙箱化,使其遠離原始作業系統存取。

## 瀏覽器擴充功能檔案

每個瀏覽器擴充功能都是靜態資源與程式碼的資料夾,並經過簽章與打包以便發佈:

| 檔案或資料夾    | 用途                                                                          |
| --------------- | ----------------------------------------------------------------------------- |
| `manifest.json` | 宣告擴充功能的名稱、版本、權限、進入點與後設資料。                            |
| 背景腳本        | 長期存活的事件處理器。Manifest V3 在 Chromium 上使用 service worker。         |
| 內容腳本        | 注入到符合條件網頁中的程式碼。                                                |
| HTML 頁面       | popup、options、側邊欄或新分頁 UI。                                           |
| 圖示與資源      | 工具列圖示、地區字串及其他靜態資源。                                          |

Extension.js 會將你的原始碼(`.ts`、`.tsx`、`.jsx`、`.vue`、`.svelte`、CSS、Less、Sass)編譯為這樣的磁碟結構,並依瀏覽器目標產生獨立資料夾。

## 擴充功能中的 JavaScript、TypeScript、HTML 與 CSS

瀏覽器擴充功能用與 Web 相同的語言建構。JavaScript 與 TypeScript 驅動邏輯。HTML 定義擴充功能頁面。CSS(或 Less、Sass、CSS modules、Tailwind)為它們加上樣式。Extension.js 透過 Rspack 並以對擴充功能感知的預設值將這些串起來,讓你不必為每個瀏覽器設定打包工具。參見 [瀏覽器擴充功能中的 JavaScript 與 TypeScript 檔案](/docs/concepts/javascript-typescript-browser-extension-files) 了解哪個副檔名落在哪裡。

## Chrome 擴充功能 vs Firefox 擴充功能

Chrome 與 Firefox 擴充功能共用 WebExtensions API,但每次建構時都有幾項差異需要留意:

- **背景:** Chrome(Manifest V3)需要 `background.service_worker`。Firefox 使用帶有非持久化事件頁面的 `background.scripts`。
- **Runtime API:** Firefox 原生公開 `browser.*`。Chromium 使用 `chrome.*`。可在 Chromium 上使用 `--polyfill` 來消弭差異。
- **權限:** 瀏覽器對幾個 `permissions` 與 `host_permissions` 條目的解讀有些不同。
- **發佈:** Chrome 透過 Chrome Web Store 發佈,Firefox 透過 addons.mozilla.org,Edge 透過 Edge Add-ons store。

Extension.js 透過 [瀏覽器特定的 manifest 欄位](/docs/features/browser-specific-fields) 與依目標的建構輸出來處理這些差異。參見 [跨瀏覽器相容性](/docs/features/cross-browser-compatibility) 了解完整流程。

## Extension.js 如何協助你

沒有框架時,建構瀏覽器擴充功能意味著手寫 `manifest.json` 並為每個瀏覽器設定打包工具。你還要手動串接存檔即重新載入、polyfill API,並為 Chrome、Firefox 與 Edge 產出獨立成品。Extension.js 會替你完成這些:

- 一個專案、一個 `manifest.json`、獨立的 `dist/<browser>` 輸出。
- TypeScript、React、Vue、Svelte、Preact 與現代 CSS 無須設定即可運作。
- `extension dev` 在存檔時重新載入擴充功能、內容腳本與 HTML 頁面。
- `extension build` 依瀏覽器目標產出可簽署發佈的成品。
