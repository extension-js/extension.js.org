瀏覽器擴充功能使用 JavaScript、TypeScript、HTML、CSS 與 `manifest.json` 來定義行為、UI、權限以及瀏覽器整合。本頁說明你在瀏覽器擴充功能專案中會看到的副檔名,以及它們各自落在哪裡。

如果你尋找的是作為 Chrome 或 Firefox 軟體附加元件的「擴充功能」,請改從 [什麼是瀏覽器擴充功能?](/docs/concepts/what-is-a-browser-extension) 開始。

## JavaScript 副檔名:`.js` 與 `.mjs`

瀏覽器擴充功能接受與其他 Web 平台相同的 JavaScript 副檔名:

| 副檔名 | 意義                                                                                |
| ------ | ----------------------------------------------------------------------------------- |
| `.js`  | 一般 JavaScript。依載入方式被視為 ES 模組或傳統腳本。                                |
| `.mjs` | ES 模組。當你想強制使用模組語意時很有用,特別是在 service worker 中。               |
| `.cjs` | CommonJS。在擴充功能原始碼中很少見;有時出現在 `extension.config.cjs` 中。          |

在 Manifest V3 中,當 `manifest.json` 在 `background` 區塊中包含 `"type": "module"` 時,背景的 `service_worker` 會以模組方式執行。詳見 [Manifest V3 疑難排解](/docs/concepts/manifest-v3)。

## TypeScript 副檔名:`.ts` 與 `.tsx`

TypeScript 在 Extension.js 中是一等的原始語言:

| 副檔名  | 意義                                                |
| ------- | --------------------------------------------------- |
| `.ts`   | 不含 JSX 的 TypeScript。                            |
| `.tsx`  | 含 JSX 的 TypeScript(React、Preact)。             |
| `.d.ts` | 型別宣告檔。不會被輸出到建構結果。                  |

你不需要從零撰寫 `tsconfig.json`。Extension.js 已內建合理預設值。`chrome.*`、`browser.*`、`import.meta.env` 以及公開環境變數鍵的型別來自 `@types/chrome` 與 Extension.js 自身的環境型別。參見 [TypeScript](/docs/languages-and-frameworks/typescript)。

## React 副檔名:`.jsx` 與 `.tsx`

瀏覽器擴充功能中的 React 使用標準的 JSX 副檔名:

| 副檔名 | 意義                       |
| ------ | -------------------------- |
| `.jsx` | 含 JSX 的 JavaScript。     |
| `.tsx` | 含 JSX 的 TypeScript。     |

React 可在擴充功能頁面(popup、options、側邊欄、新分頁)中使用,也可在注入到網頁的內容腳本中使用。參見 [React](/docs/languages-and-frameworks/react) 了解設定與 shadow-DOM 模式。

## 瀏覽器擴充功能檔案:`manifest.json`、背景、內容腳本、頁面

除了 JavaScript 與 TypeScript 原始碼之外,瀏覽器擴充功能資料夾通常還包含:

| 檔案或資料夾     | 用途                                                              |
| ---------------- | ----------------------------------------------------------------- |
| `manifest.json`  | 宣告名稱、版本、權限、進入點與後設資料。                          |
| 背景進入點       | 長期存活的事件處理器。在 Chromium 上為 Manifest V3 service worker。|
| 內容腳本         | 注入到符合 URL 樣式的網頁中的程式碼。                             |
| 擴充功能頁面     | popup、options、側邊欄或新分頁 HTML。                             |
| 地區設定         | `_locales/<lang>/messages.json` 用於翻譯字串。                    |
| 圖示與資源       | 工具列圖示、Web 可存取資源、字型、圖片。                          |

Extension.js 會把你的 `.ts`、`.tsx`、`.jsx`、`.vue`、`.svelte`、`.css`、`.less`、`.scss` 與 `.module.*` 原始碼編譯成這樣的磁碟結構。它會依瀏覽器目標產生獨立資料夾(`dist/chrome`、`dist/firefox`、`dist/edge`)。

## Extension.js 如何編譯 JavaScript 與 TypeScript 擴充功能

當你執行 `extension dev` 或 `extension build` 時,Extension.js 會:

1. 讀取 `manifest.json` 並找出每個進入點(背景、內容腳本、popup、options、側邊欄、新分頁、Web 可存取的 HTML)。
2. 解析這些進入點所參照的原始檔,包含跨 `.ts`、`.tsx`、`.jsx`、`.vue`、`.svelte` 與樣式類型的匯入。
3. 透過 Rspack 編譯,並使用對擴充功能感知的預設值:在有幫助的地方進行程式碼分割,在瀏覽器拒絕的地方則不分塊(service worker、內容腳本)。
4. 將結果輸出到 `dist/<browser>`,並使用為該目標過濾後的 manifest。

你可以用上述任何一種副檔名撰寫原始碼。Extension.js 會處理打包、polyfill、存檔即重新載入流程,以及依瀏覽器打包。
