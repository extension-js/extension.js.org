透過兩個標準的 AI 可存取介面取得 Extension.js 文件。這能讓你的編輯器與助理依據最新文件作答，而不是用過期的訓練資料。

<Note>
  本頁主要在於把**文件**提供給助理——回答「我該怎麼…」之類的問題。如果你想讓助理**操控工具鏈**（執行 dev session、串流日誌、檢查 DOM、觸發動作、重新載入），請改參考 [Chrome DevTools MCP 與 Extension.js](/zh-Hant/docs/integrations/chrome-devtools-mcp) 這個獨立的控制伺服器。
</Note>

## 代管的 MCP（Model Context Protocol）伺服器

文件網站在以下位址提供 [Model Context Protocol](https://modelcontextprotocol.io) 伺服器：

```
https://extensionjs.mintlify.app/mcp
```

把任何相容於 MCP 的用戶端指向這個 URL，就能取得搜尋文件、抓取特定頁面以及依據最新內容回答問題的工具。

### 一鍵安裝

每個頁面右上角都有情境選單，提供常見用戶端的安裝按鈕:**Claude**、**Cursor**、**VS Code** 與 **ChatGPT**。這是最快的方式。

### 手動安裝

<CodeGroup>

```json Claude Desktop (claude_desktop_config.json)
{
  "mcpServers": {
    "extensionjs": {
      "url": "https://extensionjs.mintlify.app/mcp"
    }
  }
}
```

```json Cursor (.cursor/mcp.json)
{
  "mcpServers": {
    "extensionjs": {
      "url": "https://extensionjs.mintlify.app/mcp"
    }
  }
}
```

```jsonc VS Code (settings.json)
"mcp.servers": {
  "extensionjs": {
    "url": "https://extensionjs.mintlify.app/mcp"
  }
}
```

```bash Claude Code
claude mcp add --transport http extensionjs https://extensionjs.mintlify.app/mcp
```

</CodeGroup>

安裝完後，可以問助理像 _「我要怎麼在 Extension.js 設定瀏覽器專屬的 manifest 欄位？」_ 之類的問題。助理會查詢 MCP 工具並引用文件頁面。

## llms.txt

文件提供一個靜態、機器友善的索引：

- [`/llms.txt`](https://extensionjs.mintlify.app/llms.txt):所有頁面的簡短索引
- [`/llms-full.txt`](https://extensionjs.mintlify.app/llms-full.txt):可供擷取使用的完整內容包

當你想把文件直接餵給檢索流程，或在自訂代理人中產生摘要時，這些檔案非常實用。Extension.js 在每次文件部署時都會重新產生。

## 每頁的 AI 操作

頁面層級的情境選單還提供：

- **複製頁面:** 將目前頁面以 Markdown 複製,方便貼到聊天視窗
- **以 Markdown 檢視:** 在新分頁開啟原始 Markdown 來源
- **詢問 ChatGPT / Claude / Perplexity:** 開啟對應的助理,並預先載入頁面作為情境

## 最佳實踐

- 當你希望助理跨多個頁面進行推理並保持答案即時更新時，優先使用 MCP 伺服器。
- 如果是離線工具、評估或自訂的 RAG（檢索增強生成）流程，優先使用 `llms-full.txt`。
- 若只是針對目前在閱讀的頁面快速提問，請使用每頁的操作。
