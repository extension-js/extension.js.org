將 Extension.js 與現代網頁專案常用的格式化、Lint、CSS 工具與 UI 元件工作流程串接。

## 依需求選擇

| 需求                                                   | 從這裡開始                                                    |
| ------------------------------------------------------ | ------------------------------------------------------------- |
| 在 SWC（Speedy Web Compiler）之外加入自訂轉換          | [Babel](/docs/integrations/babel)                             |
| 強制程式碼品質規則                                     | [ESLint](/docs/integrations/eslint)                           |
| 保持格式化一致                                         | [Prettier](/docs/integrations/prettier)                       |
| 透過外掛轉換 CSS                                       | [PostCSS](/docs/integrations/postcss)                         |
| 在持續整合（CI）中為樣式表進行 lint                    | [Stylelint](/docs/integrations/stylelint)                     |
| 建構可組合的 UI primitive                              | [shadcn/ui](/docs/integrations/shadcnui)                      |
| Utility-first 樣式系統                                 | [Tailwind CSS](/docs/integrations/tailwindcss)                |
| 透過 Chrome DevTools 與 AI agent 一起對擴充功能除錯    | [Chrome DevTools MCP](/docs/integrations/chrome-devtools-mcp) |

## 實務整合順序

1. 加入基線品質工具：ESLint + Prettier。
2. 加入樣式工具：PostCSS + Stylelint（如有需要再加入 Tailwind）。
3. 僅在需要時加入框架特定工具。

## 下一步

- 在 [擴充功能設定](/docs/features/extension-configuration) 中對齊專案預設值。
- 透過 [工作流程](/docs/workflows/index) 驗證變更。
