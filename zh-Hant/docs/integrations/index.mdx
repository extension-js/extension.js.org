將 Extension.js 與現代 web 專案中使用的格式化、檢查、CSS 工具和 UI 元件工作流程連結起來。

## 依需求選擇

| 需求                                                  | 從這裡開始                                                    |
| ----------------------------------------------------- | ------------------------------------------------------------- |
| 加入超出 SWC(Speedy Web Compiler)的自訂轉換         | [Babel](/docs/integrations/babel)                             |
| 強制執行程式碼品質規則                                | [ESLint](/docs/integrations/eslint)                           |
| 保持格式化一致                                        | [Prettier](/docs/integrations/prettier)                       |
| 使用外掛轉換 CSS                                      | [PostCSS](/docs/integrations/postcss)                         |
| 在持續整合(CI)中檢查樣式表                          | [Stylelint](/docs/integrations/stylelint)                     |
| 建構可組合的 UI 原語                                  | [shadcn/ui](/docs/integrations/shadcnui)                      |
| 工具優先的樣式系統                                    | [Tailwind CSS](/docs/integrations/tailwindcss)                |
| 透過 Chrome DevTools 與 AI agent 偵錯擴充功能         | [Chrome DevTools MCP](/docs/integrations/chrome-devtools-mcp) |

## 實用的整合順序

1. 加入基礎品質工具:ESLint + Prettier。
2. 加入樣式工具:PostCSS + Stylelint(如需要,加上 Tailwind)。
3. 僅在需要時加入框架特定的工具。

## 後續步驟

- 在[擴充功能設定](/docs/features/extension-configuration)中對齊專案預設值。
- 使用[工作流程](/docs/workflows/index)驗證變更。
