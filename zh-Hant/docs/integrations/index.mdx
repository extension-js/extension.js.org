把 Extension.js 與現代網頁專案中使用的格式化、lint、CSS 工具與 UI 元件工作流程連結起來。

## 依需求選擇

| 需求                                                   | 從這裡開始                                                    |
| ------------------------------------------------------ | ------------------------------------------------------------- |
| 加入 SWC(Speedy Web Compiler)以外的自訂轉換          | [Babel](/zh-Hant/docs/integrations/babel)                     |
| 強制套用程式碼品質規則                                 | [ESLint](/zh-Hant/docs/integrations/eslint)                   |
| 維持一致的格式                                         | [Prettier](/zh-Hant/docs/integrations/prettier)               |
| 使用 plugin 轉換 CSS                                   | [PostCSS](/zh-Hant/docs/integrations/postcss)                 |
| 在持續整合(CI)中對樣式表 lint                        | [Stylelint](/zh-Hant/docs/integrations/stylelint)             |
| 構建可組合的 UI 元件                                   | [shadcn/ui](/zh-Hant/docs/integrations/shadcnui)              |
| 工具優先(utility-first)的樣式系統                    | [Tailwind CSS](/zh-Hant/docs/integrations/tailwindcss)        |
| 透過 Chrome DevTools 與 AI agent 一起除錯擴充功能      | [Chrome DevTools MCP](/zh-Hant/docs/integrations/chrome-devtools-mcp) |

## 實務上的整合順序

1. 加入基本品質工具:ESLint + Prettier。
2. 加入樣式工具:PostCSS + Stylelint(以及需要時加入 Tailwind)。
3. 只在需要時加入框架專屬工具。

## 下一步

- 在 [擴充功能設定](/zh-Hant/docs/features/extension-configuration) 中對齊專案預設值。
- 用 [工作流程](/zh-Hant/docs/workflows/index) 驗證變更。
