跨 popup、options、new tab、sidebar 與 content-script 入口,用 utility class 快速建構擴充功能的 UI 介面。

Extension.js 透過 PostCSS 整合 Tailwind,並同時支援 Tailwind v3 與 v4 的設定。

## 何時適合用 Tailwind CSS

- 你需要在多個擴充功能介面上快速迭代 UI。
- 你偏好以共用 token 為基礎的 utility-first 樣式。
- 你想在頁面與 content-script 入口之間維持一致的樣式模式。

## Tailwind 能力

| 能力                       | 它能為你提供什麼                                                             |
| -------------------------- | ---------------------------------------------------------------------------- |
| 多介面樣式                 | 在擴充功能頁面與 content script 之間重用 utility class                       |
| 以 PostCSS 為基礎的整合    | 讓 Tailwind 與專案其他部分使用同一條 CSS 轉換 pipeline                       |
| 版本彈性                   | 支援 Tailwind v3 與 Tailwind v4 的 plugin 樣式                               |
| 範本入門                   | 從現成的設定起步,並逐步調整                                                 |

## 與既有擴充功能搭配使用

### 安裝

安裝必要的相依套件:

<CodeGroup>

```bash npm
npm install -D tailwindcss @tailwindcss/postcss postcss
```

```bash pnpm
pnpm install -D tailwindcss @tailwindcss/postcss postcss
```

```bash yarn
yarn add -D tailwindcss @tailwindcss/postcss postcss
```

```bash bun
bun add --dev tailwindcss @tailwindcss/postcss postcss
```

```bash bun
bun add --dev tailwindcss @tailwindcss/postcss postcss
```

</CodeGroup>

### 設定

建立 `postcss.config.js`:

```js
export default {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
```

對於 Tailwind v3 專案,請使用 `tailwindcss` 作為 PostCSS plugin,並維持 `tailwind.config.js` 的 content 列表。

### Tailwind v4 與 v3 快速設定

```js
// Tailwind v4 postcss.config.js
export default {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
```

```js
// Tailwind v3 postcss.config.js
export default {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
};
```

### Tailwind content 路徑(v3 風格)

```js
/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./pages/**/*.{html,js,ts,jsx,tsx,mdx}",
    "./scripts/**/*.{js,ts,jsx,tsx}",
    "./src/**/*.{html,js,ts,jsx,tsx,mdx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
};
```

### 使用方式

建立全域樣式表並 import Tailwind:

#### 建立全域 CSS 檔

```css
/* styles/globals.css */
@import "tailwindcss";
```

在使用 Tailwind class 的入口腳本中 import 這個檔案。

#### 在頁面腳本中(popup/options/new-tab)

```tsx
import "./styles/globals.css";

export default function MyNewTabPage() {
  return <h1 className="text-4xl font-bold">Hello, Extension.</h1>;
}
```

#### 在 content script 中

```ts
import "./styles/globals.css";

console.log("Tailwind styles loaded for content script");
```

Extension.js 會把 content-script 的 CSS 透過其 content-script 樣式 pipeline 處理,並把這些資產輸出到 `content_scripts/` 下。

## 最佳實務

- 將 Tailwind 來源 import 放在共用的樣式入口檔(`styles/globals.css`)中。
- 確保你的 content 掃描在相關時包含 `pages/`、`scripts/` 與 `src/`。
- 在 monorepo 中,確認 Tailwind 的 content 路徑是從擴充功能專案根目錄解析的。
- 盡量使用範本,以避免 PostCSS 與 Tailwind 版本之間的設定漂移。
