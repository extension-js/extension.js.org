以單一 pipeline 在擴充功能頁面與 content scripts 之間一致地轉換 CSS。當您需要 Autoprefixer 等外掛或現代 CSS 轉換時，請使用 PostCSS。

Extension.js 會偵測 PostCSS 設定並在需要時套用 `postcss-loader`。

## PostCSS 何時合適

- 您需要在擴充功能樣式中加入自動加前綴或現代 CSS 轉換。
- 您希望在 popup/options/sidebar/content scripts 之間有一致的 CSS 轉換 pipeline。
- 您使用 Tailwind 或以外掛為基礎的樣式處理。

## PostCSS 能力

| 能力                      | 為您提供的功能                                                              |
| ------------------------- | --------------------------------------------------------------------------- |
| 共享的 CSS pipeline       | 在 popup、options、new tab 與 content scripts 之間使用相同的轉換            |
| 彈性的設定探索            | 從專用檔案或 `package.json` 載入 PostCSS 設定                               |
| Tailwind 相容性           | 透過 PostCSS 外掛與 Tailwind 設定整合                                       |
| 以外掛為基礎的轉換        | 加入 `autoprefixer` 與 `postcss-preset-env` 等工具                          |

## 偵測與設定檔

Extension.js 會檢查下列 PostCSS 設定檔：

- `.postcssrc`
- `.postcssrc.json`
- `.postcssrc.yaml`
- `.postcssrc.yml`
- `postcss.config.mjs`
- `.postcssrc.js` / `.postcssrc.cjs`
- `postcss.config.js` / `postcss.config.cjs`

它也會辨識 `package.json` 中的 `postcss` 設定。

## 在既有擴充功能中使用 PostCSS

安裝 PostCSS 相依套件：

<PackageManagerTabs command="install -D postcss postcss-loader autoprefixer" />

### 建立 `postcss.config.js`

```js
export default {
  plugins: {
    autoprefixer: {},
  },
};
```

## 用法

在擴充功能頁面中正常匯入樣式：

```ts
import "./styles/globals.css";
```

對於 content scripts，在 content-script 進入點中以相同方式匯入：

```ts
import "./styles/content.css";
```

Extension.js 會在兩種情境下套用 PostCSS，但輸出方式不同（頁面採用 `css` pipeline，content scripts 採用 asset pipeline）。

## 與 Tailwind 互動

- Extension.js 可在偵測到 Tailwind 時自動觸發 PostCSS 設定。
- Extension.js 同時支援 Tailwind v3 與 v4，並依偵測到的版本選擇外掛。
- 在採用不相容 CommonJS（CJS）PostCSS 設定樣式的 ECMAScript Module（ESM）專案中，Extension.js 可能會繞過您的設定，改為注入相容性外掛。

## 最佳實務

- 保持外掛鏈小巧且明確（例如 `autoprefixer`、`postcss-preset-env`）。
- 確認設定格式（`.mjs`/`.cjs`）與您的專案模組類型相符。
- 保持 Tailwind 與 PostCSS 版本對齊，以避免外掛解析不符。
- 偏好共享的樣式進入點檔案，以在擴充功能表面間有可預測的處理。
