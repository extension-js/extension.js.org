在擴充功能介面之間保持樣式表品質一致,並盡早捕捉 CSS 問題。Stylelint 透過指令稿或持續整合(CI)執行,不在 Extension.js 的 dev/建置管線中執行。

## 何時適合使用 Stylelint

- 你在多個擴充功能介面之間維護共享的 CSS/Sass/Less。
- 你希望在 pull request 合併前進行樣式品質檢查。
- 你在團隊工作流程中需要一致的樣式表慣例。

## Stylelint 能力

| 能力                  | 它能為你提供什麼                                       |
| --------------------- | ------------------------------------------------------ |
| CSS 品質檢查          | 捕捉無效模式並強制執行樣式慣例                         |
| 預處理器支援          | 在一個工作流程中檢查 CSS、Sass 和 Less                 |
| 指令稿與 CI 整合      | 在合併和發布前執行樣式檢查                             |
| 與格式化器對齊        | 與 Prettier 結合以減少樣式工具衝突                     |

## 與既有擴充功能一起使用

安裝 Stylelint 相依套件:

<PackageManagerTabs command="install -D stylelint stylelint-config-standard-scss" />

### Stylelint 設定

在專案根目錄建立 Stylelint 設定檔(例如 `.stylelintrc.json`):

```json
{
  "extends": "stylelint-config-standard-scss",
  "rules": {
    "color-no-invalid-hex": true,
    "block-no-empty": true
  }
}
```

### 執行 Stylelint

<CodeGroup>

```bash npm
npm run lint:styles
```

```bash pnpm
pnpm lint:styles
```

```bash yarn
yarn lint:styles
```

```bash bun
bun run lint:styles
```

```bash bun
bun run lint:styles
```

</CodeGroup>

## 與 Prettier 整合

將 Stylelint 與 Prettier 整合,以同時格式化和檢查樣式。安裝以下相依套件:

<PackageManagerTabs command="install -D stylelint-prettier prettier" />

然後,更新你的 `.stylelintrc.json` 檔案以包含 Prettier:

```json
{
  "extends": [
    "stylelint-config-standard-scss",
    "stylelint-prettier/recommended"
  ]
}
```

這能讓樣式格式化和檢查保持一致。

## 最佳實務

- 保留專屬的 `lint:styles` 命令並在 CI 中執行。
- 在 monorepo 中跨擴充功能套件使用一份共享設定。
- 將 Stylelint 與 Prettier 搭配,讓樣式格式化保持確定性。

## 偵測說明

Extension.js 可以偵測 Stylelint 設定檔用於診斷/報告,但它不會將 Stylelint 注入到建置管線中。
