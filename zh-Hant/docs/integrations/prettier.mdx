以最少的手動工作在整個擴充功能程式碼庫中維持一致的格式化。Prettier 從 scripts、編輯器整合與持續整合（CI）執行。它不會在 Extension.js 建置 pipeline 內執行。

## Prettier 何時合適

- 您希望在本機與 CI 工作流程中具備確定性的格式化。
- 您正在加入新貢獻者，需要低摩擦的風格一致性。
- 您希望 ESLint 專注於正確性，而 Prettier 處理格式化。

## Prettier 能力

| 能力                       | 為您提供的功能                                            |
| -------------------------- | --------------------------------------------------------- |
| 一致的格式化               | 為 JS、TS、JSON、CSS 與 Markdown 共用同一種風格           |
| Script 與編輯器流程        | 在本機儲存時格式化，並在 CI 中強制執行                    |
| 減少 lint 衝突             | 與 `eslint-config-prettier` 搭配以避免規則重疊            |
| 低維護的預設值             | 以最少設定採用穩定的基線                                  |

## 在既有擴充功能中使用

安裝 Prettier：

<PackageManagerTabs command="install -D prettier" />

建立設定檔（例如 `.prettierrc`）：

```json
{
  "semi": false,
  "singleQuote": true,
  "trailingComma": "none",
  "printWidth": 80,
  "tabWidth": 2,
  "arrowParens": "always"
}
```

從 scripts 或手動執行 Prettier：

<CodeGroup>

```bash npm
npm run format
```

```bash pnpm
pnpm format
```

```bash yarn
yarn format
```

```bash bun
bun run format
```

```bash bun
bun run format
```

</CodeGroup>

若尚未有 format script：

## 與 ESLint 整合

為避免 ESLint 中的格式化規則衝突，請安裝：

<PackageManagerTabs command="install -D eslint-config-prettier" />

然後在 ESLint 設定鏈中加入 `eslint-config-prettier`，讓 ESLint 將格式化議題交給 Prettier 處理。

## 最佳實務

- 讓格式化在編輯器與 CI 中自動進行，而非手動。
- 在 CI 中使用 `--check`，在本機使用 `--write`。
- 在較大的儲存庫中，於 pre-commit 階段格式化暫存區檔案。
