在問題進入執行階段前讓擴充功能程式碼品質可預測。ESLint 在建置 pipeline 之外執行。請從您的 scripts 或持續整合（CI）執行它。

## ESLint 何時合適

- 您希望團隊貢獻具一致的程式碼標準。
- 您需要在 CI 上盡早捕捉常見 JS/TS 錯誤的檢查。
- 您在發佈前強制執行擴充功能特定的品質規則。

## ESLint 能力

| 能力                              | 為您提供的功能                                                            |
| --------------------------------- | ------------------------------------------------------------------------- |
| JavaScript 與 TypeScript lint     | 捕捉常見錯誤並強制執行專案標準                                            |
| Flat config 支援                  | 在新專案中使用現代的 `eslint.config.mjs`                                  |
| 對 CI 友善的執行方式              | 在本機 scripts 與 pull request pipeline 中執行 lint 檢查                  |
| 擴充功能特定的規則集              | 在團隊需要更嚴格的檢查時加入瀏覽器擴充功能外掛/規則                       |

## 在既有擴充功能中使用

安裝核心套件：

<PackageManagerTabs command="install -D eslint @eslint/js globals typescript-eslint" />

建立 `eslint.config.mjs`：

```js
import globals from "globals";
import pluginJs from "@eslint/js";
import tseslint from "typescript-eslint";

export default [
  { files: ["**/*.{js,mjs,cjs,ts,tsx,jsx}"] },
  {
    languageOptions: {
      globals: globals.browser,
    },
  },
  pluginJs.configs.recommended,
  ...tseslint.configs.recommended,
];
```

從您的 scripts 執行 lint 命令：

<CodeGroup>

```bash npm
npm run lint
```

```bash pnpm
pnpm lint
```

```bash yarn
yarn lint
```

```bash bun
bun run lint
```

```bash bun
bun run lint
```

</CodeGroup>

## 最佳實務

- 將 lint 規則放入版本控制並在 CI 中套用。
- 使用 lint-staged 或 pre-commit hook 取得更快的本機回饋。
- 僅在需要時加入框架特定的外掛（例如 React）。
