為一個或多個瀏覽器目標建立生產擴充功能產物。

`build` 會以生產模式編譯你的擴充功能,並將輸出寫入 `dist/<browser>`。

對於 monorepo/子模組專案,請參閱 [環境變數](/docs/features/environment-variables#how-it-works) 了解設定階段的環境變數解析(先在專案根目錄,再回退到工作區根目錄)。

## 何時使用 `build`

- 為 Chrome 線上應用程式商店、Edge 附加元件或 Firefox 附加元件準備擴充功能套件。
- 在持續整合 (CI) 工作中產生可重複的生產產物。
- 在送出前驗證生產打包輸出以及瀏覽器目標之間的差異。

## Build 指令能力

| 能力             | 它能提供什麼                                             |
| ---------------- | -------------------------------------------------------- |
| 生產編譯         | 為每個目標產生最佳化後的擴充功能產物                     |
| 多目標輸出       | 一條指令建置多個瀏覽器目標                               |
| 打包支援         | 建立散佈用 zip 產物,可選附帶原始碼套件                  |
| CI 友善          | 在自動化情境下保持建置輸出與命名可預期                   |

## 建置輸出

執行 `build` 之後,Extension.js 會為所選瀏覽器目標產生最佳化後的檔案。輸出位於 `dist/`,每個目標一個子資料夾。每個資料夾包含打包後的 JavaScript、CSS、HTML 以及所需的執行階段資源。

<Note>
  對於 TypeScript 專案,`build` 也會重新產生 `extension-env.d.ts` 全域型別宣告
  (與 [`dev`](/docs/commands/dev) 寫入的是同一個檔案),
  這樣不管你之前是否執行過 `dev`,CI 的 `tsc --noEmit` 都能保持乾淨。
  純 JavaScript 專案會略過此步驟。
</Note>

**範例輸出結構:**

```plaintext
dist/
├── chrome/
│   ├── manifest.json
│   ├── background/service_worker.js
│   ├── content_scripts/content-0.js
├── edge/
│   ├── manifest.json
│   ├── background/service_worker.js
│   ├── content_scripts/content-0.js
```

## 瀏覽器目標矩陣

| 目標類型     | 範例                                             | 說明                               |
| ------------ | ------------------------------------------------ | ---------------------------------- |
| 命名目標     | `chromium`、`chrome`、`edge`、`firefox`          | 建置某個特定的瀏覽器目標           |
| 引擎目標     | `chromium-based`、`gecko-based`、`firefox-based` | 適用於以引擎家族劃分的工作流程     |
| 多目標       | `chrome,firefox`                                 | 以逗號分隔的多個目標               |

## 參數與選項

| 選項                     | 別名            | 作用                                                                                                          | 預設值                   |
| ------------------------ | --------------- | ------------------------------------------------------------------------------------------------------------- | ------------------------ |
| `[path]`                 | -               | 建置一個本機擴充功能專案。                                                                                    | `process.cwd()`          |
| `--browser <browser>`    | -               | 瀏覽器/引擎目標(`chromium`、`chrome`、`edge`、`firefox`、引擎別名或以逗號分隔的多個值)。                   | `chromium`               |
| `--polyfill [boolean]`   | -               | 在 Chromium 目標上啟用 `browser.*` API 相容性 polyfill。                                                      | `false`                  |
| `--zip [boolean]`        | -               | 產生打包後的 zip 產物。                                                                                       | `false`                  |
| `--zip-source [boolean]` | -               | 在 zip 輸出中包含原始檔。                                                                                     | `false`                  |
| `--zip-filename <name>`  | -               | 設定自訂的 zip 檔名。                                                                                         | 擴充功能名稱 + 版本號    |
| `--silent [boolean]`     | -               | 隱藏建置記錄。                                                                                                | `false`                  |
| `--mode <mode>`          | -               | 覆寫打包模式(`development`、`production` 或 `none`),同時設定 `NODE_ENV`。                                  | `production`             |
| `--extensions <list>`    | -               | 以逗號分隔的伴隨擴充功能清單或商店 URL。                                                                      | 未設定                   |
| `--install [boolean]`    | -               | 缺少相依套件時安裝專案相依套件。                                                                              | 指令預設行為             |
| `--author`               | `--author-mode` | 啟用維護者診斷模式。                                                                                          | 停用                     |

## 共用的全域選項

也支援 [全域選項](/docs/workflows/global-flags)。

## 模式覆寫

`--mode` 會覆寫建置使用的打包模式以及 `NODE_ENV`。可選 `development`、`production` 或 `none`。當你需要為預先發布或除錯產生非生產打包時,可以用它來對齊 Vite/webpack 的工作流程。

<CodeGroup>

```bash npm
extension build ./my-extension --mode development
```

```bash pnpm
extension build ./my-extension --mode development
```

```bash yarn
extension build ./my-extension --mode development
```

```bash bun
extension build ./my-extension --mode development
```

```bash bun
extension build ./my-extension --mode development
```

</CodeGroup>

無效值會以錯誤結束;預設值仍然是 `production`。

## Zip 行為

| 選項             | 效果                                  | 典型用途                             |
| ---------------- | ------------------------------------- | ------------------------------------ |
| `--zip`          | 建立打包後的產物 zip                  | 商店送出/手動散佈                    |
| `--zip-filename` | 設定自訂的 zip 名稱                   | CI 命名慣例                          |
| `--zip-source`   | 在產物旁邊附加原始碼壓縮檔            | 合規/審核流水線                      |

## 最佳實務

- **檢查建置記錄:** 每次建置後檢視記錄,留意警告與遺失的資源。
- **最佳化 manifest:** 讓 `manifest.json` 相容所有目標瀏覽器。
- **有意識地命名產物:** 使用 `--zip-filename` 維持 CI 產物命名穩定。
- **驗證目標輸出:** 發布前檢查每個 `dist/<browser>` 資料夾。
