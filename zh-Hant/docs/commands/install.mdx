使用 `install` 將受管理的瀏覽器執行階段加入 Extension.js 快取。

當你想在 `dev`、`build`、`start` 或 `preview` 中使用一致的瀏覽器二進位檔時，這最為實用。支援 Chrome for Testing、Chromium、Firefox 與 Edge。

## 何時使用 `install`

- 你需要在持續整合（CI）、自動化或團隊一致的本機執行中，使用一致且可重複的瀏覽器二進位檔。
- 你想要使用 Chrome for Testing，而不是依賴系統上安裝的任何 Chrome 版本。
- 你正以受管理的 Firefox 或 Edge 執行階段建立跨瀏覽器測試。

## 標準用法

對單一瀏覽器，請使用位置引數形式：

<CodeGroup>

```bash npm
extension install <browser>
```

```bash pnpm
extension install <browser>
```

```bash yarn
extension install <browser>
```

```bash bun
extension install <browser>
```

```bash bun
extension install <browser>
```

</CodeGroup>

僅在需要多個目標、瀏覽器系列或 `all` 時才使用 `--browser`。

## Install 指令功能

| 功能                  | 提供內容                                                                       |
| --------------------- | ------------------------------------------------------------------------------ |
| 受管理瀏覽器快取      | 在 Extension.js 瀏覽器快取下提供穩定的安裝位置                                 |
| 可重複的執行階段      | 為可重複的本機執行與自動化提供一致的二進位檔                                   |
| 跨瀏覽器設定          | 以單一指令流程支援 Chrome、Chromium、Edge 與 Firefox                           |
| 路徑探索              | `--where` 顯示解析後的快取根目錄或瀏覽器專屬安裝路徑                           |

## 引數與旗標

| 旗標／引數                                                                                     | 作用                                                                                | 預設       |
| ---------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------- | ---------- |
| `[browser-name]`                                                                               | 安裝單一瀏覽器目標，例如 `chrome`、`chromium`、`edge` 或 `firefox`                  | `chromium` |
| `--browser <chrome\|chromium\|edge\|firefox\|chromium-based\|gecko-based\|firefox-based\|all>` | 覆寫位置引數的瀏覽器名稱，並支援多目標安裝                                          | 未設定     |
| `--where`                                                                                      | 印出解析後的受管理快取根目錄，或瀏覽器專屬的安裝路徑                                | 停用       |

## 範例

### 安裝 Chrome for Testing

```bash
extension install chrome
```

### 在單一指令中安裝多個目標

```bash
extension install --browser chrome,firefox
```

### 顯示 Chrome 的受管理安裝路徑

```bash
extension install chrome --where
```

## 快取位置

Extension.js 預設將受管理的瀏覽器儲存在穩定的每位使用者快取：

- macOS：`~/Library/Caches/extension.js/browsers`
- Linux：`~/.cache/extension.js/browsers` 或 `$XDG_CACHE_HOME/extension.js/browsers`
- Windows：`%LOCALAPPDATA%\extension.js\browsers`

可透過 `EXT_BROWSERS_CACHE_DIR` 覆寫快取根目錄。

## 最佳實務

- **在 CI 中使用 `install`** 來鎖定一致的瀏覽器二進位檔，而非依賴執行器提供的版本。
- **優先使用 `chrome`** 而非 `chromium`，以取得 Chrome for Testing：它更貼近穩定 Chrome 的行為。
- **使用 `--where`** 在針對受管理瀏覽器撰寫自動化前驗證快取路徑。
- `install` 只會管理 Extension.js 快取內的瀏覽器，不會修改系統的瀏覽器安裝。

## 行為說明

- `chrome` 會安裝 Chrome for Testing，而不是依賴系統的 Google Chrome 應用程式。
- `edge` 在 Linux 上可能需要具特權的互動式工作階段。

## 後續步驟

- 以 [`uninstall`](/docs/commands/uninstall) 移除受管理的瀏覽器。
- 在 [`dev`](/docs/commands/dev) 與 [`start`](/docs/commands/start) 中使用受管理的瀏覽器。
