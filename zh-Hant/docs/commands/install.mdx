使用 `install` 將受管的瀏覽器執行階段加入 Extension.js 快取。

當你想為 `dev`、`build`、`start` 或 `preview` 使用一致的瀏覽器二進位檔時,它最為實用。支援 Chrome for Testing、Chromium、Firefox 與 Edge。

## 何時使用 `install`

- 你需要在持續整合 (CI)、自動化或團隊一致的本機執行中,獲得穩定且可重複的瀏覽器二進位檔。
- 你想使用 Chrome for Testing,而不是依賴系統已安裝的任意 Chrome 版本。
- 你正在以受管的 Firefox 或 Edge 執行階段建立跨瀏覽器測試。

## 標準用法

對於單一瀏覽器,請使用位置引數形式:

<CodeGroup>

```bash npm
extension install <browser>
```

```bash pnpm
extension install <browser>
```

```bash yarn
extension install <browser>
```

```bash bun
extension install <browser>
```

```bash bun
extension install <browser>
```

</CodeGroup>

僅當你需要多個目標、瀏覽器家族或 `all` 時,才使用 `--browser`。

## Install 指令能力

| 能力               | 它能提供什麼                                                               |
| ------------------ | -------------------------------------------------------------------------- |
| 受管瀏覽器快取     | 在 Extension.js 瀏覽器快取中提供穩定的安裝位置                             |
| 可重現的執行階段   | 為可重複的本機執行與自動化提供一致的二進位檔                               |
| 跨瀏覽器搭建       | 用一條指令流為 Chrome、Chromium、Edge 與 Firefox 完成搭建                  |
| 路徑查詢           | `--where` 顯示解析得到的快取根目錄或瀏覽器特定的安裝路徑                   |

## 用法

<CodeGroup>




## 參數與選項

| 選項 / 引數                                                                                    | 作用                                                                               | 預設值     |
| ---------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------- | ---------- |
| `[browser-name]`                                                                               | 安裝單一瀏覽器目標,例如 `chrome`、`chromium`、`edge` 或 `firefox`                 | `chromium` |
| `--browser <chrome\|chromium\|edge\|firefox\|chromium-based\|gecko-based\|firefox-based\|all>` | 覆寫位置引數的瀏覽器名稱,並支援多目標安裝                                         | 未設定     |
| `--where`                                                                                      | 列印解析得到的受管快取根目錄,或瀏覽器特定的安裝路徑                               | 停用       |

## 範例

### 安裝 Chrome for Testing

```bash
extension install chrome
```

### 在一條指令中安裝多個目標

```bash
extension install --browser chrome,firefox
```

### 顯示 Chrome 的受管安裝路徑

```bash
extension install chrome --where
```

## 快取位置

預設情況下,Extension.js 將受管瀏覽器儲存在每位使用者穩定的快取中:

- macOS: `~/Library/Caches/extension.js/browsers`
- Linux: `~/.cache/extension.js/browsers` 或 `$XDG_CACHE_HOME/extension.js/browsers`
- Windows: `%LOCALAPPDATA%\extension.js\browsers`

你可以透過 `EXT_BROWSERS_CACHE_DIR` 覆寫快取根目錄。

## 最佳實務

- **在 CI 中使用 `install`** 以固定一個一致的瀏覽器二進位檔,而不是依賴執行器提供的任何版本。
- **優先使用 `chrome`** 而非 `chromium`,因為 Chrome for Testing 更貼近穩定版 Chrome 的行為。
- **使用 `--where`** 在圍繞受管瀏覽器撰寫自動化指令稿前確認快取路徑。
- `install` 只管理 Extension.js 快取內的瀏覽器,並不會修改系統瀏覽器的安裝。

## 行為說明

- `chrome` 會安裝 Chrome for Testing,而非依賴系統中的 Google Chrome 應用程式。
- 在 Linux 上,`edge` 可能需要一個具有特權的互動式工作階段。
