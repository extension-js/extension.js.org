使用 `install` 將受管理的瀏覽器執行環境加入 Extension.js 快取。

當你希望 `dev`、`build`、`start` 或 `preview` 都使用一致的瀏覽器執行檔時,這個指令最為實用。它支援 Chrome for Testing、Chromium、Firefox 與 Edge。

## 何時使用 `install`

- 你需要一致、可重複的瀏覽器執行檔,用於持續整合(CI)、自動化或讓團隊本機執行保持一致。
- 你想使用 Chrome for Testing,而不是仰賴系統上裝著的任意版本 Chrome。
- 你正在用受管理的 Firefox 或 Edge 執行環境設定跨瀏覽器測試。

## 標準用法

針對單一瀏覽器,使用位置參數形式:

<CodeGroup>

```bash npm
extension install <browser>
```

```bash pnpm
extension install <browser>
```

```bash yarn
extension install <browser>
```

```bash bun
extension install <browser>
```

```bash bun
extension install <browser>
```

</CodeGroup>

只有在需要多個目標、瀏覽器家族或 `all` 時,才使用 `--browser`。

## Install 指令的能力

| 能力                | 它能為你提供什麼                                                              |
| ------------------- | ----------------------------------------------------------------------------- |
| 受管理的瀏覽器快取  | Extension.js 瀏覽器快取下的穩定安裝位置                                       |
| 可重複的執行環境    | 為可重複的本機執行與自動化提供一致的執行檔                                    |
| 跨瀏覽器設定        | Chrome、Chromium、Edge 與 Firefox 一致的指令流程                              |
| 路徑探查            | `--where` 會顯示解析後的快取根目錄或瀏覽器專屬的安裝路徑                      |

## 參數與旗標

| 旗標 / 參數                                                                                    | 功能                                                                            | 預設值     |
| ---------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------- | ---------- |
| `[browser-name]`                                                                               | 安裝單一瀏覽器目標,例如 `chrome`、`chromium`、`edge` 或 `firefox`              | `chromium` |
| `--browser <chrome\|chromium\|edge\|firefox\|chromium-based\|gecko-based\|firefox-based\|all>` | 覆寫位置形式的瀏覽器名稱,並支援多目標安裝                                      | 未設定     |
| `--where`                                                                                      | 印出解析後的受管理快取根目錄,或瀏覽器專屬的安裝路徑                            | 關閉       |

## 範例

### 安裝 Chrome for Testing

```bash
extension install chrome
```

### 一個指令安裝多個目標

```bash
extension install --browser chrome,firefox
```

### 顯示 Chrome 的受管理安裝路徑

```bash
extension install chrome --where
```

## 快取位置

預設情況下,Extension.js 會把受管理的瀏覽器存放在每位使用者的穩定快取中:

- macOS: `~/Library/Caches/extension.js/browsers`
- Linux: `~/.cache/extension.js/browsers` 或 `$XDG_CACHE_HOME/extension.js/browsers`
- Windows: `%LOCALAPPDATA%\extension.js\browsers`

你可以用 `EXT_BROWSERS_CACHE_DIR` 覆寫快取根目錄。

## 最佳實務

- **在 CI 中使用 `install`** 來固定一致的瀏覽器執行檔,而不是仰賴 runner 所提供的版本。
- **`chrome` 比 `chromium` 更建議用於 Chrome for Testing**:它更接近穩定版 Chrome 的行為。
- **使用 `--where`** 在針對受管理瀏覽器寫腳本自動化前先驗證快取路徑。
- `install` 只管理 Extension.js 快取裡的瀏覽器,不會修改系統上的瀏覽器安裝。

## 行為說明

- `chrome` 會安裝 Chrome for Testing,而不是仰賴系統上的 Google Chrome 應用程式。
- `edge` 在 Linux 上可能需要具特權的互動式 session。
