`create` 會依據所選範本產生檔案、設定與起始指令稿,並可選擇是否安裝相依套件。

## 何時使用 `create`

- 從零開始一個新擴充功能。
- 快速建立多個概念驗證 (POC) 構想。
- 用一致的範本預設值,讓團隊成員的入門體驗一致。

## Create 指令能力

| 能力             | 它能提供什麼                                                |
| ---------------- | ----------------------------------------------------------- |
| 範本建立         | 從官方範本出發,獲得就緒的專案結構                          |
| 相依套件安裝     | 可在建立後選擇安裝所需套件                                  |
| 路徑彈性         | 依專案名稱或明確的資料夾路徑建立                            |
| 快速上手         | 從空資料夾迅速過渡到可執行的擴充功能                        |

## 用法

<CodeGroup>

```bash npm
npx extension@latest create <extension-name|extension-path> [options]
```

```bash pnpm
pnpx extension@latest create <extension-name|extension-path> [options]
```

```bash yarn
yarn dlx extension@latest create <extension-name|extension-path> [options]
```

```bash bun
bunx extension@latest create <extension-name|extension-path> [options]
```

```bash bun
bunx extension@latest create <extension-name|extension-path> [options]
```

</CodeGroup>

## 參數與選項

| 選項                  | 別名  | 作用                                       | 預設值       |
| --------------------- | ----- | ------------------------------------------ | ------------ |
| `[path or name]`      | -     | 要建立的專案資料夾/名稱。                  | 必填         |
| `--template <name>`   | `-t`  | 設定範本別名。                             | `javascript` |
| `--install [boolean]` | -     | 建立完成後安裝相依套件。                   | `false`      |

如果你想使用預設的 JavaScript 起始範本,直接省略 `--template` 即可。只有當你想使用 [官方範例](https://github.com/extension-js/examples/tree/main/examples) 中的其他技術堆疊時,才加入 `--template=<slug>`。

`init` 是同一預設起始範本的別名。只有當範例存放庫中確實存在名為 `default` 的資料夾時,才使用字面值 `default`。

## 最佳實務

- 從符合你 UI/執行階段需求的範本開始,減少後續設定上的偏差。
- 第一次執行保持精簡,在驗證基本指令流程之後再加入額外工具。
