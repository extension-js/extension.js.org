`create` 會為所選範本建立檔案、設定與起始腳本,並可選擇安裝相依套件。

## 何時使用 `create`

- 從零開始建立新擴充功能。
- 快速啟動多個概念驗證。
- 以一致的範本預設值為團隊夥伴標準化新手入門。

## Create 指令能做什麼

| 功能                  | 你會得到什麼                                                |
| --------------------- | ----------------------------------------------------------- |
| 範本建立              | 從官方範本開始,內含現成的專案結構                          |
| 安裝相依套件          | 可選擇在建立後安裝所需套件                                  |
| 路徑彈性              | 以專案名稱或明確資料夾路徑建立                              |
| 快速上手              | 從空資料夾推進到可執行的擴充功能                            |

## 用法

<CodeGroup>

```bash npm
npx extension@latest create <extension-name|extension-path> [options]
```

```bash pnpm
pnpx extension@latest create <extension-name|extension-path> [options]
```

```bash yarn
yarn dlx extension@latest create <extension-name|extension-path> [options]
```

```bash bun
bunx extension@latest create <extension-name|extension-path> [options]
```

```bash bun
bunx extension@latest create <extension-name|extension-path> [options]
```

</CodeGroup>

## 參數與 flag

| Flag                  | 別名  | 用途                                     | 預設         |
| --------------------- | ----- | ---------------------------------------- | ------------ |
| `[path or name]`      | -     | 要建立的專案資料夾/名稱。                | 必填         |
| `--template <name>`   | `-t`  | 設定範本 slug。                          | `javascript` |
| `--install [boolean]` | -     | 建立後安裝相依套件。                     | `false`      |

如果你想用預設的 JavaScript starter,完全省略 `--template`。只有當你要用其他技術棧時才加上 `--template=<slug>`,可參考 [官方範例](https://github.com/extension-js/examples/tree/main/examples)。

slug `init` 是相同預設 starter 的別名。只有當 examples 儲存庫實際存在名為 `default` 的資料夾時,才使用字面值 slug `default`。

## 共用全域選項

## 最佳實務

- 從符合你 UI/runtime 需求的範本開始,以降低設定差異。
- 第一次執行盡量精簡,確認基本指令流程後再加入其他工具鏈。
