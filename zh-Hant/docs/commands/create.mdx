`create` 會為所選的範本建立檔案、設定與起始指令稿，並可選擇安裝相依性。

## 何時使用 `create`

- 從零開始建立新的擴充功能。
- 快速產出多個概念驗證原型。
- 透過一致的範本預設值，讓團隊上手流程標準化。

## Create 指令功能

| 功能                 | 提供內容                                                       |
| -------------------- | -------------------------------------------------------------- |
| 範本建立             | 從官方範本與現成的專案結構開始                                 |
| 相依性安裝           | 建立後可選擇安裝必要的套件                                     |
| 路徑彈性             | 以專案名稱或明確的資料夾路徑建立                               |
| 快速上手             | 從空資料夾快速進入可執行的擴充功能                             |

## 用法

<CodeGroup>

```bash npm
npx extension@latest create <extension-name|extension-path> [options]
```

```bash pnpm
pnpx extension@latest create <extension-name|extension-path> [options]
```

```bash yarn
yarn dlx extension@latest create <extension-name|extension-path> [options]
```

```bash bun
bunx extension@latest create <extension-name|extension-path> [options]
```

```bash bun
bunx extension@latest create <extension-name|extension-path> [options]
```

</CodeGroup>

## 引數與旗標

| 旗標                  | 別名 | 作用                                       | 預設         |
| --------------------- | ---- | ------------------------------------------ | ------------ |
| `[path or name]`      | -    | 要建立的專案資料夾／名稱。                 | 必填         |
| `--template <name>`   | `-t` | 設定範本 slug。                            | `javascript` |
| `--install [boolean]` | -    | 建立後安裝相依性。                         | `false`      |

當你想使用預設的 JavaScript 起始範本時，完全省略 `--template`。只有在要使用[官方範例](https://github.com/extension-js/examples/tree/main/examples)中其他技術堆疊時，才加入 `--template=<slug>`。

slug `init` 是同一個預設起始範本的別名。只有當範例儲存庫中確實存在名為 `default` 的資料夾時，才使用字面上的 `default` slug。

## 共用全域選項

也支援[全域旗標](/docs/workflows/global-flags)。

## 最佳實務

- 從符合你 UI／執行階段需求的範本開始，以減少設定漂移。
- 第一次執行先保持精簡，待驗證基本指令流程後再加入額外工具。
