當你想以一條指令同時完成生產建置與立即啟動瀏覽器時,請使用 `start`。

`start` 指令會先執行生產建置,接著以與 `preview` 相同的流程啟動建置後的擴充功能。

## 何時使用 `start`

- 在編譯完成後立刻手動驗證生產行為。
- 重現監看模式與生產輸出之間的執行階段差異。
- 在本機進行類生產檢查,而不需要分別執行 `build` 與 `preview`。

## Start 指令能力

| 能力                  | 它能提供什麼                                              |
| --------------------- | --------------------------------------------------------- |
| 建置 + 啟動工作流程   | 以一步完成生產編譯與瀏覽器啟動                            |
| 目標選擇              | 直接在所選瀏覽器或引擎目標中啟動                          |
| 執行器控制            | 當你只需要建置驗證時略過瀏覽器啟動                        |
| 類生產驗證            | 檢查真實的編譯輸出,而不是監看模式狀態                    |

## 它與其他指令的差異

- `dev`:開發伺服器 + 熱模組替換 (HMR)/監看模式
- `build`:僅生產建置
- `preview`:啟動既有的已建置擴充功能,但不會建置
- `start`:依序執行 `build` + `preview`

## 自動化中繼資料

`start` 會將就緒中繼資料寫入:

- `dist/extension-js/<browser>/ready.json`

這對於使用 `--no-browser` 的自動化非常實用:

- 在啟動外部執行器前等待 `status: "ready"`。
- 將 `status: "error"` 視為確定性的失敗訊號。
- 用 `runId` 與 `startedAt` 關聯特定的執行階段工作階段。

### `--no-browser` 與就緒同步

`--no-browser` 只會停用瀏覽器啟動,不會在生產建置完成前阻擋外部執行器。

對於面向生產的 Playwright、持續整合 (CI) 與 AI 工作流程:

1. 將 `extension start --no-browser` 作為產生者處理程序。
2. 將 `extension start --wait --browser=<browser>` 作為就緒閘道。
3. 僅在出現 `status: "ready"` 之後再啟動外部瀏覽器自動化。

`--wait` 在 `error`/逾時情況下以非零狀態結束,並忽略來自已結束處理程序的過期契約(`pid` 已不存在)。
若在同一次呼叫中同時傳入 `--wait` 與 `--no-browser`,`--wait` 優先生效,指令會以僅等待模式執行。

## 記錄選項

| 選項                                                 | 作用                                                                                         | 預設值             |
| ---------------------------------------------------- | -------------------------------------------------------------------------------------------- | ------------------ |
| `--logs <off\|error\|warn\|info\|debug\|trace\|all>` | 最低記錄層級。                                                                               | `off`              |
| `--log-context <list\|all>`                          | 情境過濾器(`background`、`content`、`page`、`sidebar`、`popup`、`options`、`devtools`)。   | `all`              |
| `--log-format <pretty\|json\|ndjson>`                | 記錄輸出格式。                                                                               | `pretty`           |
| `--no-log-timestamps`                                | 在 pretty 模式中停用時間戳記。                                                               | 預設啟用時間戳記   |
| `--no-log-color`                                     | 在 pretty 模式中停用色彩。                                                                   | 預設啟用色彩       |
| `--log-url <pattern>`                                | 以 URL 子字串/正規表示式過濾記錄事件。                                                       | 未設定             |
| `--log-tab <id>`                                     | 以 tab ID 過濾記錄事件。                                                                     | 未設定             |

## 行為說明

- `start` 不會執行開發伺服器,也不會提供熱模組替換 (HMR) 或監看模式。
- `start` 面向生產模式;迭代式本機開發請使用 `dev`。
- `start` 不支援原始碼檢視選項,此類工作流程請使用 `dev --source ...`。
- 對於機器消費者,請剖析 `dist/extension-js/<browser>/ready.json`,而不要剖析終端文字。
