當你希望以單一指令進行生產建置並立即啟動瀏覽器時，使用 `start`。

`start` 指令會先執行生產建置，然後以與 `preview` 指令相同的流程啟動建置好的擴充功能。

## 何時使用 `start`

- 在編譯後立即手動驗證生產行為。
- 重現監看模式與生產輸出之間的執行階段差異。
- 在本機進行類生產檢查，而不需要分開執行 `build` 與 `preview`。

## Start 指令功能

| 功能                    | 提供內容                                                       |
| ----------------------- | -------------------------------------------------------------- |
| 建置 + 啟動工作流程     | 在單一步驟中執行生產編譯與瀏覽器啟動                           |
| 目標選擇                | 直接在所選的瀏覽器或引擎目標中啟動                             |
| 執行器控制              | 僅需建置驗證時，可略過瀏覽器啟動                               |
| 類生產驗證              | 檢查真實的已編譯輸出，而非監看模式的狀態                       |

## 與其他指令的差異

- `dev`：dev server + 模組熱替換（HMR）／監看模式
- `build`：僅進行生產建置
- `preview`：啟動既有的已建置擴充功能而不進行建置
- `start`：依序進行 `build` + `preview`

## 自動化中繼資料

`start` 會將就緒中繼資料寫入：

- `dist/extension-js/<browser>/ready.json`

在使用 `--no-browser` 時對自動化很有用：

- 在啟動外部執行器前等待 `status: "ready"`。
- 將 `status: "error"` 視為確定性的失敗訊號。
- 使用 `runId` 與 `startedAt` 對應到特定的執行階段工作階段。

### `--no-browser` 與就緒同步

`--no-browser` 僅停用瀏覽器啟動。它並不會等到生產建置完成後再阻塞外部執行器。

對於以生產為導向的 Playwright、持續整合（CI）與 AI 工作流程：

1. 將 `extension start --no-browser` 作為生產者程序執行。
2. 將 `extension start --wait --browser=<browser>` 作為就緒閘道執行。
3. 僅在 `status: "ready"` 之後啟動外部瀏覽器自動化。

`--wait` 會在 `error`／逾時時以非零結束，並忽略死掉程序留下的過期契約（`pid` 已不存在）。
若在同一個指令呼叫中同時傳入 `--wait` 與 `--no-browser`，`--wait` 優先。指令會以僅等待模式執行。

| 旗標                                                 | 作用                                                                                            | 預設                |

| `--logs <off\|error\|warn\|info\|debug\|trace\|all>` | 最低記錄層級。                                                                                  | `off`               |

| `--log-tab <id>`                                     | 以分頁 ID 篩選記錄事件。                                                                        | 未設定              |

## 行為說明

- `start` 不會執行 dev server，亦不提供模組熱替換（HMR）或監看模式。
- `start` 以生產模式為導向；如需迭代式本機開發請使用 `dev`。
- `start` 不支援原始碼檢視選項，請以 `dev --source ...` 進行該工作流程。
- 對機器消費者而言，請解析 `dist/extension-js/<browser>/ready.json`，而非終端機文字。
