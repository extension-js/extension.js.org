使用 `uninstall` 從 Extension.js 快取移除受管理的瀏覽器執行環境。

它只會移除 Extension.js 安裝在其受管理快取根目錄中的瀏覽器,不會動到系統 Chrome、系統 Edge 或任何安裝在該快取以外的瀏覽器。

## 何時使用 `uninstall`

- 你想從不再需要的受管理瀏覽器中釋放硬碟空間。
- 你想重置某個受管理瀏覽器的安裝,讓下次 `install` 強制重新下載。
- 你正在清理持續整合(CI)快取或切換瀏覽器目標。

## 標準用法

針對單一瀏覽器,使用位置參數形式:

<CodeGroup>

```bash npm
extension uninstall <browser>
```

```bash pnpm
extension uninstall <browser>
```

```bash yarn
extension uninstall <browser>
```

```bash bun
extension uninstall <browser>
```

```bash bun
extension uninstall <browser>
```

</CodeGroup>

使用 `--all` 移除所有受管理的瀏覽器目標。

## 用法

<CodeGroup>

```bash npm
extension uninstall [browser-name] [options]
```

```bash pnpm
extension uninstall [browser-name] [options]
```

## 參數與旗標

| 旗標 / 參數           | 功能                                                                            | 預設值   |
| --------------------- | ------------------------------------------------------------------------------- | -------- |
| `[browser-name]`      | 移除單一受管理瀏覽器,例如 `chrome`、`chromium`、`edge` 或 `firefox`            | 未設定   |
| `--browser <browser>` | 瀏覽器參數的顯式旗標形式                                                        | 未設定   |
| `--all`               | 移除 Extension.js 快取中所有受管理的瀏覽器執行環境                              | 關閉     |
| `--where`             | 印出解析後的快取根目錄,或瀏覽器專屬的受管理路徑                                | 關閉     |

## 範例

### 移除受管理的 Chrome for Testing

```bash
extension uninstall chrome
```

### 移除所有受管理的瀏覽器

```bash
extension uninstall --all
```

### 顯示 Firefox 的受管理解除安裝路徑

```bash
extension uninstall firefox --where
```

## 最佳實務

- **在 CI 的清理階段使用 `--all`**,於測試執行後清理受管理的瀏覽器。
- **先使用 `--where`**,在批次解除安裝寫成腳本前確認會移除什麼。
- `uninstall` 對系統瀏覽器是安全的;它只會移除 Extension.js 管理的快取資料夾。

## 行為說明

- 如果你設定了 `EXT_BROWSERS_CACHE_DIR`,uninstall 會使用該自訂的快取根目錄。
