使用 `uninstall` 從 Extension.js 快取中移除受管的瀏覽器執行階段。

它只會移除 Extension.js 安裝在受管快取根目錄中的瀏覽器,不會碰觸系統 Chrome、系統 Edge 或任何安裝在該快取之外的瀏覽器。

## 何時使用 `uninstall`

- 你想從不再需要的受管瀏覽器中回收磁碟空間。
- 你正在重置受管瀏覽器的安裝,以便下次 `install` 時強制重新下載。
- 你正在清理持續整合 (CI) 快取或切換瀏覽器目標。

## 標準用法

對於單一瀏覽器,請使用位置引數形式:

<CodeGroup>

```bash npm
extension uninstall <browser>
```

```bash pnpm
extension uninstall <browser>
```

```bash yarn
extension uninstall <browser>
```

```bash bun
extension uninstall <browser>
```

```bash bun
extension uninstall <browser>
```

</CodeGroup>

使用 `--all` 移除所有受管的瀏覽器目標。

## 用法

<CodeGroup>

```bash npm

```

```bash pnpm
extension uninstall [browser-name] [options]
```


```bash bun
extension uninstall [browser-name] [options]
```


## 參數與選項

| 選項 / 引數           | 作用                                                                               | 預設值   |
| --------------------- | ---------------------------------------------------------------------------------- | -------- |
| `[browser-name]`      | 移除單一受管瀏覽器,例如 `chrome`、`chromium`、`edge` 或 `firefox`                 | 未設定   |
| `--browser <browser>` | 瀏覽器引數的明確選項形式                                                           | 未設定   |
| `--all`               | 從 Extension.js 快取中移除所有受管的瀏覽器執行階段                                 | 停用     |
| `--where`             | 列印解析得到的快取根目錄,或瀏覽器特定的受管路徑                                   | 停用     |

## 範例

### 移除受管的 Chrome for Testing

```bash
extension uninstall chrome
```

### 移除所有受管瀏覽器

```bash
extension uninstall --all
```

### 顯示 Firefox 的受管解除安裝路徑

```bash
extension uninstall firefox --where
```

## 最佳實務

- **在 CI 收尾時使用 `--all`** 以在測試執行後清理受管瀏覽器。
- **先使用 `--where`** 在撰寫批次解除安裝指令稿前確認將移除哪些內容。
- `uninstall` 對系統瀏覽器是安全的,它只移除 Extension.js 所管理的快取資料夾。

## 行為說明

- 如果你設定了 `EXT_BROWSERS_CACHE_DIR`,uninstall 會使用該自訂快取根目錄。
