使用 `uninstall` 從 Extension.js 快取移除受管理的瀏覽器執行階段。

這只會移除 Extension.js 安裝在其受管理快取根目錄中的瀏覽器，不會影響系統 Chrome、系統 Edge 或任何安裝在該快取以外的瀏覽器。

## 何時使用 `uninstall`

- 你想釋放不再需要的受管理瀏覽器所佔用的磁碟空間。
- 你想重設受管理的瀏覽器安裝，以便在下次 `install` 時強制重新下載。
- 你正在清理持續整合（CI）快取或切換瀏覽器目標。

## 標準用法

對單一瀏覽器，請使用位置引數形式：

<CodeGroup>

```bash npm
extension uninstall <browser>
```

```bash pnpm
extension uninstall <browser>
```

```bash yarn
extension uninstall <browser>
```

```bash bun
extension uninstall <browser>
```

```bash bun
extension uninstall <browser>
```

</CodeGroup>

使用 `--all` 移除每一個受管理的瀏覽器目標。

## 用法


## 引數與旗標

| 旗標／引數            | 作用                                                                                  | 預設     |
| --------------------- | ------------------------------------------------------------------------------------- | -------- |
| `[browser-name]`      | 移除單一受管理瀏覽器，例如 `chrome`、`chromium`、`edge` 或 `firefox`                  | 未設定   |
| `--browser <browser>` | 瀏覽器引數的明確旗標形式                                                              | 未設定   |
| `--all`               | 從 Extension.js 快取中移除所有受管理的瀏覽器執行階段                                  | 停用     |
| `--where`             | 印出解析後的快取根目錄，或瀏覽器專屬的受管理路徑                                      | 停用     |

## 範例

### 移除受管理的 Chrome for Testing

```bash
extension uninstall chrome
```

### 移除所有受管理的瀏覽器

```bash
extension uninstall --all
```

### 顯示 Firefox 的受管理移除路徑

```bash
extension uninstall firefox --where
```

## 最佳實務

- **在 CI 收尾中使用 `--all`**，在測試執行後清理受管理的瀏覽器。
- **先使用 `--where`**，在針對大量移除撰寫指令稿前確認將被移除的內容。
- `uninstall` 對系統瀏覽器是安全的；它只會移除 Extension.js 受管理的快取資料夾。

## 行為說明

- 若你設定了 `EXT_BROWSERS_CACHE_DIR`，uninstall 會使用該自訂快取根目錄。

## 後續步驟

- 以 [`install`](/docs/commands/install) 重新安裝受管理的瀏覽器。
- 透過 [`dev`](/docs/commands/dev) 與 [`start`](/docs/commands/start) 檢視瀏覽器目標。
- 進一步了解以自訂二進位路徑[執行其他瀏覽器](/docs/browsers/running-other-browsers)。
