依照擴充功能開發目前所處的階段,從首次建立到正式版驗證,選擇合適的指令。

## 指令選擇

| 目標                       | 指令        | 範例                                                            |
| -------------------------- | ----------- | --------------------------------------------------------------- |
| 建立新專案                 | `create`    | `npx extension@latest create my-extension --template=new-react` |
| 以監看模式開發             | `dev`       | `extension dev --browser=firefox`                               |
| 建置正式版產物             | `build`     | `extension build --browser=chrome,firefox --zip`                |
| 建置並啟動正式版輸出       | `start`     | `extension start --browser=edge`                                |
| 只啟動既有建置輸出         | `preview`   | `extension preview --browser=chrome`                            |
| 安裝受管理的瀏覽器執行階段 | `install`   | `extension install chrome`                                      |
| 移除受管理的瀏覽器執行階段 | `uninstall` | `extension uninstall chrome`                                    |

## 典型開發者流程

1. `create` 建立專案。
2. `dev` 進行迭代開發與重新載入循環。
3. `build` 產生正式版產物。
4. `preview` 或 `start` 做接近正式版的驗證。
5. 需要明確管理 Extension.js 瀏覽器執行階段時,使用 `install` 或 `uninstall`。
