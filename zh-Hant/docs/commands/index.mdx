從首次建立到生產驗證,為擴充功能開發的每個階段選擇正確的指令。

## 指令選擇器

| 目標                                | 指令        | 範例                                                            |
| ---------------------------------- | ----------- | --------------------------------------------------------------- |
| 建立新專案                          | `create`    | `npx extension@latest create my-extension --template=new-react` |
| 以監看模式開發                      | `dev`       | `extension dev --browser=firefox`                               |
| 建置生產產物                        | `build`     | `extension build --browser=chrome,firefox --zip`                |
| 建置並啟動生產輸出                  | `start`     | `extension start --browser=edge`                                |
| 僅啟動既有建置輸出                  | `preview`   | `extension preview --browser=chrome`                            |
| 安裝受管的瀏覽器 runtime            | `install`   | `extension install chrome`                                      |
| 移除受管的瀏覽器 runtime            | `uninstall` | `extension uninstall chrome`                                    |

## 典型開發流程

1. `create` 建立專案。
2. `dev` 進行迭代式編碼與重新載入循環。
3. `build` 產生生產產物。
4. `preview` 或 `start` 做類似生產的驗證。
5. `install` 或 `uninstall` 用於明確管理 Extension.js 的瀏覽器 runtime。
