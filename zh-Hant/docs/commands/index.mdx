依擴充功能開發目前所處的階段，從首次建立到生產驗證，選擇合適的指令。

## 指令選擇器

| 目的                                  | 指令        | 範例                                                            |
| ------------------------------------- | ----------- | --------------------------------------------------------------- |
| 建立新專案                            | `create`    | `npx extension@latest create my-extension --template=new-react` |
| 以監看模式開發                        | `dev`       | `extension dev --browser=firefox`                               |
| 建置生產產出物                        | `build`     | `extension build --browser=chrome,firefox --zip`                |
| 建置並啟動生產輸出                    | `start`     | `extension start --browser=edge`                                |
| 僅啟動既有的建置輸出                  | `preview`   | `extension preview --browser=chrome`                            |
| 安裝受管理的瀏覽器執行階段            | `install`   | `extension install chrome`                                      |
| 移除受管理的瀏覽器執行階段            | `uninstall` | `extension uninstall chrome`                                    |

## 典型的開發流程

1. 使用 `create` 建立專案。
2. 使用 `dev` 進行迭代開發與重新載入循環。
3. 使用 `build` 產出生產產出物。
4. 使用 `preview` 或 `start` 進行類生產驗證。
5. 需要明確管理 Extension.js 瀏覽器執行階段時，使用 `install` 或 `uninstall`。
