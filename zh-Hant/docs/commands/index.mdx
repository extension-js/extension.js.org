為目前的擴充功能開發階段選擇合適的指令,涵蓋從第一次建立到生產驗證的整個流程。

## 指令選擇器

| 目標                               | 指令        | 範例                                                            |
| ---------------------------------- | ----------- | --------------------------------------------------------------- |
| 建立一個新專案                     | `create`    | `npx extension@latest create my-extension --template=new-react` |
| 在監看模式下開發                   | `dev`       | `extension dev --browser=firefox`                               |
| 建置生產產物                       | `build`     | `extension build --browser=chrome,firefox --zip`                |
| 建置並啟動生產輸出                 | `start`     | `extension start --browser=edge`                                |
| 僅啟動既有的建置輸出               | `preview`   | `extension preview --browser=chrome`                            |
| 安裝受管的瀏覽器執行階段           | `install`   | `extension install chrome`                                      |
| 移除受管的瀏覽器執行階段           | `uninstall` | `extension uninstall chrome`                                    |

## 典型開發者流程

1. 用 `create` 建立。
2. 用 `dev` 進行迭代式編碼與重新載入循環。
3. 用 `build` 建置生產產物。
4. 用 `preview` 或 `start` 進行類生產驗證。
5. 當你需要明確管理 Extension.js 瀏覽器執行階段時,使用 `install` 或 `uninstall`。
