[Plasmo](https://www.plasmo.com) 是以 Parcel 為基礎、採用檔案慣例路由的瀏覽器擴充功能框架。它的發佈節奏已經放慢,工具鏈相較於目前的打包工具也已老化,這是許多團隊重新評估它的原因。本指南會將一個典型的 Plasmo 專案遷移到 Extension.js,而不必重寫你的 UI 程式碼。

## 什麼會改變、什麼不變

**維持不變:** 你的 React 元件、Tailwind 設定、測試、`chrome.*` API 呼叫,以及 `@plasmohq/storage`(如果你有用)(它包覆 `chrome.storage`,在這裡同樣可運作)。

**會改變:**

- 檔案慣例進入點(`popup.tsx`、`contents/*.ts`)變成真正 `manifest.json` 中的明確條目。
- `package.json` 中的 `manifest` 欄位移到 `manifest.json`。
- `plasmo dev` / `plasmo build` / `plasmo package` 變成 `extension dev` / `extension build --zip`。
- `PLASMO_PUBLIC_*` 環境變數變成 `EXTENSION_PUBLIC_*`。

## 步驟 1:安裝 Extension.js

```bash
npm install extension@latest --save-dev
npm uninstall plasmo
```

## 步驟 2:撰寫 manifest

Plasmo 從 `package.json` 加上檔案慣例產生 manifest。Extension.js 把 `manifest.json` 視為唯一事實來源。請逐項翻譯:

| Plasmo 慣例                          | `manifest.json` 條目                                      |
| ------------------------------------ | --------------------------------------------------------- |
| `popup.tsx` 或 `src/popup.tsx`       | `"action": {"default_popup": "popup/index.html"}`         |
| `options.tsx`                        | `"options_ui": {"page": "options/index.html"}`            |
| `newtab.tsx`                         | `"chrome_url_overrides": {"newtab": "newtab/index.html"}` |
| `background.ts`                      | `"background": {"service_worker": "background.ts"}`       |
| 帶 CS 設定的 `contents/inline.ts`    | `"content_scripts": [{...}]` 條目                         |
| `package.json` 中的 `manifest` 欄位  | 直接合併到 `manifest.json`                                |

HTML 進入點是載入你 React 元件的純 HTML 檔。一個 popup 看起來會像:

```html popup/index.html
<!doctype html>
<html>
  <body>
    <div id="root"></div>
    <script src="./index.tsx" type="module"></script>
  </body>
</html>
```

manifest 與 script 標籤中的副檔名維持 `.ts`/`.tsx`。Extension.js 會在建構時編譯它們。

## 步驟 3:轉換內容腳本

Plasmo 讀取匯出的 `PlasmoCSConfig` 取得 matches:

```ts contents/inline.tsx
import type { PlasmoCSConfig } from "plasmo";

export const config: PlasmoCSConfig = {
  matches: ["https://example.com/*"],
};
```

在 Extension.js 中,matches 寫在 manifest 裡,而該檔案就是一個普通模組:

```json manifest.json
{
  "content_scripts": [
    {
      "matches": ["https://example.com/*"],
      "js": ["content/inline.tsx"]
    }
  ]
}
```

如果你用過 Plasmo CSUI(帶 `getRootContainer` 與錨點的內容腳本 UI),請改用一般 DOM 程式碼掛載你的元件。建立一個容器元素、把它附加到你想錨定的位置,然後渲染進去。參見 [內容腳本](/docs/implementation-guide/content-scripts) 了解完整模式,包含為樣式隔離使用 shadow DOM。

## 步驟 4:環境變數與訊息傳遞

- 在你的 `.env` 檔案與程式碼中,把 `PLASMO_PUBLIC_*` 改名為 `EXTENSION_PUBLIC_*`。參見 [環境變數](/docs/features/environment-variables)。
- `@plasmohq/storage` 可以原封不動繼續使用。
- `@plasmohq/messaging` 依賴 Plasmo 的 `background/messages/*` 建構慣例。請在背景腳本中用標準的 `chrome.runtime.onMessage` 監聽器來取代處理器。參見 [訊息傳遞](/docs/implementation-guide/messaging)。

## 步驟 5:更新 package.json 腳本

```json
{
  "scripts": {
    "dev": "extension dev",
    "build": "extension build",
    "start": "extension start"
  }
}
```

Plasmo 使用 `--target=firefox-mv2` 的地方,Extension.js 直接以瀏覽器為目標:

```bash
extension build --browser=chrome,firefox --zip
```

你會得到 `dist/chrome` 與 `dist/firefox`,搭配各瀏覽器正確的 manifest,以及可上架 Chrome Web Store 與 addons.mozilla.org 的 `.zip` 壓縮檔。

## 步驟 6:驗證

```bash
extension dev --browser=chrome
```

檢查 popup、options、內容腳本與背景行為,接著用 `--browser=firefox` 對 Firefox 做相同的檢查。如果你的程式碼呼叫 `chrome.*` 且你希望相同的原始碼能在 Firefox 上執行,請使用 `--polyfill`。

## 常見地雷

- **`assets/` 中的圖示:** Plasmo 會自動由 `assets/icon.png` 產生各種圖示尺寸。Extension.js 使用 manifest 中 `"icons"` 所宣告的內容;請輸出你需要的尺寸。參見 [圖示](/docs/implementation-guide/icons)。
- **`~` 匯入別名:** Plasmo 把 `~` 別名對應到專案根目錄。請在 `tsconfig.json` 的 paths 中對應它,或改用相對匯入。參見 [路徑解析](/docs/features/path-resolution)。
- **儲存 hook:** 來自 `@plasmohq/storage/hook` 的 `useStorage` 不需改動即可運作;它只依賴 React 與 `chrome.storage`。
