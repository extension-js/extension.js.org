無需離開 Extension.js 流程,就能自訂打包行為。

不必分叉預設設定,就能解決自訂打包需求。你可以直接加入 loader(檔案類型轉換)、外掛(建置時擴充)與別名(import 捷徑)。Extension.js 以 [Rspack](https://rspack.dev) 進行建置——一個以 Rust 為基礎的 JavaScript 打包器(`@rspack/core`)——你可以直接擴充所產生的設定。

## 運作方式

在專案根目錄建立下列任一檔案:

- `extension.config.js`
- `extension.config.mjs`
- `extension.config.cjs`

使用 `config` 鍵來修補所產生的打包器設定。

### `config` 能力

| 能力                                          | 功能                                                                        |
| --------------------------------------------- | --------------------------------------------------------------------------- |
| 函式 hook(`config: (config) => config`)      | 在建置前完全掌控,讀取與修改所產生的 Rspack 設定。                          |
| 物件合併(`config: { ... }`)                 | 將額外設定合併進所產生的基礎設定。                                          |
| 規則(`config.module.rules`)                | 為檔案類型加入或修改 loader 規則。                                          |
| 外掛(`config.plugins`)                     | 加入編譯時的外掛與轉換。                                                    |
| 解析(`config.resolve`)                     | 加入別名與模組解析行為。                                                    |

### 選項 1:函式 hook(建議)

```js
export default {
  config: (config) => {
    config.module.rules.push({
      test: /\.mdx$/,
      use: ["babel-loader", "@mdx-js/loader"],
    });
    return config;
  },
};
```

### 選項 2:物件合併

`config` 也可以是物件。Extension.js 會將它合併到基礎設定。

```js
export default {
  config: {
    resolve: {
      alias: {
        "@ui": "/absolute/path/to/ui",
      },
    },
  },
};
```

## Rspack 為主,相容 webpack

Extension.js 原生使用 Rspack,但你仍能使用 webpack 生態系中的多數工具。

- 設定型別擴充自 `@rspack/core` 的 `Configuration`。
- 多數 webpack 的 loader/外掛可透過相容層運作。
- 某些 webpack 的內部實作/外掛並非與 Rspack 1:1 相容。

使用 Rspack 原生外掛的範例:

```js
import { DefinePlugin } from "@rspack/core";

export default {
  config: (config) => {
    config.plugins = [
      ...config.plugins,
      new DefinePlugin({
        __FEATURE_FLAG__: JSON.stringify("enabled"),
      }),
    ];
    return config;
  },
};
```

## 何時使用

- 為專案特定的檔案類型加入自訂 loader/規則。
- 加入用於編譯時轉換與診斷的外掛。
- 覆寫 Extension.js 一等選項未提供的 resolve 別名與模組行為。

## 最佳實務

- **優先使用一等選項**:在使用底層打包器覆寫之前,先採用 `browser` / `commands` 設定鍵。
- **最小化修補**:只更動你需要的部分,然後回傳設定。
- **讓外掛與 Rspack 對齊**:有 Rspack 原生外掛時,優先選用。
- **在所有目標上驗證**:設定變更後,為你的瀏覽器矩陣測試 `dev`、`start`、`preview` 與 `build`。
