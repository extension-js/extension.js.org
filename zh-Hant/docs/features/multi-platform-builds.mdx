為多個瀏覽器建置與封裝同一個擴充功能,並讓結果具可預期性。

將同一個擴充功能交付到 Chrome、Edge 與 Firefox,而無需維護多份建置指令稿。Extension.js 會為每個目標執行正式建置,並把產物寫入 `dist/<browser>`。它也可以為發行產生 zip 套件。

## 運作方式

執行一次正式建置:

<CodeGroup>

```bash npm
npx extension build
```

```bash pnpm
pnpx extension build
```

```bash yarn
yarn dlx extension build
```

```bash bun
bunx extension build
```

```bash bun
bunx extension build
```

</CodeGroup>

若未自行覆寫,預設的瀏覽器目標為 `chromium`。

你可以針對特定瀏覽器/引擎建置:

<CodeGroup>

```bash npm

```

```bash pnpm

```

```bash yarn

```

```bash bun
bunx extension build --browser=chrome
```

```bash bun
bunx extension build --browser=chrome
```

</CodeGroup>

<CodeGroup>

```bash npm
npx extension build --browser=firefox
```

```bash pnpm
pnpx extension build --browser=firefox
```

```bash yarn
yarn dlx extension build --browser=firefox
```

```bash bun
bunx extension build --browser=firefox
```


</CodeGroup>

支援的值包括:

- `chrome`
- `edge`
- `firefox`
- `chromium`
- `chromium-based`
- `gecko-based` / `firefox-based`(別名)

你也可以在單一命令中執行一組建置矩陣:

<CodeGroup>

```bash npm

```

```bash pnpm

```

```bash yarn

```

```bash bun
bunx extension build --browser=chrome,edge,firefox
```

```bash bun
bunx extension build --browser=chrome,edge,firefox
```

</CodeGroup>

這會依序為 `chrome`、`edge` 與 `firefox` 進行建置。

## 輸出配置

每個目標各自寫入其資料夾:

- `dist/chrome`
- `dist/edge`
- `dist/firefox`
- `dist/chromium`
- `dist/chromium-based`
- `dist/gecko-based`

### 建置能力

| 選項                    | 功能                                                                                |
| ----------------------- | ----------------------------------------------------------------------------------- |
| `--browser=<target>`    | 為特定瀏覽器或引擎家族建置。                                                        |
| `--zip`                 | 為每個目標產物建立發行用 zip。                                                      |
| `--zip-filename=<name>` | 覆寫預設的 zip 檔名。                                                               |
| `--zip-source`          | 同時建立原始碼封存檔。                                                              |
| `--polyfill`            | 為 Chromium 家族目標啟用相容性行為,以支援 `browser.*` API 的使用。                 |

## 產生 zip 檔

使用 `--zip` 為每個目標產物建立發行用 zip:

<CodeGroup>

```bash npm
npx extension build --zip
```

```bash pnpm
pnpx extension build --zip
```

```bash yarn
yarn dlx extension build --zip
```

```bash bun
bunx extension build --zip
```

```bash bun
bunx extension build --zip
```

</CodeGroup>

預設情況下,zip 檔名會使用清理後的 manifest `name` 與 `version`,例如:

- `my-extension-1.0.0.zip`

自訂檔名:

<CodeGroup>

```bash npm
npx extension build --zip --zip-filename=my-release
```

```bash pnpm
pnpx extension build --zip --zip-filename=my-release
```

```bash yarn
yarn dlx extension build --zip --zip-filename=my-release
```

```bash bun
bunx extension build --zip --zip-filename=my-release
```

```bash bun
bunx extension build --zip --zip-filename=my-release
```

</CodeGroup>

這會在目標的 `dist/<browser>` 資料夾中產生 `my-release.zip`。

## 最佳實務

- **在持續整合(CI)中針對每個目標分別建置**:把每個瀏覽器的產物視為獨立成品。
- **使用瀏覽器矩陣命令做平台一致性檢查**:在單一流程步驟中儘早抓出目標特有問題。
- **有目的地封裝**:`--zip` 用於商店上架,`--zip-source` 用於可追溯的原始碼成品。
- **將目標設定明確化**:以 `extension.config.*` 的命令/瀏覽器預設值達成可重現的建置。
