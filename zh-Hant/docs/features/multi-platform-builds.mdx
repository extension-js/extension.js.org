為多個瀏覽器建構並打包同一份擴充功能，並擁有可預測的輸出。

無需維護分別的建構腳本，即可將同一份擴充功能交付到 Chrome、Edge 與 Firefox。Extension.js 為每個目標執行 production 建構，並將成品寫入 `dist/<browser>`。它也能產生用於發行的 zip 套件。

## 運作方式

執行 production 建構：

<CodeGroup>

```bash npm
npx extension build
```

```bash pnpm
pnpx extension build
```

```bash yarn
yarn dlx extension build
```

```bash bun
bunx extension build
```

```bash bun
bunx extension build
```

</CodeGroup>

除非你另外指定，否則預設的瀏覽器目標是 `chromium`。

## 瀏覽器選擇

你可以鎖定特定的瀏覽器/引擎：

<CodeGroup>

```bash npm

```

```bash pnpm
pnpx extension build --browser=chrome
```

```bash yarn
yarn dlx extension build --browser=chrome
```

```bash bun
bunx extension build --browser=chrome
```

```bash bun
bunx extension build --browser=chrome
```

</CodeGroup>

<CodeGroup>

```bash npm
npx extension build --browser=firefox
```

```bash pnpm
pnpx extension build --browser=firefox
```

```bash yarn
yarn dlx extension build --browser=firefox
```

```bash bun
bunx extension build --browser=firefox
```

```bash bun
bunx extension build --browser=firefox
```

</CodeGroup>

支援的值包括：

- `chrome`
- `edge`
- `firefox`
- `chromium`
- `chromium-based`
- `gecko-based` / `firefox-based`（別名）

你也可以在單一指令中執行建構矩陣：

<CodeGroup>

```bash npm

```

```bash pnpm

```

```bash yarn

```

```bash bun
bunx extension build --browser=chrome,edge,firefox
```

```bash bun
bunx extension build --browser=chrome,edge,firefox
```

</CodeGroup>

這會依序為 `chrome`、`edge` 與 `firefox` 進行建構。

## 輸出佈局

每個目標寫入各自的資料夾：

- `dist/chrome`
- `dist/edge`
- `dist/firefox`
- `dist/chromium`
- `dist/chromium-based`
- `dist/gecko-based`

### 建構能力

| 選項                    | 功能                                                                                 |
| ----------------------- | ------------------------------------------------------------------------------------ |
| `--browser=<target>`    | 為特定瀏覽器或引擎家族進行建構。                                                     |
| `--zip`                 | 為每個目標輸出建立發行 zip。                                                         |
| `--zip-filename=<name>` | 覆寫預設的 zip 檔名。                                                                |
| `--zip-source`          | 同時建立原始碼封存。                                                                 |
| `--polyfill`            | 為 Chromium 系列目標啟用 `browser.*` API 相容行為。                                  |

## 產生 zip 檔案

使用 `--zip` 從每個目標輸出產生發行 zip：

<CodeGroup>

```bash npm
npx extension build --zip
```

```bash pnpm
pnpx extension build --zip
```

```bash yarn
yarn dlx extension build --zip
```

```bash bun
bunx extension build --zip
```

```bash bun
bunx extension build --zip
```

</CodeGroup>

預設情況下，zip 名稱使用淨化過的 manifest `name` + `version`，例如：

- `my-extension-1.0.0.zip`

自訂檔名：

<CodeGroup>

```bash npm
npx extension build --zip --zip-filename=my-release
```

```bash pnpm
pnpx extension build --zip --zip-filename=my-release
```

```bash yarn
yarn dlx extension build --zip --zip-filename=my-release
```

```bash bun
bunx extension build --zip --zip-filename=my-release
```

```bash bun
bunx extension build --zip --zip-filename=my-release
```

</CodeGroup>

這會在目標 `dist/<browser>` 資料夾中建立 `my-release.zip`。

## 最佳實踐

- **在持續整合（CI）中分別建構每個目標**：將每個瀏覽器輸出視為獨立成品。
- **使用瀏覽器矩陣指令進行同等性檢查**：在單一流水線步驟中及早發現特定目標的問題。
- **有意地打包**：使用 `--zip` 進行商店上架，使用 `--zip-source` 產生可追溯的原始碼成品。
- **明確設定目標**：使用 `extension.config.*` 的指令/瀏覽器預設值來確保建構可重現。
