以可預期的輸出,為多個瀏覽器建置與封裝同一份擴充功能。

把同一份擴充功能發佈到 Chrome、Edge 與 Firefox,而不必維護多份建置腳本。Extension.js 會針對每個目標執行正式建置,並把產物寫到 `dist/<browser>`。也可以為發佈產生 zip 封裝。

## 運作方式

執行一次正式建置:

<CodeGroup>

```bash npm
npx extension build
```

```bash pnpm
pnpx extension build
```

```bash yarn
yarn dlx extension build
```

```bash bun
bunx extension build
```

```bash bun
bunx extension build
```

</CodeGroup>

除非你覆寫,預設的瀏覽器目標是 `chromium`。

## 瀏覽器選擇

你可以指定特定的瀏覽器/引擎作為目標:

<CodeGroup>

```bash npm
npx extension build --browser=chrome
```

```bash pnpm
pnpx extension build --browser=chrome
```

```bash yarn
yarn dlx extension build --browser=chrome
```

```bash bun
bunx extension build --browser=chrome
```

```bash bun
bunx extension build --browser=chrome
```

</CodeGroup>

<CodeGroup>

```bash npm
npx extension build --browser=firefox
```

```bash pnpm
pnpx extension build --browser=firefox
```

```bash yarn
yarn dlx extension build --browser=firefox
```

```bash bun
bunx extension build --browser=firefox
```

```bash bun
bunx extension build --browser=firefox
```

</CodeGroup>

支援的值包含:

- `chrome`
- `edge`
- `firefox`
- `chromium`
- `chromium-based`
- `gecko-based` / `firefox-based`(別名)

你也可以在一個命令中跑建置矩陣:

<CodeGroup>

```bash npm
npx extension build --browser=chrome,edge,firefox
```

```bash pnpm
pnpx extension build --browser=chrome,edge,firefox
```

```bash yarn
yarn dlx extension build --browser=chrome,edge,firefox
```

```bash bun
bunx extension build --browser=chrome,edge,firefox
```

```bash bun
bunx extension build --browser=chrome,edge,firefox
```

</CodeGroup>

這會依序為 `chrome`、`edge` 與 `firefox` 建置。

## 輸出結構

每個目標都會寫到自己的資料夾:

- `dist/chrome`
- `dist/edge`
- `dist/firefox`
- `dist/chromium`
- `dist/chromium-based`
- `dist/gecko-based`

### 建置能力

| 選項                    | 用途                                                                                 |
| ----------------------- | ------------------------------------------------------------------------------------ |
| `--browser=<target>`    | 為特定瀏覽器或引擎家族建置。                                                          |
| `--zip`                 | 為每個目標的輸出建立發佈用的 zip。                                                    |
| `--zip-filename=<name>` | 覆寫預設的 zip 檔名。                                                                |
| `--zip-source`          | 在輸出旁邊建立原始碼壓縮檔。                                                          |
| `--polyfill`            | 為 Chromium 家族目標中使用的 `browser.*` API 啟用相容性行為。                          |

## 產生 zip 檔案

用 `--zip` 從每個目標的輸出產生發佈用 zip:

<CodeGroup>

```bash npm
npx extension build --zip
```

```bash pnpm
pnpx extension build --zip
```

```bash yarn
yarn dlx extension build --zip
```

```bash bun
bunx extension build --zip
```

```bash bun
bunx extension build --zip
```

</CodeGroup>

預設情況下,zip 名稱會使用清理過的 manifest `name` + `version`,例如:

- `my-extension-1.0.0.zip`

自訂檔名:

<CodeGroup>

```bash npm
npx extension build --zip --zip-filename=my-release
```

```bash pnpm
pnpx extension build --zip --zip-filename=my-release
```

```bash yarn
yarn dlx extension build --zip --zip-filename=my-release
```

```bash bun
bunx extension build --zip --zip-filename=my-release
```

```bash bun
bunx extension build --zip --zip-filename=my-release
```

</CodeGroup>

這會在目標的 `dist/<browser>` 資料夾內建立 `my-release.zip`。

如果你的程式碼依賴 Gecko 風格的 `browser.*` API,而你又需要 Chromium 的相容性,請啟用 `--polyfill`:

<CodeGroup>

```bash npm
npx extension build --polyfill
```

```bash pnpm
pnpx extension build --polyfill
```

```bash yarn
yarn dlx extension build --polyfill
```

```bash bun
bunx extension build --polyfill
```

```bash bun

```

</CodeGroup>

## 最佳實務

- **在持續整合(CI)上為每個目標分別建置**:把每個瀏覽器的輸出視為獨立產物。
- **用瀏覽器矩陣命令做一致性檢查**:在單一管線步驟中早期發現目標專屬問題。
- **有目的地封裝**:`--zip` 用於上架商店,`--zip-source` 用於可追溯的原始碼產物。
- **讓目標設定保持顯式**:用 `extension.config.*` 的命令/瀏覽器預設值,做可重現的建置。
