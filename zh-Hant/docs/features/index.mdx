了解讓 Extension.js 在真實世界的瀏覽器擴充功能開發中實用的核心平台能力。

## 功能選擇器

| 目標                                            | 從這裡開始                                                                            |
| ----------------------------------------------- | ------------------------------------------------------------------------------------- |
| 用同一份原始碼出貨到多個瀏覽器                  | [跨瀏覽器相容性](/zh-Hant/docs/features/cross-browser-compatibility)                  |
| 處理瀏覽器專屬的 manifest 差異                  | [瀏覽器專屬的 manifest 欄位](/zh-Hant/docs/features/browser-specific-fields)         |
| 把非 manifest 的資產有條理地組織                | [特殊資料夾](/zh-Hant/docs/features/special-folders)                                  |
| 管理環境相關的行為                              | [環境變數](/zh-Hant/docs/features/environment-variables)                              |
| 集中管理建置/執行期預設值                       | [擴充功能設定](/zh-Hant/docs/features/extension-configuration)                        |
| 安全地擴充打包工具行為                          | [Rspack 設定](/zh-Hant/docs/features/rspack-configuration)                            |
| 了解開發模式下的更新行為                        | [頁面重新載入與 hot module replacement(HMR)](/zh-Hant/docs/features/reload-and-hmr) |
| 避免原始碼與輸出之間的路徑漂移                  | [路徑解析](/zh-Hant/docs/features/path-resolution)                                   |
| 為目標矩陣建置與打包                            | [多平台建置](/zh-Hant/docs/features/multi-platform-builds)                            |
| 檢視遙測隱私契約                                | [遙測與隱私](/zh-Hant/docs/features/telemetry-and-privacy)                            |

## 建議的學習順序

1. 驗證瀏覽器輸出的保證(`cross-browser-compatibility`)。
2. 固定專案預設值(`extension-configuration`)。
3. 加入進階行為(`rspack-configuration`、`environment-variables`)。
4. 最佳化發布流程(`multi-platform-builds`)。
