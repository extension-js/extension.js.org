使用單一設定檔來設定瀏覽器預設值、指令行為與打包器自訂內容。

在團隊間共享瀏覽器預設值、指令選項與建構設定。不再重複輸入 CLI 旗標。Extension.js 會從專案根目錄讀取 `extension.config.js`（或 `.mjs` / `.cjs`），並將設定套用到每個指令與打包器上。

## 運作方式

在專案根目錄加入 `extension.config.js`（在典型設定中與 `package.json` 同層）。

支援的檔名：

- `extension.config.js`
- `extension.config.mjs`
- `extension.config.cjs`

頂層鍵：

| 鍵                  | 說明                                                                         |
| ------------------- | ---------------------------------------------------------------------------- |
| `browser`           | 依瀏覽器目標索引的瀏覽器特定預設值。                                         |
| `commands`          | 各指令的預設值（`dev`、`start`、`preview`、`build`）。                       |
| `extensions`        | 在各指令間共用的伴隨擴充功能（僅載入）。                                     |
| `transpilePackages` | 在打包前要編譯的套件（適用於 monorepo/workspace 設定）。                     |
| `config`            | 擴充／覆寫產生的 Rspack 設定的鉤子。                                         |

### 設定檔的環境載入

`extension.config.*` 在 Node 中執行，應從 `process.env.*` 讀取值。

- Extension.js 會在評估 `extension.config.*` 之前預載 env 檔案。
- 它會先檢查專案資料夾。
- 在 monorepo 中，若 Extension.js 找不到專案本地的 `.env*` 檔案，會回退至最近的工作區根目錄。工作區根目錄是包含 `pnpm-workspace.yaml` 的資料夾。
- 在設定檔中，建議使用內建的 env 預載，而不是匯入 `dotenv`。

### 瀏覽器設定

需要為每個目標設定不同的瀏覽器預設值嗎？使用 `browser`：

```js
export default {
  browser: {
    chrome: {
      profile: "path/to/profile",
      preferences: { darkMode: true },
      browserFlags: ["--disable-web-security"],
      excludeBrowserFlags: ["--mute-audio"],
    },
    "chromium-based": {
      chromiumBinary: "/path/to/brave-or-other-chromium-binary",
    },
    firefox: {
      geckoBinary: "/path/to/firefox",
    },
  },
};
```

支援的瀏覽器鍵包括：`chrome`、`edge`、`firefox`、`chromium`、`chromium-based`、`gecko-based`、`firefox-based`。

常見瀏覽器欄位：

- `profile`、`persistProfile`
- `preferences`
- `browserFlags`、`excludeBrowserFlags`
- `chromiumBinary`、`geckoBinary`
- `extensions`（僅載入的伴隨擴充功能）

#### 瀏覽器目標能力

| 鍵                    | 功能                                                                     |
| --------------------- | ------------------------------------------------------------------------ |
| `profile`             | 設定啟動時使用的瀏覽器設定檔路徑（或設定檔行為）。                       |
| `persistProfile`      | 在多次執行間重用設定檔資料。                                             |
| `preferences`         | 套用瀏覽器偏好設定的覆寫。                                               |
| `browserFlags`        | 為瀏覽器程序加入啟動旗標。                                               |
| `excludeBrowserFlags` | 移除你不想要的預設啟動旗標。                                             |
| `chromiumBinary`      | 使用自訂 Chromium 系列的二進位路徑。                                     |
| `geckoBinary`         | 使用自訂 Gecko/Firefox 二進位路徑。                                      |
| `extensions`          | 與主擴充功能一起載入伴隨擴充功能。                                       |

### 指令設定

使用 `commands` 為每個指令定義預設值：

```js
export default {
  transpilePackages: ["@workspace/ui"],
  commands: {
    dev: {
      browser: "chrome",
      polyfill: true,
      logLevel: "info",
      persistProfile: true,
      browserFlags: ["--headless=new"],
      extensions: { dir: "./extensions" },
    },
    start: {
      browser: "firefox",
      source: "https://example.com",
    },
    preview: {
      browser: "edge",
      noBrowser: false,
    },
    build: {
      browser: "chrome",
      zip: true,
      zipSource: true,
      zipFilename: "my-extension.zip",
      transpilePackages: ["@workspace/ui", "@workspace/icons"],
    },
  },
};
```

注意：

- Extension.js 會將頂層的 `extensions` 與 `transpilePackages` 合併至指令預設值中。
- 各指令的值會覆寫頂層的值。
- `start` 指令在內部會依序執行 `build` 與 `preview`。Extension.js 會套用 `commands.start` 的設定，包括啟動瀏覽器的選項，例如 `profile`、`browserFlags` 與 `startingUrl`。你也可以將建構相關設定放在 `commands.build`。

#### 指令能力（共用）

| 鍵                    | 適用於                             | 功能                                                       |
| --------------------- | ---------------------------------- | ---------------------------------------------------------- |
| `browser`             | `dev`、`start`、`preview`、`build` | 設定瀏覽器或瀏覽器家族目標。                               |
| `profile`             | `dev`、`start`、`preview`          | 設定瀏覽器設定檔路徑/行為。                                |
| `persistProfile`      | `dev`、`start`、`preview`          | 在多次執行間重用瀏覽器設定檔資料。                         |
| `chromiumBinary`      | `dev`、`start`、`preview`          | 使用自訂 Chromium 系列的二進位檔。                         |
| `geckoBinary`         | `dev`、`start`、`preview`          | 使用自訂 Gecko 系列的二進位檔。                            |
| `polyfill`            | `dev`、`start`、`build`            | 在相關情境下啟用相容性 polyfill 行為。                     |
| `preferences`         | `dev`、`start`、`preview`          | 套用瀏覽器偏好設定的覆寫。                                 |
| `browserFlags`        | `dev`、`start`、`preview`          | 為瀏覽器程序加入啟動旗標。                                 |
| `excludeBrowserFlags` | `dev`、`start`、`preview`          | 移除你不想要的預設啟動旗標。                               |
| `startingUrl`         | `dev`、`start`、`preview`          | 在瀏覽器啟動時開啟特定 URL。                               |
| `port`                | `dev`、`start`、`preview`          | 設定 runner/DevTools 連接埠。                              |
| `host`                | `dev`、`start`、`preview`          | 設定 dev 伺服器主機。Docker/dev 容器時使用 `0.0.0.0`。     |
| `noBrowser`           | `dev`、`start`、`preview`          | 停用瀏覽器啟動。                                           |
| `extensions`          | `dev`、`start`、`preview`、`build` | 載入伴隨擴充功能（或擴充功能清單）。                       |
| `install`             | `dev`、`start`、`build`            | 執行前安裝缺少的相依套件。                                 |
| `transpilePackages`   | `dev`、`start`、`preview`、`build` | 編譯列出的 workspace/外部套件。                            |

#### `build` 指令能力

| 鍵            | 功能                                                                |
| ------------- | ------------------------------------------------------------------- |
| `zip`         | 產生封裝的 zip 成品。                                               |
| `zipSource`   | 產生用於審查/合規流程的原始碼套件/封存輸出。                        |
| `zipFilename` | 設定自訂 zip 輸出檔名。                                             |
| `silent`      | 減少建構日誌輸出。                                                  |

#### `dev` 指令能力

| 鍵                    | 功能                                                     |
| --------------------- | -------------------------------------------------------- |
| `noOpen`              | 執行 dev 伺服器但不自動開啟瀏覽器。                      |
| `source`              | 從遠端來源 URL 啟用原始碼檢查流程。                      |
| `watchSource`         | 當監看的更新發生時重新執行原始碼檢查。                   |
| `sourceFormat`        | 設定原始碼輸出格式（`pretty`、`json`、`ndjson`）。       |
| `sourceSummary`       | 輸出精簡的原始碼摘要。                                   |
| `sourceMeta`          | 在原始碼輸出中包含頁面中繼資料。                         |
| `sourceProbe`         | 以特定 CSS 選擇器探測原始碼輸出。                        |
| `sourceTree`          | 控制精簡的擴充功能樹輸出。                               |
| `sourceConsole`       | 在原始碼輸出中包含主控台摘要。                           |
| `sourceDom`           | 在原始碼輸出中包含 DOM 快照/差異。                       |
| `sourceMaxBytes`      | 限制原始碼輸出載荷的大小。                               |
| `sourceRedact`        | 控制原始碼遮蔽策略。                                     |
| `sourceIncludeShadow` | 控制 Shadow DOM 包含策略。                               |
| `sourceDiff`          | 在原始碼監看更新中包含差異中繼資料。                     |

#### 日誌能力

| 鍵           | 適用於                    | 功能                                                       |
| ------------ | ------------------------- | ---------------------------------------------------------- |
| `logs`       | `dev`、`start`、`preview` | 設定最低日誌層級（`off` 至 `all`）。                       |
| `logContext` | `dev`、`start`、`preview` | 依情境過濾日誌（`background`、`content` 等）。             |
| `logFormat`  | `dev`、`start`、`preview` | 設定輸出格式（`pretty`、`json`、`ndjson`）。               |
| `logUrl`     | `dev`、`start`、`preview` | 依 URL 樣式過濾日誌。                                      |
| `logTab`     | `dev`、`start`、`preview` | 依分頁 ID 過濾日誌。                                       |

### Rspack 設定

需要進階的打包器自訂嗎？使用 `config` 修補產生的 Rspack 設定：

```js
export default {
  config: (config) => {
    config.module.rules.push({
      test: /\.mdx$/,
      use: ["babel-loader", "@mdx-js/loader"],
    });
    return config;
  },
};
```

`config` 也可以是物件，Extension.js 會將其合併到產生的設定之上。

## 完整範例

```js
export default {
  browser: {
    chrome: { browser: "chrome", profile: "default" },
    firefox: { browser: "firefox", persistProfile: true },
    "chromium-based": {
      browser: "chromium-based",
      chromiumBinary:
        "/Applications/Brave Browser.app/Contents/MacOS/Brave Browser",
    },
  },
  commands: {
    dev: {
      browser: "chrome",
      polyfill: true,
      host: "0.0.0.0",
      extensions: { dir: "./extensions" },
    },
    start: { browser: "edge" },
    preview: { browser: "firefox" },
    build: {
      zipFilename: "extension.zip",
      zip: true,
      zipSource: true,
    },
  },
  extensions: { dir: "./extensions" },
  transpilePackages: ["@workspace/ui"],
  config: (config) => config,
};
```

## 最佳實踐

- **將瀏覽器特定的值保留在 `browser`**：讓指令定義專注於工作流程，而不是瀏覽器內部細節。
- **謹慎使用頂層預設值**：將共用的 `extensions` / `transpilePackages` 放在根層級，只在必要時覆寫。
- **優先使用 `chromiumBinary`/`geckoBinary` 名稱**：它們與目前的指令與型別介面一致。
- **保持 `config` 鉤子簡潔**：僅加入第一線 Extension.js 選項未涵蓋的內容。
