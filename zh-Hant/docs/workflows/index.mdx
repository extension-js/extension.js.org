## 發行信心循環

1. 執行有針對性的本機檢查（`dev` + 疑難排解）。
2. 建置正式產物（`build` / `start`）。
3. 透過端對端（E2E）與煙霧測試驗證關鍵流程。
4. 在送交商店前執行 CI 範本。
