## 發佈信心循環

1. 執行有針對性的本機檢查(`dev` + 疑難排解)。
2. 建置正式版產物(`build` / `start`)。
3. 用端對端(E2E)與煙霧檢查驗證關鍵流程。
4. 在上架前執行 CI 範本。
