用一套現代化工作流為 Chrome、Edge 與 Firefox 建置瀏覽器擴充功能。Extension.js 為你處理 manifest 編譯、瀏覽器專屬輸出、重新載入行為與封裝，讓你專注在產品功能而不是建置工具。

使用熟悉的 Web 工具，例如 [TypeScript](/zh-Hant/docs/languages-and-frameworks/typescript)、[React](/zh-Hant/docs/languages-and-frameworks/react)、[Vue](/zh-Hant/docs/languages-and-frameworks/vue) 與 [Tailwind CSS](/zh-Hant/docs/integrations/tailwindcss)，同時保留直接存取原生擴充功能 API 的能力。

<Note>
  正在比較工具？看看 Extension.js 與 WXT 等其他框架的差異:[Extension.js vs WXT](/zh-Hant/docs/compare/extension-js-vs-wxt)。
</Note>

<Info>
  **CLI 版本:** 執行 `extension --version`（或 `npx extension@latest --version`）查看你正在使用的建置版本。[npm 上的 extension 套件](https://www.npmjs.com/package/extension)會列出最新發佈版本。[GitHub releases](https://github.com/extension-js/extension.js/releases) 提供更新紀錄樣式的說明，文件不會綁定到特定的修補版號。
</Info>

## 挑選合適的指令

| 目標                              | 指令      | 範例                                                            |
| --------------------------------- | --------- | --------------------------------------------------------------- |
| 建立專案鷹架                       | `create`  | `npx extension@latest create my-extension --template=new-react` |
| 以監看模式進行開發                 | `dev`     | `extension dev --browser=firefox`                               |
| 建置正式版產物                     | `build`   | `extension build --browser=chrome,firefox --zip`                |
| 建置並啟動正式版輸出               | `start`   | `extension start --browser=edge`                                |
| 啟動既有的建置輸出                 | `preview` | `extension preview --browser=chrome`                            |

使用 `create` 指令建立新專案鷹架，並用 `--template` 從特定技術堆疊的基線開始。

<CodeGroup>

```bash npm
npx extension@latest create <your-extension-name>
```

```bash pnpm
pnpx extension@latest create <your-extension-name>
```

```bash yarn
yarn dlx extension@latest create <your-extension-name>
```

```bash bun
bunx extension@latest create <your-extension-name>
```

```bash bun
bunx extension@latest create <your-extension-name>
```

</CodeGroup>

如果你已經有一個擴充功能，安裝 `extension` 套件並一次性接好腳本即可。這能讓本機開發、測試與發佈建置保持一致。

### 步驟 1:將 `extension` 套件安裝為 `devDependency`

<CodeGroup>

```bash npm
npm install extension@latest --save-dev
```

```bash pnpm
pnpm add extension@latest --save-dev
```

```bash yarn
yarn add extension@latest --dev
```

```bash bun
bun add --dev extension@latest
```

```bash bun
bun add --dev extension@latest
```

</CodeGroup>

### 步驟 2:把 npm 腳本指向 `extension` 指令

```json package.json
{
  "scripts": {
    "build": "extension build",
    "dev": "extension dev",
    "start": "extension start"
  },
  "devDependencies": {
    "extension": "latest"
  }
}
```

完成。你的擴充功能已經可以使用。

- 執行 `npm run dev` 進行日常迭代與監看模式。
- 執行 `npm run start` 進行正式版建置並立刻啟動。
- 執行 `npm run build` 產出可上架的正式版產物。

## 最佳實踐

- **保持單一指令流程:** 將 `create` → `dev` → `build` 作為預設迴圈。
- **明確指定瀏覽器目標:** 發佈前以 `--browser=chrome,firefox` 進行驗證。
- **集中管理預設值:** 把共用的指令/瀏覽器設定放在 `extension.config.js`。
