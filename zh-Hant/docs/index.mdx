以一套現代化的工作流程為 Chrome、Edge 與 Firefox 建構瀏覽器擴充功能。Extension.js 處理 manifest 編譯、各瀏覽器專用的輸出、重新載入行為與封裝。讓你專注於產品功能，而不是建構工具。

使用熟悉的網頁開發工具，例如 [TypeScript](/docs/languages-and-frameworks/typescript)、[React](/docs/languages-and-frameworks/react)、[Vue](/docs/languages-and-frameworks/vue) 與 [Tailwind CSS](/docs/integrations/tailwindcss)。同時保有對原生擴充功能 API 的直接存取。

<Note>
  在比較工具嗎？看看 Extension.js 與 WXT 及其他框架的對比，請參考 [Extension.js 與 WXT 比較](/docs/compare/extension-js-vs-wxt)。
</Note>

<Info>
  **CLI 版本：** 執行 `extension --version`（或 `npx extension@latest
  --version`）可查看你正在使用的建構版本。[npm 上的 extension 套件](https://www.npmjs.com/package/extension)會列出最新發佈版本。[GitHub releases](https://github.com/extension-js/extension.js/releases) 追蹤類似變更日誌的說明，而不會把本文件綁定到特定的修訂版號。
</Info>

## 選擇合適的命令

| 目標                               | 命令      | 範例                                                          |
| ---------------------------------- | --------- | --------------------------------------------------------------- |
| 建立新專案                        | `create`  | `npx extension@latest create my-extension --template=new-react` |
| 以監看模式開發            | `dev`     | `extension dev --browser=firefox`                               |
| 建構正式環境產物         | `build`   | `extension build --browser=chrome,firefox --zip`                |
| 建構並啟動正式環境輸出 | `start`   | `extension start --browser=edge`                                |
| 啟動既有的建構輸出       | `preview` | `extension preview --browser=chrome`                            |

使用 `create` 命令來建立新專案，並透過 `--template` 從特定技術棧的基線開始。

<CodeGroup>

```bash npm
npx extension@latest create <your-extension-name>
```

```bash pnpm
pnpx extension@latest create <your-extension-name>
```

```bash yarn
yarn dlx extension@latest create <your-extension-name>
```

```bash bun
bunx extension@latest create <your-extension-name>
```

```bash bun
bunx extension@latest create <your-extension-name>
```

</CodeGroup>

如果你已經有擴充功能，安裝 `extension` 套件並把腳本串接一次即可。這能讓本地開發、測試與發行建構保持一致。

### 步驟 1：將 `extension` 套件安裝為 `devDependency`

<CodeGroup>

```bash npm
npm install extension@latest --save-dev
```

```bash pnpm
pnpm add extension@latest --save-dev
```

```bash yarn
yarn add extension@latest --dev
```

```bash bun
bun add --dev extension@latest
```

```bash bun
bun add --dev extension@latest
```

</CodeGroup>

### 步驟 2：將你的 npm 腳本連結到 `extension` 命令

```json package.json
{
  "scripts": {
    "build": "extension build",
    "dev": "extension dev",
    "start": "extension start"
  },
  "devDependencies": {
    "extension": "latest"
  }
}
```

完成。你的擴充功能已準備好可以使用。

- 執行 `npm run dev` 用於每日迭代與監看模式。
- 執行 `npm run start` 進行正式建構並立即啟動。
- 執行 `npm run build` 產出可上架的正式環境產物。

## 最佳實務

- **保持單一命令流程：** 將 `create` → `dev` → `build` 作為預設循環。
- **明確指定瀏覽器目標：** 發行前以 `--browser=chrome,firefox` 進行驗證。
- **集中管理預設值：** 把共用的命令／瀏覽器設定放在 `extension.config.js`。
