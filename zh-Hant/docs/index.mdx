用一個現代化的工作流為 Chrome、Edge 與 Firefox 建構瀏覽器擴充功能。Extension.js 處理 manifest 編譯、特定瀏覽器的輸出、重新載入行為以及打包。專注於產品功能，而非建置工具。

使用你熟悉的 Web 工具，如 [TypeScript](/docs/languages-and-frameworks/typescript)、[React](/docs/languages-and-frameworks/react)、[Vue](/docs/languages-and-frameworks/vue) 與 [Tailwind CSS](/docs/integrations/tailwindcss)。同時保留對原生擴充功能 API 的直接存取。

<Note>
  正在比較工具？參見 Extension.js 與 WXT 及其他框架的對比：[Extension.js vs WXT](/docs/compare/extension-js-vs-wxt)。
</Note>

<Info>
  **CLI 版本：** 執行 `extension --version`（或 `npx extension@latest
  --version`）查看你正在使用的建置版本。[npm 上的 extension 套件](https://www.npmjs.com/package/extension) 列出最新發布。
  [GitHub releases](https://github.com/extension-js/extension.js/releases) 以變更紀錄風格記錄說明，且不會讓文件綁定特定修補版號。
</Info>

## 選擇正確的指令

| 目標                               | 指令       | 範例                                                            |
| ---------------------------------- | --------- | --------------------------------------------------------------- |
| 建立專案鷹架                       | `create`  | `npx extension@latest create my-extension --template=new-react` |
| 以監看模式開發                     | `dev`     | `extension dev --browser=firefox`                               |
| 建置生產產物                        | `build`   | `extension build --browser=chrome,firefox --zip`                |
| 建置並啟動生產輸出                  | `start`   | `extension start --browser=edge`                                |
| 啟動既有的建置輸出                  | `preview` | `extension preview --browser=chrome`                            |

使用 `create` 指令建立新專案鷹架，並透過 `--template` 從特定技術堆疊的基準範本起步。

<CodeGroup>

```bash npm
npx extension@latest create <your-extension-name>
```

```bash pnpm
pnpx extension@latest create <your-extension-name>
```

```bash yarn
yarn dlx extension@latest create <your-extension-name>
```

```bash bun
bunx extension@latest create <your-extension-name>
```

bunx extension@latest create <your-extension-name>

如果你已經有一個擴充功能，安裝 `extension` 套件並一次設定好指令稿即可。這能讓本機開發、測試與發行建置保持一致。

### 步驟 1：將 `extension` 套件安裝為 `devDependency`

<CodeGroup>

```bash npm
npm install extension@latest --save-dev
```

```bash pnpm
pnpm add extension@latest --save-dev
```

```bash yarn
yarn add extension@latest --dev
```

```bash bun
bun add --dev extension@latest
```

```bash bun
bun add --dev extension@latest
```

</CodeGroup>

### 步驟 2：將你的 npm 指令稿連結到 `extension` 指令

```json package.json
{
  "scripts": {
    "build": "extension build",
    "dev": "extension dev",
    "start": "extension start"
  },
  "devDependencies": {
    "extension": "latest"
  }
}
```

完成。你的擴充功能已經可以使用了。

- 執行 `npm run dev` 進行日常迭代與監看模式開發。
- 執行 `npm run start` 進行生產建置並立即啟動。
- 執行 `npm run build` 產生可上架的生產產物。

## 最佳實務

- **保持單一指令流：** 以 `create` → `dev` → `build` 作為預設循環。
- **明確指定目標瀏覽器：** 發行前使用 `--browser=chrome,firefox` 進行驗證。
- **集中管理預設設定：** 將共享的指令／瀏覽器設定放在 `extension.config.js` 中。
