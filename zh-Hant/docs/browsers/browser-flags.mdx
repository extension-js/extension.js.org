為除錯、自動化與 runtime 實驗控制瀏覽器啟動行為。

不必修改擴充功能原始碼也能調校瀏覽器啟動行為。Extension.js 會合併 `extension.config.*` 中的瀏覽器 flag,並在 `dev`、`preview` 與 `start` 流程中套用。

## Firefox 有 `firefox://flags` 嗎?

Firefox 並沒有像 Chromium 瀏覽器那樣使用 `firefox://flags`,Chromium 使用 `chrome://flags` 或 `edge://flags`。Firefox 透過 `about:config`(preferences)提供 runtime 開關,並從命令列接受啟動 flag。

如果你搜尋了 `firefox://flags`、`firefox //flags`、`browser://flags`、`mozilla://flags` 或 `about flags firefox`,你大概是想在擴充功能開發期間改變瀏覽器行為。在 Extension.js 中,有兩個地方可以做到:

- 使用 [`browserFlags`](/zh-Hant/docs/browsers/browser-flags) 設定 Extension.js 傳遞給瀏覽器執行檔的啟動 flag。
- 使用 [Firefox preferences](/zh-Hant/docs/browsers/browser-preferences) 取得原本要寫在 `about:config` 中、可重現的 Gecko runtime 行為。

兩者都能用於同一個 Extension.js 專案的 Chrome、Edge 與 Firefox 擴充功能開發。

## 運作方式

在 `extension.config.*` 中設定 flag:

- `browser.<target>.browserFlags`
- `commands.dev|start|preview.browserFlags`
- 選用 `excludeBrowserFlags` 來移除預設值或使用者 flag(行為依目標是 Chromium 或 Firefox 而不同)。

覆寫順序:瀏覽器預設值 → 指令預設值 → CLI 選擇的指令 context。

## Flag 能做什麼

| 設定鍵                                 | 用途                                                     |
| -------------------------------------- | -------------------------------------------------------- |
| `browser.<target>.browserFlags`        | 為特定瀏覽器目標設定預設啟動 flag。                      |
| `commands.dev.browserFlags`            | 為 `dev` 執行新增或覆寫 flag。                           |
| `commands.start.browserFlags`          | 為 `start` 執行新增或覆寫 flag。                         |
| `commands.preview.browserFlags`        | 為 `preview` 執行新增或覆寫 flag。                       |
| `browser.<target>.excludeBrowserFlags` | 為某個目標移除符合的預設或使用者 flag。                  |
| `commands.<name>.excludeBrowserFlags`  | 在指令專屬 context 中移除 flag。                         |

### 範例設定

```js
export default {
  browser: {
    chrome: {
      browserFlags: ["--disable-web-security", "--auto-open-devtools-for-tabs"],
      excludeBrowserFlags: ["--mute-audio"],
    },
    firefox: {
      browserFlags: ["--devtools", "--new-instance"],
      excludeBrowserFlags: ["--devtools"],
    },
  },
};
```

## Chromium 與 Firefox 行為差異

- **Chromium 家族(`chrome`、`edge`、`chromium`、`chromium-based`)**
  - 先從內部預設 flag 集開始,再附加你的 `browserFlags`。
  - `excludeBrowserFlags` 會移除符合的預設 flag(完全比對過濾)。
  - Extension.js 會管理 `--load-extension=...`,並從使用者提供的 flag 中過濾掉它。

- **Firefox/Gecko 家族(`firefox`、`gecko-based` / `firefox-based`)**
  - 使用使用者提供的 `browserFlags`(不像 Chromium 那樣帶有大量預設 flag)。
  - `excludeBrowserFlags` 透過前綴比對移除使用者 flag。

## 最佳實務

- **只加必要 flag**:盡量縮短 flag 清單,以降低不穩定或不可攜的設定。
- **優先用 `excludeBrowserFlags` 而非取代預設值**:只移除與你工作流衝突的項目。
- **不要手動傳入 `--load-extension`**:Extension.js 會在內部管理擴充功能的載入 flag。
- **依瀏覽器家族驗證**:在 Chromium 上有效的 flag,可能在 Firefox 中無效或被忽略。
