為偵錯、自動化與執行階段實驗控制瀏覽器啟動行為。

不需要修改擴充功能原始碼即可調整瀏覽器啟動行為。Extension.js 會合併你 `extension.config.*` 中的瀏覽器旗標,並在 `dev`、`preview` 與 `start` 流程中套用。

## Firefox 有 `firefox://flags` 嗎?

Firefox 並不像 Chromium 系瀏覽器使用 `chrome://flags` 或 `edge://flags` 那樣使用 `firefox://flags`。Firefox 透過 `about:config`(偏好設定)暴露執行階段開關,並接受從命令列傳入的啟動旗標。

如果你搜尋的是 `firefox://flags`、`firefox //flags`、`browser://flags`、`mozilla://flags` 或 `about flags firefox`,你大概是想在擴充功能開發期間修改瀏覽器的行為。在 Extension.js 中,可以在兩個地方做這件事:

- 使用 [`browserFlags`](/docs/browsers/browser-flags) 設定 Extension.js 傳給瀏覽器執行檔的啟動期旗標。
- 使用 [Firefox 偏好設定](/docs/browsers/browser-preferences) 設定原本應放在 `about:config` 中、可重複套用的 Gecko 執行階段行為。

兩者都可以在同一個 Extension.js 專案中用於 Chrome、Edge 與 Firefox 的擴充功能開發。

## 運作方式

在 `extension.config.*` 中設定旗標:

- `browser.<target>.browserFlags`
- `commands.dev|start|preview.browserFlags`
- 選用的 `excludeBrowserFlags`,用來移除預設或使用者旗標(具體行為取決於你目標是 Chromium 或 Firefox)。

覆寫順序:瀏覽器預設值 → 命令預設值 → CLI 選取的命令情境。

## 旗標能力

| 設定鍵                                 | 作用                                                     |
| -------------------------------------- | -------------------------------------------------------- |
| `browser.<target>.browserFlags`        | 為特定瀏覽器目標設定預設啟動旗標。                       |
| `commands.dev.browserFlags`            | 為 `dev` 執行新增或覆寫旗標。                            |
| `commands.start.browserFlags`          | 為 `start` 執行新增或覆寫旗標。                          |
| `commands.preview.browserFlags`        | 為 `preview` 執行新增或覆寫旗標。                        |
| `browser.<target>.excludeBrowserFlags` | 移除某個目標中符合的預設或使用者旗標。                   |
| `commands.<name>.excludeBrowserFlags`  | 在特定命令情境中移除旗標。                               |

### 範例設定

```js
export default {
  browser: {
    chrome: {
      browserFlags: ["--disable-web-security", "--auto-open-devtools-for-tabs"],
      excludeBrowserFlags: ["--mute-audio"],
    },
    firefox: {
      browserFlags: ["--devtools", "--new-instance"],
      excludeBrowserFlags: ["--devtools"],
    },
  },
};
```

## Chromium 與 Firefox 行為比較

- **Chromium 家族(`chrome`、`edge`、`chromium`、`chromium-based`)**
  - 從內部預設旗標集開始,然後附加你的 `browserFlags`。
  - `excludeBrowserFlags` 會移除符合的預設旗標(完全比對篩選)。
  - Extension.js 管理 `--load-extension=...`,並會從使用者提供的旗標中將其過濾掉。

- **Firefox/Gecko 家族(`firefox`、`gecko-based` / `firefox-based`)**
  - 使用使用者提供的 `browserFlags`(不像 Chromium 那樣附帶大量預設旗標)。
  - `excludeBrowserFlags` 透過前置詞比對移除使用者旗標。

### Chromium 預設旗標

| ---------------------------------------------------------- | ----------------------------------------------------------------------- |

| `--disable-client-side-phishing-detection`                 | 停用釣魚偵測                                                            |

| `--disable-component-extensions-with-background-pages`     | 停用不受 `--disable-extensions` 影響的內建擴充功能                      |
| `--disable-default-apps`                                   | 停用預設應用的安裝                                                      |

| `--disable-features=Translate`                             | 停用 Chrome 翻譯                                                        |

| `--no-default-browser-check`                               | 停用預設瀏覽器檢查彈窗                                                  |

| `--disable-breakpad`                                       | 停用當機傾印收集                                                        |
| `--disable-component-update`                               | 停用元件更新                                                            |
| `--disable-domain-reliability`                             | 停用網域可靠性監控                                                      |
| `--no-pings`                                               | 停用超連結稽核 ping                                                     |
| `--enable-features=SidePanelUpdates`                       | 確保側邊面板可見                                                        |

## 最佳實務

- **只新增必要的旗標**:盡量縮短旗標清單,減少不穩定或不可攜的設定。
- **優先使用 `excludeBrowserFlags` 而非取代預設值**:只移除與你工作流衝突的內容。
- **不要手動傳入 `--load-extension`**:Extension.js 會在內部管理擴充功能載入旗標。
- **對每個瀏覽器家族都進行驗證**:在 Chromium 上有效的旗標在 Firefox 中可能無效或被忽略。
