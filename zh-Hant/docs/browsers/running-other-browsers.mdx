執行熱門的 Chromium 與 Gecko 分支,可以用名稱(Extension.js
會為你定位已安裝的執行檔)或提供明確的執行檔路徑。

從同一份 Extension.js 工作流程測試 Brave、Opera、Vivaldi、Yandex、Waterfox 與 LibreWolf。直接指定分支名稱,或在 `dev`、`start` 與 `preview` 中透過執行檔旗標與 `extension.config.*` 指向任何自訂執行檔。

## 依名稱執行分支

這些分支是一級的瀏覽器目標。把名稱傳給 `--browser`,Extension.js 會自動在你的系統上找到已安裝的執行檔,並透過其引擎家族的啟動器執行:

| 瀏覽器目標  | 引擎家族 | 自動定位 |
| ----------- | -------- | -------- |
| `brave`     | Chromium | 是       |
| `opera`     | Chromium | 是       |
| `vivaldi`   | Chromium | 是       |
| `yandex`    | Chromium | 是       |
| `waterfox`  | Gecko    | 是       |
| `librewolf` | Gecko    | 是       |

```bash
extension dev --browser=brave
```

```bash
extension dev --browser=waterfox
```

如果該瀏覽器未安裝,Extension.js 會以安裝指引結束。具名分支會繼承其家族的 manifest 鍵,因此 `chromium:`/`firefox:` 前綴的欄位能正確解析(參見 [瀏覽器專屬 manifest 欄位](/docs/features/browser-specific-fields))。

## 執行自訂執行檔

若要執行沒有內建定位器的瀏覽器,或覆寫已定位的執行檔,使用下列其中一個旗標:

- `--chromium-binary <path>`
- `--gecko-binary <path>`(別名:`--firefox-binary <path>`)

不論你選擇的具名瀏覽器目標為何,這些執行檔旗標都會覆寫 Extension.js 啟動時所使用的瀏覽器執行檔。

## 執行檔功能

| 選項 / 鍵                         | 用途                                     |
| --------------------------------- | ---------------------------------------- |
| `--chromium-binary <path>`        | 啟動自訂的 Chromium 系列瀏覽器執行檔。   |
| `--gecko-binary <path>`           | 啟動自訂的 Gecko 系列瀏覽器執行檔。      |
| `--firefox-binary <path>`         | `--gecko-binary` 的別名。                |
| `browser.<target>.chromiumBinary` | 在設定中設定預設的自訂 Chromium 執行檔。 |
| `browser.<target>.geckoBinary`    | 在設定中設定預設的自訂 Gecko 執行檔。    |
| `commands.<name>.chromiumBinary`  | 設定特定指令使用的自訂 Chromium 執行檔。 |
| `commands.<name>.geckoBinary`     | 設定特定指令使用的自訂 Gecko 執行檔。    |

### CLI 範例

```bash
extension dev --browser=chromium-based --chromium-binary="/path/to/brave"
```

```bash
extension dev --browser=firefox --gecko-binary="/path/to/firefox-developer-edition"
```

也可以搭配 `start` 與 `preview` 使用。

## 在 `extension.config.*` 中設定

```js
export default {
  browser: {
    "chromium-based": {
      chromiumBinary: "/path/to/custom-chromium-browser",
    },
    "gecko-based": {
      geckoBinary: "/path/to/custom-gecko-browser",
    },
  },
};
```

你也可以把執行檔路徑放在指令區塊中:

```js
export default {
  commands: {
    dev: {
      chromiumBinary: "/path/to/custom-chromium-browser",
    },
    preview: {
      geckoBinary: "/path/to/custom-gecko-browser",
    },
  },
};
```

## 目標對應行為

執行檔提示會對應到引擎目標:

- `chromiumBinary` → `chromium-based`
- `geckoBinary` / `firefoxBinary` → `gecko-based`

如果兩者都提供,Extension.js 會優先解析 Chromium 執行檔。

## 可用的瀏覽器

有內建定位器的分支可依名稱執行;其他則以執行檔旗標執行:

| 瀏覽器名稱                    | 類型                  | 執行方式                                   | 官方網站                                                  |
| ----------------------------- | --------------------- | ------------------------------------------ | --------------------------------------------------------- |
| **Brave**                     | Chromium-based 瀏覽器 | `--browser=brave` 或 `--chromium-binary`   | [brave.com](https://brave.com)                            |
| **Opera**                     | Chromium-based 瀏覽器 | `--browser=opera` 或 `--chromium-binary`   | [opera.com](https://www.opera.com)                        |
| **Vivaldi**                   | Chromium-based 瀏覽器 | `--browser=vivaldi` 或 `--chromium-binary` | [vivaldi.com](https://vivaldi.com)                        |
| **Yandex**                    | Chromium-based 瀏覽器 | `--browser=yandex` 或 `--chromium-binary`  | [browser.yandex.com](https://browser.yandex.com)          |
| **Waterfox**                  | Gecko-based 瀏覽器    | `--browser=waterfox` 或 `--gecko-binary`   | [Waterfox](https://www.waterfox.net)                      |
| **LibreWolf**                 | Gecko-based 瀏覽器    | `--browser=librewolf` 或 `--gecko-binary`  | [librewolf.net](https://librewolf.net)                    |
| **Firefox Developer Edition** | Gecko-based 瀏覽器    | `--gecko-binary`                           | [firefox.com](https://www.mozilla.org/firefox/developer/) |

## 重要限制

- `chromium-based` 需要有效的 `chromiumBinary` 路徑。
- `gecko-based` / `firefox-based` 需要有效的 `geckoBinary` 路徑。
- 無效路徑會立即失敗,並回報明確的 CLI/執行階段錯誤。
- `build` 不接受執行檔旗標。基於執行檔的啟動方式只能用在 `dev`、`start` 與 `preview`。

## 最佳實務

- **執行檔搭配明確的瀏覽器目標**:使用 `--browser=chromium-based` 或 `--browser=gecko-based` 表達清楚意圖。
- **使用絕對路徑**:避免 shell 路徑解析造成的問題。
- **在持續整合(CI)Runner 中固定版本**:讓瀏覽器執行檔路徑在自動化檢查中保持確定性。
- **小心搭配設定檔/旗標**:沿用具名瀏覽器目標所使用的設定檔與旗標策略。
