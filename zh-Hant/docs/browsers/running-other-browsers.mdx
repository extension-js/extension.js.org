透過提供明確的二進位檔路徑,執行預設命名目標之外的 Chromium 與 Gecko 家族瀏覽器。

在同一個 Extension.js 工作流中測試自訂瀏覽器二進位檔(例如 Brave、Vivaldi 或 Waterfox)。在 `dev`、`start` 與 `preview` 中使用二進位檔旗標與 `extension.config.*`。

## 運作方式

使用下列任一旗標:

- `--chromium-binary <path>`
- `--gecko-binary <path>`(別名:`--firefox-binary <path>`)

無論你選擇哪一個命名瀏覽器目標,這些二進位檔旗標都會覆寫 Extension.js 啟動的瀏覽器二進位檔。

## 二進位檔能力

| 選項 / 鍵                         | 作用                                              |
| --------------------------------- | ------------------------------------------------- |
| `--chromium-binary <path>`        | 啟動自訂 Chromium 家族瀏覽器二進位檔。            |
| `--gecko-binary <path>`           | 啟動自訂 Gecko 家族瀏覽器二進位檔。               |
| `--firefox-binary <path>`         | `--gecko-binary` 的別名。                         |
| `browser.<target>.chromiumBinary` | 在設定中設定預設的自訂 Chromium 二進位檔。        |
| `browser.<target>.geckoBinary`    | 在設定中設定預設的自訂 Gecko 二進位檔。           |
| `commands.<name>.chromiumBinary`  | 設定特定命令的自訂 Chromium 二進位檔。            |
| `commands.<name>.geckoBinary`     | 設定特定命令的自訂 Gecko 二進位檔。               |

### CLI 範例

```bash
extension dev --browser=chromium-based --chromium-binary="/path/to/brave"
```

```bash
extension dev --browser=firefox --gecko-binary="/path/to/firefox-developer-edition"
```

也可以搭配 `start` 與 `preview` 使用。

## 在 `extension.config.*` 中設定

```js
export default {
  browser: {
    "chromium-based": {
      chromiumBinary: "/path/to/custom-chromium-browser",
    },
    "gecko-based": {
      geckoBinary: "/path/to/custom-gecko-browser",
    },
  },
};
```

也可以把二進位檔路徑放在命令區塊中:

```js
export default {
  commands: {
    dev: {
      chromiumBinary: "/path/to/custom-chromium-browser",
    },
    preview: {
      geckoBinary: "/path/to/custom-gecko-browser",
    },
  },
};
```

## 目標映射行為

二進位檔提示會映射到引擎目標:

- `chromiumBinary` → `chromium-based`
- `geckoBinary` / `firefoxBinary` → `gecko-based`

若同時提供兩者,Extension.js 會優先解析 Chromium 二進位檔。

## 可用的瀏覽器

可用二進位檔旗標執行的常見瀏覽器:

| 瀏覽器名稱                    | 類型                  | CLI 旗標            | 官方網站                                                  |
| ----------------------------- | --------------------- | ------------------- | --------------------------------------------------------- |
| **Brave**                     | 基於 Chromium 的瀏覽器 | `--chromium-binary` | [brave.com](https://brave.com)                            |
| **Opera**                     | 基於 Chromium 的瀏覽器 | `--chromium-binary` | [opera.com](https://www.opera.com)                        |
| **Vivaldi**                   | 基於 Chromium 的瀏覽器 | `--chromium-binary` | [vivaldi.com](https://vivaldi.com)                        |
| **Waterfox**                  | 基於 Gecko 的瀏覽器    | `--gecko-binary`    | [Waterfox](https://www.waterfox.net)                      |
| **Firefox Developer Edition** | 基於 Gecko 的瀏覽器    | `--gecko-binary`    | [firefox.com](https://www.mozilla.org/firefox/developer/) |

## 重要限制

- `chromium-based` 需要有效的 `chromiumBinary` 路徑。
- `gecko-based` / `firefox-based` 需要有效的 `geckoBinary` 路徑。
- 無效路徑會立即失敗,並給出明確的 CLI/執行階段錯誤。
- `build` 不接受二進位檔旗標。僅 `dev`、`start` 與 `preview` 支援基於二進位檔的啟動方式。

## 最佳實務

- **將二進位檔搭配明確的瀏覽器目標使用**:使用 `--browser=chromium-based` 或 `--browser=gecko-based` 讓意圖更明確。
- **使用絕對路徑**:避免依賴 shell 的路徑解析問題。
- **在持續整合(CI)執行器中釘住版本**:讓瀏覽器二進位檔路徑在自動化檢查中保持一致。
- **謹慎組合 profile/旗標**:沿用與命名瀏覽器目標相同的 profile 與旗標策略。
