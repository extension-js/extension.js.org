透過受管理、持久或自訂 profile 控制開發期間的瀏覽器狀態隔離。

依你的工作流讓瀏覽器工作階段保持隔離或持久化。Extension.js 會以感知 profile 的預設值啟動瀏覽器。你可以選擇乾淨的執行、可重複使用的狀態,或明確的本地 profile 路徑。

## 運作方式

Extension.js 依下列順序選擇 profile 模式:

1. 明確的 `profile` 路徑(若有提供)
2. 受管理 profile 模式(預設)
   - 預設為暫時性(短暫)
   - 當 `persistProfile: true` 時持久化
3. 當 `EXTENSION_USE_SYSTEM_PROFILE=true` 時使用系統 profile 模式

## Profile 能力

| 設定 / 選項                         | 作用                                                            |
| ----------------------------------- | --------------------------------------------------------------- |
| `browser.<target>.profile`          | 為該瀏覽器目標使用明確的 profile 資料夾。                       |
| `commands.dev.profile`              | 僅為 `dev` 使用明確 profile。                                   |
| `commands.start.profile`            | 僅為 `start` 使用明確 profile。                                 |
| `commands.preview.profile`          | 僅為 `preview` 使用明確 profile。                               |
| `browser.<target>.persistProfile`   | 在該目標的多次執行間重複使用受管理 profile 狀態。               |
| `commands.<name>.persistProfile`    | 在特定命令情境中重複使用受管理 profile 狀態。                   |
| `--profile=/abs/path`               | 透過 CLI 覆寫,指定明確 profile 路徑。                          |
| `EXTENSION_USE_SYSTEM_PROFILE=true` | 使用作業系統/瀏覽器的系統 profile 而非受管理 profile。          |

## Profile 模式

| 模式                | 啟用方式                                        | 典型用途                                       |
| ------------------- | ----------------------------------------------- | ---------------------------------------------- |
| 受管理暫時          | 預設                                            | 狀態隔離的乾淨執行                             |
| 受管理持久          | `persistProfile: true`                          | 需要穩定瀏覽器狀態的迭代偵錯                   |
| 明確自訂 profile    | `profile: "/abs/path"` 或 `--profile=/abs/path` | 重複使用既有 profile                           |
| 系統 profile        | `EXTENSION_USE_SYSTEM_PROFILE=true`             | 以作業系統/瀏覽器預設 profile 啟動             |

Extension.js 會在以下位置建立受管理 profile:

- `dist/extension-js/profiles/<browser>-profile/<...>`

持久模式則使用:

- `dist/extension-js/profiles/<browser>-profile/dev`

## 在 `extension.config.*` 中設定

```js
export default {
  browser: {
    chrome: {
      // 使用你自己的 profile 資料夾:
      profile: "path/to/custom/profile",
    },
    firefox: {
      // 在多次工作階段間保留穩定的受管理 profile:
      persistProfile: true,
    },
  },
};
```

也可以依命令來限定 profile 預設值:

```js
export default {
  commands: {
    dev: {
      persistProfile: true,
    },
  },
};
```

## CLI 用法

直接使用明確 profile 路徑:

```bash
extension dev --browser=chrome --profile=/path/to/custom/profile
```

`start` 與 `preview` 用法類似。

## 生命週期說明

- Extension.js 為每次執行建立暫時性受管理 profile。
- Extension.js 在多次執行間重複使用持久受管理 profile(`dev`)。
- Extension.js 會自動移除過期的受管理 profile 資料夾。可設定 `EXTENSION_TMP_PROFILE_MAX_AGE_HOURS` 控制最長保留時間。

## 最佳實務

- **基準測試使用受管理暫時 profile**:減少隱藏狀態與不穩定的重現。
- **長時間偵錯工作階段使用 `persistProfile`**:在多次執行間保留驗證/工作階段/devtools 狀態。
- **依瀏覽器家族隔離自訂 profile**:避免跨瀏覽器污染。
- **有意識地使用系統 profile 模式**:適合重現問題,但隔離性不如受管理 profile。
