透過明確的瀏覽器目標設定流程,讓同一份擴充功能程式碼跨 Chrome、Edge、Firefox
與自訂執行檔執行。

## Chrome 擴充功能開發

開發 Chrome 擴充功能時,將目標設為 `chrome`(或設為 `chromium` 以使用預設的 Chromium 執行檔):`extension dev --browser=chrome`。Extension.js 會將擴充功能載入到一個全新、隔離的 profile,並套用一組合理的 [Chrome 啟動 flag](/zh-Hant/docs/browsers/browser-flags),讓重新載入行為可預期。

## Firefox 擴充功能開發

開發 Firefox 擴充功能時,將目標設為 `firefox`:`extension dev --browser=firefox`。Manifest V3 的 background 腳本會編譯為非持久性的 `scripts` 陣列(Firefox 不使用 `service_worker`),而 Firefox preferences 則取代了 Chromium 中 `chrome://flags` 的概念。請參考 [瀏覽器 preferences](/zh-Hant/docs/browsers/browser-preferences)。

## Edge 擴充功能開發

開發 Edge 擴充功能時,將目標設為 `edge`:`extension dev --browser=edge`。Edge 共用 Chromium 引擎,因此多數 flag 與 API 與 Chrome 相同,但 Extension.js 仍會輸出獨立的 `dist/edge` 產物以利發佈。

## 跨瀏覽器擴充功能開發

開發跨瀏覽器擴充功能時,只需在單一指令中執行多個目標(`extension dev --browser=chrome,firefox`),並透過 [帶有瀏覽器前綴的 manifest 欄位](/zh-Hant/docs/features/browser-specific-fields) 處理瀏覽器差異。一個專案、一份 `manifest.json`,每個瀏覽器都有獨立的產物。完整流程請參考 [跨瀏覽器相容性](/zh-Hant/docs/features/cross-browser-compatibility)。

## 實務上的目標策略

1. 日常檢查使用具名目標(`chrome`、`edge`、`firefox`)。
2. 發版驗證使用逗號分隔的多目標。
3. 只有在需要自訂執行檔時才使用引擎目標。
4. 把瀏覽器差異放在帶有瀏覽器前綴的 manifest 欄位裡。
