透過明確的瀏覽器目標工作流,在 Chrome、Edge、Firefox 與自訂二進位檔中執行同一份擴充功能程式碼庫。

## Chrome 擴充功能開發

對於 Chrome 擴充功能開發,目標是 `chrome`(或使用 `chromium` 來搭配預設 Chromium 二進位檔):`extension dev --browser=chrome`。Extension.js 會把擴充功能載入一個全新且隔離的 profile,並套用一組合理的 [Chrome 啟動旗標](/docs/browsers/browser-flags),讓重新載入行為可預期。

## Firefox 擴充功能開發

對於 Firefox 擴充功能開發,目標是 `firefox`:`extension dev --browser=firefox`。Manifest V3 背景指令稿會被編譯成非持久的 `scripts` 陣列(Firefox 不使用 `service_worker`),而 Firefox 偏好設定取代了 Chromium 中 `chrome://flags` 的概念。詳見[瀏覽器偏好設定](/docs/browsers/browser-preferences)。

## Edge 擴充功能開發

對於 Edge 擴充功能開發,目標是 `edge`:`extension dev --browser=edge`。Edge 共用 Chromium 引擎,因此大部分旗標與 API 與 Chrome 相同,但 Extension.js 仍會為發行產生獨立的 `dist/edge` 產出。

## 跨瀏覽器擴充功能開發

對於跨瀏覽器擴充功能開發,可在單一命令中執行多個目標(`extension dev --browser=chrome,firefox`),並透過[帶瀏覽器前置詞的 manifest 欄位](/docs/features/browser-specific-fields)保留瀏覽器差異。一個專案、一份 `manifest.json`、每個瀏覽器各自獨立的產出。完整流程請參見[跨瀏覽器相容性](/docs/features/cross-browser-compatibility)。

## 實用的目標策略

1. 日常檢查使用命名目標(`chrome`、`edge`、`firefox`)。
2. 發行驗證使用以逗號分隔的多目標。
3. 只有在需要自訂二進位檔時才使用引擎目標。
4. 透過帶瀏覽器前置詞的 manifest 欄位保留瀏覽器差異。
