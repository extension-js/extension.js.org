透過明確的瀏覽器目標工作流程，在 Chrome、Edge、Firefox 和自訂二進位檔上執行同一個擴充功能程式碼庫。

## Chrome 擴充功能開發

針對 Chrome 擴充功能開發，請以 `chrome` 為目標（或以 `chromium` 為預設 Chromium 二進位檔的目標）：`extension dev --browser=chrome`。Extension.js 會將擴充功能載入到全新、隔離的設定檔中，並套用一組合理的 [Chrome 啟動旗標](/docs/browsers/browser-flags)，讓重新載入的行為更具可預測性。

## Firefox 擴充功能開發

針對 Firefox 擴充功能開發，請以 `firefox` 為目標：`extension dev --browser=firefox`。Manifest V3 的背景指令稿會被編譯為非持久性的 `scripts` 陣列（Firefox 不使用 `service_worker`），並由 Firefox 偏好設定取代 Chromium 的 `chrome://flags` 概念。請參閱[瀏覽器偏好設定](/docs/browsers/browser-preferences)。

## Edge 擴充功能開發

針對 Edge 擴充功能開發，請以 `edge` 為目標：`extension dev --browser=edge`。Edge 共用 Chromium 引擎，因此大多數旗標與 API 都與 Chrome 一致，但 Extension.js 仍會輸出獨立的 `dist/edge` 產出物供發佈使用。

## 跨瀏覽器擴充功能開發

針對跨瀏覽器擴充功能開發，可在單一指令中執行多個目標（`extension dev --browser=chrome,firefox`），並透過[以瀏覽器為前綴的 manifest 欄位](/docs/features/browser-specific-fields)處理瀏覽器間的差異。一個專案、一份 `manifest.json`，並按瀏覽器產出不同的輸出。完整流程請參閱[跨瀏覽器相容性](/docs/features/cross-browser-compatibility)。

## 實用的目標策略

1. 日常檢查時使用具名目標（`chrome`、`edge`、`firefox`）。
2. 發佈驗證時使用以逗號分隔的目標。
3. 僅在需要自訂二進位檔時才使用引擎目標。
4. 透過以瀏覽器為前綴的 manifest 欄位處理瀏覽器差異。
