透過單一 CLI 工作流在主要瀏覽器中執行並測試你的擴充功能。

從同一個 CLI 在 Chrome、Edge、Firefox 與自訂瀏覽器二進位檔中驗證同一個擴充功能。

當你需要在同一個 Extension.js 專案中測試 Chrome 擴充功能、Firefox 擴充功能或 Edge 擴充功能時,可以參考此頁面。

## 選擇正確的目標

| 目標             | 適用情境                                            | 範例                                                                        |
| ---------------- | --------------------------------------------------- | --------------------------------------------------------------------------- |
| `chromium`       | 快速的預設本地開發                                  | `extension dev --browser=chromium`                                          |
| `chrome`         | 驗證 Chrome 特定行為                                | `extension dev --browser=chrome`                                            |
| `edge`           | 驗證 Edge 發行行為                                  | `extension dev --browser=edge`                                              |
| `firefox`        | 驗證 Gecko 相容性與 API                             | `extension dev --browser=firefox`                                           |
| `chrome,firefox` | 跨兩個主要引擎的發行檢查                            | `extension build --browser=chrome,firefox`                                  |
| `chromium-based` | 執行自訂 Chromium 二進位檔(Brave、Arc 等)         | `extension dev --browser=chromium-based --chromium-binary=/path/to/browser` |
| `gecko-based`    | 執行自訂 Firefox 家族二進位檔                       | `extension dev --browser=gecko-based --gecko-binary=/path/to/browser`       |
| `safari`         | 在 macOS 上建置 Safari 應用(需要 Xcode)           | `extension build --browser=safari`                                          |

## 運作方式

在 `dev`、`start`、`preview` 與 `build` 中使用 `--browser` 選擇目標。

若沒有指定瀏覽器,CLI 會預設使用 `chromium`。

`safari`(以及其別名 `webkit-based`)是個例外:它是一個 **alpha** 階段、僅支援 macOS 的建置目標,僅 `build` 與 `dev` 支援,`preview` 與 `start` 並不支援。詳見[建置 Safari 擴充功能](/docs/browsers/safari)。

## 支援的瀏覽器

命名瀏覽器目標:

| 瀏覽器       | 用法                                   |
| ------------ | -------------------------------------- |
| **Chrome**   | `npx extension dev --browser=chrome`   |
| **Edge**     | `npx extension dev --browser=edge`     |
| **Firefox**  | `npx extension dev --browser=firefox`  |
| **Chromium** | `npx extension dev --browser=chromium` |

基於引擎的目標(需要自訂二進位檔):

| 引擎目標                          | 用法                                                                            |
| --------------------------------- | ------------------------------------------------------------------------------- |
| **Chromium-based**                | `npx extension dev --browser=chromium-based --chromium-binary=/path/to/browser` |
| **Gecko-based**(`firefox-based`) | `npx extension dev --browser=gecko-based --gecko-binary=/path/to/browser`       |

Extension.js 在內部將 `firefox-based` 視為 Gecko 引擎目標。

## Safari 與其他 WebKit 目標

除了 Chromium 家族與 Firefox(Gecko 引擎)以外,Extension.js 還能把你的擴充功能建置成 macOS 上的 **Safari** 應用。

| 目標                          | 用法                                         |
| ----------------------------- | -------------------------------------------- |
| **Safari**                    | `npx extension build --browser=safari`       |
| **WebKit-based**(引擎別名)  | `npx extension build --browser=webkit-based` |

Safari 是**建置目標**:支援 `build` 與 `dev`,但不支援 `preview` 與 `start`(Safari 擴充功能無法被自動載入到執行中的瀏覽器)。它需要 macOS 與完整的 Xcode 應用。完整流程、需求以及如何在 Safari 中啟用擴充功能,請參見[建置 Safari 擴充功能](/docs/browsers/safari)。

## 多瀏覽器選擇

你可以在單一命令中執行多個命名瀏覽器:

```bash
npx extension dev --browser=chrome,firefox
```

使用以逗號分隔的值即可依序執行多個命名目標(例如 `--browser=chrome,edge,firefox`)。

## 限制與行為

- `chromium-based` 需要 `--chromium-binary`。
- `gecko-based` / `firefox-based` 需要 `--gecko-binary`。
- 基於引擎的目標會被導向相同的 Chromium/Firefox 執行器,並採用感知引擎的行為。

## 最佳實務

- **日常迭代使用命名瀏覽器**:`chrome`、`edge` 與 `firefox` 是日常測試最快的路徑。
- **有意識地使用基於引擎的模式**:只有在需要驗證自訂二進位檔時才使用 `chromium-based` / `gecko-based`。
- **依瀏覽器隔離 profile**:減少偵錯時的跨瀏覽器狀態外洩。
- **搭配瀏覽器特定欄位**:使用帶瀏覽器前置詞的 manifest 鍵來表達真正的行為差異。
