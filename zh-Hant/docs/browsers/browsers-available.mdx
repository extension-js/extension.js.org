透過單一 CLI 流程,在主要瀏覽器上執行與測試你的擴充功能。

在單一 CLI 中,於 Chrome、Edge、Firefox 與自訂瀏覽器執行檔上驗證同一份擴充功能。

當你需要在同一個 Extension.js 專案中測試 Chrome、Firefox 或 Edge 擴充功能時,就使用本頁。

## 選擇正確的目標

| 目標             | 適用時機                                            | 範例                                                                        |
| ---------------- | --------------------------------------------------- | --------------------------------------------------------------------------- |
| `chromium`       | 快速、預設的本機開發                                | `extension dev --browser=chromium`                                          |
| `chrome`         | 驗證 Chrome 專屬行為                                | `extension dev --browser=chrome`                                            |
| `edge`           | 驗證 Edge 發佈相關行為                              | `extension dev --browser=edge`                                              |
| `firefox`        | 驗證 Gecko 相容性與 API                             | `extension dev --browser=firefox`                                           |
| `chrome,firefox` | 兩大主流引擎的發版檢查                              | `extension build --browser=chrome,firefox`                                  |
| `chromium-based` | 執行自訂 Chromium 執行檔(Brave、Arc 等)           | `extension dev --browser=chromium-based --chromium-binary=/path/to/browser` |
| `gecko-based`    | 執行自訂 Firefox 系列執行檔                         | `extension dev --browser=gecko-based --gecko-binary=/path/to/browser`       |
| `safari`         | 在 macOS 上建置 Safari app(需要 Xcode)            | `extension build --browser=safari`                                          |

## 運作方式

在 `dev`、`start`、`preview` 與 `build` 中使用 `--browser` 來選擇目標。

如果你沒有指定瀏覽器,CLI 預設為 `chromium`。

`safari`(以及其別名 `webkit-based`)是個例外:它是 **alpha**、僅限 macOS 的建置目標,只支援 `build` 與 `dev` — 不支援 `preview` 或 `start`。請參考 [建置 Safari 擴充功能](/zh-Hant/docs/browsers/safari)。

## 支援的瀏覽器

具名瀏覽器目標:

| 瀏覽器       | 用法                                   |
| ------------ | -------------------------------------- |
| **Chrome**   | `npx extension dev --browser=chrome`   |
| **Edge**     | `npx extension dev --browser=edge`     |
| **Firefox**  | `npx extension dev --browser=firefox`  |
| **Chromium** | `npx extension dev --browser=chromium` |

引擎目標(需要自訂執行檔):

| 引擎目標                          | 用法                                                                            |
| --------------------------------- | ------------------------------------------------------------------------------- |
| **Chromium-based**                | `npx extension dev --browser=chromium-based --chromium-binary=/path/to/browser` |
| **Gecko-based**(`firefox-based`) | `npx extension dev --browser=gecko-based --gecko-binary=/path/to/browser`       |

Extension.js 內部會把 `firefox-based` 視為 Gecko 引擎目標。

## Safari 與其他 WebKit 目標

除了 Chromium 家族與 Firefox(Gecko 引擎)之外,Extension.js 也能在 macOS 上把你的擴充功能建置為 **Safari** app。

| 目標                            | 用法                                         |
| ------------------------------- | -------------------------------------------- |
| **Safari**                      | `npx extension build --browser=safari`       |
| **WebKit-based**(引擎別名)    | `npx extension build --browser=webkit-based` |

Safari 是 **建置目標**:`build` 與 `dev` 受支援,但 `preview` 與 `start` 不支援(Safari 擴充功能無法自動載入到執行中的瀏覽器)。它需要 macOS 與完整的 Xcode app。完整工作流程、需求,以及如何在 Safari 中啟用擴充功能,請參考 [建置 Safari 擴充功能](/zh-Hant/docs/browsers/safari)。

## 多瀏覽器選擇

你可以在單一指令中執行多個具名瀏覽器:

```bash
npx extension dev --browser=chrome,firefox
```

使用逗號分隔的值依序執行多個具名目標(例如 `--browser=chrome,edge,firefox`)。

## 限制與行為

- `chromium-based` 需要 `--chromium-binary`。
- `gecko-based` / `firefox-based` 需要 `--gecko-binary`。
- 引擎目標會路由到相同的 Chromium/Firefox runner,並帶有引擎感知的行為。

## 最佳實務

- **日常迭代使用具名瀏覽器**:`chrome`、`edge` 與 `firefox` 是日常測試最快的路徑。
- **有目的地使用引擎模式**:只有在驗證自訂執行檔時才優先使用 `chromium-based` / `gecko-based`。
- **保持各瀏覽器的 profile 獨立**:在除錯時減少跨瀏覽器的狀態外洩。
- **搭配瀏覽器專屬欄位使用**:用瀏覽器前綴的 manifest 鍵處理真正的行為差異。
