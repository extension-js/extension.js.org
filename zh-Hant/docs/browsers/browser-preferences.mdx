在開發期間取得可重現的瀏覽器行為(例如首頁預設值、devtools 設定或通知行為),
而不需修改擴充功能程式碼。

Extension.js 會從 `extension.config.*` 讀取 `preferences`,並在 Firefox 與 Gecko 瀏覽器啟動時套用。

## 運作方式

在 `extension.config.js`(或 `.mjs` / `.cjs`)中設定 preferences,可放在:

- `browser.<target>.preferences`
- `commands.dev|start|preview.preferences`

指令層級的值可覆寫瀏覽器預設值。

## Preference 能做什麼

| Preference 鍵                  | 用途                                                    |
| ------------------------------ | ------------------------------------------------------- |
| `browser.<target>.preferences` | 為特定瀏覽器目標設定預設 preferences。                  |
| `commands.dev.preferences`     | 為 `dev` 執行設定或覆寫 preferences。                   |
| `commands.start.preferences`   | 為 `start` 執行設定或覆寫 preferences。                 |
| `commands.preview.preferences` | 為 `preview` 執行設定或覆寫 preferences。               |

## Firefox 與 Gecko 系列行為

### 範例設定

```js
export default {
  browser: {
    firefox: {
      preferences: {
        "browser.startup.homepage": "https://developer.mozilla.org",
        "devtools.theme": "dark",
        "dom.webnotifications.enabled": false,
      },
    },
  },
};
```

在 Firefox/Gecko 流程中,Extension.js 會把 `user.js` 檔案寫入啟用中的 profile(受管或明確指定的 profile),並合併:

- 開發與 runtime 行為所需的內部基準 preferences。
- 你自訂的 `preferences` 值(發生鍵衝突時以你的值為準)。

如果你啟用系統 profile 模式(`EXTENSION_USE_SYSTEM_PROFILE=true`),Extension.js 不會寫入受管 profile 檔案。

## Chromium 家族行為

Chromium 家族的啟動(`chrome`、`edge`、`chromium`、`chromium-based`)主要使用 **瀏覽器 flag**。

`preferences` 仍可存在於合併後的設定物件中。然而,控制 Chromium 啟動行為的是 flag 與 profile 選項。Chromium 不使用 Firefox 風格的 `user.js` preference 檔。

對 Chromium 的自訂,建議使用:

- `browserFlags`
- `excludeBrowserFlags`
- `profile` / `persistProfile`

## 深色模式預設值

除非你已自行定義相關鍵,否則 Extension.js 會注入深色模式的預設值:

- Chromium 家族:深色模式啟動 flag
- Firefox/Gecko 家族:深色模式 preference 鍵(用於 UI 與內容色彩配置)

你明確設定的 `preferences`/flag 會覆寫這些預設值。

## 介面範例

```js
export default {
  commands: {
    dev: {
      browser: "firefox",
      preferences: {
        "devtools.theme": "dark",
      },
    },
  },
};
```

### 搭配自訂 profile 的範例

```js
export default {
  browser: {
    firefox: {
      profile: "path/to/custom-profile",
      preferences: {
        "browser.startup.homepage": "https://example.com",
      },
    },
  },
};
```

## 更詳盡的 preference 參考

可用的 Firefox preferences 清單十分完整,可以在 [Firefox 原始碼](https://searchfox.org/mozilla-central/source/) 中瀏覽。Mozilla 在 `all.js` 或 `firefox.js` 中定義了許多預設值。

## 最佳實務

- **優先使用瀏覽器作用域的 preferences**:把 Firefox/Gecko 的 preference 鍵放在瀏覽器目標的設定區塊裡。
- **臨時實驗用指令覆寫**:把短期的 preference 調整放在 `commands.dev`。
- **保持 profile 隔離**:用獨立 profile 取得可重現的除錯。
- **Chromium 調校使用 flag**:在 Chromium 家族目標上,把 flag 視為主要的設定介面。
