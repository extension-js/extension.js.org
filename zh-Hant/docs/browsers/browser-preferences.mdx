在開發期間獲得可重複的瀏覽器行為(例如首頁預設值、devtools 設定或通知行為),而不必修改擴充功能程式碼。

Extension.js 會讀取 `extension.config.*` 中的 `preferences`,並在啟動 Firefox 與 Gecko 瀏覽器時套用。

## 運作方式

在 `extension.config.js`(或 `.mjs` / `.cjs`)中以下位置設定偏好設定:

- `browser.<target>.preferences`
- `commands.dev|start|preview.preferences`

命令層級的值可以覆寫瀏覽器預設值。

## 偏好設定能力

| 偏好設定鍵                     | 作用                                                    |
| ------------------------------ | ------------------------------------------------------- |
| `browser.<target>.preferences` | 為特定瀏覽器目標設定預設偏好設定。                      |
| `commands.dev.preferences`     | 為 `dev` 執行設定或覆寫偏好設定。                       |
| `commands.start.preferences`   | 為 `start` 執行設定或覆寫偏好設定。                     |
| `commands.preview.preferences` | 為 `preview` 執行設定或覆寫偏好設定。                   |

## Firefox 與 Gecko 系瀏覽器的行為

### 範例設定

```js
export default {
  browser: {
    firefox: {
      preferences: {
        "browser.startup.homepage": "https://developer.mozilla.org",
        "devtools.theme": "dark",
        "dom.webnotifications.enabled": false,
      },
    },
  },
};
```

在 Firefox/Gecko 流程中,Extension.js 會在使用中的 profile(受管理或明確指定的 profile)裡寫入一個 `user.js` 檔案,並合併:

- 開發與執行階段行為所需的內部基準偏好設定。
- 你的自訂 `preferences` 值(當鍵衝突時,以你的值為準)。

如果你啟用了系統 profile 模式(`EXTENSION_USE_SYSTEM_PROFILE=true`),Extension.js 不會寫入受管理的 profile 檔案。

## Chromium 家族的行為

Chromium 家族的啟動(`chrome`、`edge`、`chromium`、`chromium-based`)主要使用**瀏覽器旗標**。

`preferences` 仍可以存在於合併後的設定物件中。但實際控制 Chromium 啟動行為的是旗標與 profile 選項。Chromium 不使用 Firefox 風格的 `user.js` 偏好設定檔。

對於 Chromium 客製化,優先使用:

- `browserFlags`
- `excludeBrowserFlags`
- `profile` / `persistProfile`

## 深色模式預設值

除非你已自行定義對應的鍵,Extension.js 會注入深色模式預設值:

- Chromium 家族:深色模式啟動旗標
- Firefox/Gecko 家族:深色模式偏好設定鍵(用於 UI 與內容配色)

你明確設定的 `preferences`/旗標會覆寫這些預設值。

## 介面範例

```js
export default {
  commands: {
    dev: {
      browser: "firefox",
      preferences: {
        "devtools.theme": "dark",
      },
    },
  },
};
```

### 自訂 profile 範例

```js
export default {
  browser: {
    firefox: {
      profile: "path/to/custom-profile",
      preferences: {
        "browser.startup.homepage": "https://example.com",
      },
    },
  },
};
```

## 更詳細的偏好設定參考

若要查看可用 Firefox 偏好設定的完整清單,可瀏覽 [Firefox 原始碼](https://searchfox.org/mozilla-central/source/)。Mozilla 在 `all.js` 或 `firefox.js` 中定義了許多預設值。

## 最佳實務

- **優先使用瀏覽器作用域的偏好設定**:把 Firefox/Gecko 偏好設定鍵放在針對特定瀏覽器的設定區塊中。
- **用命令覆寫做暫時性實驗**:把短期的偏好設定調整放在 `commands.dev` 中。
- **保持 profile 隔離**:使用獨立的 profile 以便可重現偵錯。
- **在 Chromium 上用旗標做調整**:把旗標作為 Chromium 家族目標的主要設定面。
