在開發期間取得可重複的瀏覽器行為（例如首頁預設值、開發者工具設定或通知行為），
而無需修改擴充功能程式碼。

Extension.js 會從 `extension.config.*` 讀取 `preferences`，並在啟動 Firefox 與 Gecko 瀏覽器時套用。

## 運作方式

在 `extension.config.js`（或 `.mjs` / `.cjs`）的下列位置設定偏好：

- `browser.<target>.preferences`
- `commands.dev|start|preview.preferences`

指令層級的值可以覆寫瀏覽器預設值。

## 偏好設定功能

| 偏好鍵                         | 作用                                                    |
| ------------------------------ | ------------------------------------------------------- |
| `browser.<target>.preferences` | 為特定瀏覽器目標設定預設偏好。                          |
| `commands.dev.preferences`     | 為 `dev` 執行設定或覆寫偏好。                           |
| `commands.start.preferences`   | 為 `start` 執行設定或覆寫偏好。                         |
| `commands.preview.preferences` | 為 `preview` 執行設定或覆寫偏好。                       |

## Firefox 與 Gecko 系列的行為

### 設定範例

```js
export default {
  browser: {
    firefox: {
      preferences: {
        "browser.startup.homepage": "https://developer.mozilla.org",
        "devtools.theme": "dark",
        "dom.webnotifications.enabled": false,
      },
    },
  },
};
```

在 Firefox/Gecko 流程中，Extension.js 會在使用中的設定檔（受管理或明確指定的設定檔）中寫入 `user.js` 檔案，並合併：

- 開發與執行階段行為所需的內部基準偏好。
- 你自訂的 `preferences` 值（鍵衝突時以你的值為主）。

若啟用系統設定檔模式（`EXTENSION_USE_SYSTEM_PROFILE=true`），Extension.js 不會寫入受管理的設定檔檔案。

## Chromium 系列的行為

Chromium 系列啟動（`chrome`、`edge`、`chromium`、`chromium-based`）主要使用**瀏覽器旗標**。

`preferences` 仍可存在於合併後的設定物件中，但實際上是由旗標與設定檔選項來控制 Chromium 的啟動行為。Chromium 並不使用 Firefox 風格的 `user.js` 偏好檔。

如需自訂 Chromium，建議使用：

- `browserFlags`
- `excludeBrowserFlags`
- `profile` / `persistProfile`

## 深色模式預設值

除非你已自行定義對應的鍵值，Extension.js 會注入深色模式預設值：

- Chromium 系列：深色模式啟動旗標
- Firefox/Gecko 系列：深色模式偏好鍵（用於 UI 與內容色彩配置）

你明確設定的 `preferences`／旗標會覆寫這些預設值。

## 介面範例

```js
export default {
  commands: {
    dev: {
      browser: "firefox",
      preferences: {
        "devtools.theme": "dark",
      },
    },
  },
};
```

### 搭配自訂設定檔的範例

```js
export default {
  browser: {
    firefox: {
      profile: "path/to/custom-profile",
      preferences: {
        "browser.startup.homepage": "https://example.com",
      },
    },
  },
};
```

## 更詳盡的偏好參考

如需完整的 Firefox 偏好清單，可瀏覽 [Firefox 原始碼](https://searchfox.org/mozilla-central/source/)。Mozilla 在 `all.js` 或 `firefox.js` 中定義了許多預設值。

## 最佳實務

- **優先採用瀏覽器層級的偏好設定**：將 Firefox/Gecko 的偏好鍵放在以瀏覽器為目標的設定區塊中。
- **以指令覆寫進行臨時實驗**：將短期的偏好調整放在 `commands.dev` 中。
- **保持設定檔的隔離**：使用獨立設定檔以利可重現的偵錯。
- **以旗標調校 Chromium**：在 Chromium 系列目標上，將旗標視為主要的設定介面。
