Extension.js 會在您 manifest 旁探索語系檔案，並驗證每個已宣告的語系都有 `messages.json`。它會將語系 JSON 資產輸出到特定瀏覽器的建置中。在開發期間，任何語系檔案的編輯都會被捕捉，且不需要完整重新啟動。

## 範本範例

### `action-locales`

![action-locales 範本截圖](/images/examples/action-locales/screenshot.png)

透過 `_locales` 支援查看本地化的擴充功能後設資料與 UI 字串。

## 語系能力

| 能力                 | 為您提供的功能                                                       |
| -------------------- | -------------------------------------------------------------------- |
| 語系探索             | 從 manifest 位置偵測 `_locales/<locale>/messages.json`               |
| 驗證                 | 捕捉缺少的預設語系檔案與未解析的 `__MSG_*__` 索引鍵                 |
| 建置輸出對應         | 將語系檔案輸出至預期的擴充功能輸出結構中                             |
| 開發監看支援         | 在開發期間於語系檔案變更時重新載入                                   |

## 預期的結構

```plaintext
manifest.json
_locales/
  en/
    messages.json
  fr/
    messages.json
```

`manifest.json` 中的 `default_locale` 應對應到既有的 `_locales/<default>/messages.json`。

## `manifest.json` 中的範例語系宣告

以下是如何在 `manifest.json` 中宣告語系：

```json
{
  "manifest_version": 3,
  "name": "My Extension",
  "version": "1.0.0",
  "default_locale": "en",
  "description": "__MSG_extension_description__"
}
```

接著您會在 `_locales` 資料夾中為每種語系包含 JSON 檔案：

```plaintext
_locales/
└── en/
    └── messages.json
```

## 範例 `messages.json` 檔案

翻譯用的範例 `messages.json` 檔案：

```json
{
  "extension_name": {
    "message": "My Extension"
  },
  "extension_description": {
    "message": "This is a localized description of my extension."
  }
}
```

## 輸出路徑

Extension.js 將語系 JSON 檔案輸出至：

```plaintext
_locales/<locale>/messages.json
```

## 開發行為

- Extension.js 將語系 JSON 檔案加入編譯相依性並監看它們。
- 語系變更會觸發擴充功能重新載入行為（硬重新載入），而非元件式的 hot module replacement（HMR）。
- 當所需的語系檔案缺失或無效時，Extension.js 會以可採取行動的診斷訊息驗證失敗。

## 驗證行為

Extension.js 驗證：

- 當專案使用 `_locales` 時 `default_locale` 是否存在
- `_locales/<default>/messages.json` 是否存在
- 語系檔案的 JSON 有效性
- 對照預設語系索引鍵的 manifest 中 `__MSG_*__` 參照

### 排查缺少的語系索引鍵

若您的 manifest 使用 `__MSG_extension_description__`，請確認預設語系檔案包含 `extension_description`：

```json
{
  "extension_description": {
    "message": "Localized extension description"
  }
}
```

若預設語系沒有定義該索引鍵，Extension.js 會出現診斷訊息以說明此不一致。

## 最佳實務

- 在各語系中保持 `messages.json` 索引鍵一致。
- 先更新預設語系，再將索引鍵推廣至其他語系。
- 在持續整合（CI）中驗證語系 JSON，以在打包前捕捉格式錯誤的檔案。
- 將語系檔案放在靠近 manifest（`manifestDir/_locales`）的位置，以便可預測地解析。
