讓擴充功能保有能力,同時不索取超過功能所需的特權。

Extension.js 會編譯並驗證你的擴充功能,但瀏覽器仍會強制執行 `manifest.json` 中的 `permissions`、`host_permissions` 與 `optional_*` 欄位。良好的權限設計能提升信任,並減少應用商店審核中的問題。

## 權限策略

| 需求                          | 優先使用                          |
| --------------------------- | ----------------------------- |
| 擴充功能正常運作所需的核心 API 存取           | `permissions`                 |
| 安裝時所需的主機存取                  | `host_permissions`            |
| 可稍後再請求的能力                   | `optional_permissions`        |
| 僅部分使用者需要的網域存取                | `optional_host_permissions`   |

## 常見模式

### 背景驅動的功能

使用 `permissions` 宣告所需的瀏覽器 API,並僅加入功能所需的主機模式:

```json
{
  "manifest_version": 3,
  "permissions": ["storage", "tabs", "scripting"],
  "host_permissions": ["https://example.com/*"]
}
```

### 可選能力

如果首次啟動體驗不依賴某項功能,將其設為可選:

```json
{
  "manifest_version": 3,
  "permissions": ["storage"],
  "optional_permissions": ["notifications"],
  "optional_host_permissions": ["https://api.example.com/*"]
}
```

## 實用規則

- 只在功能確實需要時申請 API 權限。
- 將 `host_permissions` 限制在可運作的最小來源集合中。
- 將次要或進階功能優先放入可選權限。
- 每次新增背景行為、內容指令稿注入或遠端 API 呼叫時,重新檢視權限。

## 依功能類型的權限設計

| 功能                              | 通常涉及                                                   |
| ------------------------------- | ------------------------------------------------------ |
| 在已知網站上的內容指令稿增強                    | 針對該網站的 `host_permissions`,有時還需要 `scripting`               |
| 僅 popup 或 options UI              | 通常無需任何 host 權限                                         |
| 程式化注入                            | `scripting` 加上對應的 host 權限                              |
| 跨分頁工作流                          | `tabs`,有時還需要 `activeTab`                              |
| 持久化設定                            | `storage`                                              |
| 通知或標記                            | `notifications` 與 `action` 相關的 manifest 欄位             |

## 常見錯誤

- 在只需要一兩個來源時,卻使用 `<all_urls>` 這類過於寬泛的萬用字元。
- 在原可採用更狹窄的使用者觸發流程時,把 `host_permissions` 加入功能中。
- 忘記內容指令稿、web 可存取資源,以及透過 `chrome.scripting` 注入指令稿三者各自有不同的安全意涵。
- 在功能文件中只列出權限,卻未說明每個權限存在的原因。

## 複查清單

1. 列出每一項面向使用者的功能。
2. 將每項功能對應到它所需的 API 與主機權限。
3. 將非核心權限盡可能移至可選權限。
4. 移除舊實驗殘留的過時權限。
5. 修改後,重新測試安裝提示與瀏覽器商店預期行為。
