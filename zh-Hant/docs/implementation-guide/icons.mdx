透過在 Extension.js 可驗證、改寫並可預期輸出的 manifest 欄位中宣告圖示,讓擴充功能品牌與 action UI 保持一致。

Extension.js 會處理 `manifest.json` 中的圖示路徑,解析 public/相對路徑,輸出圖示資源,並在開發期間監看圖示檔案。

## 範本範例

![action 範本截圖](/images/examples/action/screenshot.png)

## 圖示能力

| 能力                | 你可以獲得什麼                                       |
| ----------------- | --------------------------------------------- |
| 支援 manifest 圖示欄位  | 驗證並編譯支援之擴充功能介面的圖示                              |
| 路徑解析              | 一致處理相對路徑與 public 根路徑                          |
| 開發期 watch 更新       | 在本機開發時針對圖示變動重新編譯                              |
| 瀏覽器安全輸出           | 在擴充功能產物中輸出圖示資源,參考保持穩定                          |

## 支援的圖示欄位

| Manifest 欄位                 | 預期的檔案類型             |
| ----------------------------- | --------------------- |
| `action.default_icon`         | .png、.jpg、.svg      |
| `browser_action.default_icon` | .png、.jpg、.svg      |
| `icons`                       | .png、.jpg、.svg      |
| `page_action.default_icon`    | .png、.jpg、.svg      |
| `sidebar_action.default_icon` | .png、.jpg、.svg (\*) |
| `browser_action.theme_icons`  | .png、.jpg、.svg      |

<Note>
  目前部分瀏覽器對 `sidebar_action.default_icon` 的 `.svg`
  支援仍不完整。在此情境使用 SVG 前請確認瀏覽器相容性。
</Note>

## `manifest.json` 中的圖示宣告範例

```json
{
  "manifest_version": 3,
  "name": "My Extension",
  "version": "1.0.0",
  "icons": {
    "16": "icons/icon-16.png",
    "48": "icons/icon-48.png",
    "128": "icons/icon-128.png"
  },
  "action": {
    "default_icon": "icons/action-icon.png"
  }
}
```

## 輸出路徑

典型的圖示輸出:

```plaintext
icons/<filename>
browser_action/<theme-icon-filename>   # for theme_icons
```

## 路徑行為

- Extension.js 從 manifest 所在資料夾解析相對圖示路徑。
- 以 `/` 開頭以及 `public/...` 都依擴充功能 public 根語意解析。
- Extension.js 可以監看 public 資料夾中的資源,而不需要透過圖示功能本身重新輸出它們。

## 開發期行為

- 修改既有的圖示檔案會觸發重新編譯。
- 修改 manifest 中的圖示入口參考可能需要重新啟動開發伺服器。
- 缺少必要的圖示檔案會產生建置錯誤(部分可選圖示群組可能只是發出警告)。

## 最佳實務

- 明確宣告 manifest 圖示欄位,而非依賴偶然的資源匯入。
- 提供多種圖示尺寸(`16`、`32`、`48`、`128`),讓 UI 在各瀏覽器介面上更銳利。
- 保持圖示檔名穩定,以減少開發期間的 manifest 變動。
- 有意識地使用 public 根路徑,並測試產生的 manifest 輸出路徑。
