## 本節涵蓋什麼

- 僅看 CLI 文件容易忽略的撰寫契約。
- 由原始檔驅動、會影響擴充功能載入與重新載入方式的執行時行為。
- 即使在封裝後仍然重要的瀏覽器擴充功能架構問題,例如權限、訊息傳遞、儲存,以及背景、內容指令稿與頁面情境之間的隔離。
- 無論你是手動開發或與 AI 編碼代理協作,都適用的實用指引。

## 重要的心智模型

Extension.js 在編譯、路徑規範化、manifest 改寫與開發體驗方面提供協助。它不會消除底層的瀏覽器擴充功能邊界:

- 內容指令稿仍受頁面情境限制。
- 背景 service worker 仍有生命週期限制。
- Web 可存取資源仍需明確公開。
- 權限仍決定擴充功能能做什麼。
- 訊息傳遞與儲存仍需明確設計。

## 開發新功能的建議閱讀順序

1. 從 [manifest.json](/docs/implementation-guide/manifest-json) 開始,理解你正在宣告的擴充功能介面。
2. 閱讀對應的功能頁面,例如 [內容指令稿](/docs/implementation-guide/content-scripts) 或 [背景](/docs/implementation-guide/background)。
3. 補齊配套內容:[權限](/docs/implementation-guide/permissions-and-host-permissions)、[訊息傳遞](/docs/implementation-guide/messaging)、[儲存](/docs/implementation-guide/storage) 與 [web 可存取資源](/docs/implementation-guide/web-accessible-resources)。
4. 在 [可預期的路徑解析](/docs/features/path-resolution) 中驗證路徑假設。
5. 在進入 watch 模式後修改入口前,先查看 [開發期更新行為](/docs/workflows/dev-update-behavior)。
