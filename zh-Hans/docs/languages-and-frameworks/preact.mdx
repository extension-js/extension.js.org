让扩展 UI 的体积更小，同时保留类 React 的开发体验和快速本地迭代。

Extension.js 会从依赖中检测 Preact。它会自动配置 JSX/TSX 转换、React 兼容性别名以及 fast-refresh 支持。你的 React 导入会通过这些别名映射到 Preact。

## Preact 适合什么场景

- 你想为 popup/sidebar/新标签页等界面减小 UI bundle 体积。
- 你喜欢 React 风格的组件，但想要更轻量的运行时。
- 你在为低端设备上的扩展启动速度与 UI 响应性能做优化。

## 在已有扩展中使用

按下面的步骤把 Preact 加到一个已有扩展中。

### 安装

安装所需依赖：

<PackageManagerTabs command="install preact" />

Preact 自带 TypeScript 类型，因此你不需要单独安装 `@types/preact` 包。

### 配置

Extension.js 期望 Preact 文件使用以下扩展名：

- 如果你不启用 TypeScript：`*.jsx`
- 如果你启用 TypeScript：`*.tsx`

## 开发期行为

当 Extension.js 检测到 Preact 时，会配置：

- 兼容性别名（例如 `react` → `preact/compat`）。
- 针对 Preact 调整的 JSX 处理。
- 通过 `@rspack/plugin-preact-refresh` 在开发期集成 refresh。

如果项目缺少可选的 refresh 依赖，Extension.js 会替你安装并提示重启。

### 排错

- **缺少 refresh 依赖的警告：** 安装 `@rspack/plugin-preact-refresh`（如有提示，再安装相关包）。
- **安装后提示重启：** 停止并重新运行 `extension dev`，让开发服务器干净地重新加载 refresh 接线。
- **没有检测到 Preact 集成：** 确认 `preact` 出现在 `dependencies` 或 `devDependencies` 中。

## 使用示例

### 在新标签页扩展中

要在新标签页扩展中使用 Preact，把它作为 `<script>` 引入 HTML 文件：

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>New Extension</title>
  </head>
  <body>
    <noscript>You need to enable JavaScript to run this extension.</noscript>
    <div id="root"></div>
  </body>
  <script src="./index.tsx"></script>
</html>
```

```tsx
import { h, render } from "preact";
import App from "./App";

const root = document.getElementById("root");

render(<App />, root);
```

```tsx
export default function App() {
  return <h1>Hello, Preact Extension!</h1>;
}
```

### 在 `content_script` 文件中

在 content scripts 中，通过创建 HTML 元素并渲染到其中，把 Preact 注入页面：

```tsx
import { h, render } from "preact";
import App from "./App";
import "./content.css";

const rootDiv = document.createElement("div");
rootDiv.id = "extension-root";
document.body.appendChild(rootDiv);

render(<App />, rootDiv);
```
