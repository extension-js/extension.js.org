构建更小的扩展 UI 产物,同时保留类似 React 的开发体验和快速的本地迭代。

Extension.js 会从你的依赖中检测到 Preact,自动配置 JSX/TSX 转换、React 兼容别名以及 fast-refresh 支持。你的 React 导入将通过这些别名映射到 Preact。

## 何时适合使用 Preact

- 你希望弹窗/侧边栏/新标签页等界面有更小的 UI 产物体积。
- 你喜欢 React 风格的组件,但希望更轻量的运行时。
- 你正在优化扩展在低端设备上的启动速度与 UI 响应性。

## 在已有扩展中使用

按以下步骤将 Preact 添加到现有扩展中。

### 安装

安装所需依赖:

<PackageManagerTabs command="install preact" />

Preact 自带 TypeScript 类型,因此不需要单独安装 `@types/preact` 包。

### 配置

Extension.js 期望 Preact 文件使用以下扩展名:

- 如果你没有启用 TypeScript:`*.jsx`
- 如果你启用了 TypeScript:`*.tsx`

## 开发行为

当 Extension.js 检测到 Preact 时,它会配置:

- 兼容别名(例如 `react` → `preact/compat`)。
- 针对 Preact 调整过的 JSX 处理。
- 开发时基于 `@rspack/plugin-preact-refresh` 的 refresh 集成。

如果你的项目缺少可选的 refresh 依赖,Extension.js 会自动安装并要求你重启。

### 故障排查

- **缺少 refresh 依赖警告:** 安装 `@rspack/plugin-preact-refresh`(以及若有提示的相关包)。
- **安装后提示重启:** 停止并重新运行 `extension dev`,让开发服务器能干净地重新加载 refresh 配线。
- **未检测到 Preact 集成:** 确认 `preact` 出现在 `dependencies` 或 `devDependencies` 中。

## 使用示例

### 在新标签页扩展中

要在新标签页扩展中使用 Preact,在 HTML 文件中以 `<script>` 引入:

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>New Extension</title>
  </head>
  <body>
    <noscript>You need to enable JavaScript to run this extension.</noscript>
    <div id="root"></div>
  </body>
  <script src="./index.tsx"></script>
</html>
```

```tsx
import { h, render } from "preact";
import App from "./App";

const root = document.getElementById("root");

render(<App />, root);
```

```tsx
export default function App() {
  return <h1>Hello, Preact Extension!</h1>;
}
```

### 在 `content_script` 文件中

对于内容脚本,通过创建一个 HTML 元素并向其中渲染来在页面注入 Preact:

```tsx
import { h, render } from "preact";
import App from "./App";
import "./content.css";

const rootDiv = document.createElement("div");
rootDiv.id = "extension-root";
document.body.appendChild(rootDiv);

render(<App />, rootDiv);
```
