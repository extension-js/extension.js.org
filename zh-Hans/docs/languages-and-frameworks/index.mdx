为你的扩展界面选择合适的语言和 UI 框架策略，然后利用 Extension.js 的默认配置将搭建成本降到最低。

## 按目标选择

| 目标                                | 从这里开始                                                                               |
| ----------------------------------- | ---------------------------------------------------------------------------------------- |
| 强类型保障和更安全的重构            | [TypeScript](/docs/languages-and-frameworks/typescript)                                  |
| 拥有大型生态的丰富 UI 组件          | [React](/docs/languages-and-frameworks/react)                                            |
| 更小的类 React 运行时               | [Preact](/docs/languages-and-frameworks/preact)                                          |
| Vue 单文件组件开发工作流            | [Vue.js](/docs/languages-and-frameworks/vue)                                             |
| 现代模块语法                        | [ECMAScript modules](/docs/languages-and-frameworks/ecmascript-modules)                  |
| 默认作用域化的样式                  | [CSS modules](/docs/languages-and-frameworks/css-modules)                                |
| Sass 或 Less 样式工作流             | [Sass](/docs/languages-and-frameworks/sass), [Less](/docs/languages-and-frameworks/less) |
| 浏览器安全的 Node 兼容性决策        | [Node APIs in browser extensions](/docs/languages-and-frameworks/node)                   |
| 重性能负载的 Wasm 任务              | [WebAssembly](/docs/languages-and-frameworks/webassembly)                                |

## 推荐进阶路线

1. 选择语言/运行时（`TypeScript`、`React`、`Vue` 或 `Preact`）。
2. 选择样式方案（`CSS modules`、`Sass` 或 `Less`）。
3. 仅在需要时再添加构建/运行时约束（`Node APIs`、`WebAssembly`）。
