使用 Sass 通过变量、嵌套和 SCSS modules 来扩展浏览器扩展样式,同时保持单一的构建流水线。

Extension.js 通过基于 Rspack 的样式流水线支持 `.scss` 和 `.sass`,以及 SCSS 模块变体(`.module.scss`、`.module.sass`)。

## 何时适合使用 Sass

- 你的设计体系已经依赖 Sass 变量和 mixin。
- 你的组件驱动型扩展 UI 需要模块化样式。
- 你希望与 Web 应用中已有的 Sass 工作流保持一致。

## 在已有扩展中使用

按以下步骤将 Sass 添加到现有扩展中。

### 在 HTML 文件中使用示例

在扩展页面(弹窗/选项页/新标签页)中,通过入口脚本引入 Sass:

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>New Extension</title>
    <script src="./main.ts"></script>
  </head>
  <body>
    <h1 class="title">Hello, Extension.</h1>
  </body>
</html>
```

```ts
import "./styles/globals.scss";

console.log("Sass loaded");
```

## Sass modules

像 CSS modules 一样,用 Sass modules 在本地为样式设置作用域。使用 `.module.scss` 或 `.module.sass` 即可启用作用域,避免样式表中的命名冲突。

## 在已有扩展中使用 Sass modules

要启用 Sass modules,将你的 Sass 文件重命名为带有 `.module.scss` 或 `.module.sass` 的后缀。这会自动将样式作用域限制到你的组件。

```diff
- button.scss
+ button.module.scss
```

### 使用示例

重命名 Sass 文件后,在脚本中导入:

```ts
import styles from "./styles/button.module.scss";

const element = document.createElement("h1");
element.className = styles.primaryColor;
element.innerText = "Hello, Extension!";
document.body.appendChild(element);
```

在 Sass 文件中:

```scss
/* styles/button.module.scss */

$primary-color: #64ff00;

:export {
  primaryColor: $primary-color;
}
```

## React/Preact 使用示例

将 Sass module 导入到你的 React 或 Preact 组件中,并使用作用域化的类名:

```jsx
import styles from "./styles/button.module.scss";

export default function MyComponent() {
  return <h1 className={styles.primaryColor}>Hello, Extension!</h1>;
}
```

## Vue 使用示例

在 Vue SFC 中,推荐使用 `lang="scss"` 的模块样式:

```vue
<template>
  <h1 :class="$style.primaryColor">Hello, Extension!</h1>
</template>

<style lang="scss" module>
@import "./styles/button.module.scss";
</style>
```

## Svelte 使用示例

在 Svelte 中,导入 Sass module 的映射并应用类名:

```svelte
<script>
  import styles from "./styles/button.module.scss";
</script>

<h1 class={styles.primaryColor}>Hello, Extension!</h1>
```

## 行为说明

- 在扩展页面中,`.module.scss` 和 `.module.sass` 会为 JS/TS 导入导出类名映射。
- 在内容脚本中,Extension.js 会将 Sass 输出为 CSS 资源,不带 JavaScript 类名映射。
- 如果你的项目缺少 Sass 工具链,Extension.js 会安装可选依赖并要求重启。
