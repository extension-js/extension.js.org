使用 Sass 通过变量、嵌套和 SCSS modules 扩展浏览器扩展的样式能力，同时保持单一的构建工作流。

Extension.js 通过基于 Rspack 的样式管线支持 `.scss` 和 `.sass`，以及 SCSS 模块变体（`.module.scss`、`.module.sass`）。

## 适合使用 Sass 的场景

- 你的设计系统已经依赖 Sass 变量和 mixin。
- 你需要为组件驱动的扩展 UI 提供模块化样式。
- 你希望与 Web 应用中已有的 Sass 工作流保持一致。

## 模板示例

### `new-sass`

![new-sass template screenshot](/images/examples/new-sass/screenshot.png)

启动一个已配置 Sass 支持的新标签页扩展。

<CodeGroup>

```bash npm
npx extension@latest create my-extension --template=new-sass
```

```bash pnpm
pnpx extension@latest create my-extension --template=new-sass
```

```bash yarn

```

```bash bun

```

```bash bun

```

</CodeGroup>

## 在现有扩展中使用

按以下步骤为现有扩展添加 Sass 支持。

### 安装

安装所需依赖：

<PackageManagerTabs command="install -D sass sass-loader" />

### 在 HTML 文件中使用示例

在扩展页面（弹窗/选项页/新标签页）中，通过入口脚本导入 Sass：

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>New Extension</title>
    <script src="./main.ts"></script>
  </head>
  <body>
    <h1 class="title">Hello, Extension.</h1>
  </body>
</html>
```

```ts
import "./styles/globals.scss";

console.log("Sass loaded");
```

## Sass modules

像 CSS Modules 一样使用 Sass modules，可对样式进行本地作用域化。使用 `.module.scss` 或 `.module.sass` 即可启用作用域并避免样式表中的命名冲突。

## 在现有扩展中使用 Sass modules

要启用 Sass modules，请将 Sass 文件重命名为包含 `.module.scss` 或 `.module.sass`。这会自动将样式作用域限定到你的组件中。

```diff
- button.scss
+ button.module.scss
```

### 用法示例

重命名 Sass 文件后，将其导入到脚本中：

```ts
import styles from "./styles/button.module.scss";

const element = document.createElement("h1");
element.className = styles.primaryColor;
element.innerText = "Hello, Extension!";
document.body.appendChild(element);
```

在 Sass 文件中：

```scss
/* styles/button.module.scss */

$primary-color: #64ff00;

:export {
  primaryColor: $primary-color;
}
```

## React/Preact 用法示例

在 React 或 Preact 组件中导入 Sass 模块，并使用作用域化的类名：

```jsx
import styles from "./styles/button.module.scss";

export default function MyComponent() {
  return <h1 className={styles.primaryColor}>Hello, Extension!</h1>;
}
```

## Vue 用法示例

在 Vue SFC 中，优先使用带 `lang="scss"` 的 module 样式：

```vue
<template>
  <h1 :class="$style.primaryColor">Hello, Extension!</h1>
</template>

<style lang="scss" module>
@import "./styles/button.module.scss";
</style>
```

## Svelte 用法示例

在 Svelte 中，导入 Sass 模块映射并应用类名：

```svelte
<script>
  import styles from "./styles/button.module.scss";
</script>

<h1 class={styles.primaryColor}>Hello, Extension!</h1>
```

## 行为说明

- 在扩展页面中，`.module.scss` 和 `.module.sass` 会为 JS/TS 导入导出类映射。
- 在内容脚本中，Extension.js 会将 Sass 作为 CSS 资源输出，但不会附带 JavaScript 类名映射。
- 如果项目缺少 Sass 工具链，Extension.js 会安装可选依赖并提示你重启。
