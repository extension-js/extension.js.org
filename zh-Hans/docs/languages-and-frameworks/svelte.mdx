用 Svelte 组件构建扩展 UI，同时把框架接线交给 Extension.js 处理。

Extension.js 会从项目依赖中检测 Svelte，并自动配置 `.svelte` 解析、loader 设置以及 Svelte client 别名。

## Svelte 适合什么场景

- 你希望为 popup、options、sidebar 或新标签页 UI 使用更小的运行时。
- 你倾向于组件优先、语法简洁的开发方式。
- 你想要一个对扩展页面与 content-script UI 挂载都适用的框架方案。

<PackageManagerTabs command="install -D svelte-loader typescript" />

## 开发期行为

当 Extension.js 检测到 Svelte 时，会：

- 启用 `.svelte` 文件解析。
- 在 JS 框架管线中配置 Svelte loader。
- 应用 Svelte client 别名（例如 `svelte`、`svelte/store` 与 `svelte/reactivity`）。

如果项目缺少可选工具，Extension.js 会按需安装并提示重启。

## 使用示例

### 在扩展页面中

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>Svelte Extension</title>
  </head>
  <body>
    <div id="app"></div>
  </body>
  <script src="./main.ts"></script>
</html>
```

```ts
import App from "./App.svelte";

new App({
  target: document.getElementById("app")!,
});
```

### 在 `content_script` 文件中

在 content scripts 中，创建一个挂载节点，并从默认导出返回一个清理函数。这样 Extension.js 在开发期间可以安全地重新挂载：

```ts
import App from "./App.svelte";

export default function main() {
  const target = document.createElement("div");
  target.id = "extension-root";
  document.body.appendChild(target);

  const app = new App({ target });

  return () => {
    app.$destroy();
    target.remove();
  };
}
```

## 最佳实践

- 让 Svelte 组件专注于 UI，把浏览器 API 的编排放进专门的模块。
- 在 content scripts 中始终返回清理函数，让重新挂载的行为保持可预期。
- 先在扩展页面中使用 Svelte，等功能边界清晰后再扩展到 content-script UI。
