使用 Svelte 组件构建扩展 UI，同时由 Extension.js 负责框架接线。

Extension.js 会从项目依赖中检测 Svelte，并自动配置 `.svelte` 解析、loader 以及 Svelte 客户端别名。

## 适合使用 Svelte 的场景

- 你希望为弹窗、选项页、侧边栏或新标签页 UI 使用更小的运行时。
- 你倾向以组件为先、语法简洁的编写方式。
- 你希望选择的框架既能用于扩展页面，也能用于内容脚本 UI 挂载点。

## 在现有扩展中使用

安装 Svelte：

<PackageManagerTabs command="install svelte" />

对于在现有项目中进行显式配置，还需安装 Svelte loader：

<PackageManagerTabs command="install -D svelte-loader typescript" />

## 开发行为

当 Extension.js 检测到 Svelte 时，会：

- 启用 `.svelte` 文件解析。
- 在 JS 框架管线中配置 Svelte loader。
- 应用 Svelte 客户端别名（例如 `svelte`、`svelte/store` 和 `svelte/reactivity`）。

如果项目缺少可选工具链，Extension.js 会自动安装所需依赖并提示你重启。

## 用法示例

### 在扩展页面中

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>Svelte Extension</title>
  </head>
  <body>
    <div id="app"></div>
  </body>
  <script src="./main.ts"></script>
</html>
```

```ts
import App from "./App.svelte";

new App({
  target: document.getElementById("app")!,
});
```

### 在 `content_script` 文件中

对于内容脚本，创建挂载节点，并从默认导出中返回一个清理函数。这样 Extension.js 可以在开发模式下安全地重新挂载：

```ts
import App from "./App.svelte";

export default function main() {
  const target = document.createElement("div");
  target.id = "extension-root";
  document.body.appendChild(target);

  const app = new App({ target });

  return () => {
    app.$destroy();
    target.remove();
  };
}
```

## 最佳实践

- 让 Svelte 组件专注于 UI，将浏览器 API 编排逻辑放到独立的模块中。
- 在内容脚本中始终返回清理函数，使重新挂载行为保持可预测。
- 先在扩展页面中使用 Svelte，等到功能边界明确后再扩展到内容脚本 UI。
