通过一套同时支持模块作用域样式的统一流水线,在浏览器扩展页面和 UI 组件中使用 Less CSS。

Extension.js 会检测 Less 的使用,为 Rspack 构建配置预处理器,并在浏览器扩展中同时支持 `.less` 和 `.module.less` 文件。

## 何时适合使用 Less

- 你正在迁移现有的基于 Less 的 Web 代码库。
- 你希望以较低的迁移成本获得变量与嵌套。
- 你需要模块作用域样式,同时保留 Less 的写法约定。

## 在已有扩展中使用

按以下步骤将 Less 添加到现有扩展中。

### 安装

安装所需依赖:

<PackageManagerTabs command="install -D less less-loader" />

### 在 HTML 文件中使用示例

在扩展页面(弹窗/选项页/新标签页)中,通过脚本入口引入 Less:

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>New Extension</title>
    <script src="./main.ts"></script>
  </head>
  <body>
    <h1 class="title">Hello, Extension.</h1>
  </body>
</html>
```

```ts
import "./styles/globals.less";

console.log("Less loaded");
```

### 在 `content_script` 文件中

```ts
import "./styles/content.less";

console.log("Content styles loaded");
```

## 浏览器扩展中的 Less modules

像 CSS modules 一样,用 Less modules 在本地为类名设置作用域。这能默认避免全局样式冲突,而这一点在扩展中尤为重要,因为内容脚本会运行在宿主页面的 CSS 作用域之内。

![content-less-modules 模板截图](/images/examples/content-less-modules/screenshot.png)

## 在已有扩展中使用 Less modules

要启用 Less modules,将 `.less` 文件重命名为带 `.module.less` 后缀。这会自动为类名启用本地作用域。

```diff
- myStyles.less
+ myStyles.module.less
```

### 使用示例

重命名 Less 文件后,你可以在脚本中导入它:

```ts
import styles from "./styles/myStyles.module.less";

const element = document.createElement("h1");
element.className = styles.primary;
element.innerText = "Hello, Extension!";
document.body.appendChild(element);
```

## React/Preact 使用示例

要在 React 或 Preact 中使用 Less modules,只需导入 Less module,并像在任何组件中那样应用样式:

```jsx
import styles from "./styles/myStyles.module.less";

export default function MyComponent() {
  return <h1 className={styles.primary}>Hello, Extension!</h1>;
}
```

## Vue 使用示例

在 Vue 中,推荐使用 `lang="less"` 和 `module` 的单文件组件(SFC)样式:

```vue
<template>
  <h1 :class="$style.primary">Hello, Extension!</h1>
</template>

<style lang="less" module>
@import "./styles/myStyles.module.less";
</style>
```

## Svelte 使用示例

在 Svelte 中,导入 Less module 的映射并应用类名:

```svelte
<script>
  import styles from "./styles/myStyles.module.less";
</script>

<h1 class={styles.primary}>Hello, Extension!</h1>
```

## 行为说明

- 在扩展页面上下文中,`.module.less` 会按预期导出类名映射。
- 在内容脚本中,Extension.js 会将样式作为 CSS 资源输出,并以样式表的形式注入页面。
- 如果你的项目缺少 Less 工具链,Extension.js 会安装可选依赖并要求重启。
