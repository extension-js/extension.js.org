浏览器扩展是一种为浏览器增加功能的软件。扩展可以改变页面渲染方式,也可以在浏览器 chrome 内部添加新的 UI:工具栏、弹窗、侧边栏和新标签页。它们还可以响应网络与标签页事件,并调用普通网页无法使用的特权浏览器 API。

Extension.js 是一个 JavaScript 框架,用于从单个项目跨 Chrome、Firefox 和 Edge 构建浏览器扩展。

## 浏览器扩展能做什么

具体的 API 覆盖面因浏览器而异,但大多数扩展会做以下中的一项或多项:

- **改变页面内容。** 将 CSS 或脚本注入到特定 URL(内容脚本)。
- **添加浏览器 UI。** 渲染来自工具栏 action 的弹窗、侧边栏、选项页或自定义新标签页。
- **响应浏览器事件。** 监听标签更新、导航、下载、闹钟,或来自页面的消息。
- **存储数据。** 读写跨会话保留的扩展作用域存储。
- **调用特权 API。** 管理 cookie、声明式网络请求、书签、历史、下载或通知,受 `permissions` 控制。

扩展不能做用户未授权的事,浏览器也会将扩展沙箱隔离,远离原始的操作系统访问。

## 浏览器扩展文件

每个浏览器扩展都是静态资源和代码的文件夹,经过签名和打包以便分发:

| 文件或文件夹     | 用途                                                                              |
| ---------------- | --------------------------------------------------------------------------------- |
| `manifest.json`  | 声明扩展的名称、版本、权限、入口点和元数据。                                      |
| 后台脚本         | 长期存活的事件处理程序。Manifest V3 在 Chromium 上使用 service worker。           |
| 内容脚本         | 注入到匹配网页的代码。                                                            |
| HTML 页面        | 弹窗、选项、侧边栏或新标签页 UI。                                                 |
| 图标和资源       | 工具栏图标、区域字符串及其他静态资源。                                            |

Extension.js 把你的源代码(`.ts`、`.tsx`、`.jsx`、`.vue`、`.svelte`、CSS、Less、Sass)编译为这种磁盘布局,并按浏览器目标产出独立文件夹。

## 扩展中的 JavaScript、TypeScript、HTML 和 CSS

浏览器扩展使用与 Web 相同的语言构建。JavaScript 和 TypeScript 驱动逻辑。HTML 定义扩展页面。CSS(或 Less、Sass、CSS modules、Tailwind)为它们添加样式。Extension.js 通过 Rspack 并使用对扩展感知的默认值将这一切串联起来,你不必为每个浏览器配置打包器。参见 [浏览器扩展中的 JavaScript 与 TypeScript 文件](/docs/concepts/javascript-typescript-browser-extension-files) 了解哪个文件扩展名落在哪里。

## Chrome 扩展与 Firefox 扩展

Chrome 和 Firefox 扩展共享 WebExtensions API,但每次构建时都需要注意几处差异:

- **后台:** Chrome(Manifest V3)需要 `background.service_worker`。Firefox 使用带有非持久化事件页面的 `background.scripts`。
- **Runtime API:** Firefox 原生暴露 `browser.*`。Chromium 使用 `chrome.*`。可以在 Chromium 上使用 `--polyfill` 弥合差异。
- **权限:** 浏览器对几个 `permissions` 和 `host_permissions` 条目的评估方式不同。
- **分发:** Chrome 通过 Chrome Web Store 分发,Firefox 通过 addons.mozilla.org,Edge 通过 Edge Add-ons store。

Extension.js 通过 [浏览器特定的清单字段](/docs/features/browser-specific-fields) 和按目标构建输出来处理这些差异。参见 [跨浏览器兼容性](/docs/features/cross-browser-compatibility) 了解完整流水线。

## Extension.js 如何帮助你

如果没有框架,构建浏览器扩展意味着手写 `manifest.json` 并为每个浏览器配置打包器。你还要手动连接保存即重载、polyfill API,并为 Chrome、Firefox 和 Edge 产出独立构件。Extension.js 替你完成这些:

- 一个项目、一个 `manifest.json`、独立的 `dist/<browser>` 输出。
- TypeScript、React、Vue、Svelte、Preact 和现代 CSS 无需配置即可工作。
- `extension dev` 在保存时重载扩展、内容脚本和 HTML 页面。
- `extension build` 按浏览器目标产出可签名发布的构件。
