浏览器扩展用 JavaScript、TypeScript、HTML、CSS 与一份 `manifest.json` 来定义行为、UI、权限以及浏览器集成。本页讲解你会在浏览器扩展项目里看到的文件扩展名，以及它们分别落到哪里。

如果你想了解的是"扩展"作为 Chrome 或 Firefox 附加软件的概念，请先看[什么是浏览器扩展？](/zh-Hans/docs/concepts/what-is-a-browser-extension)。

## JavaScript 文件扩展名：`.js` 与 `.mjs`

浏览器扩展接受与 Web 平台其他部分一致的 JavaScript 文件扩展名：

| 文件扩展名 | 含义                                                                              |
| ---------- | --------------------------------------------------------------------------------- |
| `.js`      | 普通 JavaScript。按加载方式被当作 ES 模块或经典脚本。                              |
| `.mjs`     | ES 模块。当你想强制使用模块语义时（尤其是 service worker 中）很有用。               |
| `.cjs`     | CommonJS。扩展源代码中不常见；偶尔出现在 `extension.config.cjs` 里。               |

在 Manifest V3 中，只有当 `manifest.json` 的 `background` 块包含 `"type": "module"` 时，后台 `service_worker` 才以模块形式运行。详情见 [Manifest V3 排错](/zh-Hans/docs/concepts/manifest-v3)。

## TypeScript 文件扩展名：`.ts` 与 `.tsx`

TypeScript 在 Extension.js 中是一等的源代码语言：

| 文件扩展名 | 含义                                              |
| ---------- | ------------------------------------------------- |
| `.ts`      | 不含 JSX 的 TypeScript。                          |
| `.tsx`     | 含 JSX 的 TypeScript（React、Preact）。           |
| `.d.ts`    | 类型声明文件，不会被输出到构建产物中。              |

你不需要从零开始写 `tsconfig.json`。Extension.js 提供了合理的默认值。`chrome.*`、`browser.*`、`import.meta.env` 以及公开 env 键的类型来自 `@types/chrome` 与 Extension.js 自带的环境类型。详见 [TypeScript](/zh-Hans/docs/languages-and-frameworks/typescript)。

## React 文件扩展名：`.jsx` 与 `.tsx`

浏览器扩展中的 React 使用标准的 JSX 文件扩展名：

| 文件扩展名 | 含义                       |
| ---------- | -------------------------- |
| `.jsx`     | 含 JSX 的 JavaScript。     |
| `.tsx`     | 含 JSX 的 TypeScript。     |

React 既可用于扩展页面（popup、options、侧边栏、新标签页），也可用于注入到页面中的 content script。配置方式与 shadow DOM 模式见 [React](/zh-Hans/docs/languages-and-frameworks/react)。

## 浏览器扩展文件：`manifest.json`、后台、content script、页面

除了 JavaScript 与 TypeScript 源码以外，浏览器扩展文件夹通常还包含：

| 文件或目录      | 用途                                                              |
| --------------- | ----------------------------------------------------------------- |
| `manifest.json` | 声明名称、版本、权限、入口与元数据。                              |
| 后台入口         | 长时间存活的事件处理者。Chromium 上 Manifest V3 的 service worker。 |
| Content script  | 注入到匹配 URL 的页面中的代码。                                    |
| 扩展页面         | popup、options、侧边栏或新标签页 HTML。                           |
| 本地化文件       | `_locales/<lang>/messages.json` 中的翻译字符串。                  |
| 图标与资源       | 工具栏图标、web 可访问资源、字体、图片。                          |

Extension.js 把你的 `.ts`、`.tsx`、`.jsx`、`.vue`、`.svelte`、`.css`、`.less`、`.scss` 与 `.module.*` 源码编译成上述磁盘布局，并按浏览器目标分别产出文件夹（`dist/chrome`、`dist/firefox`、`dist/edge`）。

## Extension.js 如何编译 JavaScript 与 TypeScript 扩展

当你运行 `extension dev` 或 `extension build` 时，Extension.js 会：

1. 读取 `manifest.json`，找到每一个入口（后台、content script、popup、options、侧边栏、新标签页、web 可访问 HTML）。
2. 解析这些入口引用的源文件，包括跨 `.ts`、`.tsx`、`.jsx`、`.vue`、`.svelte` 与各类样式表的 import。
3. 通过 Rspack 编译，并应用面向扩展的默认值：在能拆分代码的地方拆分，在浏览器不接受的地方（service worker、content script）不拆分。
4. 把结果输出到 `dist/<browser>`，并附带按目标过滤后的 manifest。

你只需用上述任意一种文件扩展名编写源码，Extension.js 负责打包、polyfill、保存即重载循环与按浏览器打包。
