浏览器扩展使用 JavaScript、TypeScript、HTML、CSS 以及 `manifest.json` 来定义行为、UI、权限和浏览器集成。本页介绍你在浏览器扩展项目中会看到的文件扩展名，以及它们各自的归宿。

如果你想了解的是“扩展”作为 Chrome 或 Firefox 软件附加组件的含义，请改从 [什么是浏览器扩展？](/docs/concepts/what-is-a-browser-extension) 开始。

## JavaScript 文件扩展名：`.js` 与 `.mjs`

浏览器扩展接受与 Web 平台其他部分相同的 JavaScript 文件扩展名：

| 文件扩展名 | 含义                                                                                                |
| ---------- | --------------------------------------------------------------------------------------------------- |
| `.js`      | 普通 JavaScript。根据加载方式被视作 ES 模块或经典脚本。                                             |
| `.mjs`     | ES 模块。在希望强制使用模块语义时很有用，尤其是在 service worker 中。                                |
| `.cjs`     | CommonJS。在扩展源代码中很少见；偶尔出现在 `extension.config.cjs` 中。                              |

在 Manifest V3 中，当 `manifest.json` 的 `background` 块包含 `"type": "module"` 时，后台 `service_worker` 以模块方式运行。详见 [Manifest V3 排错](/docs/concepts/manifest-v3)。

## TypeScript 文件扩展名：`.ts` 与 `.tsx`

TypeScript 在 Extension.js 中是一等公民的源语言：

| 文件扩展名 | 含义                                            |
| ---------- | ----------------------------------------------- |
| `.ts`      | 不含 JSX 的 TypeScript。                        |
| `.tsx`     | 包含 JSX 的 TypeScript（React、Preact）。       |
| `.d.ts`    | 类型声明文件。不会输出到构建产物中。            |

你不需要从零开始编写 `tsconfig.json`。Extension.js 提供合理的默认值。`chrome.*`、`browser.*`、`import.meta.env` 以及公开环境变量键的类型来自 `@types/chrome` 和 Extension.js 自带的环境类型。参见 [TypeScript](/docs/languages-and-frameworks/typescript)。

## React 文件扩展名：`.jsx` 与 `.tsx`

浏览器扩展中的 React 使用标准的 JSX 文件扩展名：

| 文件扩展名 | 含义                       |
| ---------- | -------------------------- |
| `.jsx`     | 包含 JSX 的 JavaScript。   |
| `.tsx`     | 包含 JSX 的 TypeScript。   |

React 既可用于扩展页面（popup、options、侧边栏、新标签页），也可用于注入到网页中的内容脚本。设置和 shadow DOM 模式参见 [React](/docs/languages-and-frameworks/react)。

## 浏览器扩展文件：`manifest.json`、后台、内容脚本、页面

除了 JavaScript 和 TypeScript 源码外，浏览器扩展文件夹通常还包含：

| 文件或目录       | 用途                                                              |
| ---------------- | ----------------------------------------------------------------- |
| `manifest.json`  | 声明名称、版本、权限、入口点和元数据。                            |
| 后台入口         | 长期运行的事件处理器。Chromium 上 Manifest V3 使用 service worker。|
| 内容脚本         | 注入到匹配指定 URL 模式的网页中的代码。                           |
| 扩展页面         | popup、options、侧边栏或新标签页的 HTML。                         |
| 本地化           | `_locales/<lang>/messages.json` 中的翻译字符串。                   |
| 图标与资源       | 工具栏图标、可被网页访问的资源、字体、图片。                       |

Extension.js 会将你的 `.ts`、`.tsx`、`.jsx`、`.vue`、`.svelte`、`.css`、`.less`、`.scss` 和 `.module.*` 源码编译为这种磁盘布局。它会按浏览器目标分别生成单独的文件夹（`dist/chrome`、`dist/firefox`、`dist/edge`）。

## Extension.js 如何编译 JavaScript 和 TypeScript 扩展

当你运行 `extension dev` 或 `extension build` 时，Extension.js 会：

1. 读取 `manifest.json` 并找到每一个入口点（后台、内容脚本、popup、options、侧边栏、新标签页、可被网页访问的 HTML）。
2. 解析这些入口引用的源文件，包括跨 `.ts`、`.tsx`、`.jsx`、`.vue`、`.svelte` 以及样式表类型的导入。
3. 通过 Rspack 编译，并应用面向扩展的默认设置：在有益的地方做代码拆分，在浏览器不允许的地方（service worker、内容脚本）不进行分块。
4. 将结果输出到 `dist/<browser>`，并附带为该目标过滤过的清单。

你可以使用上述任意一种文件扩展名编写源码。Extension.js 负责处理打包、polyfill、保存即重载的循环以及按浏览器打包。
