使用一套现代化的工作流，为 Chrome、Edge 和 Firefox 构建浏览器扩展。Extension.js 负责处理 manifest 编译、浏览器特定的产物输出、reload 行为以及打包，让你专注于产品功能而非构建工具。

继续使用你熟悉的 Web 工具，如 [TypeScript](/docs/languages-and-frameworks/typescript)、[React](/docs/languages-and-frameworks/react)、[Vue](/docs/languages-and-frameworks/vue) 与 [Tailwind CSS](/docs/integrations/tailwindcss)，同时直接访问原生扩展 API。

<Note>
  在对比工具？查看 Extension.js 与 WXT 等其他框架的对比，请参阅 [Extension.js vs WXT](/docs/compare/extension-js-vs-wxt)。
</Note>

<Info>
  **CLI 版本：**运行 `extension --version`（或 `npx extension@latest --version`）查看你当前使用的构建版本。[npm 上的 extension 包](https://www.npmjs.com/package/extension) 列出最新发布。[GitHub releases](https://github.com/extension-js/extension.js/releases) 以变更日志风格记录说明，且不会让这些文档绑定到具体的补丁版本。
</Info>

## 选择合适的命令

| 目标                              | 命令     | 示例                                                            |
| ---------------------------------- | --------- | --------------------------------------------------------------- |
| 搭建项目脚手架                     | `create`  | `npx extension@latest create my-extension --template=new-react` |
| 以 watch 模式进行开发              | `dev`     | `extension dev --browser=firefox`                               |
| 构建生产产物                       | `build`   | `extension build --browser=chrome,firefox --zip`                |
| 构建并启动生产产物                 | `start`   | `extension start --browser=edge`                                |
| 启动已有的构建产物                 | `preview` | `extension preview --browser=chrome`                            |

使用 `create` 命令搭建新项目，并通过 `--template` 从某个具体技术栈的基线开始。

<CodeGroup>

```bash npm
npx extension@latest create <your-extension-name>
```

```bash pnpm
pnpx extension@latest create <your-extension-name>
```

```bash yarn
yarn dlx extension@latest create <your-extension-name>
```

```bash bun
bunx extension@latest create <your-extension-name>
```

```bash bun
bunx extension@latest create <your-extension-name>
```

</CodeGroup>

要查看所有受支持模板的列表，请浏览[模板](/docs/getting-started/templates)页面。

## 在现有扩展中使用 Extension.js

如果你已经有一个扩展，安装 `extension` 包并一次性配置好脚本即可。这能让本地开发、测试和发布构建保持一致。

### 第 1 步：将 `extension` 包安装为 `devDependency`

<CodeGroup>

```bash npm
npm install extension@latest --save-dev
```

```bash pnpm
pnpm add extension@latest --save-dev
```

```bash yarn
yarn add extension@latest --dev
```

```bash bun
bun add --dev extension@latest
```

```bash bun
bun add --dev extension@latest
```

</CodeGroup>

### 第 2 步：将你的 npm 脚本链接到 `extension` 命令

```json package.json
{
  "scripts": {
    "build": "extension build",
    "dev": "extension dev",
    "start": "extension start"
  },
  "devDependencies": {
    "extension": "latest"
  }
}
```

完成。你的扩展已就绪可用。

- 运行 `npm run dev` 进行日常迭代和 watch 模式开发。
- 运行 `npm run start` 进行生产构建并立即启动。
- 运行 `npm run build` 生成可上架的生产产物。

## 最佳实践

- **保持一条命令流：**默认循环使用 `create` → `dev` → `build`。
- **显式指定目标浏览器：**在发布前使用 `--browser=chrome,firefox` 进行验证。
- **集中管理默认值：**将共享的命令/浏览器设置放入 `extension.config.js`。
