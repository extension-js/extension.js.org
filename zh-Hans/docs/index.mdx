用一个现代化的工作流为 Chrome、Edge 和 Firefox 构建浏览器扩展。Extension.js 处理 manifest 编译、面向特定浏览器的输出、重新加载行为以及打包。专注于产品功能，而不是构建工具。

使用你熟悉的 Web 工具，如 [TypeScript](/docs/languages-and-frameworks/typescript)、[React](/docs/languages-and-frameworks/react)、[Vue](/docs/languages-and-frameworks/vue) 和 [Tailwind CSS](/docs/integrations/tailwindcss)。同时保留对原生扩展 API 的直接访问。

<Note>
  在比较工具？参见 Extension.js 与 WXT 及其他框架的对比：[Extension.js vs WXT](/docs/compare/extension-js-vs-wxt)。
</Note>

<Info>
  **CLI 版本：** 运行 `extension --version`（或 `npx extension@latest
  --version`）查看你正在使用的构建版本。 [npm 上的 extension 包](https://www.npmjs.com/package/extension) 列出了最新发布。
  [GitHub releases](https://github.com/extension-js/extension.js/releases) 以变更日志风格记录说明，且不会让文档绑定到特定补丁号。
</Info>

## 选择正确的命令

| 目标                               | 命令       | 示例                                                            |
| ---------------------------------- | --------- | --------------------------------------------------------------- |
| 脚手架创建项目                      | `create`  | `npx extension@latest create my-extension --template=new-react` |
| 以监听模式开发                      | `dev`     | `extension dev --browser=firefox`                               |
| 构建生产产物                        | `build`   | `extension build --browser=chrome,firefox --zip`                |
| 构建并启动生产输出                  | `start`   | `extension start --browser=edge`                                |
| 启动已有的构建输出                  | `preview` | `extension preview --browser=chrome`                            |

使用 `create` 命令搭建新项目，并通过 `--template` 选择一个特定技术栈的基线模板开始。

<CodeGroup>

```bash npm
npx extension@latest create <your-extension-name>
```

```bash pnpm
pnpx extension@latest create <your-extension-name>
```

```bash yarn
yarn dlx extension@latest create <your-extension-name>
```

```bash bun
bunx extension@latest create <your-extension-name>
```

```bash bun
bunx extension@latest create <your-extension-name>
```

</CodeGroup>

要查看所有支持模板的列表，请浏览 [模板](/docs/getting-started/templates) 页面。

如果你已经有一个扩展，安装 `extension` 包并一次性配置好脚本即可。这能让本地开发、测试和发布构建保持一致。

### 步骤 1：将 `extension` 包安装为 `devDependency`

<CodeGroup>

```bash npm
npm install extension@latest --save-dev
```

```bash pnpm
pnpm add extension@latest --save-dev
```

```bash yarn
yarn add extension@latest --dev
```

```bash bun
bun add --dev extension@latest
```

```bash bun
bun add --dev extension@latest
```

</CodeGroup>

### 步骤 2：将你的 npm 脚本链接到 `extension` 命令

```json package.json
{
  "scripts": {
    "build": "extension build",
    "dev": "extension dev",
    "start": "extension start"
  },
  "devDependencies": {
    "extension": "latest"
  }
}
```

完成。你的扩展已经可以使用了。

- 运行 `npm run dev` 进行日常迭代和监听模式开发。
- 运行 `npm run start` 进行生产构建并立即启动。
- 运行 `npm run build` 生成可上架的生产产物。

## 最佳实践

- **保持单一命令流：** 以 `create` → `dev` → `build` 作为默认循环。
- **明确目标浏览器：** 发布前用 `--browser=chrome,firefox` 进行验证。
- **集中管理默认配置：** 把共享的命令/浏览器设置放到 `extension.config.js` 中。
