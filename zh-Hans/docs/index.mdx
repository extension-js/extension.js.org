用一套现代工作流为 Chrome、Edge 和 Firefox 构建浏览器扩展。Extension.js 帮你处理 manifest 编译、按浏览器分别产出、重载行为以及打包。把精力放在产品功能上，而不是构建工具上。

继续使用熟悉的 Web 工具，例如 [TypeScript](/zh-Hans/docs/languages-and-frameworks/typescript)、[React](/zh-Hans/docs/languages-and-frameworks/react)、[Vue](/zh-Hans/docs/languages-and-frameworks/vue) 和 [Tailwind CSS](/zh-Hans/docs/integrations/tailwindcss)，并保留对原生扩展 API 的直接访问。

<Note>
  在做工具选型？在 [Extension.js vs WXT](/zh-Hans/docs/compare/extension-js-vs-wxt) 中查看 Extension.js 与 WXT 等框架的对比。
</Note>

<Info>
  **CLI 版本：** 运行 `extension --version`（或 `npx extension@latest --version`）查看当前使用的版本。[npm 上的 extension 包](https://www.npmjs.com/package/extension)显示最新的发布版本。[GitHub releases](https://github.com/extension-js/extension.js/releases) 以类似更新日志的形式记录变化，不需要把文档绑定到具体的补丁号上。
</Info>

## 选择合适的命令

| 目标                               | 命令      | 示例                                                              |
| ---------------------------------- | --------- | --------------------------------------------------------------- |
| 创建项目脚手架                     | `create`  | `npx extension@latest create my-extension --template=new-react` |
| 在 watch 模式下开发                | `dev`     | `extension dev --browser=firefox`                               |
| 构建生产产物                       | `build`   | `extension build --browser=chrome,firefox --zip`                |
| 构建并启动生产产物                 | `start`   | `extension start --browser=edge`                                |
| 启动已有的构建产物                 | `preview` | `extension preview --browser=chrome`                            |

使用 `create` 命令创建一个新项目，并通过 `--template` 从特定技术栈的基线开始。

<CodeGroup>

```bash npm
npx extension@latest create <your-extension-name>
```

```bash pnpm
pnpx extension@latest create <your-extension-name>
```

```bash yarn
yarn dlx extension@latest create <your-extension-name>
```

```bash bun
bunx extension@latest create <your-extension-name>
```

```bash bun
bunx extension@latest create <your-extension-name>
```

</CodeGroup>

如果你已经有一个扩展，安装 `extension` 包并一次性接好脚本即可。这能让本地开发、测试与发布构建保持一致。

### 第 1 步：把 `extension` 包安装为 `devDependency`

<CodeGroup>

```bash npm
npm install extension@latest --save-dev
```

```bash pnpm
pnpm add extension@latest --save-dev
```

```bash yarn
yarn add extension@latest --dev
```

```bash bun
bun add --dev extension@latest
```

```bash bun
bun add --dev extension@latest
```

</CodeGroup>

### 第 2 步：把你的 npm 脚本指向 `extension` 命令

```json package.json
{
  "scripts": {
    "build": "extension build",
    "dev": "extension dev",
    "start": "extension start"
  },
  "devDependencies": {
    "extension": "latest"
  }
}
```

完成。你的扩展已经可以使用了。

- 运行 `npm run dev` 进行日常迭代和 watch 模式。
- 运行 `npm run start` 进行生产构建并立即启动。
- 运行 `npm run build` 生成可上架的生产产物。

## 最佳实践

- **保持单一命令流程：** 默认循环使用 `create` → `dev` → `build`。
- **显式指定目标浏览器：** 发布前用 `--browser=chrome,firefox` 验证。
- **集中默认值：** 把共享的命令/浏览器设置放在 `extension.config.js` 里。
