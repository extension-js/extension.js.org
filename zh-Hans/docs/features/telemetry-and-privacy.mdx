Extension.js 收集极少量的匿名遥测数据,用以了解你运行了哪些命令以及哪些命令失败。它从不收集源代码、文件路径、URL 或项目内容。

隐私门槛被有意设得很严格:

- 总共两类事件:`command_executed` 和 `command_failed`
- 每个事件三个属性:`command`、`success`、`version`
- 通过环境变量、CLI 标志或持久化的同意命令选择退出
- Extension.js 对事件进行采样并限量,以充分保持在 [PostHog](https://posthog.com/)(开源分析)免费层之内

## Extension.js 收集的内容

每次 CLI 运行,至多发送以下之一:

| 事件               | 是否采样                                  | 属性                                  |
| ------------------ | ---------------------------------------- | -------------------------------------- |
| `command_executed` | 20%(可配置,见下文)                     | `command`、`success: true`、`version`  |
| `command_failed`   | 100%(Extension.js 始终发送失败)         | `command`、`success: false`、`version` |

附加在每个事件上的通用上下文:`os`(`darwin`/`linux`/`win32`)、`arch`、`node_major`、`is_ci`。仅此而已。

## 量级控制

三个独立的控制项限制离开机器的数据量:

- **采样:** Extension.js 默认按 20% 对 `command_executed` 采样。可通过 `EXTENSION_TELEMETRY_SAMPLE_RATE` 覆盖(0.0–1.0)。失败事件永不采样。
- **每次运行上限:** 每个 CLI 进程至多 **3 个事件**。可通过 `EXTENSION_TELEMETRY_MAX_EVENTS` 覆盖。
- **去重(抑制重复):** Extension.js 在 60 秒内丢弃重复的 `(event, command, success)` 元组。可通过 `EXTENSION_TELEMETRY_DEBOUNCE_MS` 覆盖。

## Extension.js 从不收集的内容

Extension.js 遥测契约明确排除:

- 源代码、manifest 内容、HTML 输出或 `package.json` 内容
- 仓库名称、Git remote、GitHub 组织/用户名、分支名、提交 SHA 或预览 URL
- 依赖列表、权限列表或自由格式的项目标识符
- 环境变量值、文件系统路径或机器本地 URL
- 堆栈跟踪、错误消息或自由文本错误名
- IP 地址(Extension.js 在每个载荷上将 `$ip` 设为 `null`)

## 选择退出

三种方式禁用遥测,按优先级顺序列出:

```bash
# 1. Environment variable (wins over everything else)
EXTENSION_TELEMETRY_DISABLED=1 extension dev   # preferred
EXTENSION_TELEMETRY=0 extension dev            # back-compat, also honored

# 2. Per-run flag
extension dev --no-telemetry

# 3. Persistent consent file via the telemetry command
extension telemetry disable
extension telemetry enable
extension telemetry status    # show current state (default when no argument)
```

同意文件位于 `$XDG_CONFIG_HOME/extensionjs/telemetry/consent`(或对应平台的等价位置)。

## 默认行为

遥测是**选择退出制**。在没有上述任一覆盖项的首次运行中,Extension.js 会打印一行说明,告诉你如何禁用它。它还会记录一个 `enabled` 同意标记,以避免提示重复出现。

持续集成 (CI) 环境遵循相同规则。在 CI 环境中设置 `EXTENSION_TELEMETRY=0` 可在所有地方禁用遥测。

## 本地审计日志

Extension.js 会把它考虑发送的每个事件(无论是否真的发送)追加到同意文件旁的 `events.jsonl`。你可以随时查看,也可以随意删除。

## 最佳实践

- 当政策要求不进行遥测时,在 CI 中使用 `EXTENSION_TELEMETRY_DISABLED=1`。
- 把隐私回退视作产品回退。
- 阅读仓库级契约,了解精确的事件列表。
