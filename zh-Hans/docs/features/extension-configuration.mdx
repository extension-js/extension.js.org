用一个配置文件设置浏览器默认值、命令行为，以及打包器的自定义选项。

在团队间共享浏览器默认值、命令选项与构建设置。不必再重复传 CLI 选项。Extension.js 会读取项目根目录下的 `extension.config.js`（或 `.mjs` / `.cjs`），并把其中的设置应用到每个命令以及打包器上。

## 工作原理

在项目根目录（通常与 `package.json` 同级）放置 `extension.config.js`。

支持的文件名：

- `extension.config.js`
- `extension.config.mjs`
- `extension.config.cjs`

顶层键：

| 键                  | 描述                                                                              |
| ------------------- | --------------------------------------------------------------------------------- |
| `browser`           | 按浏览器目标分别配置的默认值。                                                    |
| `commands`          | 各命令（`dev`、`start`、`preview`、`build`）的默认值。                            |
| `extensions`        | 跨命令生效的配套扩展（仅加载）。                                                  |
| `transpilePackages` | 在打包前需要先编译的包（对 monorepo/工作区结构很有用）。                          |
| `config`            | 用于扩展或覆盖生成的 Rspack 配置的钩子。                                          |

### 配置文件的环境加载

`extension.config.*` 在 Node 中运行，应该从 `process.env.*` 读取值。

- Extension.js 会在求值 `extension.config.*` 之前预加载 env 文件。
- 它会先检查项目目录。
- 在 monorepo 中，如果 Extension.js 在项目本地找不到 `.env*` 文件，会回退到最近的工作区根目录。工作区根目录是包含 `pnpm-workspace.yaml` 的文件夹。
- 优先使用内建的 env 预加载，而不是在配置文件中导入 `dotenv`。

### 浏览器配置

需要对不同目标使用不同的浏览器默认值？使用 `browser`：

```js
export default {
  browser: {
    chrome: {
      profile: "path/to/profile",
      preferences: { darkMode: true },
      browserFlags: ["--disable-web-security"],
      excludeBrowserFlags: ["--mute-audio"],
    },
    "chromium-based": {
      chromiumBinary: "/path/to/brave-or-other-chromium-binary",
    },
    firefox: {
      geckoBinary: "/path/to/firefox",
    },
  },
};
```

支持的浏览器键包括：`chrome`、`edge`、`firefox`、`chromium`、`chromium-based`、`gecko-based`、`firefox-based`。

常见的浏览器字段：

- `profile`、`persistProfile`
- `preferences`
- `browserFlags`、`excludeBrowserFlags`
- `chromiumBinary`、`geckoBinary`
- `extensions`（配套的仅加载扩展）

#### 浏览器目标能力

| 键                    | 作用                                                                |
| --------------------- | ------------------------------------------------------------------- |
| `profile`             | 设置启动时使用的浏览器 profile 路径（或 profile 行为）。            |
| `persistProfile`      | 在多次运行之间复用 profile 数据。                                  |
| `preferences`         | 应用浏览器偏好设置覆盖。                                            |
| `browserFlags`        | 为浏览器进程添加启动标志。                                          |
| `excludeBrowserFlags` | 移除你不想要的默认启动标志。                                        |
| `chromiumBinary`      | 使用自定义的 Chromium 系列二进制路径。                              |
| `geckoBinary`         | 使用自定义的 Gecko/Firefox 二进制路径。                             |
| `extensions`          | 与主扩展一起加载配套扩展。                                          |

### 命令配置

用 `commands` 为每个命令定义默认值：

```js
export default {
  transpilePackages: ["@workspace/ui"],
  commands: {
    dev: {
      browser: "chrome",
      polyfill: true,
      logLevel: "info",
      persistProfile: true,
      browserFlags: ["--headless=new"],
      extensions: { dir: "./extensions" },
    },
    start: {
      browser: "firefox",
      source: "https://example.com",
    },
    preview: {
      browser: "edge",
      noBrowser: false,
    },
    build: {
      browser: "chrome",
      zip: true,
      zipSource: true,
      zipFilename: "my-extension.zip",
      transpilePackages: ["@workspace/ui", "@workspace/icons"],
    },
  },
};
```

注意：

- Extension.js 会把顶层的 `extensions` 与 `transpilePackages` 合并进各命令的默认值。
- 各命令下的值会覆盖顶层值。
- `start` 命令内部会先运行 `build`，再运行 `preview`。Extension.js 会应用 `commands.start` 中的设置，包括 `profile`、`browserFlags`、`startingUrl` 等浏览器启动选项。构建相关的设置也可以放进 `commands.build`。

#### 命令通用能力

| 键                    | 适用范围                           | 作用                                                                |
| --------------------- | ---------------------------------- | ------------------------------------------------------------------- |
| `browser`             | `dev`、`start`、`preview`、`build` | 设置浏览器或浏览器家族目标。                                        |
| `profile`             | `dev`、`start`、`preview`          | 设置浏览器 profile 路径/行为。                                      |
| `persistProfile`      | `dev`、`start`、`preview`          | 在多次运行之间复用浏览器 profile 数据。                            |
| `chromiumBinary`      | `dev`、`start`、`preview`          | 使用自定义的 Chromium 系列二进制。                                  |
| `geckoBinary`         | `dev`、`start`、`preview`          | 使用自定义的 Gecko 系列二进制。                                     |
| `polyfill`            | `dev`、`start`、`build`            | 在相关场景启用兼容性 polyfill 行为。                                |
| `preferences`         | `dev`、`start`、`preview`          | 应用浏览器偏好设置覆盖。                                            |
| `browserFlags`        | `dev`、`start`、`preview`          | 为浏览器进程添加启动标志。                                          |
| `excludeBrowserFlags` | `dev`、`start`、`preview`          | 移除你不想要的默认启动标志。                                        |
| `startingUrl`         | `dev`、`start`、`preview`          | 浏览器启动时打开特定 URL。                                          |
| `port`                | `dev`、`start`、`preview`          | 设置 runner/DevTools 端口。                                         |
| `host`                | `dev`、`start`、`preview`          | 设置 dev 服务器主机。Docker/dev 容器中使用 `0.0.0.0`。              |
| `noBrowser`           | `dev`、`start`、`preview`          | 禁用浏览器启动。                                                    |
| `extensions`          | `dev`、`start`、`preview`、`build` | 加载配套扩展（或扩展列表）。                                        |
| `install`             | `dev`、`start`、`build`            | 在运行之前安装缺失的依赖。                                          |
| `transpilePackages`   | `dev`、`start`、`preview`、`build` | 对列出的工作区/外部包进行编译转换。                                |

#### `build` 命令能力

| 键            | 作用                                                              |
| ------------- | ----------------------------------------------------------------- |
| `zip`         | 生成打包好的 zip 产物。                                           |
| `zipSource`   | 额外输出一份源码归档，便于审查/合规流程。                         |
| `zipFilename` | 设置自定义的 zip 输出文件名。                                     |
| `silent`      | 减少构建日志输出。                                                |

#### `dev` 命令能力

| 键                    | 作用                                                       |
| --------------------- | ---------------------------------------------------------- |
| `noOpen`              | 运行 dev 服务器但不自动打开浏览器。                       |
| `source`              | 从远程 source URL 启用源检查流程。                        |
| `watchSource`         | 当被监听的内容更新时重新运行源检查。                      |
| `sourceFormat`        | 设置源输出格式（`pretty`、`json`、`ndjson`）。            |
| `sourceSummary`       | 输出紧凑的源摘要。                                         |
| `sourceMeta`          | 在源输出中包含页面元数据。                                |
| `sourceProbe`         | 用特定 CSS 选择器探测源输出。                             |
| `sourceTree`          | 控制紧凑的扩展树输出。                                    |
| `sourceConsole`       | 在源输出中包含 console 摘要。                             |
| `sourceDom`           | 在源输出中包含 DOM 快照/差异。                            |
| `sourceMaxBytes`      | 限制源输出载荷的大小。                                    |
| `sourceRedact`        | 控制源脱敏策略。                                           |
| `sourceIncludeShadow` | 控制 Shadow DOM 包含策略。                                |
| `sourceDiff`          | 在源 watch 更新时包含 diff 元数据。                       |

#### 日志能力

| 键           | 适用范围                  | 作用                                                       |
| ------------ | ------------------------- | ---------------------------------------------------------- |
| `logs`       | `dev`、`start`、`preview` | 设置最小日志级别（从 `off` 到 `all`）。                   |
| `logContext` | `dev`、`start`、`preview` | 按上下文过滤日志（`background`、`content` 等）。          |
| `logFormat`  | `dev`、`start`、`preview` | 设置输出格式（`pretty`、`json`、`ndjson`）。              |
| `logUrl`     | `dev`、`start`、`preview` | 按 URL 模式过滤日志。                                      |
| `logTab`     | `dev`、`start`、`preview` | 按 tab ID 过滤日志。                                       |

### Rspack 配置

需要更进一步定制打包器？使用 `config` 来对生成的 Rspack 配置做补丁：

```js
export default {
  config: (config) => {
    config.module.rules.push({
      test: /\.mdx$/,
      use: ["babel-loader", "@mdx-js/loader"],
    });
    return config;
  },
};
```

`config` 也可以是一个对象，Extension.js 会把它合并到生成的配置之上。

## 完整示例

```js
export default {
  browser: {
    chrome: { browser: "chrome", profile: "default" },
    firefox: { browser: "firefox", persistProfile: true },
    "chromium-based": {
      browser: "chromium-based",
      chromiumBinary:
        "/Applications/Brave Browser.app/Contents/MacOS/Brave Browser",
    },
  },
  commands: {
    dev: {
      browser: "chrome",
      polyfill: true,
      host: "0.0.0.0",
      extensions: { dir: "./extensions" },
    },
    start: { browser: "edge" },
    preview: { browser: "firefox" },
    build: {
      zipFilename: "extension.zip",
      zip: true,
      zipSource: true,
    },
  },
  extensions: { dir: "./extensions" },
  transpilePackages: ["@workspace/ui"],
  config: (config) => config,
};
```

## 最佳实践

- **把浏览器相关的值放在 `browser` 中**：让命令定义聚焦于工作流，而不是浏览器细节。
- **有意识地使用顶层默认值**：在根上放共享的 `extensions` / `transpilePackages`，仅在必要时覆盖。
- **优先使用 `chromiumBinary`/`geckoBinary` 命名**：它们与当前命令和类型层保持一致。
- **`config` 钩子保持最小化**：只加 Extension.js 一等公民选项覆盖不到的内容。
