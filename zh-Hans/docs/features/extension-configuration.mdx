使用一个配置文件来设置浏览器默认值、命令行为和打包器自定义。

在团队范围内共享浏览器默认值、命令选项和构建设置,停止反复输入 CLI 标志。Extension.js 会从项目根目录读取 `extension.config.js`(或 `.mjs` / `.cjs`),并把设置应用到所有命令以及打包器。

## 工作原理

在项目根目录(通常与 `package.json` 同级)添加 `extension.config.js`。

支持的文件名:

- `extension.config.js`
- `extension.config.mjs`
- `extension.config.cjs`

顶层键:

| 键                  | 描述                                                                       |
| ------------------- | --------------------------------------------------------------------------- |
| `browser`           | 按浏览器目标划分的浏览器特定默认值。                                          |
| `commands`          | 按命令划分的默认值(`dev`、`start`、`preview`、`build`)。                    |
| `extensions`        | 跨命令应用的伴随扩展(仅加载)。                                              |
| `transpilePackages` | 在打包前进行编译的包(适用于 monorepo/工作区设置)。                          |
| `config`            | 扩展/覆盖生成的 Rspack 配置的钩子。                                          |

### 配置文件的环境加载

`extension.config.*` 运行在 Node 中,应当从 `process.env.*` 读取值。

- Extension.js 会在求值 `extension.config.*` 之前预加载环境文件。
- 它先检查项目目录。
- 在 monorepo 中,如果 Extension.js 在项目本地找不到 `.env*` 文件,会回退到最近的工作区根。工作区根是包含 `pnpm-workspace.yaml` 的目录。
- 优先使用内置环境预加载,而不是在配置文件中导入 `dotenv`。

### 浏览器配置

需要按目标使用不同的浏览器默认值?使用 `browser`:

```js
export default {
  browser: {
    chrome: {
      profile: "path/to/profile",
      preferences: { darkMode: true },
      browserFlags: ["--disable-web-security"],
      excludeBrowserFlags: ["--mute-audio"],
    },
    "chromium-based": {
      chromiumBinary: "/path/to/brave-or-other-chromium-binary",
    },
    firefox: {
      geckoBinary: "/path/to/firefox",
    },
  },
};
```

支持的浏览器键包括:`chrome`、`edge`、`firefox`、`chromium`、`chromium-based`、`gecko-based`、`firefox-based`。

常用的浏览器字段:

- `profile`、`persistProfile`
- `preferences`
- `browserFlags`、`excludeBrowserFlags`
- `chromiumBinary`、`geckoBinary`
- `extensions`(仅加载的伴随扩展)

#### 浏览器目标能力

| 键                    | 作用                                                                  |
| --------------------- | --------------------------------------------------------------------- |
| `profile`             | 设置启动时使用的浏览器 profile 路径(或 profile 行为)。               |
| `persistProfile`      | 在多次运行之间复用 profile 数据。                                       |
| `preferences`         | 应用浏览器首选项覆盖。                                                  |
| `browserFlags`        | 为浏览器进程添加启动标志。                                              |
| `excludeBrowserFlags` | 移除你不想要的默认启动标志。                                            |
| `chromiumBinary`      | 使用自定义的 Chromium 系二进制路径。                                    |
| `geckoBinary`         | 使用自定义的 Gecko/Firefox 二进制路径。                                 |
| `extensions`          | 与主扩展一起加载伴随扩展。                                              |

### 命令配置

使用 `commands` 为每个命令定义默认值:

```js
export default {
  transpilePackages: ["@workspace/ui"],
  commands: {
    dev: {
      browser: "chrome",
      polyfill: true,
      logLevel: "info",
      persistProfile: true,
      browserFlags: ["--headless=new"],
      extensions: { dir: "./extensions" },
    },
    start: {
      browser: "firefox",
      source: "https://example.com",
    },
    preview: {
      browser: "edge",
      noBrowser: false,
    },
    build: {
      browser: "chrome",
      zip: true,
      zipSource: true,
      zipFilename: "my-extension.zip",
      transpilePackages: ["@workspace/ui", "@workspace/icons"],
    },
  },
};
```

注意事项:

- Extension.js 会把顶层的 `extensions` 和 `transpilePackages` 合并到命令默认值中。
- 按命令的值会覆盖顶层的值。
- `start` 命令在内部依次执行 `build` 和 `preview`。Extension.js 会应用 `commands.start` 中的设置,包括 `profile`、`browserFlags`、`startingUrl` 等浏览器启动选项。你也可以把构建相关的设置放进 `commands.build`。

#### 命令通用能力

| 键                    | 适用于                              | 作用                                                            |
| --------------------- | ---------------------------------- | -------------------------------------------------------------- |
| `browser`             | `dev`、`start`、`preview`、`build` | 设置浏览器或浏览器系列目标。                                      |
| `profile`             | `dev`、`start`、`preview`          | 设置浏览器 profile 路径/行为。                                    |
| `persistProfile`      | `dev`、`start`、`preview`          | 在多次运行之间复用浏览器 profile 数据。                            |
| `chromiumBinary`      | `dev`、`start`、`preview`          | 使用自定义的 Chromium 系二进制。                                  |
| `geckoBinary`         | `dev`、`start`、`preview`          | 使用自定义的 Gecko 系二进制。                                     |
| `polyfill`            | `dev`、`start`、`build`            | 在相关场景下启用兼容性 polyfill 行为。                             |
| `preferences`         | `dev`、`start`、`preview`          | 应用浏览器首选项覆盖。                                            |
| `browserFlags`        | `dev`、`start`、`preview`          | 为浏览器进程添加启动标志。                                        |
| `excludeBrowserFlags` | `dev`、`start`、`preview`          | 移除你不想要的默认启动标志。                                      |
| `startingUrl`         | `dev`、`start`、`preview`          | 浏览器启动时打开指定 URL。                                        |
| `port`                | `dev`、`start`、`preview`          | 设置 runner/DevTools 端口。                                       |
| `host`                | `dev`、`start`、`preview`          | 设置开发服务器主机。Docker/dev 容器中使用 `0.0.0.0`。              |
| `noBrowser`           | `dev`、`start`、`preview`          | 禁用浏览器启动。                                                  |
| `extensions`          | `dev`、`start`、`preview`、`build` | 加载伴随扩展(或扩展列表)。                                      |
| `install`             | `dev`、`start`、`build`            | 运行前安装缺失依赖。                                              |
| `transpilePackages`   | `dev`、`start`、`preview`、`build` | 转译列出的工作区/外部包。                                          |

#### `build` 命令能力

| 键            | 作用                                                                |
| ------------- | ------------------------------------------------------------------- |
| `zip`         | 生成打包好的 zip 构件。                                              |
| `zipSource`   | 为审查/合规流程额外输出源代码包/归档。                                |
| `zipFilename` | 自定义 zip 输出文件名。                                              |
| `silent`      | 减少构建日志输出。                                                   |

#### `dev` 命令能力

| 键                    | 作用                                                       |
| --------------------- | ---------------------------------------------------------- |
| `noOpen`              | 运行开发服务器但不自动打开浏览器。                          |
| `source`              | 启用来自远程源 URL 的 source 检查流程。                     |
| `watchSource`         | 当监视的更新发生时重新运行 source 检查。                    |
| `sourceFormat`        | 设置 source 输出格式(`pretty`、`json`、`ndjson`)。         |
| `sourceSummary`       | 输出紧凑的 source 摘要。                                    |
| `sourceMeta`          | 在 source 输出中包含页面元数据。                            |
| `sourceProbe`         | 使用特定 CSS 选择器探测 source 输出。                       |
| `sourceTree`          | 控制紧凑的扩展树输出。                                      |
| `sourceConsole`       | 在 source 输出中包含 console 摘要。                         |
| `sourceDom`           | 在 source 输出中包含 DOM 快照/差异。                        |
| `sourceMaxBytes`      | 限制 source 输出载荷大小。                                  |
| `sourceRedact`        | 控制 source 脱敏策略。                                      |
| `sourceIncludeShadow` | 控制 Shadow DOM 的包含策略。                                |
| `sourceDiff`          | 在 source watch 更新中包含 diff 元数据。                    |

#### 日志能力

| 键           | 适用于                     | 作用                                                       |
| ------------ | ------------------------- | ---------------------------------------------------------- |
| `logs`       | `dev`、`start`、`preview` | 设置最低日志等级(`off` 到 `all`)。                         |
| `logContext` | `dev`、`start`、`preview` | 按上下文过滤日志(`background`、`content` 等)。             |
| `logFormat`  | `dev`、`start`、`preview` | 设置输出格式(`pretty`、`json`、`ndjson`)。                 |
| `logUrl`     | `dev`、`start`、`preview` | 按 URL 模式过滤日志。                                       |
| `logTab`     | `dev`、`start`、`preview` | 按 tab ID 过滤日志。                                        |

### Rspack 配置

需要高级的打包器自定义?使用 `config` 修补生成的 Rspack 配置:

```js
export default {
  config: (config) => {
    config.module.rules.push({
      test: /\.mdx$/,
      use: ["babel-loader", "@mdx-js/loader"],
    });
    return config;
  },
};
```

`config` 也可以是一个对象,Extension.js 会将其合并到生成的配置之上。

## 完整示例

```js
export default {
  browser: {
    chrome: { browser: "chrome", profile: "default" },
    firefox: { browser: "firefox", persistProfile: true },
    "chromium-based": {
      browser: "chromium-based",
      chromiumBinary:
        "/Applications/Brave Browser.app/Contents/MacOS/Brave Browser",
    },
  },
  commands: {
    dev: {
      browser: "chrome",
      polyfill: true,
      host: "0.0.0.0",
      extensions: { dir: "./extensions" },
    },
    start: { browser: "edge" },
    preview: { browser: "firefox" },
    build: {
      zipFilename: "extension.zip",
      zip: true,
      zipSource: true,
    },
  },
  extensions: { dir: "./extensions" },
  transpilePackages: ["@workspace/ui"],
  config: (config) => config,
};
```

## 最佳实践

- **把浏览器特定值放在 `browser` 中**:让命令定义聚焦在工作流上,而非浏览器内部细节。
- **有意识地使用顶层默认值**:把共享的 `extensions` / `transpilePackages` 放在根级,仅在需要时覆盖。
- **优先使用 `chromiumBinary`/`geckoBinary` 名称**:它们与当前的命令和类型契合。
- **保持 `config` 钩子最小化**:只添加 Extension.js 一等选项未覆盖的内容。
