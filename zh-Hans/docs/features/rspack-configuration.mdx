在不离开 Extension.js 工作流的前提下自定义打包行为。

无需 fork 默认设置即可解决自定义打包需求。直接添加 loader(文件类型转换)、插件(构建时扩展)和别名(导入快捷方式)。Extension.js 使用 [Rspack](https://rspack.dev)(基于 Rust 的 JavaScript 打包器,`@rspack/core`)构建,你可以直接扩展其生成的配置。

## 工作原理

在项目根目录创建以下文件之一:

- `extension.config.js`
- `extension.config.mjs`
- `extension.config.cjs`

使用 `config` 键修补生成的打包器配置。

### `config` 能力

| 能力                                          | 作用                                                                       |
| -------------------------------------------- | -------------------------------------------------------------------------- |
| 函数钩子(`config: (config) => config`)        | 在构建前完全控制读取并修改生成的 Rspack 配置。                                |
| 对象合并(`config: { ... }`)                   | 将额外配置合并到生成的基础配置中。                                            |
| 规则(`config.module.rules`)                  | 为文件类型添加或修改 loader 规则。                                            |
| 插件(`config.plugins`)                       | 添加编译时插件与转换。                                                        |
| 解析(`config.resolve`)                       | 添加别名和模块解析行为。                                                      |

### 方式 1:函数钩子(推荐)

```js
export default {
  config: (config) => {
    config.module.rules.push({
      test: /\.mdx$/,
      use: ["babel-loader", "@mdx-js/loader"],
    });
    return config;
  },
};
```

### 方式 2:对象合并

`config` 也可以是一个对象。Extension.js 会把它合并到基础配置中。

```js
export default {
  config: {
    resolve: {
      alias: {
        "@ui": "/absolute/path/to/ui",
      },
    },
  },
};
```

## Rspack 优先,兼容 webpack

Extension.js 以 Rspack 原生为主,但你仍可使用大部分 webpack 生态。

- 配置类型扩展自 `@rspack/core` 的 `Configuration`。
- 许多 webpack 的 loader/插件通过兼容层可用。
- 部分 webpack 内部/插件与 Rspack 并非 1:1 兼容。

使用 Rspack 原生插件的示例:

```js
import { DefinePlugin } from "@rspack/core";

export default {
  config: (config) => {
    config.plugins = [
      ...config.plugins,
      new DefinePlugin({
        __FEATURE_FLAG__: JSON.stringify("enabled"),
      }),
    ];
    return config;
  },
};
```

## 何时使用

- 为项目特定文件类型添加自定义 loader/规则。
- 添加用于编译时转换和诊断的插件。
- 覆盖 Extension.js 一等选项未暴露的 resolve 别名与模块行为。

## 最佳实践

- **优先使用一等选项**:在使用底层打包器覆盖之前,先用 `browser` / `commands` 配置键。
- **最小化修补**:只修改你需要的部分,然后返回配置。
- **保持插件感知 Rspack**:可用时优先选择 Rspack 原生插件。
- **在所有目标上验证**:修改配置后,在你的浏览器矩阵上测试 `dev`、`start`、`preview` 和 `build`。
