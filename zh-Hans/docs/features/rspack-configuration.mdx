无需离开 Extension.js 工作流就能自定义打包行为。

解决自定义打包需求,而无需 fork 默认设置。可以直接添加 loader(文件类型转换)、plugin(构建时扩展)和 alias(导入快捷方式)。Extension.js 使用 [Rspack](https://rspack.dev) 进行构建,这是一个基于 Rust 的 JavaScript 打包器(`@rspack/core`),你可以直接扩展生成的配置。

## 工作原理

在项目根目录创建以下任一文件:

- `extension.config.js`
- `extension.config.mjs`
- `extension.config.cjs`

使用 `config` 键来修补生成的打包器配置。

### `config` 能力

| 能力                                          | 作用                                                                        |
| --------------------------------------------- | --------------------------------------------------------------------------- |
| 函数钩子(`config: (config) => config`)      | 在构建前完全控制并读取/修改生成的 Rspack 配置。                             |
| 对象合并(`config: { ... }`)                 | 把额外配置合并进生成的基础配置。                                            |
| 规则(`config.module.rules`)                | 为文件类型添加或修改 loader 规则。                                          |
| 插件(`config.plugins`)                     | 添加编译时插件与转换。                                                      |
| 解析(`config.resolve`)                     | 添加别名与模块解析行为。                                                    |

### 选项 1:函数钩子(推荐)

```js
export default {
  config: (config) => {
    config.module.rules.push({
      test: /\.mdx$/,
      use: ["babel-loader", "@mdx-js/loader"],
    });
    return config;
  },
};
```

### 选项 2:对象合并

`config` 也可以是一个对象。Extension.js 会将其合并到基础配置中。

```js
export default {
  config: {
    resolve: {
      alias: {
        "@ui": "/absolute/path/to/ui",
      },
    },
  },
};
```

## Rspack 优先,webpack 兼容

Extension.js 原生使用 Rspack,但你仍可以使用大量 webpack 生态系统。

- 配置类型扩展自 `@rspack/core` 的 `Configuration`。
- 许多 webpack 的 loader/plugin 通过兼容层可用。
- 某些 webpack 的内部实现/插件并不与 Rspack 完全 1:1 兼容。

下面是一个使用 Rspack 原生插件的示例:

```js
import { DefinePlugin } from "@rspack/core";

export default {
  config: (config) => {
    config.plugins = [
      ...config.plugins,
      new DefinePlugin({
        __FEATURE_FLAG__: JSON.stringify("enabled"),
      }),
    ];
    return config;
  },
};
```

## 何时使用

- 为项目特定文件类型添加自定义 loader/规则。
- 添加用于编译时转换与诊断的插件。
- 覆盖 Extension.js 一等选项未暴露的 resolve 别名与模块行为。

## 最佳实践

- **优先使用一等选项**:在使用底层打包器覆盖之前,先用 `browser` / `commands` 配置键。
- **最小化补丁**:只改你需要的那部分,然后返回配置。
- **让插件兼容 Rspack**:有可用的 Rspack 原生插件时优先使用。
- **在所有目标上验证**:配置变更后,为你的浏览器矩阵测试 `dev`、`start`、`preview` 与 `build`。
