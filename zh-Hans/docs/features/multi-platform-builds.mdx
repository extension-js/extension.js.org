为多个浏览器构建并打包同一个扩展,产出结果可预测。

将同一扩展交付到 Chrome、Edge 与 Firefox,无需维护多套构建脚本。Extension.js 会为每个目标运行生产构建,并将产物写入 `dist/<browser>`。它也可以为分发生成 zip 包。

## 工作原理

运行一次生产构建:

<CodeGroup>

```bash npm
npx extension build
```

```bash pnpm
pnpx extension build
```

```bash yarn
yarn dlx extension build
```

```bash bun
bunx extension build
```

```bash bun
bunx extension build
```

</CodeGroup>

如果你未做覆盖,默认浏览器目标为 `chromium`。

## 选择浏览器

你可以针对特定浏览器/引擎进行构建:

<CodeGroup>

```bash npm

```

```bash pnpm

```

```bash yarn
yarn dlx extension build --browser=chrome
```

```bash bun
bunx extension build --browser=chrome
```

```bash bun
bunx extension build --browser=chrome
```

</CodeGroup>

<CodeGroup>

```bash npm
npx extension build --browser=firefox
```

```bash pnpm
pnpx extension build --browser=firefox
```

```bash yarn
yarn dlx extension build --browser=firefox
```

```bash bun
bunx extension build --browser=firefox
```

```bash bun
bunx extension build --browser=firefox
```

</CodeGroup>

支持的取值包括:

- `chrome`
- `edge`
- `firefox`
- `chromium`
- `chromium-based`
- `gecko-based` / `firefox-based`(别名)

你也可以在一条命令中运行一组构建矩阵:

<CodeGroup>

```bash npm

```

```bash pnpm
pnpx extension build --browser=chrome,edge,firefox
```

```bash yarn
yarn dlx extension build --browser=chrome,edge,firefox
```

```bash bun
bunx extension build --browser=chrome,edge,firefox
```

```bash bun
bunx extension build --browser=chrome,edge,firefox
```

</CodeGroup>

这将依次为 `chrome`、`edge` 和 `firefox` 构建。

## 输出布局

每个目标会写入各自的目录:

- `dist/chrome`
- `dist/edge`
- `dist/firefox`
- `dist/chromium`
- `dist/chromium-based`
- `dist/gecko-based`

### 构建能力

| 选项                    | 作用                                                                                |
| ----------------------- | ----------------------------------------------------------------------------------- |
| `--browser=<target>`    | 为特定浏览器或引擎家族构建。                                                        |
| `--zip`                 | 为每个目标产物创建分发用 zip。                                                      |
| `--zip-filename=<name>` | 覆盖默认的 zip 文件名。                                                             |
| `--zip-source`          | 同时创建源代码归档。                                                                |
| `--polyfill`            | 为 Chromium 家族目标启用兼容行为,以支持 `browser.*` API 的使用。                   |

## 生成 zip 文件

使用 `--zip` 为每个目标产物生成分发用 zip:

<CodeGroup>

```bash npm
npx extension build --zip
```

```bash pnpm
pnpx extension build --zip
```

```bash yarn
yarn dlx extension build --zip
```

```bash bun
bunx extension build --zip
```

```bash bun
bunx extension build --zip
```

</CodeGroup>

默认情况下,zip 名称使用经过清洗的 manifest `name` 加 `version`,例如:

- `my-extension-1.0.0.zip`

自定义文件名:

<CodeGroup>

```bash npm
npx extension build --zip --zip-filename=my-release
```

```bash pnpm
pnpx extension build --zip --zip-filename=my-release
```

```bash yarn
yarn dlx extension build --zip --zip-filename=my-release
```

```bash bun
bunx extension build --zip --zip-filename=my-release
```

```bash bun
bunx extension build --zip --zip-filename=my-release
```

</CodeGroup>

这会在目标的 `dist/<browser>` 目录中创建 `my-release.zip`。

## 最佳实践

- **在持续集成(CI)中按目标分别构建**:将每个浏览器产物视为独立产物。
- **使用浏览器矩阵命令做一致性检查**:在一个流水线步骤中尽早发现目标专属问题。
- **有目的地打包**:`--zip` 用于商店上传,`--zip-source` 用于可追溯的源代码归档。
- **保持目标配置显式化**:用 `extension.config.*` 的命令/浏览器默认值实现可复现构建。
