为多个浏览器构建并打包同一份扩展，输出可预测。

把同一份扩展发布到 Chrome、Edge 与 Firefox，不必维护多份构建脚本。Extension.js 会按目标分别运行生产构建，并把产物写入 `dist/<browser>`。它还可以为分发生成 zip 包。

## 工作原理

运行生产构建：

<CodeGroup>

```bash npm
npx extension build
```

```bash pnpm
pnpx extension build
```

```bash yarn
yarn dlx extension build
```

```bash bun
bunx extension build
```

```bash bun
bunx extension build
```

</CodeGroup>

除非你显式覆盖，默认的浏览器目标是 `chromium`。

## 浏览器选择

你可以指定具体的浏览器/引擎目标：

<CodeGroup>

```bash npm
npx extension build --browser=chrome
```

```bash pnpm
pnpx extension build --browser=chrome
```

```bash yarn
yarn dlx extension build --browser=chrome
```

```bash bun
bunx extension build --browser=chrome
```

```bash bun
bunx extension build --browser=chrome
```

</CodeGroup>

<CodeGroup>

```bash npm
npx extension build --browser=firefox
```

```bash pnpm
pnpx extension build --browser=firefox
```

```bash yarn
yarn dlx extension build --browser=firefox
```

```bash bun
bunx extension build --browser=firefox
```

```bash bun
bunx extension build --browser=firefox
```

</CodeGroup>

支持的取值包括：

- `chrome`
- `edge`
- `firefox`
- `chromium`
- `chromium-based`
- `gecko-based` / `firefox-based`（别名）

你也可以在一条命令中运行构建矩阵：

<CodeGroup>

```bash npm
npx extension build --browser=chrome,edge,firefox
```

```bash pnpm
pnpx extension build --browser=chrome,edge,firefox
```

```bash yarn
yarn dlx extension build --browser=chrome,edge,firefox
```

```bash bun
bunx extension build --browser=chrome,edge,firefox
```

```bash bun
bunx extension build --browser=chrome,edge,firefox
```

</CodeGroup>

这会依次为 `chrome`、`edge` 与 `firefox` 进行构建。

## 输出布局

每个目标都写入各自的文件夹：

- `dist/chrome`
- `dist/edge`
- `dist/firefox`
- `dist/chromium`
- `dist/chromium-based`
- `dist/gecko-based`

### 构建能力

| 选项                    | 作用                                                                                |
| ----------------------- | ----------------------------------------------------------------------------------- |
| `--browser=<target>`    | 为特定浏览器或引擎家族进行构建。                                                    |
| `--zip`                 | 为每个目标输出生成一个分发用 zip。                                                  |
| `--zip-filename=<name>` | 覆盖默认的 zip 文件名。                                                             |
| `--zip-source`          | 在输出之外额外创建一个源码归档。                                                    |
| `--polyfill`            | 为 Chromium 系列目标启用 `browser.*` API 使用的兼容性行为。                         |

## 生成 zip 文件

使用 `--zip` 从各目标输出生成一个分发用 zip：

<CodeGroup>

```bash npm
npx extension build --zip
```

```bash pnpm
pnpx extension build --zip
```

```bash yarn
yarn dlx extension build --zip
```

```bash bun
bunx extension build --zip
```

```bash bun
bunx extension build --zip
```

</CodeGroup>

默认情况下，zip 名称由清理过的 manifest `name` 加 `version` 拼接而成，例如：

- `my-extension-1.0.0.zip`

自定义文件名：

<CodeGroup>

```bash npm
npx extension build --zip --zip-filename=my-release
```

```bash pnpm
pnpx extension build --zip --zip-filename=my-release
```

```bash yarn
yarn dlx extension build --zip --zip-filename=my-release
```

```bash bun
bunx extension build --zip --zip-filename=my-release
```

```bash bun
bunx extension build --zip --zip-filename=my-release
```

</CodeGroup>

这会在对应的 `dist/<browser>` 文件夹中创建 `my-release.zip`。

## 为浏览器 API 打 polyfill

如果你的代码使用 Gecko 风格的 `browser.*` API，并且需要兼容 Chromium，请启用 `--polyfill`：

<CodeGroup>



```bash yarn
yarn dlx extension build --polyfill
```

```bash bun
bunx extension build --polyfill
```


</CodeGroup>

## 最佳实践

- **在持续集成 (CI) 中按目标构建**：把每个浏览器输出当作独立产物。
- **使用浏览器矩阵命令做一致性检查**：在一个流水线步骤里尽早发现目标特有的问题。
- **有意识地打包**：用 `--zip` 上传应用商店，用 `--zip-source` 生成可追溯的源码产物。
- **保持目标配置显式**：用 `extension.config.*` 的 command/browser 默认值，让构建可复现。
