浏览器扩展很难调试,因为有趣的状态存在你不容易触达的地方:会进入空闲的 MV3 service worker、隔离的内容脚本世界,以及一失去焦点就关闭的弹窗。Extension.js 在你运行的开发会话中打开一个小巧的 **本地控制通道**,让你(或一个 AI 代理,或一个 CI 任务)可以直接到达这些上下文 —— 读取它们记录的内容、检查它们渲染的内容、调用它们,并触发用户会触发的事件。

它是本地的,无需账户,观察始终免费。任何 _修改_ 状态的操作都是按会话主动启用的。

## 启用

启动一个带有所需控制标志的开发会话:

```bash
# 只读始终可用。添加 control 以对扩展执行操作:
pnpm extension dev --allow-control

# 添加 eval 以在某个上下文中运行任意表达式:
pnpm extension dev --allow-control --allow-eval
```

下面的每个操作都针对该会话,无论你是在读取还是在执行操作,寻址上下文的方式都一致。

## 你可以做什么

| 命令                                       | 作用                                                                                                              | 门控              |
| ------------------------------------------ | ----------------------------------------------------------------------------------------------------------------- | ----------------- |
| `extension logs`                           | 在一个有序时间线上聚合每个上下文(SW、content、popup、options、sidebar、devtools)的日志                          | —                 |
| `extension inspect`                        | 读取某个面的实时(注入后)DOM                                                                                     | —                 |
| `extension storage get \| set`             | 读取或写入你的扩展的 `chrome.storage`                                                                             | `--allow-control` |
| `extension reload`                         | 重载扩展或某个标签页                                                                                              | `--allow-control` |
| `extension open <popup\|options\|sidebar>` | 打开一个扩展面                                                                                                    | `--allow-control` |
| `extension open action`                    | [触发工具栏 action](/docs/debugging/trigger-actions-and-commands) —— 打开其弹窗,或重放 `onClicked`                | `--allow-control` |
| `extension open command --name <cmd>`      | [重放某个键盘快捷键命令](/docs/debugging/trigger-actions-and-commands)                                            | `--allow-control` |
| `extension eval "<expr>" --context <c>`    | 在某个上下文中求值表达式并返回值                                                                                  | `--allow-eval`    |

## 寻址上下文

读取和执行操作共用一套词汇 —— 你给出面的名字,Extension.js 会在它已经追踪的会话中将其解析。

| 寻址                                                | 含义                                            |
| --------------------------------------------------- | ----------------------------------------------- |
| `--context background`                              | service worker(或 MV2 后台页面)               |
| `--context popup` / `options` / `sidebar`           | 该扩展面(若已打开)                            |
| `--context content --url "https://shop.example/*"`  | 匹配页面上的隔离内容世界                        |
| `--context content --tab 7`                         | 指定标签页内的内容世界                          |

## 示例:我的后台处理程序真的运行了吗?

```bash
pnpm extension eval "globalThis.__lastSyncAt" --context background --allow-eval
```

```json
{ "ok": true, "value": 1718000000000 }
```

你的表达式触发的任何 `console.log` 都会同时通过 `extension logs` 流出,并按序列相关联 —— 你能同时看到返回值 _和_ 副作用。

## 跨浏览器支持

Extension.js 通过 **浏览器内伴侣** 进行调试,而不是通过 Chrome DevTools Protocol,因此核心循环可以到达 **Chrome 和 Firefox** 上你自己的面 —— 旧的"Firefox 用 RDP,不受支持"的墙对这些工具来说已经被推倒了。

| 能力                                                                              | Chromium | Firefox      |
| --------------------------------------------------------------------------------- | -------- | ------------ |
| `logs`、`inspect`、`eval`、`storage`、`reload`、`open`(含 `action`/`command`)   | ✓        | ✓(已验证)  |
| `inspect --deep-dom`(closed shadow roots)                                        | ✓        | —(使用 CDP)|
| `extension_list_extensions`(MCP 工具)                                           | ✓        | —(使用 CDP)|

(`extension_list_extensions` 是 MCP 工具而不是 CLI 动词 —— 它通过 DevTools Protocol 连接,因此仅支持 Chromium。)

## 安全

这些门控是有意的,不是繁文缛节:

- **观察无需任何门控。** 读取日志和 DOM 始终可用。
- **有限的操作需要 `--allow-control`。** `storage`、`reload` 和 `open` 会修改状态,因此按会话主动启用。
- **`eval` 需要 `--allow-eval` _以及_ 一个按会话的 token。** Token 写入 `dist/` 之外的 `0600` 权限文件,因此永远不会进入构建 —— 随机的本地进程无法悄悄驱动你的 service worker。
- **不会触达生产。** 控制通道仅在 `dev`/`preview` 期间存在;它通过一个不会出现在已构建包中的端口门控。

## 与 AI 代理配合

相同的操作通过 [`@extension.dev/mcp`](/docs/integrations/chrome-devtools-mcp) 作为 MCP 工具暴露(`extension_logs`、`extension_eval`、`extension_storage`、`extension_reload`、`extension_open`、`extension_list_extensions`)。门控完全相同 —— 助手可以自由观察,但只有在你按会话启用时才会执行操作。
