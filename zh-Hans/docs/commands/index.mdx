为当前扩展开发阶段选择合适的命令,覆盖从首次脚手架到生产验证的整个过程。

## 命令选择器

| 目标                               | 命令        | 示例                                                            |
| ---------------------------------- | ----------- | --------------------------------------------------------------- |
| 脚手架一个新项目                   | `create`    | `npx extension@latest create my-extension --template=new-react` |
| 在监听模式下开发                   | `dev`       | `extension dev --browser=firefox`                               |
| 构建生产产物                       | `build`     | `extension build --browser=chrome,firefox --zip`                |
| 构建并启动生产输出                 | `start`     | `extension start --browser=edge`                                |
| 仅启动已有的构建输出               | `preview`   | `extension preview --browser=chrome`                            |
| 安装受管的浏览器运行时             | `install`   | `extension install chrome`                                      |
| 移除受管的浏览器运行时             | `uninstall` | `extension uninstall chrome`                                    |

## 典型开发者流程

1. 用 `create` 进行脚手架。
2. 用 `dev` 进行迭代式编码和重载循环。
3. 用 `build` 构建生产产物。
4. 用 `preview` 或 `start` 进行类生产验证。
5. 当需要显式管理 Extension.js 浏览器运行时时,使用 `install` 或 `uninstall`。
