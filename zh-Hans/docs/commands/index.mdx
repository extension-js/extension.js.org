针对扩展开发当前所处的阶段——从首次脚手架到生产验证——选择合适的命令。

## 命令选择器

| 目标                               | 命令        | 示例                                                            |
| ---------------------------------- | ----------- | --------------------------------------------------------------- |
| 脚手架一个新项目                   | `create`    | `npx extension@latest create my-extension --template=new-react` |
| 以监听模式开发                     | `dev`       | `extension dev --browser=firefox`                               |
| 构建生产产物                       | `build`     | `extension build --browser=chrome,firefox --zip`                |
| 构建并启动生产输出                 | `start`     | `extension start --browser=edge`                                |
| 仅启动已有构建输出                 | `preview`   | `extension preview --browser=chrome`                            |
| 安装托管的浏览器运行时             | `install`   | `extension install chrome`                                      |
| 移除托管的浏览器运行时             | `uninstall` | `extension uninstall chrome`                                    |

## 典型的开发者流程

1. `create` 进行脚手架。
2. `dev` 进行迭代编码与重载循环。
3. `build` 产出生产产物。
4. `preview` 或 `start` 做类生产环境的验证。
5. 当你需要显式管理 Extension.js 浏览器运行时,使用 `install` 或 `uninstall`。
