根据当前扩展开发所处的阶段——从初次脚手架到生产验证——选择合适的命令。

## 命令选择器

| 目标                              | 命令        | 示例                                                            |
| --------------------------------- | ----------- | --------------------------------------------------------------- |
| 搭建一个新项目                    | `create`    | `npx extension@latest create my-extension --template=new-react` |
| 进入 watch 模式开发               | `dev`       | `extension dev --browser=firefox`                               |
| 构建生产产物                      | `build`     | `extension build --browser=chrome,firefox --zip`                |
| 构建并启动生产输出                | `start`     | `extension start --browser=edge`                                |
| 只启动已有的构建输出              | `preview`   | `extension preview --browser=chrome`                            |
| 安装一个受管理的浏览器运行时      | `install`   | `extension install chrome`                                      |
| 移除一个受管理的浏览器运行时      | `uninstall` | `extension uninstall chrome`                                    |

## 典型的开发流程

1. `create` 搭建骨架。
2. `dev` 迭代编码与重载循环。
3. `build` 产出生产产物。
4. `preview` 或 `start` 做类生产验证。
5. 需要显式管理 Extension.js 浏览器运行时时，用 `install` 或 `uninstall`。
