为扩展开发的当前阶段选择正确的命令，从最初的脚手架到生产环境校验。

## 命令选择器

| 目标                                 | 命令        | 示例                                                            |
| ------------------------------------ | ----------- | --------------------------------------------------------------- |
| 生成一个新项目                       | `create`    | `npx extension@latest create my-extension --template=new-react` |
| 在监听模式下开发                     | `dev`       | `extension dev --browser=firefox`                               |
| 构建生产产物                         | `build`     | `extension build --browser=chrome,firefox --zip`                |
| 构建并启动生产产物                   | `start`     | `extension start --browser=edge`                                |
| 只启动已有的构建产物                 | `preview`   | `extension preview --browser=chrome`                            |
| 安装一个受管浏览器运行时             | `install`   | `extension install chrome`                                      |
| 移除一个受管浏览器运行时             | `uninstall` | `extension uninstall chrome`                                    |

## 典型的开发流程

1. `create` 生成项目。
2. `dev` 进行迭代式编码与重载循环。
3. `build` 构建生产产物。
4. `preview` 或 `start` 进行近似生产环境的校验。
5. 当你需要显式管理 Extension.js 浏览器运行时时，使用 `install` 或 `uninstall`。
