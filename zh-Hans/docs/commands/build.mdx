为一个或多个浏览器目标创建生产扩展产物。

`build` 以生产模式编译你的扩展,并将输出写入 `dist/<browser>`。

对于 monorepo/submodule 项目,请参见[环境变量](/docs/features/environment-variables#how-it-works)了解配置期的环境变量解析(先在项目根目录解析,然后回退到工作区根目录)。

## 何时使用 `build`

- 为 Chrome 网上应用店、Edge Add-ons 或 Firefox Add-ons 准备扩展包。
- 在持续集成(CI)任务中产出可重复的生产产物。
- 在提交前验证生产打包输出以及不同浏览器目标间的差异。

## Build 命令能力

| 能力                   | 它能带给你什么                                                  |
| ---------------------- | --------------------------------------------------------------- |
| 生产编译               | 为每个目标生成优化后的扩展产物                                  |
| 多目标输出             | 在一条命令中构建多个浏览器目标                                  |
| 打包支持               | 生成发布用的 zip 产物,可选附带源码包                           |
| 适合 CI 的行为         | 在自动化中保持构建输出与命名可预测                              |

## 构建输出

运行 `build` 后,Extension.js 会为所选浏览器目标生成优化后的文件。输出落在 `dist/` 中,每个目标一个子文件夹。每个文件夹包含打包后的 JavaScript、CSS、HTML 以及所需的运行时资源。

<Note>
  对于 TypeScript 项目,`build` 还会重新生成 `extension-env.d.ts`
  全局类型声明(与 [`dev`](/docs/commands/dev) 写入的是同一份文件),这样无论你是否先运行过
  `dev`,CI 中的 `tsc --noEmit` 都能保持干净。纯 JavaScript 项目会跳过这一步。
</Note>

**输出结构示例:**

```plaintext
dist/
├── chrome/
│   ├── manifest.json
│   ├── background/service_worker.js
│   ├── content_scripts/content-0.js
├── edge/
│   ├── manifest.json
│   ├── background/service_worker.js
│   ├── content_scripts/content-0.js
```

## 浏览器目标矩阵

| 目标形式       | 示例                                             | 说明                               |
| -------------- | ------------------------------------------------ | ---------------------------------- |
| 具名目标       | `chromium`、`chrome`、`edge`、`firefox`          | 构建一个特定的浏览器目标           |
| 引擎目标       | `chromium-based`、`gecko-based`、`firefox-based` | 适用于引擎系列工作流               |
| 多目标         | `chrome,firefox`                                 | 逗号分隔的多个目标                 |

## 参数与标志

| 标志                     | 别名            | 它的作用                                                                                                | 默认值                  |
| ------------------------ | --------------- | ------------------------------------------------------------------------------------------------------- | ----------------------- |
| `[path]`                 | -               | 构建本地扩展项目。                                                                                      | `process.cwd()`         |
| `--browser <browser>`    | -               | 浏览器/引擎目标(`chromium`、`chrome`、`edge`、`firefox`、引擎别名,或逗号分隔的多个值)。              | `chromium`              |
| `--polyfill [boolean]`   | -               | 为 Chromium 目标启用 `browser.*` API 兼容 polyfill。                                                    | `false`                 |
| `--zip [boolean]`        | -               | 创建打包的 zip 产物。                                                                                   | `false`                 |
| `--zip-source [boolean]` | -               | 在 zip 输出中包含源文件。                                                                               | `false`                 |
| `--zip-filename <name>`  | -               | 设置自定义的 zip 文件名。                                                                               | 扩展名称 + 版本号       |
| `--silent [boolean]`     | -               | 隐藏构建日志。                                                                                          | `false`                 |
| `--mode <mode>`          | -               | 覆盖打包器模式(`development`、`production` 或 `none`)。同时设置 `NODE_ENV`。                          | `production`            |
| `--extensions <list>`    | -               | 逗号分隔的伴随扩展或商店 URL。                                                                          | 未设置                  |
| `--install [boolean]`    | -               | 缺失时安装项目依赖。                                                                                    | 命令的默认行为          |
| `--author`               | `--author-mode` | 启用维护者诊断。                                                                                        | 禁用                    |

## 共享的全局选项

同样支持[全局标志](/docs/workflows/global-flags)。

## 模式覆盖

`--mode` 用于覆盖构建的打包器模式和 `NODE_ENV`。接受 `development`、`production` 或 `none`。当你需要为预发或调试生成非生产打包时(类似 Vite/webpack 的工作流),可以使用它。

<CodeGroup>

```bash npm
extension build ./my-extension --mode development
```

```bash pnpm
extension build ./my-extension --mode development
```

```bash yarn
extension build ./my-extension --mode development
```

```bash bun
extension build ./my-extension --mode development
```

```bash bun
extension build ./my-extension --mode development
```

</CodeGroup>

无效值会以错误退出;默认值仍为 `production`。

## zip 行为

| 选项             | 效果                                    | 典型用途                             |
| ---------------- | --------------------------------------- | ------------------------------------ |
| `--zip`          | 创建打包的产物 zip                      | 商店提交/手动分发                    |
| `--zip-filename` | 设置自定义 zip 名称                     | CI 命名约定                          |
| `--zip-source`   | 在产物之外附加源码归档                  | 合规/审查流水线                      |

## 最佳实践

- **检查构建日志:** 在每次构建后查看日志中的警告和缺失资源。
- **优化你的 manifest:** 让 `manifest.json` 与每个目标浏览器兼容。
- **有意命名产物:** 使用 `--zip-filename` 让 CI 产物命名稳定。
- **验证目标输出:** 发布前检查每个 `dist/<browser>` 文件夹。
