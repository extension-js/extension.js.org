为一个或多个浏览器目标创建生产扩展产物。

`build` 以生产模式编译你的扩展,并将输出写入 `dist/<browser>`。

对于 monorepo/子模块项目,请参阅 [环境变量](/docs/features/environment-variables#how-it-works) 了解配置阶段的环境变量解析(先在项目根目录,再回退到工作区根目录)。

## 何时使用 `build`

- 为 Chrome 网上应用店、Edge 加载项或 Firefox 附加组件准备扩展包。
- 在持续集成 (CI) 任务中生成可重复的生产产物。
- 在提交之前验证生产打包输出以及浏览器目标之间的差异。

## Build 命令能力

| 能力             | 它能提供什么                                             |
| ---------------- | -------------------------------------------------------- |
| 生产编译         | 为每个目标生成优化后的扩展产物                           |
| 多目标输出       | 一条命令构建多个浏览器目标                               |
| 打包支持         | 创建分发用 zip 产物,可选附带源码包                      |
| CI 友好          | 在自动化场景下保持构建输出和命名可预测                   |

## 构建输出

运行 `build` 后,Extension.js 会为所选浏览器目标生成优化后的文件。输出位于 `dist/`,每个目标一个子文件夹。每个文件夹包含打包后的 JavaScript、CSS、HTML 以及所需的运行时资源。

<Note>
  对于 TypeScript 项目,`build` 也会重新生成 `extension-env.d.ts` 全局类型声明
  (与 [`dev`](/docs/commands/dev) 写入的是同一个文件),
  这样无论你之前是否运行过 `dev`,CI 中的 `tsc --noEmit` 都能保持干净。
  纯 JavaScript 项目会跳过这一步。
</Note>

**示例输出结构:**

```plaintext
dist/
├── chrome/
│   ├── manifest.json
│   ├── background/service_worker.js
│   ├── content_scripts/content-0.js
├── edge/
│   ├── manifest.json
│   ├── background/service_worker.js
│   ├── content_scripts/content-0.js
```

## 浏览器目标矩阵

| 目标类型     | 示例                                             | 说明                               |
| ------------ | ------------------------------------------------ | ---------------------------------- |
| 命名目标     | `chromium`、`chrome`、`edge`、`firefox`          | 构建某一个具体的浏览器目标         |
| 引擎目标     | `chromium-based`、`gecko-based`、`firefox-based` | 适用于按引擎家族划分的工作流       |
| 多目标       | `chrome,firefox`                                 | 用逗号分隔的多个目标               |

## 参数与选项

| 选项                     | 别名            | 作用                                                                                                          | 默认值                   |
| ------------------------ | --------------- | ------------------------------------------------------------------------------------------------------------- | ------------------------ |
| `[path]`                 | -               | 构建一个本地扩展项目。                                                                                        | `process.cwd()`          |
| `--browser <browser>`    | -               | 浏览器/引擎目标(`chromium`、`chrome`、`edge`、`firefox`、引擎别名或逗号分隔的多个值)。                     | `chromium`               |
| `--polyfill [boolean]`   | -               | 在 Chromium 目标上启用 `browser.*` API 兼容性 polyfill。                                                      | `false`                  |
| `--zip [boolean]`        | -               | 生成打包后的 zip 产物。                                                                                       | `false`                  |
| `--zip-source [boolean]` | -               | 在 zip 输出中包含源文件。                                                                                     | `false`                  |
| `--zip-filename <name>`  | -               | 设置自定义 zip 文件名。                                                                                       | 扩展名 + 版本号          |
| `--silent [boolean]`     | -               | 静默构建日志。                                                                                                | `false`                  |
| `--mode <mode>`          | -               | 覆盖打包模式(`development`、`production` 或 `none`),同时设置 `NODE_ENV`。                                  | `production`             |
| `--extensions <list>`    | -               | 用逗号分隔的伴随扩展列表或商店 URL。                                                                          | 未设置                   |
| `--install [boolean]`    | -               | 缺少依赖时安装项目依赖。                                                                                      | 命令默认行为             |
| `--author`               | `--author-mode` | 启用维护者诊断模式。                                                                                          | 禁用                     |

## 共享的全局选项

也支持 [全局选项](/docs/workflows/global-flags)。

## 模式覆盖

`--mode` 会覆盖构建使用的打包模式和 `NODE_ENV`。可选 `development`、`production` 或 `none`。当你需要为预发布或调试生成非生产打包时,可以用它来对齐 Vite/webpack 的工作流。

<CodeGroup>

```bash npm
extension build ./my-extension --mode development
```

```bash pnpm
extension build ./my-extension --mode development
```

```bash yarn
extension build ./my-extension --mode development
```

```bash bun
extension build ./my-extension --mode development
```

```bash bun
extension build ./my-extension --mode development
```

</CodeGroup>

无效值会以错误退出;默认值仍然是 `production`。

## Zip 行为

| 选项             | 效果                                  | 典型用途                             |
| ---------------- | ------------------------------------- | ------------------------------------ |
| `--zip`          | 创建打包后的产物 zip                  | 商店提交/手动分发                    |
| `--zip-filename` | 设置自定义 zip 名称                   | CI 命名规范                          |
| `--zip-source`   | 在产物旁边附加源码归档                | 合规/审查流水线                      |

## 最佳实践

- **检查构建日志:** 每次构建后查看日志,关注警告和缺失的资源。
- **优化 manifest:** 让 `manifest.json` 兼容所有目标浏览器。
- **有意识地为产物命名:** 使用 `--zip-filename` 保证 CI 产物命名稳定。
- **验证目标输出:** 发布前检查每个 `dist/<browser>` 文件夹。
