使用 `install` 将受管的浏览器运行时添加到 Extension.js 缓存。

当你想为 `dev`、`build`、`start` 或 `preview` 使用一致的浏览器二进制时,它最为有用。它支持 Chrome for Testing、Chromium、Firefox 和 Edge。

## 何时使用 `install`

- 你需要在持续集成 (CI)、自动化或团队一致的本地运行中获得稳定、可重复的浏览器二进制。
- 你想使用 Chrome for Testing,而不是依赖系统已安装的任意 Chrome 版本。
- 你正在用受管的 Firefox 或 Edge 运行时搭建跨浏览器测试。

## 标准用法

对于单个浏览器,使用位置参数形式:

<CodeGroup>

```bash npm
extension install <browser>
```

```bash pnpm
extension install <browser>
```

```bash yarn
extension install <browser>
```

```bash bun
extension install <browser>
```

```bash bun
extension install <browser>
```

</CodeGroup>

仅当你需要多个目标、浏览器家族或 `all` 时才使用 `--browser`。

## Install 命令能力

| 能力               | 它能提供什么                                                               |
| ------------------ | -------------------------------------------------------------------------- |
| 受管浏览器缓存     | 在 Extension.js 浏览器缓存中获得稳定的安装位置                             |
| 可复现的运行时     | 用于可重复的本地运行和自动化的一致二进制                                   |
| 跨浏览器搭建       | 用一条命令流为 Chrome、Chromium、Edge 和 Firefox 完成搭建                  |
| 路径查询           | `--where` 显示解析得到的缓存根目录或浏览器特定的安装路径                   |

## 用法


## 参数与选项

| 选项 / 参数                                                                                    | 作用                                                                               | 默认值     |
| ---------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------- | ---------- |
| `[browser-name]`                                                                               | 安装单个浏览器目标,例如 `chrome`、`chromium`、`edge` 或 `firefox`                 | `chromium` |
| `--browser <chrome\|chromium\|edge\|firefox\|chromium-based\|gecko-based\|firefox-based\|all>` | 覆盖位置参数的浏览器名,并支持多目标安装                                           | 未设置     |
| `--where`                                                                                      | 打印解析得到的受管缓存根目录,或浏览器特定的安装路径                               | 禁用       |

## 示例

### 安装 Chrome for Testing

```bash
extension install chrome
```

### 在一条命令中安装多个目标

```bash
extension install --browser chrome,firefox
```

### 显示 Chrome 的受管安装路径

```bash
extension install chrome --where
```

## 缓存位置

默认情况下,Extension.js 将受管浏览器存储在每个用户稳定的缓存中:

- macOS: `~/Library/Caches/extension.js/browsers`
- Linux: `~/.cache/extension.js/browsers` 或 `$XDG_CACHE_HOME/extension.js/browsers`
- Windows: `%LOCALAPPDATA%\extension.js\browsers`

你可以通过 `EXT_BROWSERS_CACHE_DIR` 覆盖缓存根目录。

## 最佳实践

- **在 CI 中使用 `install`** 来锁定一个一致的浏览器二进制,而不是依赖运行器提供的任何版本。
- **优先使用 `chrome`** 而非 `chromium`,因为 Chrome for Testing 更贴近稳定版 Chrome 的行为。
- **使用 `--where`** 在围绕受管浏览器编写自动化脚本前确认缓存路径。
- `install` 仅管理 Extension.js 缓存内的浏览器,不会修改系统浏览器的安装。

## 行为说明

- `chrome` 会安装 Chrome for Testing,而不是依赖系统中的 Google Chrome 应用。
- 在 Linux 上,`edge` 可能需要一个具有特权的交互式会话。
