使用 `install` 将一个托管的浏览器运行时加入 Extension.js 缓存。

当你希望在 `dev`、`build`、`start` 或 `preview` 中使用一致的浏览器二进制时，它最为有用。支持 Chrome for Testing、Chromium、Firefox 和 Edge。

## 何时使用 `install`

- 你需要在持续集成（CI）、自动化或团队协作的本地运行中使用一致、可重复的浏览器二进制。
- 你希望使用 Chrome for Testing，而不是依赖系统上恰好安装的某个 Chrome 版本。
- 你正在用托管的 Firefox 或 Edge 运行时搭建跨浏览器测试。

## 标准用法

针对单个浏览器，使用位置参数形式：

<CodeGroup>

```bash npm
extension install <browser>
```

```bash pnpm
extension install <browser>
```

```bash yarn
extension install <browser>
```

```bash bun
extension install <browser>
```

```bash bun
extension install <browser>
```

</CodeGroup>

只有在需要多个目标、浏览器家族或 `all` 时才使用 `--browser`。

## install 命令的能力

| 能力             | 你能得到什么                                                                |
| ---------------- | --------------------------------------------------------------------------- |
| 托管浏览器缓存   | 在 Extension.js 浏览器缓存下提供稳定的安装位置                              |
| 可重复的运行时   | 让本地运行与自动化都使用一致的二进制                                        |
| 跨浏览器搭建     | 用一条命令流程搞定 Chrome、Chromium、Edge 与 Firefox                        |
| 路径发现         | `--where` 会显示已解析的缓存根目录或某个浏览器的具体安装路径                |

## 参数与标志

| 标志 / 参数                                                                                    | 作用                                                                              | 默认值     |
| ---------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------- | ---------- |
| `[browser-name]`                                                                               | 安装单个浏览器目标，例如 `chrome`、`chromium`、`edge` 或 `firefox`                | `chromium` |
| `--browser <chrome\|chromium\|edge\|firefox\|chromium-based\|gecko-based\|firefox-based\|all>` | 覆盖位置参数中的浏览器名称，并支持多目标安装                                      | 未设置     |
| `--where`                                                                                      | 打印已解析的托管缓存根目录，或某个浏览器具体的安装路径                            | 禁用       |

## 示例

### 安装 Chrome for Testing

```bash
extension install chrome
```

### 在一条命令中安装多个目标

```bash
extension install --browser chrome,firefox
```

### 显示 Chrome 的托管安装路径

```bash
extension install chrome --where
```

## 缓存位置

默认情况下，Extension.js 将托管浏览器存储在一个稳定的、按用户隔离的缓存中：

- macOS：`~/Library/Caches/extension.js/browsers`
- Linux：`~/.cache/extension.js/browsers` 或 `$XDG_CACHE_HOME/extension.js/browsers`
- Windows：`%LOCALAPPDATA%\extension.js\browsers`

你可以用 `EXT_BROWSERS_CACHE_DIR` 覆盖缓存根目录。

## 最佳实践

- **在 CI 中使用 `install`**，固定一个一致的浏览器二进制，而不是依赖 runner 提供的版本。
- **更倾向于 `chrome`** 而不是 `chromium`，因为 Chrome for Testing 的行为更接近稳定版 Chrome。
- **使用 `--where`** 在围绕托管浏览器编写自动化脚本前，先确认缓存路径。
- `install` 只管理 Extension.js 缓存中的浏览器，它不会修改系统的浏览器安装。

## 行为说明

- `chrome` 会安装 Chrome for Testing，而不是依赖系统中的 Google Chrome 应用。
- 在 Linux 上，`edge` 可能需要一个有特权的交互式会话。
