使用 `uninstall` 从 Extension.js 缓存中移除托管的浏览器运行时。

该命令只会移除 Extension.js 安装在其托管缓存根目录下的浏览器。它不会动到系统的 Chrome、系统的 Edge，或任何安装在该缓存之外的浏览器。

## 何时使用 `uninstall`

- 你想为不再需要的托管浏览器回收磁盘空间。
- 你正在重置某个托管浏览器的安装，以便下次 `install` 时强制全新下载。
- 你正在清理持续集成（CI）缓存或切换浏览器目标。

## 标准用法

针对单个浏览器，使用位置参数形式：

<CodeGroup>

```bash npm
extension uninstall <browser>
```

```bash pnpm
extension uninstall <browser>
```

```bash yarn
extension uninstall <browser>
```

```bash bun
extension uninstall <browser>
```

```bash bun
extension uninstall <browser>
```

</CodeGroup>

使用 `--all` 可以移除全部托管的浏览器目标。

## 用法

<CodeGroup>

```bash npm
extension uninstall [browser-name] [options]
```

```bash pnpm
extension uninstall [browser-name] [options]
```

```bash yarn
extension uninstall [browser-name] [options]
```

```bash bun
extension uninstall [browser-name] [options]
```

```bash bun
extension uninstall [browser-name] [options]
```

</CodeGroup>

## 参数与标志

| 标志 / 参数           | 作用                                                                              | 默认值   |
| --------------------- | --------------------------------------------------------------------------------- | -------- |
| `[browser-name]`      | 移除某个托管的浏览器，例如 `chrome`、`chromium`、`edge` 或 `firefox`              | 未设置   |
| `--browser <browser>` | 浏览器参数的显式标志形式                                                          | 未设置   |
| `--all`               | 从 Extension.js 缓存中移除所有托管的浏览器运行时                                  | 禁用     |
| `--where`             | 打印已解析的缓存根目录，或某个浏览器的具体托管路径                                | 禁用     |

## 示例

### 移除托管的 Chrome for Testing

```bash
extension uninstall chrome
```

### 移除所有托管的浏览器

```bash
extension uninstall --all
```

### 显示 Firefox 的托管卸载路径

```bash
extension uninstall firefox --where
```

## 最佳实践

- **在 CI 收尾阶段使用 `--all`**，在测试运行后清理托管的浏览器。
- **先使用 `--where`**，在脚本化批量卸载之前确认会被移除的内容。
- 对于系统浏览器，`uninstall` 是安全的；它只会移除由 Extension.js 管理的缓存文件夹。

## 行为说明

- 如果设置了 `EXT_BROWSERS_CACHE_DIR`，卸载会使用那个自定义缓存根目录。
