使用 `uninstall` 从 Extension.js 缓存中移除托管的浏览器运行时。

此命令只会移除 Extension.js 安装在其托管缓存根目录下的浏览器,不会触及系统 Chrome、系统 Edge,或在该缓存之外安装的任何浏览器。

## 何时使用 `uninstall`

- 你想从不再需要的托管浏览器中收回磁盘空间。
- 你正在重置托管浏览器的安装,以便下次 `install` 时强制重新下载。
- 你正在清理持续集成(CI)缓存或切换浏览器目标。

## 规范用法

对于单个浏览器,使用位置参数形式:

<CodeGroup>

```bash npm
extension uninstall <browser>
```

```bash pnpm
extension uninstall <browser>
```

```bash yarn
extension uninstall <browser>
```

```bash bun
extension uninstall <browser>
```

```bash bun
extension uninstall <browser>
```

</CodeGroup>

使用 `--all` 移除每个托管浏览器目标。

## 用法

<CodeGroup>

```bash npm
extension uninstall [browser-name] [options]
```

```bash pnpm
extension uninstall [browser-name] [options]
```

```bash yarn
extension uninstall [browser-name] [options]
```

```bash bun
extension uninstall [browser-name] [options]
```

```bash bun
extension uninstall [browser-name] [options]
```

</CodeGroup>

## 参数与标志

| 标志 / 参数           | 它的作用                                                                            | 默认值   |
| --------------------- | ----------------------------------------------------------------------------------- | -------- |
| `[browser-name]`      | 移除单个托管浏览器,如 `chrome`、`chromium`、`edge` 或 `firefox`                    | 未设置   |
| `--browser <browser>` | 浏览器参数的显式标志形式                                                            | 未设置   |
| `--all`               | 移除 Extension.js 缓存中的所有托管浏览器运行时                                      | 禁用     |
| `--where`             | 打印已解析的缓存根目录,或特定浏览器的托管路径                                      | 禁用     |

## 示例

### 移除托管的 Chrome for Testing

```bash
extension uninstall chrome
```

### 移除所有托管浏览器

```bash
extension uninstall --all
```

### 显示 Firefox 的托管卸载路径

```bash
extension uninstall firefox --where
```

## 最佳实践

- **在 CI 清理中使用 `--all`**,在测试运行后清理托管浏览器。
- **先使用 `--where`** 在脚本化批量卸载之前确认将要移除的内容。
- `uninstall` 对系统浏览器是安全的;它只会移除 Extension.js 托管的缓存文件夹。

## 行为说明

- 如果你设置了 `EXT_BROWSERS_CACHE_DIR`,uninstall 会使用该自定义缓存根目录。
