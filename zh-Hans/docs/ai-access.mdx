通过两种标准的 AI 可访问入口来获取 Extension.js 文档。这能让你的编辑器和助手基于最新文档而非过时的训练数据来回答问题。

<Note>
  本页面讲的是把**文档**提供给助手 —— 用最新内容回答“我该怎么做……”。如果你希望让助手**驱动工具链**（运行 dev 会话、流式查看日志、检查 DOM、触发操作、reload），请参阅 [Chrome DevTools MCP with
  Extension.js](/docs/integrations/chrome-devtools-mcp) 中介绍的独立控制服务器。
</Note>

## 托管的 MCP（Model Context Protocol）服务器

文档站点在以下地址暴露了一个 [Model Context Protocol](https://modelcontextprotocol.io) 服务器：

```
https://extensionjs.mintlify.app/mcp
```

将任何兼容 MCP 的客户端指向该 URL，它就会获得一系列工具，能够搜索文档、获取特定页面，并基于最新内容回答问题。

### 一键安装

每个页面都有一个上下文菜单（位于页面右上角），其中包含常见客户端的安装按钮：**Claude**、**Cursor**、**VS Code** 和 **ChatGPT**。使用它们是最快的方式。

### 手动安装

<CodeGroup>

```json Claude Desktop (claude_desktop_config.json)
{
  "mcpServers": {
    "extensionjs": {
      "url": "https://extensionjs.mintlify.app/mcp"
    }
  }
}
```

```json Cursor (.cursor/mcp.json)
{
  "mcpServers": {
    "extensionjs": {
      "url": "https://extensionjs.mintlify.app/mcp"
    }
  }
}
```

```jsonc VS Code (settings.json)
"mcp.servers": {
  "extensionjs": {
    "url": "https://extensionjs.mintlify.app/mcp"
  }
}
```

```bash Claude Code
claude mcp add --transport http extensionjs https://extensionjs.mintlify.app/mcp
```

</CodeGroup>

安装后，可以向你的助手提问，比如 _“我该如何在 Extension.js 中配置浏览器特定的 manifest 字段？”_ 助手会调用 MCP 工具并引用文档页面。

## llms.txt

文档在以下地址提供了一个静态、对机器友好的索引：

- [`/llms.txt`](https://extensionjs.mintlify.app/llms.txt)：所有页面的简短索引
- [`/llms-full.txt`](https://extensionjs.mintlify.app/llms-full.txt)：用于摄入的完整内容包

当你想把文档直接喂给检索流水线，或在自定义 agent 中对其进行总结时，可以使用它们。Extension.js 在每次文档部署时都会重新生成这些文件。

## 页面级 AI 操作

页面级上下文菜单还提供：

- **复制页面：**把当前页面以 Markdown 形式复制，便于粘贴到聊天中
- **以 Markdown 查看：**在新标签页中打开原始 Markdown 源文件
- **询问 ChatGPT / Claude / Perplexity：**打开相应的助手，并预先以该页面作为上下文

## 最佳实践

- 当你希望助手跨多个页面进行推理并保持答案最新时，优先使用 MCP 服务器。
- 对于离线工具、评估或自定义 RAG（检索增强生成）流水线，优先使用 `llms-full.txt`。
- 对于你正在阅读的页面的快速问题，使用页面级操作即可。
