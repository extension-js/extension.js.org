使用实用类跨 popup、options、新标签页、sidebar 和内容脚本入口快速构建扩展 UI 表面。

Extension.js 通过 PostCSS 集成 Tailwind,并同时支持 Tailwind v3 和 v4 配置。

## 何时适合使用 Tailwind CSS

- 你需要跨多个扩展表面快速迭代 UI。
- 你偏好以共享 token 的实用类优先样式。
- 你希望在页面和内容脚本入口间保持一致的样式模式。

## Tailwind 能力

| 能力                      | 提供给你的功能                                                                |
| ------------------------- | ----------------------------------------------------------------------------- |
| 多表面样式                | 在扩展页面和内容脚本之间复用实用类                                            |
| 基于 PostCSS 的集成       | 把 Tailwind 放在与项目其余部分相同的 CSS 转换流水线中                         |
| 版本灵活性                | 支持 Tailwind v3 与 Tailwind v4 的插件模式                                    |
| 模板上手                  | 从现成配置起步并增量调整                                                      |

## 在现有扩展中使用

### 安装

安装所需依赖:

<CodeGroup>

```bash npm
npm install -D tailwindcss @tailwindcss/postcss postcss
```

```bash pnpm
pnpm install -D tailwindcss @tailwindcss/postcss postcss
```

```bash yarn
yarn add -D tailwindcss @tailwindcss/postcss postcss
```

```bash bun
bun add --dev tailwindcss @tailwindcss/postcss postcss
```

```bash bun
bun add --dev tailwindcss @tailwindcss/postcss postcss
```

</CodeGroup>

### 配置

创建 `postcss.config.js`:

```js
export default {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
```

对于 Tailwind v3 项目,使用 `tailwindcss` 作为 PostCSS 插件,并保留 `tailwind.config.js` 的 content 列表。

### Tailwind v4 与 v3 的快速配置

```js
// Tailwind v4 postcss.config.js
export default {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
```

```js
// Tailwind v3 postcss.config.js
export default {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
};
```

### Tailwind content 路径(v3 风格)

```js
/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./pages/**/*.{html,js,ts,jsx,tsx,mdx}",
    "./scripts/**/*.{js,ts,jsx,tsx}",
    "./src/**/*.{html,js,ts,jsx,tsx,mdx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
};
```

### 用法

创建一个全局样式表并导入 Tailwind:

#### 创建全局 CSS 文件

```css
/* styles/globals.css */
@import "tailwindcss";
```

在使用 Tailwind 类的入口脚本中导入此文件。

#### 在页面脚本中(popup/options/新标签页)

```tsx
import "./styles/globals.css";

export default function MyNewTabPage() {
  return <h1 className="text-4xl font-bold">Hello, Extension.</h1>;
}
```

#### 在内容脚本中

```ts
import "./styles/globals.css";

console.log("Tailwind styles loaded for content script");
```

Extension.js 通过其内容脚本样式流水线路由内容脚本的 CSS,并把这些资源输出到 `content_scripts/` 下。

## 最佳实践

- 把 Tailwind 源导入保留在共享的样式入口文件中(`styles/globals.css`)。
- 当相关时,确保你的 content 扫描包含 `pages/`、`scripts/` 和 `src/`。
- 在 monorepo 中,验证 Tailwind 的 content 路径从扩展项目根解析。
- 尽可能使用模板,以避免 PostCSS 与 Tailwind 版本之间的配置漂移。

## 后续步骤

- 为额外的 CSS 转换配置 [PostCSS](/docs/integrations/postcss)。
