使用实用类在 popup、options、new tab、sidebar 和内容脚本入口中快速构建扩展 UI 界面。

Extension.js 通过 PostCSS 集成 Tailwind,并同时支持 Tailwind v3 和 v4 设置。

## 何时适合使用 Tailwind CSS

- 你需要在多个扩展界面之间快速迭代 UI。
- 你偏好实用优先的样式编写方式,并使用共享 token。
- 你希望在页面和内容脚本入口之间使用一致的样式模式。

## Tailwind 能力

| 能力                       | 它能为你提供什么                                                               |
| -------------------------- | ------------------------------------------------------------------------------ |
| 多界面样式                 | 在扩展页面和内容脚本之间复用实用类                                             |
| 基于 PostCSS 的集成        | 让 Tailwind 与项目的其他部分使用相同的 CSS 转换管道                            |
| 版本灵活性                 | 支持 Tailwind v3 和 Tailwind v4 的插件模式                                     |
| 模板入门                   | 从现成的设置开始,然后逐步调整                                                 |

## 与现有扩展一起使用

### 安装

安装所需依赖:

<CodeGroup>

```bash npm
npm install -D tailwindcss @tailwindcss/postcss postcss
```

```bash pnpm
pnpm install -D tailwindcss @tailwindcss/postcss postcss
```

```bash yarn
yarn add -D tailwindcss @tailwindcss/postcss postcss
```

```bash bun
bun add --dev tailwindcss @tailwindcss/postcss postcss
```

```bash bun
bun add --dev tailwindcss @tailwindcss/postcss postcss
```

</CodeGroup>

### 配置

创建 `postcss.config.js`:

```js
export default {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
```

对于 Tailwind v3 项目,使用 `tailwindcss` 作为 PostCSS 插件,并保留 `tailwind.config.js` 的 content 列表。

### Tailwind v4 vs v3 快速设置

```js
// Tailwind v4 postcss.config.js
export default {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
```

```js
// Tailwind v3 postcss.config.js
export default {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
};
```

### Tailwind content 路径(v3 风格)

```js
/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./pages/**/*.{html,js,ts,jsx,tsx,mdx}",
    "./scripts/**/*.{js,ts,jsx,tsx}",
    "./src/**/*.{html,js,ts,jsx,tsx,mdx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
};
```

### 用法

创建全局样式表并导入 Tailwind:

#### 创建全局 CSS 文件

```css
/* styles/globals.css */
@import "tailwindcss";
```

在使用 Tailwind 类的入口脚本中导入此文件。

#### 在页面脚本中(popup/options/new-tab)

```tsx
import "./styles/globals.css";

export default function MyNewTabPage() {
  return <h1 className="text-4xl font-bold">Hello, Extension.</h1>;
}
```

#### 在内容脚本中

```ts
import "./styles/globals.css";

console.log("Tailwind styles loaded for content script");
```

Extension.js 通过其内容脚本样式管道路由内容脚本的 CSS,并在 `content_scripts/` 下发出这些资源。

## 最佳实践

- 将 Tailwind 源导入保留在共享样式入口文件中(`styles/globals.css`)。
- 确保你的内容扫描在相关时包括 `pages/`、`scripts/` 和 `src/`。
- 在 monorepo 中,验证 Tailwind 的 content 路径从扩展项目根解析。
- 尽可能使用模板,以避免 PostCSS 与 Tailwind 版本之间的配置漂移。
