在扩展各表面间保持样式表质量一致,并尽早捕获 CSS 问题。Stylelint 通过脚本或持续集成(CI)运行,而不是通过 Extension.js 的开发/构建流水线。

## 何时适合使用 Stylelint

- 你在多个扩展表面间维护共享的 CSS/Sass/Less。
- 你希望在 PR 合并前进行样式质量检查。
- 你需要在团队工作流中保持一致的样式表约定。

## Stylelint 能力

| 能力                      | 提供给你的功能                                       |
| ------------------------- | ---------------------------------------------------- |
| CSS 质量检查              | 捕获无效模式并强制执行样式约定                       |
| 预处理器支持              | 在一个工作流中 lint CSS、Sass 和 Less                |
| 脚本与 CI 集成            | 在合并和发布前运行样式检查                           |
| 格式化器对齐              | 与 Prettier 结合以减少样式工具冲突                   |

## 在现有扩展中使用

安装 Stylelint 依赖:

<PackageManagerTabs command="install -D stylelint stylelint-config-standard-scss" />

### Stylelint 配置

在项目根目录创建一个 Stylelint 配置文件(例如 `.stylelintrc.json`):

```json
{
  "extends": "stylelint-config-standard-scss",
  "rules": {
    "color-no-invalid-hex": true,
    "block-no-empty": true
  }
}
```

### 运行 Stylelint

<CodeGroup>

```bash npm
npm run lint:styles
```

```bash pnpm
pnpm lint:styles
```

```bash yarn
yarn lint:styles
```

```bash bun
bun run lint:styles
```

```bash bun
bun run lint:styles
```

</CodeGroup>

## 与 Prettier 集成

把 Stylelint 与 Prettier 集成以同时格式化和 lint 样式。安装以下依赖:

<PackageManagerTabs command="install -D stylelint-prettier prettier" />

然后,更新你的 `.stylelintrc.json` 文件以包含 Prettier:

```json
{
  "extends": [
    "stylelint-config-standard-scss",
    "stylelint-prettier/recommended"
  ]
}
```

这让样式的格式化与 lint 关注点保持对齐。

## 最佳实践

- 保留一个专用的 `lint:styles` 命令并在 CI 中运行它。
- 在 monorepo 中,在多个扩展包之间使用同一份共享配置。
- 让 Stylelint 与 Prettier 配对,使样式格式化保持确定性。

## 检测说明

Extension.js 可以检测 Stylelint 配置文件以进行诊断/报告,但它不会把 Stylelint 注入构建流水线。
