shadcn/ui 在扩展中的工作方式与在 web 应用中相同,只要你的项目使用了启用 Tailwind 的 React 设置。组件位于你的源代码树中(而不是 `node_modules`),因此你可以完全控制样式和行为。

## 何时适合使用 shadcn/ui

- 你需要在 sidebar、popup 和 options 界面之间复用 UI 原语。
- 你的产品代码中已经使用 React + Tailwind。
- 你希望团队可以完全自定义的源代码所有组件。

## shadcn/ui 能力

| 能力                       | 它能为你提供什么                                                    |
| -------------------------- | ------------------------------------------------------------------- |
| 源代码所有的 UI 组件       | 让生成的 `components/ui/*` 文件在你的项目中受版本控制               |
| React + Tailwind 对齐      | 与现代 web 应用栈使用相同的编写模型                                 |
| 扩展界面复用               | 在 sidebar、popup 和 options 页面之间共享组件原语                   |
| 渐进式采用                 | 从模板开始,然后逐步扩大组件覆盖范围                                |

## 技术栈假设

- React + TypeScript
- Tailwind CSS
- PostCSS(`@tailwindcss/postcss`)
- 本地 `components/ui/*` 组件文件(shadcn 风格的结构)

## 在现有项目中手动设置

1. 设置 Tailwind 和 PostCSS(参见 [Tailwind CSS](/docs/integrations/tailwindcss))。
2. 在你的项目中初始化 shadcn 并生成组件。
3. 将生成的 UI 组件放在你的扩展源代码中(例如 `src/components/ui/`)。
4. 在 UI 渲染所在的扩展入口中导入你的 Tailwind 样式表。

### PostCSS 示例(Tailwind v4)

```js
export default {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
```

### `components.json` 示例

```json
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "style": "new-york",
  "tsx": true,
  "tailwind": {
    "css": "sidebar/styles.css",
    "cssVariables": true
  }
}
```

## 用法

通过本地导入使用生成的组件:

```tsx
import { Button } from "@/components/ui/button";
import "./styles.css";

export default function MyComponent() {
  return <Button className="text-lg font-semibold">Click Me</Button>;
}
```

## 内容脚本注意事项

- 扩展页面(popup/options/sidebar/new tab)是最容易开始使用 shadcn/ui 的地方。
- 对于内容脚本,在内容脚本入口点中导入你的样式入口。
- Extension.js 通过其 CSS 管道发出内容脚本样式,它们仍可能与宿主页面的样式冲突。

如果你需要为内容脚本提供更强的隔离,请将 UI 挂载到 Shadow DOM 根中(一种封装样式和标记的浏览器 API)。有意地为类名设定作用域。

## 最佳实践

- 将 shadcn/ui 视为源代码所有的组件,而不是作为运行时 UI 包导入。
- 让 Tailwind 配置/内容路径与你的扩展文件夹(`pages`、`scripts`、`src`)对齐。
- 先从一个界面(如 sidebar 或 popup)开始,然后再添加内容脚本渲染的复杂性。
- 将共享 token 和工具辅助函数保留在专用文件中(`lib/utils`、设计 token、主题变量)。
