把 Extension.js 与现代 Web 项目中使用的格式化、Linting、CSS 工具和 UI 组件工作流连接起来。

## 按需选择

| 需求                                                       | 从这里开始                                                    |
| ---------------------------------------------------------- | ------------------------------------------------------------- |
| 添加 SWC(Speedy Web Compiler)之外的自定义转换            | [Babel](/docs/integrations/babel)                             |
| 强制执行代码质量规则                                       | [ESLint](/docs/integrations/eslint)                           |
| 保持格式化一致                                             | [Prettier](/docs/integrations/prettier)                       |
| 使用插件转换 CSS                                           | [PostCSS](/docs/integrations/postcss)                         |
| 在持续集成(CI)中对样式表进行 lint                        | [Stylelint](/docs/integrations/stylelint)                     |
| 构建可组合的 UI 原语                                       | [shadcn/ui](/docs/integrations/shadcnui)                      |
| 实用类优先的样式体系                                       | [Tailwind CSS](/docs/integrations/tailwindcss)                |
| 通过 Chrome DevTools 让 AI 代理调试扩展                    | [Chrome DevTools MCP](/docs/integrations/chrome-devtools-mcp) |

## 实用的集成顺序

1. 添加基础质量工具:ESLint + Prettier。
2. 添加样式工具:PostCSS + Stylelint(如果需要还可以加 Tailwind)。
3. 仅在必要时添加特定框架的工具。

## 后续步骤

- 在 [扩展配置](/docs/features/extension-configuration) 中对齐项目默认值。
- 通过 [工作流](/docs/workflows/index) 校验变更。
