将 Extension.js 与现代 Web 项目中常用的格式化、Lint、CSS 工具链以及 UI 组件工作流连接起来。

## 按需求选择

| 需求                                                            | 从这里开始                                                            |
| --------------------------------------------------------------- | --------------------------------------------------------------------- |
| 添加超出 SWC（Speedy Web Compiler）范围的自定义转换             | [Babel](/zh-Hans/docs/integrations/babel)                             |
| 强制代码质量规则                                                | [ESLint](/zh-Hans/docs/integrations/eslint)                           |
| 保持格式一致                                                    | [Prettier](/zh-Hans/docs/integrations/prettier)                       |
| 用插件转换 CSS                                                  | [PostCSS](/zh-Hans/docs/integrations/postcss)                         |
| 在持续集成（CI）中 lint 样式表                                  | [Stylelint](/zh-Hans/docs/integrations/stylelint)                     |
| 构建可组合的 UI 基础元素                                        | [shadcn/ui](/zh-Hans/docs/integrations/shadcnui)                      |
| 工具类优先的样式体系                                            | [Tailwind CSS](/zh-Hans/docs/integrations/tailwindcss)                |
| 借助 Chrome DevTools 用 AI agent 调试扩展                       | [Chrome DevTools MCP](/zh-Hans/docs/integrations/chrome-devtools-mcp) |

## 实用的集成顺序

1. 添加基线质量工具：ESLint + Prettier。
2. 添加样式工具：PostCSS + Stylelint（必要时还有 Tailwind）。
3. 仅在需要时再添加框架特定的工具。

## 下一步

- 在 [Extension 配置](/zh-Hans/docs/features/extension-configuration) 中对齐项目默认值。
- 通过 [工作流](/zh-Hans/docs/workflows/index) 验证变更。
