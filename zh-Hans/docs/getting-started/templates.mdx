通过官方模板生成扩展，无需从零搭建目录结构和配置，让你更快进入开发状态。

## 模板能力

- **开箱即用的项目结构：** 自带可工作的 manifest、脚本以及源码布局。
- **针对技术栈的配置：** 根据框架与运行时需求选择模板。
- **更顺畅的上手过程：** 在几分钟内从想法走到可运行的扩展。
- **一致的默认值：** 减少不同项目和团队成员之间的配置漂移。

## 快速上手

<CodeGroup>

```bash npm
npx extension@latest create my-extension --template=<slug>
```

```bash pnpm
pnpx extension@latest create my-extension --template=<slug>
```

```bash yarn
yarn dlx extension@latest create my-extension --template=<slug>
```

```bash bun
bunx extension@latest create my-extension --template=<slug>
```

```bash bun
bunx extension@latest create my-extension --template=<slug>
```

</CodeGroup>

## 工作原理

当你运行 `create --template=<slug>` 时，Extension.js 会拉取所选模板，生成项目文件，并可选地安装依赖。

## 更多模板

在 examples 仓库中查看官方模板源码：

- [extension-js/examples](https://github.com/extension-js/examples/tree/main/examples)

模板文件夹名称会直接映射到 `--template=<slug>`。

## 最佳实践

- 使用模板快速生成项目。
- 选择一个你已经熟悉的技术栈对应的模板作为起点。
- 选择最契合项目需求的浏览器扩展上下文。
