通过从官方模板搭建扩展项目,无需从零构建目录结构和配置,从而更快开始。

## 模板能力

- **现成的项目结构:** 提供可工作的 manifest、脚本和源码布局。
- **针对技术栈的预设:** 按框架和运行时需求选择模板。
- **更快上手:** 几分钟从想法走到可运行的扩展。
- **一致的默认值:** 减少项目和团队成员之间的配置漂移。

## 快速使用

<CodeGroup>

```bash npm
npx extension@latest create my-extension --template=<slug>
```

```bash pnpm
pnpx extension@latest create my-extension --template=<slug>
```

```bash yarn
yarn dlx extension@latest create my-extension --template=<slug>
```

```bash bun
bunx extension@latest create my-extension --template=<slug>
```

```bash bun
bunx extension@latest create my-extension --template=<slug>
```

</CodeGroup>

将 `<slug>` 替换为你想使用的模板。

## 工作原理

当你运行 `create --template=<slug>` 时,Extension.js 会获取所选模板、生成项目文件,并可选地安装依赖。

## 更多模板

在示例仓库中查看官方模板源码:

- [extension-js/examples](https://github.com/extension-js/examples/tree/main/examples)

模板文件夹的名称直接对应 `--template=<slug>`。

## 最佳实践

- 使用模板快速搭建项目。
- 选择你已经熟悉的技术栈所对应的模板作为起点。
- 选择最契合你项目需求的浏览器扩展上下文。
