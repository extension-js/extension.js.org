通过从官方模板脚手架扩展,而不是从零搭建目录结构与配置,更快地起步。

## 模板能力

- **现成的项目结构:** 从可用的 manifest、脚本和源代码布局开始。
- **针对技术栈的设置:** 根据框架和运行时需求选择模板。
- **更快的上手:** 几分钟从想法走到可运行的扩展。
- **一致的默认值:** 减少项目间和团队成员间的配置漂移。

## 快速使用

<CodeGroup>

```bash npm
npx extension@latest create my-extension --template=<slug>
```

```bash pnpm
pnpx extension@latest create my-extension --template=<slug>
```

```bash yarn
yarn dlx extension@latest create my-extension --template=<slug>
```

```bash bun
bunx extension@latest create my-extension --template=<slug>
```

```bash bun
bunx extension@latest create my-extension --template=<slug>
```

</CodeGroup>

把 `<slug>` 替换为你想作为起点的模板。

## 工作原理

当你运行 `create --template=<slug>` 时,Extension.js 会获取所选模板、脚手架项目文件,并可选地安装依赖。

## 更多模板

可在示例仓库中找到官方模板源:

- [extension-js/examples](https://github.com/extension-js/examples/tree/main/examples)

模板文件夹名直接对应 `--template=<slug>`。

## 最佳实践

- 使用模板快速脚手架项目。
- 从你已经熟悉的技术栈对应的模板开始。
- 选择最符合项目需求的浏览器扩展上下文。
