## 30 秒运行任意 GitHub 上的扩展

下面的小节覆盖了两种路径。你会了解 Extension.js 把什么视为项目根目录，以及它如何处理像"裸示例名"这样的歧义输入。

## `dev` 如何解析项目

`extension dev` 接受一个可选的第一参数。解析规则如下：

- **无参数：** 把当前工作目录当作扩展项目。
- **本地路径：** Extension.js 把它视为相对于当前工作目录的路径（例如克隆仓库后的 `./my-extension` 或 `../samples/page-redder`）。
- **`https://github.com/...` URL：** Extension.js 会把该仓库（或 tree 路径）拉取到你的工作目录，然后从解出的文件夹构建。详情见[项目检测与输入](https://github.com/extension-js/extension.js/blob/main/programs/develop/README.md#project-detection-and-inputs)。
- **其他 `http(s)` URL：** Extension.js 把该 URL 视为 ZIP 包，下载后在本地解压。

像 `sample.page-redder` 这样的裸名**不会**触发远程获取；它只是当前目录下的一个文件夹名。要么使用完整的 GitHub URL，要么先克隆再把 `dev` 指向该路径。

## 快速运行 Chrome Extension Samples

使用 Chrome Extension Samples 仓库里的示例来验证你的环境并快速熟悉工作流。

### 方式 A：完整 GitHub tree URL（推荐）

1. 打开终端。
2. `cd` 到希望"被克隆/解压的项目"放置的目录（通常是一个空文件夹）。
3. 运行：

<CodeGroup>

```bash npm
npx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash pnpm
pnpx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash yarn
yarn dlx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash bun
bunx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash bun
bunx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

</CodeGroup>

Extension.js 会拉取这个示例、构建它，并启动一个加载了该扩展的 Chrome。浏览 [Chrome Extension Samples](https://github.com/GoogleChrome/chrome-extensions-samples) 获取更多可以试的 URL。要面向其他浏览器，追加 `--browser=firefox` 或 `--browser=edge`。

## 先克隆，再用本地路径 `dev`

当你想要稳定的文件夹布局（而不是一次性试试）时，先克隆仓库，再把 `dev` 指向本地路径：

```bash
git clone https://github.com/GoogleChrome/chrome-extensions-samples.git
cd chrome-extensions-samples/functional-samples/sample.page-redder
npx extension@latest dev .
```

## 在 Edge 上通过 polyfill 运行 Mozilla 扩展

你可以在 Edge 上运行 MDN WebExtensions 的示例。把 GitHub tree URL 传过来，并启用 polyfill（兼容层）。polyfill 会把 Firefox 风格的 `browser.*` 调用映射成 Chrome 风格的 `chrome.*` 调用，让面向 Firefox 的扩展也能在基于 Chromium 的浏览器中运行。

<CodeGroup>

```bash npm
npx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash pnpm
pnpx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash yarn
yarn dlx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash bun
bunx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash bun
bunx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

</CodeGroup>

### 先克隆，再用 polyfill + Edge

```bash
git clone https://github.com/mdn/webextensions-examples.git
cd webextensions-examples/apply-css
npx extension@latest dev . --browser=edge --polyfill=true
```

这个例子使用了 MDN WebExtensions Examples 中的 [Apply CSS](https://github.com/mdn/webextensions-examples/tree/main/apply-css)。

## 从零创建一个最小扩展

使用 `extension create` 加 `init` 模板，得到最轻量的起点：只有 manifest 与图标，没有任何框架或 UI 表面。

### `init`

![init template screenshot](/images/examples/init/screenshot.png)

<CodeGroup>

```bash npm
npx extension@latest create my-extension --template=init
```

```bash pnpm
pnpx extension@latest create my-extension --template=init
```

```bash yarn
yarn dlx extension@latest create my-extension --template=init
```

```bash bun
bunx extension@latest create my-extension --template=init
```

```bash bun

```

</CodeGroup>

## 实用小贴士

- **使用 TypeScript：** 在项目根目录添加 `tsconfig.json`。
- **使用 React：** 在 `package.json` 中加入 `react` 与 `react-dom`。
- 一个为 React 配置好的 `tsconfig.json` 就能同时启用 TypeScript + React 开发。
- 如果需要处理 manifest 中未声明的资源，可以了解[特殊文件夹](/zh-Hans/docs/features/special-folders)。

## 最佳实践

- **使用 `extension` 包**，用一条工具链完成构建、运行与打包。
- **使用 `--browser`** 在开发时指定目标浏览器。
- **使用 `--polyfill`**，把面向 Mozilla 的示例适配到基于 Chromium 的浏览器。
- **优先使用完整 GitHub URL** 做一次性尝试；**克隆 + 本地路径**用于需要稳定文件夹布局的场景。
