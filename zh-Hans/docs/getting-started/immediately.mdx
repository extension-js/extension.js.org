## 30 秒内运行任何 GitHub 扩展

下面的章节涵盖两种路径。你将了解 Extension.js 把什么视为项目根目录,以及它如何解析像“裸的示例名称”这样模糊的输入。

## `dev` 如何解析项目

`extension dev` 接受一个可选的第一个参数。解析方式如下:

- **无参数:** 将当前工作目录作为扩展项目使用。
- **本地路径:** Extension.js 会将其视为相对于当前工作目录的路径(例如,在克隆仓库后使用 `./my-extension` 或 `../samples/page-redder`)。
- **`https://github.com/...` URL:** Extension.js 会将该仓库(或 tree 路径)抓取到你的工作目录,然后从解压后的目录进行构建。详情参见[项目检测与输入](https://github.com/extension-js/extension.js/blob/main/programs/develop/README.md#project-detection-and-inputs)。
- **其他 `http(s)` URL:** Extension.js 会将该 URL 视为一个 ZIP 压缩包,下载并在本地解压。

像 `sample.page-redder` 这样的“裸名称”**不是**远程抓取;它仅仅是当前目录下的一个文件夹名。请使用完整的 GitHub URL,或者先克隆再让 `dev` 指向该路径。

## 快速运行 Chrome 扩展示例

使用 Chrome Extension Samples 仓库中的示例来验证你的环境,并快速熟悉工作流。

### 方式 A:完整的 GitHub tree URL(推荐)

1. 打开终端。
2. `cd` 到希望放置克隆/解压后项目的目录(通常是一个空目录)。
3. 运行:

<CodeGroup>

```bash npm
npx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash pnpm
pnpx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash yarn
yarn dlx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash bun
bunx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash bun
bunx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

</CodeGroup>

Extension.js 会抓取该示例、进行构建,并启动 Chrome 加载该扩展。浏览 [Chrome Extension Samples](https://github.com/GoogleChrome/chrome-extensions-samples) 获取更多可尝试的 URL。如需指定其他浏览器,可附加 `--browser=firefox` 或 `--browser=edge`。

## 先克隆,再 `dev` 一个本地路径

当你希望拥有稳定的目录结构(而不是一次性试用)时,可以克隆仓库并让 `dev` 指向本地路径:

```bash
git clone https://github.com/GoogleChrome/chrome-extensions-samples.git
cd chrome-extensions-samples/functional-samples/sample.page-redder
npx extension@latest dev .
```

## 通过 polyfill 在 Edge 中运行 Mozilla 扩展

你可以在 Edge 中运行 MDN WebExtensions 示例。传入 GitHub tree URL 并启用 polyfill(一个兼容层)。polyfill 会将 Firefox 风格的 `browser.*` 调用映射为 Chrome 风格的 `chrome.*` 调用,这样面向 Firefox 的扩展也能在基于 Chromium 的浏览器中工作。

<CodeGroup>

```bash npm
npx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash pnpm
pnpx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash yarn
yarn dlx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash bun
bunx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash bun
bunx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

</CodeGroup>

### 先克隆,然后在 Edge 中使用 polyfill

```bash
git clone https://github.com/mdn/webextensions-examples.git
cd webextensions-examples/apply-css
npx extension@latest dev . --browser=edge --polyfill=true
```

此示例参考 MDN WebExtensions Examples 中的 [Apply CSS](https://github.com/mdn/webextensions-examples/tree/main/apply-css) 示例。

## 快速提示

- **使用 TypeScript:** 在你的项目根目录添加一个 `tsconfig.json` 文件。
- **使用 React:** 在 `package.json` 中添加 `react` 和 `react-dom`。
- 一个针对 React 配置好的 `tsconfig.json` 即可启用 TypeScript + React 开发。
- 如果你需要处理 manifest 中未声明的资源,可以了解更多关于[特殊文件夹](/docs/features/special-folders)的内容。

## 最佳实践

- **使用 `extension` 包**通过一套工具链来构建、运行并打包你的扩展。
- **使用 `--browser`** 在开发时指定目标浏览器。
- **使用 `--polyfill`** 将面向 Mozilla 的示例适配到基于 Chromium 的浏览器。
- **一次性试用优先使用完整的 GitHub URL**;**需要稳定目录结构时,先克隆再使用本地路径**。
