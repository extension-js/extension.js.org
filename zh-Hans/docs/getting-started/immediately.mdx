## 30 秒内运行任意 GitHub 扩展

下面的章节覆盖了两条路径。你将了解 Extension.js 把什么视作项目根,以及它如何解析像裸样例名这样的模糊输入。

## `dev` 如何解析项目

`extension dev` 接受一个可选的首参数。解析规则如下:

- **无参数:** 把当前工作目录作为扩展项目。
- **本地路径:** Extension.js 把它视作相对于当前工作目录的路径(例如,克隆仓库后的 `./my-extension` 或 `../samples/page-redder`)。
- **`https://github.com/...` URL:** Extension.js 会把仓库(或 tree 路径)抓取到你的工作目录,然后从提取的目录构建。详情参阅[项目检测与输入](https://github.com/extension-js/extension.js/blob/main/programs/develop/README.md#project-detection-and-inputs)。
- **其他 `http(s)` URL:** Extension.js 会把该 URL 视作 ZIP 归档,下载并在本地解压。

像 `sample.page-redder` 这样的裸名称**不是**远程抓取;它只是当前目录下的一个文件夹名。请使用完整的 GitHub URL,或者先克隆再让 `dev` 指向该路径。

## 快速运行 Chrome 扩展样例

使用 Chrome 扩展样例仓库中的样例来验证你的设置,并快速熟悉工作流。

### 方式 A:完整 GitHub tree URL(推荐)

1. 打开终端。
2. `cd` 到希望克隆/解压项目出现的目录(通常是一个空目录)。
3. 运行:

<CodeGroup>

```bash npm
npx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash pnpm
pnpx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash yarn
yarn dlx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash bun
bunx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

```bash bun
bunx extension@latest dev https://github.com/GoogleChrome/chrome-extensions-samples/tree/main/functional-samples/sample.page-redder
```

</CodeGroup>

Extension.js 会获取样例、构建它,并以加载扩展的方式启动 Chrome。浏览 [Chrome 扩展样例](https://github.com/GoogleChrome/chrome-extensions-samples)以尝试更多 URL。如需指向其他浏览器,追加 `--browser=firefox` 或 `--browser=edge`。

## 先克隆,再让 `dev` 指向本地路径

当你需要稳定的目录布局(而非一次性试用)时,先克隆仓库再让 `dev` 指向本地路径:

```bash
git clone https://github.com/GoogleChrome/chrome-extensions-samples.git
cd chrome-extensions-samples/functional-samples/sample.page-redder
npx extension@latest dev .
```

## 通过 polyfill 在 Edge 中运行 Mozilla 扩展

你可以在 Edge 中运行 MDN WebExtensions 示例。传入 GitHub tree URL 并启用 polyfill(兼容层)。该 polyfill 会把 Firefox 风格的 `browser.*` 调用映射为 Chrome 风格的 `chrome.*` 调用,这样你面向 Firefox 的扩展也能在 Chromium 系浏览器上工作。

<CodeGroup>

```bash npm
npx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash pnpm
pnpx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash yarn
yarn dlx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash bun
bunx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

```bash bun
bunx extension@latest dev https://github.com/mdn/webextensions-examples/tree/main/apply-css --browser=edge --polyfill=true
```

</CodeGroup>

### 先克隆,再使用 polyfill + Edge

```bash
git clone https://github.com/mdn/webextensions-examples.git
cd webextensions-examples/apply-css
npx extension@latest dev . --browser=edge --polyfill=true
```

本示例沿用 MDN WebExtensions Examples 中的 [Apply CSS](https://github.com/mdn/webextensions-examples/tree/main/apply-css) 示例。

使用 `extension create` 配合 `init` 模板获得最轻量的起点:仅包含 manifest 和图标,没有框架或 UI 表面。

## 快速提示

- **使用 TypeScript:** 在项目根添加 `tsconfig.json`。
- **使用 React:** 把 `react` 和 `react-dom` 添加到 `package.json`。
- 为 React 配置过的 `tsconfig.json` 可启用 TypeScript + React 创作。
- 如果你需要处理 manifest 未声明的资产,请进一步了解[特殊文件夹](/docs/features/special-folders)。

## 最佳实践

- **使用 `extension` 包**,以一套工具链构建、运行和打包你的扩展。
- **使用 `--browser`** 在开发时面向特定浏览器。
- **使用 `--polyfill`** 把面向 Mozilla 的示例适配到 Chromium 系浏览器。
- **一次性试用使用完整 GitHub URL**;当你希望有稳定的目录布局时,使用**克隆 + 本地路径**。
