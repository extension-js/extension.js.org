让扩展保持能力的同时,不索取超出功能所需的特权。

Extension.js 会编译并校验你的扩展,但浏览器仍会强制执行 `manifest.json` 中的 `permissions`、`host_permissions` 和 `optional_*` 字段。良好的权限设计能提升信任,并减少应用商店审核中的问题。

## 权限策略

| 需求                          | 优先使用                          |
| --------------------------- | ----------------------------- |
| 扩展正常工作所需的核心 API 访问           | `permissions`                 |
| 安装时所需的主机访问                  | `host_permissions`            |
| 可稍后再请求的能力                   | `optional_permissions`        |
| 只有部分用户需要的域名访问                | `optional_host_permissions`   |

## 常见模式

### 后台驱动的功能

使用 `permissions` 声明所需的浏览器 API,并仅添加功能所需的主机模式:

```json
{
  "manifest_version": 3,
  "permissions": ["storage", "tabs", "scripting"],
  "host_permissions": ["https://example.com/*"]
}
```

### 可选能力

如果首次启动体验不依赖某功能,将其设为可选:

```json
{
  "manifest_version": 3,
  "permissions": ["storage"],
  "optional_permissions": ["notifications"],
  "optional_host_permissions": ["https://api.example.com/*"]
}
```

## 实用规则

- 仅在功能确实需要时申请 API 权限。
- 把 `host_permissions` 限制在可工作的最小源集合中。
- 把次要或高级功能优先放到可选权限里。
- 每次新增后台行为、内容脚本注入或远程 API 调用时,重新审视权限。

## 按功能类型的权限设计

| 功能                              | 通常涉及                                                   |
| ------------------------------- | ------------------------------------------------------ |
| 在已知站点上的内容脚本增强                    | 针对该站点的 `host_permissions`,有时还需要 `scripting`               |
| 仅 popup 或 options UI              | 通常无需任何 host 权限                                         |
| 编程式注入                            | `scripting` 加上对应的 host 权限                              |
| 跨标签页工作流                          | `tabs`,有时还需要 `activeTab`                              |
| 持久化设置                            | `storage`                                              |
| 通知或徽标                            | `notifications` 和与 `action` 相关的 manifest 字段             |

## 常见错误

- 在只需要一两个源时,却使用 `<all_urls>` 这类宽泛通配符。
- 在本可用更窄的用户触发流程时,把 `host_permissions` 加进功能里。
- 忘记内容脚本、web 可访问资源,以及通过 `chrome.scripting` 注入脚本这三者各有不同的安全含义。
- 在功能文档中只列出权限,而没有解释每个权限存在的原因。

## 复审清单

1. 列出每一项面向用户的功能。
2. 把每项功能映射到它所需的 API 和主机权限。
3. 把非核心权限尽可能移到可选权限。
4. 移除旧实验残留的过时权限。
5. 修改后,重新测试安装提示与浏览器商店预期行为。
