为规则集与 managed schema 等扩展功能使用 JSON 资源,并获得校验与可预测的输出处理。

Extension.js 会处理 manifest 引用的 JSON 资源,并为已知的功能类型进行校验。在开发期间会监视它们,并在需要时输出产物资源。

## JSON 能力

| 能力                  | 你可以获得什么                                                                       |
| ------------------- | ----------------------------------------------------------------------------- |
| 支持 manifest 资源声明    | 干净地处理 Declarative Net Request (DNR) 规则集与 managed schema 文件                    |
| JSON 校验             | 对格式错误的 JSON 和非法资源结构提前失败                                                       |
| 开发期依赖跟踪             | 在被引用的 JSON 资源变化时重新编译                                                          |
| 占位符替换               | 在输出的 JSON 中解析受支持的 `$EXTENSION_*` 占位符                                          |

## JSON 支持

常见的 manifest 字段:

| Manifest 字段                              | 期望的文件类型 |
| ---------------------------------------- | ------- |
| `declarative_net_request.rule_resources` | .json   |
| `storage.managed_schema`                 | .json   |

## `manifest.json` 中的 JSON 声明示例

在 `manifest.json` 中声明 JSON 资源的示例:

```json
{
  "manifest_version": 3,
  "name": "My Extension",
  "version": "1.0.0",
  "declarative_net_request": {
    "rule_resources": [
      {
        "id": "ruleset_1",
        "enabled": true,
        "path": "rules/ruleset.json"
      }
    ]
  },
  "storage": {
    "managed_schema": "schema/storage-schema.json"
  }
}
```

## 输出路径

Extension.js 按功能上下文输出 JSON 资源(例如 DNR 和 storage schema 输出):

```plaintext
declarative_net_request/<ruleset-id>.json
storage/managed_schema.json
```

## 开发期行为

- Extension.js 把 manifest 引用的 JSON 文件作为文件依赖跟踪。
- 编辑 JSON 会触发重新编译。
- 部分与 JSON 相关的结构性 manifest 变化可能需要重启开发服务器。

## 校验行为

Extension.js 会校验:

- 引用的 JSON 资源的语法
- DNR 规则集的结构(基于数组的预期)
- managed storage schema 的结构(基于对象的预期)

## 环境占位符

Extension.js 在编译期间会替换输出的 `.json` 资源中的 `$EXTENSION_*` 占位符,使用与静态 HTML 占位符替换相同的处理流程。

## 最佳实践

- 让 manifest 引用的 JSON 专注于浏览器要求的资源格式。
- 在打包前的持续集成 (CI) 中校验 JSON 资源。
- 在 manifest 中优先使用稳定的文件路径,以减少开发期间 “需要重启” 的结构性变更。
- 对于代码中消费的应用数据,可单独考虑 JS/TS 模块中的 JSON 导入,与 manifest 资源 JSON 分开处理。
