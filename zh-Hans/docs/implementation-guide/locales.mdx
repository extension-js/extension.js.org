Extension.js 会在 manifest 旁发现 locale 文件,并校验每个声明的 locale 是否都有 `messages.json`。它会把 locale JSON 资源输出到各浏览器特定的构建中。在开发期间,它能在不完全重启的情况下捕获对任意 locale 文件的编辑。

## Locale 能力

| 能力                | 提供给你的功能                                                       |
| ------------------- | -------------------------------------------------------------------- |
| Locale 发现         | 从 manifest 位置检测 `_locales/<locale>/messages.json`               |
| 校验                | 捕获缺失的默认 locale 文件和未解析的 `__MSG_*__` 键                  |
| 构建输出映射        | 按预期的扩展输出结构输出 locale 文件                                 |
| 开发期监听支持      | 在开发期间对 locale 文件变更重新加载                                 |

## 期望结构

```plaintext
manifest.json
_locales/
  en/
    messages.json
  fr/
    messages.json
```

`manifest.json` 中的 `default_locale` 应当映射到一个存在的 `_locales/<default>/messages.json`。

## `manifest.json` 中的 locales 声明示例

下面是如何在 `manifest.json` 中声明 locales:

```json
{
  "manifest_version": 3,
  "name": "My Extension",
  "version": "1.0.0",
  "default_locale": "en",
  "description": "__MSG_extension_description__"
}
```

然后你需要在 `_locales` 文件夹内为每个 locale 添加 JSON 文件:

```plaintext
_locales/
└── en/
    └── messages.json
```

## 示例 `messages.json` 文件

用于翻译的 `messages.json` 示例:

```json
{
  "extension_name": {
    "message": "My Extension"
  },
  "extension_description": {
    "message": "This is a localized description of my extension."
  }
}
```

## 输出路径

Extension.js 把 locale JSON 文件输出到:

```plaintext
_locales/<locale>/messages.json
```

## 开发行为

- Extension.js 把 locale JSON 文件加入编译依赖并监听它们。
- Locale 变更触发扩展重载行为(硬重载),而不是组件式的热模块替换(HMR)。
- 当必需的 locale 文件缺失或无效时,Extension.js 会校验失败并给出可操作的诊断信息。

## 校验行为

Extension.js 会校验:

- 当项目使用 `_locales` 时,`default_locale` 是否存在
- `_locales/<default>/messages.json` 是否存在
- locale 文件的 JSON 是否有效
- manifest 中的 `__MSG_*__` 引用是否对应默认 locale 中的键

### 排查缺失的 locale 键

如果你的 manifest 使用 `__MSG_extension_description__`,请确保默认 locale 文件包含 `extension_description`:

```json
{
  "extension_description": {
    "message": "Localized extension description"
  }
}
```

如果默认 locale 没有定义该键,Extension.js 会给出说明不匹配原因的诊断。

## 最佳实践

- 在各 locale 间保持 `messages.json` 键的一致。
- 先更新默认 locale,再把键传播到其他 locale。
- 在持续集成(CI)中校验 locale JSON,以在打包前捕获错误文件。
- 把 locale 文件放在 manifest 附近(`manifestDir/_locales`),以获得可预测的解析。
