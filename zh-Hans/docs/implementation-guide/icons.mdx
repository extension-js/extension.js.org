通过在 Extension.js 可以校验、重写并可预测输出的 manifest 字段中声明图标,让扩展品牌与 action UI 保持一致。

Extension.js 会处理 `manifest.json` 中的图标路径,解析 public/相对路径,输出图标资源,并在开发期间监听图标文件。

![action 模板截图](/images/examples/action/screenshot.png)

## 图标能力

| 能力              | 你可以获得什么                                       |
| --------------- | --------------------------------------------- |
| 支持 manifest 图标字段 | 校验并编译受支持扩展界面的图标                                |
| 路径解析            | 一致处理相对路径与 public 根路径                          |
| 开发期 watch 更新     | 在本地开发时对图标变化进行重新编译                              |
| 浏览器安全输出         | 在扩展产物中输出图标资源,引用保持稳定                            |

## 支持的图标字段

| Manifest 字段                 | 期望的文件类型             |
| ----------------------------- | --------------------- |
| `action.default_icon`         | .png、.jpg、.svg      |
| `browser_action.default_icon` | .png、.jpg、.svg      |
| `icons`                       | .png、.jpg、.svg      |
| `page_action.default_icon`    | .png、.jpg、.svg      |
| `sidebar_action.default_icon` | .png、.jpg、.svg (\*) |
| `browser_action.theme_icons`  | .png、.jpg、.svg      |

<Note>
  目前部分浏览器对 `sidebar_action.default_icon` 的 `.svg`
  支持仍不完整。在此场景使用 SVG 前请检查浏览器兼容性。
</Note>

## `manifest.json` 中的图标声明示例

```json
{
  "manifest_version": 3,
  "name": "My Extension",
  "version": "1.0.0",
  "icons": {
    "16": "icons/icon-16.png",
    "48": "icons/icon-48.png",
    "128": "icons/icon-128.png"
  },
  "action": {
    "default_icon": "icons/action-icon.png"
  }
}
```

## 输出路径

典型的图标输出:

```plaintext
icons/<filename>
browser_action/<theme-icon-filename>   # for theme_icons
```

## 路径行为

- Extension.js 从 manifest 所在文件夹解析相对图标路径。
- 以 `/` 开头以及 `public/...` 都按扩展 public 根语义解析。
- Extension.js 可以监视 public 文件夹中的资源,而不必通过图标功能本身重新输出它们。

## 开发期行为

- 修改已有的图标文件会触发重新编译。
- 修改 manifest 中的图标入口引用可能需要重启开发服务器。
- 缺失必需的图标文件会产生构建错误(部分可选图标组可能只是发出警告)。

## 最佳实践

- 显式声明 manifest 图标字段,而不要依赖偶然的资源导入。
- 提供多种图标尺寸(`16`、`32`、`48`、`128`),让 UI 在各浏览器界面更清晰。
- 保持图标文件名稳定,以减少开发期间的 manifest 变动。
- 有意识地使用 public 根路径,并测试生成的 manifest 输出路径。
