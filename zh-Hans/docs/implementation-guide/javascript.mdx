使用一条 JavaScript/TypeScript 流水线在后台、内容脚本和 UI 页面间发布扩展行为。

Extension.js 从 `manifest.json`、HTML 页面和特殊文件夹收集脚本入口。它使用默认基于 SWC(Speedy Web Compiler)的配置进行编译。

## JavaScript 能力

| 能力                  | 提供给你的功能                                                  |
| --------------------- | --------------------------------------------------------------- |
| 共享的 JS/TS 流水线   | 用一套默认转换器配置编译扩展脚本                                |
| 入口发现              | 从 manifest、页面和特殊文件夹加载脚本入口                       |
| 运行时安全的输出      | 为每个扩展上下文输出可预测的脚本产物                            |
| 路径规范化            | 把受支持的扩展路径字面量重写为输出安全的路径                    |

## 支持的脚本入口

| Manifest 字段               | 期望文件类型                 |
| --------------------------- | ---------------------------- |
| `background.service_worker` | `.js`、`.ts`、`.mjs`、`.tsx` |
| `background.scripts`        | `.js`、`.ts`、`.mjs`、`.tsx` |
| `content_scripts.js`        | `.js`、`.ts`、`.mjs`、`.tsx` |
| `user_scripts.api_script`   | `.js`、`.ts`、`.mjs`、`.tsx` |

## `manifest.json` 中的脚本声明示例

`manifest.json` 中的脚本声明示例:

```json
{
  "manifest_version": 3,
  "name": "My Extension",
  "version": "1.0.0",
  "background": {
    "service_worker": "./scripts/background.ts"
  },
  "content_scripts": [
    {
      "matches": ["<all_urls>"],
      "js": ["./scripts/content-script.ts"]
    }
  ]
}
```

## 开发行为

- **内容脚本:** Extension.js 注入热模块替换(HMR)/重新挂载流程,实现快速更新。
- **扩展页面:** 页面脚本遵循页面 HMR 行为。
- **后台/service worker:** 脚本更新可能根据变更类型触发硬扩展重载。
- **入口列表变更:** 修改 manifest 中的脚本结构可能需要重启开发服务器。

## 脚本位置与约定

- 在项目根目录使用 `scripts/` 用于以脚本为中心的扩展入口。
- 使用 `pages/` 用于带有自身脚本的 HTML 入口。
- 保持 manifest 引用的路径稳定;不要在不更新 manifest 的情况下移动入口文件。

## 转换与打包默认值

- SWC 是 JS/TS/JSX/TSX 的默认转换器。
- Extension.js 使用浏览器优先的 `package.json` 字段解析(`browser`、`module`、`main`)。
- Extension.js 可预测地输出入口,默认不会把代码拆分到单独的输出文件中。
- 路径解析会重写受支持 API 中的静态扩展路径字面量。

## 动态导入注意事项

- 内容脚本的动态导入有运行时约束;Extension.js 在可能时使用 loader 兜底方案。
- Service worker 的惰性加载有浏览器限制;在 service worker 启动时必须立即可用的代码中,优先使用静态导入。

## 最佳实践

- 保持入口脚本精简,把功能逻辑放入共享模块。
- 对扩展 API 优先使用静态导入路径,使路径解析能安全地规范化它们。
- 避免在内容脚本和 service worker 中不必要的动态导入。
- 在开发工作流中,把 manifest 脚本列表的编辑视为结构性变更。
