为调试、自动化和运行时实验控制浏览器的启动行为。

无需修改扩展源代码即可调整浏览器启动行为。Extension.js 会合并来自 `extension.config.*` 的浏览器标志,并在 `dev`、`preview` 和 `start` 流程中应用它们。

## Firefox 有 `firefox://flags` 吗?

Firefox 不像 Chromium 系浏览器使用 `chrome://flags` 或 `edge://flags` 那样使用 `firefox://flags`。Firefox 通过 `about:config`(偏好设置)暴露运行时开关,并接受来自命令行的启动标志。

如果你搜索了 `firefox://flags`、`firefox //flags`、`browser://flags`、`mozilla://flags` 或 `about flags firefox`,那么你大概是想在扩展开发期间改变浏览器行为。在 Extension.js 中,可以在以下两个位置完成:

- 使用 [`browserFlags`](/docs/browsers/browser-flags) 设置 Extension.js 传递给浏览器二进制文件的启动期标志。
- 使用 [Firefox 偏好设置](/docs/browsers/browser-preferences)来配置原本会写在 `about:config` 中的可复现 Gecko 运行时行为。

这两者都可以在同一个 Extension.js 项目中用于 Chrome、Edge 和 Firefox 的扩展开发。

## 工作原理

在 `extension.config.*` 中配置标志:

- `browser.<target>.browserFlags`
- `commands.dev|start|preview.browserFlags`
- 可选的 `excludeBrowserFlags` 用于移除默认或用户标志(行为取决于目标是 Chromium 还是 Firefox)。

覆盖顺序:浏览器默认值 → 命令默认值 → 由 CLI 选定的命令上下文。

## 标志能力

| 配置键                                 | 它的作用                                                 |
| -------------------------------------- | -------------------------------------------------------- |
| `browser.<target>.browserFlags`        | 为特定浏览器目标设置默认启动标志。                       |
| `commands.dev.browserFlags`            | 为 `dev` 运行添加或覆盖标志。                            |
| `commands.start.browserFlags`          | 为 `start` 运行添加或覆盖标志。                          |
| `commands.preview.browserFlags`        | 为 `preview` 运行添加或覆盖标志。                        |
| `browser.<target>.excludeBrowserFlags` | 为目标移除匹配的默认或用户标志。                         |
| `commands.<name>.excludeBrowserFlags`  | 在特定命令上下文中移除标志。                             |

### 配置示例

```js
export default {
  browser: {
    chrome: {
      browserFlags: ["--disable-web-security", "--auto-open-devtools-for-tabs"],
      excludeBrowserFlags: ["--mute-audio"],
    },
    firefox: {
      browserFlags: ["--devtools", "--new-instance"],
      excludeBrowserFlags: ["--devtools"],
    },
  },
};
```

## Chromium 与 Firefox 的行为差异

- **Chromium 系(`chrome`、`edge`、`chromium`、`chromium-based`)**
  - 从内部的默认标志集开始,然后追加你的 `browserFlags`。
  - `excludeBrowserFlags` 用于移除匹配的默认标志(精确匹配过滤)。
  - Extension.js 管理 `--load-extension=...` 并将其从用户提供的标志中过滤掉。

- **Firefox/Gecko 系(`firefox`、`gecko-based` / `firefox-based`)**
  - 使用用户提供的 `browserFlags`(不像 Chromium 那样有庞大的默认标志包)。
  - `excludeBrowserFlags` 通过前缀匹配移除用户标志。

| `--disable-component-update`                               | 禁用组件更新                                                            |
| `--disable-domain-reliability`                             | 禁用域可靠性监控                                                        |
| `--no-pings`                                               | 禁用超链接审计 ping                                                     |
| `--enable-features=SidePanelUpdates`                       | 确保侧边栏可见                                                          |

## 最佳实践

- **只添加必要的标志**:尽量缩短标志列表,减少不稳定或不可移植的设置。
- **优先使用 `excludeBrowserFlags` 而不是替换默认值**:只移除与你的工作流冲突的项。
- **不要手动传递 `--load-extension`**:Extension.js 在内部管理扩展加载相关的标志。
- **按浏览器系验证**:在 Chromium 中生效的标志在 Firefox 中可能无效或被忽略。
