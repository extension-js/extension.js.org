为调试、自动化和运行时实验控制浏览器的启动行为。

在不修改扩展源代码的情况下调整浏览器启动行为。Extension.js 会合并你 `extension.config.*` 中的浏览器标志,并在 `dev`、`preview` 和 `start` 流程中应用它们。

## Firefox 有 `firefox://flags` 吗?

Firefox 并不像 Chromium 系浏览器使用 `chrome://flags` 或 `edge://flags` 那样使用 `firefox://flags`。Firefox 通过 `about:config`(首选项)暴露运行时开关,并接受从命令行传入的启动标志。

如果你搜索的是 `firefox://flags`、`firefox //flags`、`browser://flags`、`mozilla://flags` 或 `about flags firefox`,你大概率是想在扩展开发期间修改浏览器的行为。在 Extension.js 中,可以在两个地方做这件事:

- 使用 [`browserFlags`](/docs/browsers/browser-flags) 设置 Extension.js 传给浏览器可执行文件的启动期标志。
- 使用 [Firefox 首选项](/docs/browsers/browser-preferences) 设置原本应放在 `about:config` 中的可重复的 Gecko 运行时行为。

二者都可以在同一个 Extension.js 项目中用于 Chrome、Edge 和 Firefox 的扩展开发。

## 工作原理

在 `extension.config.*` 中配置标志:

- `browser.<target>.browserFlags`
- `commands.dev|start|preview.browserFlags`
- 可选的 `excludeBrowserFlags`,用于移除默认或用户标志(具体行为取决于你目标是 Chromium 还是 Firefox)。

覆盖顺序:浏览器默认值 → 命令默认值 → CLI 选择的命令上下文。

## 标志能力

| 配置键                                 | 作用                                                     |
| -------------------------------------- | -------------------------------------------------------- |
| `browser.<target>.browserFlags`        | 为某个特定浏览器目标设置默认启动标志。                   |
| `commands.dev.browserFlags`            | 为 `dev` 运行添加或覆盖标志。                            |
| `commands.start.browserFlags`          | 为 `start` 运行添加或覆盖标志。                          |
| `commands.preview.browserFlags`        | 为 `preview` 运行添加或覆盖标志。                        |
| `browser.<target>.excludeBrowserFlags` | 移除某个目标中匹配的默认或用户标志。                     |
| `commands.<name>.excludeBrowserFlags`  | 在特定命令上下文中移除标志。                             |

### 示例配置

```js
export default {
  browser: {
    chrome: {
      browserFlags: ["--disable-web-security", "--auto-open-devtools-for-tabs"],
      excludeBrowserFlags: ["--mute-audio"],
    },
    firefox: {
      browserFlags: ["--devtools", "--new-instance"],
      excludeBrowserFlags: ["--devtools"],
    },
  },
};
```

## Chromium 与 Firefox 行为对比

- **Chromium 家族(`chrome`、`edge`、`chromium`、`chromium-based`)**
  - 从内部默认标志集开始,然后追加你的 `browserFlags`。
  - `excludeBrowserFlags` 移除匹配的默认标志(完全匹配过滤)。
  - Extension.js 管理 `--load-extension=...`,并会从用户提供的标志中将其过滤掉。

- **Firefox/Gecko 家族(`firefox`、`gecko-based` / `firefox-based`)**
  - 使用用户提供的 `browserFlags`(不像 Chromium 那样附带大量默认标志)。
  - `excludeBrowserFlags` 通过前缀匹配移除用户标志。

| `--mute-audio`                                             | 静音所有音频                                                            |

| `--disable-component-update`                               | 禁用组件更新                                                            |
| `--disable-domain-reliability`                             | 禁用域名可靠性监控                                                      |
| `--no-pings`                                               | 禁用超链接审计 ping                                                     |
| `--enable-features=SidePanelUpdates`                       | 确保侧边面板可见                                                        |

## 最佳实践

- **只添加必要的标志**:尽量缩短标志列表,减少不稳定或不可移植的配置。
- **优先使用 `excludeBrowserFlags` 而不是替换默认值**:只移除与你工作流冲突的内容。
- **不要手动传入 `--load-extension`**:Extension.js 在内部管理扩展加载标志。
- **针对每个浏览器家族进行验证**:在 Chromium 上有效的标志在 Firefox 中可能无效或被忽略。
