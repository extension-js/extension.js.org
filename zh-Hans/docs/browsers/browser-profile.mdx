在开发期间，通过受管、持久或自定义 profile 来控制浏览器状态的隔离。

根据你的工作流让浏览器会话保持隔离或持久。Extension.js 在启动浏览器时会应用与 profile 相关的默认行为。可以选择干净启动、复用状态，或者使用显式的本地 profile 路径。

## 工作原理

Extension.js 按以下顺序选择 profile 模式：

1. 显式的 `profile` 路径（如果提供了）
2. 受管 profile 模式（默认）
   - 默认临时（短暂）
   - 当 `persistProfile: true` 时持久化
3. 当 `EXTENSION_USE_SYSTEM_PROFILE=true` 时使用系统 profile 模式

## Profile 能力

| 配置 / 选项                         | 作用                                                            |
| ----------------------------------- | --------------------------------------------------------------- |
| `browser.<target>.profile`          | 为该浏览器目标使用一个显式 profile 文件夹。                     |
| `commands.dev.profile`              | 仅在 `dev` 中使用显式 profile。                                 |
| `commands.start.profile`            | 仅在 `start` 中使用显式 profile。                               |
| `commands.preview.profile`          | 仅在 `preview` 中使用显式 profile。                             |
| `browser.<target>.persistProfile`   | 在目标上跨运行复用受管 profile 状态。                           |
| `commands.<name>.persistProfile`    | 在命令特定上下文中复用受管 profile 状态。                       |
| `--profile=/abs/path`               | 通过 CLI 覆盖显式 profile 路径。                                |
| `EXTENSION_USE_SYSTEM_PROFILE=true` | 使用操作系统 / 浏览器的系统 profile，而不是受管 profile。       |

## Profile 模式

| 模式                | 启用方式                                          | 典型用途                                       |
| ------------------- | ------------------------------------------------- | ---------------------------------------------- |
| 受管临时            | 默认                                              | 状态隔离的干净运行                              |
| 受管持久            | `persistProfile: true`                            | 在稳定浏览器状态下进行迭代调试                  |
| 显式自定义 profile  | `profile: "/abs/path"` 或 `--profile=/abs/path`   | 复用一个已有的 profile                          |
| 系统 profile        | `EXTENSION_USE_SYSTEM_PROFILE=true`               | 以 OS / 浏览器的默认 profile 启动               |

Extension.js 在以下位置创建受管 profile：

- `dist/extension-js/profiles/<browser>-profile/<...>`

持久模式使用：

- `dist/extension-js/profiles/<browser>-profile/dev`

## 在 `extension.config.*` 中配置

```js
export default {
  browser: {
    chrome: {
      // 使用你自己的 profile 文件夹：
      profile: "path/to/custom/profile",
    },
    firefox: {
      // 为重复会话保留一个稳定的受管 profile：
      persistProfile: true,
    },
  },
};
```

你也可以按命令限定 profile 默认值：

```js
export default {
  commands: {
    dev: {
      persistProfile: true,
    },
  },
};
```

## CLI 用法

直接使用显式 profile 路径：

```bash
extension dev --browser=chrome --profile=/path/to/custom/profile
```

与 `start` 和 `preview` 的用法类似。

## 生命周期说明

- Extension.js 为每次运行创建临时的受管 profile。
- Extension.js 在多次运行之间复用持久的受管 profile（`dev`）。
- Extension.js 会自动清理旧的受管 profile 文件夹。可设置 `EXTENSION_TMP_PROFILE_MAX_AGE_HOURS` 控制最长保留时长。

## 最佳实践

- **基线测试使用受管临时 profile**：减少隐藏状态以及难以复现的问题。
- **长时间调试使用 `persistProfile`**：在多次运行之间保留登录 / 会话 / devtools 状态。
- **按浏览器家族保留各自的自定义 profile**：避免跨浏览器污染。
- **谨慎使用系统 profile 模式**：适合复现，但隔离性弱于受管 profile。
