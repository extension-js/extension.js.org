通过显式的浏览器目标工作流，在 Chrome、Edge、Firefox
以及自定义二进制文件上运行同一份扩展代码。

## Chrome 扩展开发

进行 Chrome 扩展开发时，目标设为 `chrome`（或对应默认 Chromium 二进制的 `chromium`）：`extension dev --browser=chrome`。Extension.js 会把扩展加载到一个全新、隔离的 profile 中，并应用一组合理的 [Chrome 启动 flag](/zh-Hans/docs/browsers/browser-flags)，让重载行为更可预期。

## Firefox 扩展开发

进行 Firefox 扩展开发时，目标设为 `firefox`：`extension dev --browser=firefox`。Manifest V3 background 脚本会编译为非持久化的 `scripts` 数组（Firefox 不使用 `service_worker`），并以 Firefox preferences 取代 Chromium 的 `chrome://flags` 概念。参见[浏览器 preferences](/zh-Hans/docs/browsers/browser-preferences)。

## Edge 扩展开发

进行 Edge 扩展开发时，目标设为 `edge`：`extension dev --browser=edge`。Edge 与 Chromium 共用同一引擎，因此大多数 flag 与 API 都和 Chrome 一致，但 Extension.js 仍会为它单独产出 `dist/edge` 构建产物用于分发。

## 跨浏览器扩展开发

进行跨浏览器扩展开发时，可在一条命令中运行多个目标（`extension dev --browser=chrome,firefox`），并把浏览器之间的差异保留在[带浏览器前缀的 manifest 字段](/zh-Hans/docs/features/browser-specific-fields)里。一个项目、一份 `manifest.json`，每个浏览器对应独立的产物。完整流程参见[跨浏览器兼容性](/zh-Hans/docs/features/cross-browser-compatibility)。

## 实用的目标策略

1. 日常调试使用具名目标（`chrome`、`edge`、`firefox`）。
2. 发版校验使用逗号分隔的多个目标。
3. 仅在需要自定义二进制文件时使用引擎目标。
4. 把浏览器差异保留在带浏览器前缀的 manifest 字段里。
