通过显式的浏览器目标工作流,在 Chrome、Edge、Firefox 和自定义二进制中运行同一份扩展代码库。

## Chrome 扩展开发

对于 Chrome 扩展开发,目标是 `chrome`(或 `chromium` 使用默认的 Chromium 二进制):`extension dev --browser=chrome`。Extension.js 会将扩展加载到一个全新的隔离 profile 中,并应用一套合理的 [Chrome 启动标志](/docs/browsers/browser-flags),让重载行为更可预测。

## Firefox 扩展开发

对于 Firefox 扩展开发,目标是 `firefox`:`extension dev --browser=firefox`。Manifest V3 后台脚本会被编译为非持久的 `scripts` 数组(Firefox 不使用 `service_worker`),并且 Firefox 首选项替代了 Chromium 中 `chrome://flags` 的概念。参见[浏览器首选项](/docs/browsers/browser-preferences)。

## Edge 扩展开发

对于 Edge 扩展开发,目标是 `edge`:`extension dev --browser=edge`。Edge 共用 Chromium 引擎,因此大多数标志和 API 与 Chrome 一致,但 Extension.js 仍会为发行生成独立的 `dist/edge` 产物。

## 跨浏览器扩展开发

对于跨浏览器扩展开发,可在一条命令中运行多个目标(`extension dev --browser=chrome,firefox`),并通过[浏览器前缀的 manifest 字段](/docs/features/browser-specific-fields)保留浏览器差异。一个项目、一份 `manifest.json`、每个浏览器一份独立产物。完整流水线请参见[跨浏览器兼容性](/docs/features/cross-browser-compatibility)。

## 实用的目标选择策略

1. 日常检查使用命名目标(`chrome`、`edge`、`firefox`)。
2. 发布验证使用逗号分隔的多个目标。
3. 只有在需要自定义二进制时才使用引擎目标。
4. 通过浏览器前缀的 manifest 字段保留浏览器差异。
