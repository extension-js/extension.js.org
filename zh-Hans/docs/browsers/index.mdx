通过明确的浏览器目标设定工作流,在 Chrome、Edge、Firefox 和自定义二进制文件之间运行同一份扩展代码库。

## Chrome 扩展开发

对于 Chrome 扩展开发,将目标设为 `chrome`(或将 `chromium` 用于默认 Chromium 二进制):`extension dev --browser=chrome`。Extension.js 会将扩展加载到一个全新的、隔离的配置文件中,并应用一组合理的 [Chrome 启动标志](/docs/browsers/browser-flags),让重载行为可预测。

## Firefox 扩展开发

对于 Firefox 扩展开发,将目标设为 `firefox`:`extension dev --browser=firefox`。Manifest V3 的后台脚本会编译成非持久化的 `scripts` 数组(Firefox 不使用 `service_worker`),Firefox 偏好设置取代了 Chromium 中 `chrome://flags` 的概念。参见[浏览器偏好设置](/docs/browsers/browser-preferences)。

## Edge 扩展开发

对于 Edge 扩展开发,将目标设为 `edge`:`extension dev --browser=edge`。Edge 共享 Chromium 引擎,因此大多数标志和 API 与 Chrome 一致,但 Extension.js 仍会为分发输出单独的 `dist/edge` 产物。

## 跨浏览器扩展开发

对于跨浏览器扩展开发,可在一条命令中运行多个目标(`extension dev --browser=chrome,firefox`),并将浏览器之间的差异保留在[带浏览器前缀的 manifest 字段](/docs/features/browser-specific-fields)中。一个项目、一个 `manifest.json`,为每个浏览器产出不同的输出。完整流水线请参见[跨浏览器兼容性](/docs/features/cross-browser-compatibility)。

## 实用的目标策略

1. 日常检查使用具名目标(`chrome`、`edge`、`firefox`)。
2. 发布前验证使用逗号分隔的多个目标。
3. 仅在需要自定义二进制时使用引擎目标。
4. 把浏览器之间的差异保留在带浏览器前缀的 manifest 字段中。
