<Update label="v2.0.0-alpha" description="2024 年 9 月 9 日" tags={["release", "alpha"]}>

![Extension.js 2.0.0-alpha 横幅](/images/blog/extension-2-0-0-alpha.png)

Extension.js 的目标，是让跨浏览器扩展开发尽可能简单。

自从在 [HackerNews 上获得关注](https://hnrankings.info/40210627/)以来，我们做了大量内部改进，让这个包更小、更快，对开发者更友好。

对我而言，“非常简单”意味着非常低的认知负担，意味着熟悉感。开发者熟悉复制/粘贴。作为一名开发者，如果我能复制/粘贴代码，并在没有任何配置的情况下立即得到可视化反馈，体验就会非常轻松。即使你一开始不理解代码，快速的可视化反馈也清晰地表明事情**应该**如何运作。这就是 Extension.js v2 的期望：上手非常简单。

**版本 2.0.0-alpha**

- 30 多个面向流行 JavaScript 框架、CSS 预处理器和扩展 API 的模板。[在 GitHub 上查看](https://github.com/extension-js/examples/tree/main/examples)。
- 支持在 manifest.json 中添加仅对目标浏览器生效的按浏览器划分字段，例如 `{chrome:service_worker: "sw.js"}`
- `extension.config.js` —— 用于需要访问编译流程的高级场景。[查看配置文档](/docs/features/extension-configuration)。
- 得益于 [SWC (Speedy Web Compiler)](https://swc.rs/)，编译速度更快。
- 全新改版的文档站点（就是这个！）。
- 全新的模板站点（即将推出）。

alpha 版本在我们完成文档和模板站点的同时，专注于修复 bug 并测试所有现有功能。请持续关注本博客，或在 X 上关注我们以获取后续更新。

下一个版本，我们将专注于让所有现有功能稳定到足以撰写文档并进行测试。

感谢阅读，回头见。
