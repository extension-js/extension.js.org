<Update label="v2.0.0-alpha" description="September 9, 2024" tags={["release", "alpha"]}>

![Extension.js 2.0.0-alpha banner](/images/blog/extension-2-0-0-alpha.png)

Extension.js 想做的是：让跨浏览器扩展开发尽可能简单。

自从被 [HackerNews 推荐](https://hnrankings.info/40210627/)以来，我们在内部做了大量改进，让这个包更小、更快，对开发者也更友好。

对我来说，"非常简单"意味着非常低的认知成本，意味着熟悉感。开发者熟悉复制/粘贴。作为开发者，如果我能复制/粘贴一段代码，立刻看到它意味着什么的可视反馈，并且不需要做任何配置，我就会觉得这非常简单。即使你一开始看不懂代码，快速的可视反馈也清晰地告诉你这些东西**预期**怎么运作。这就是 Extension.js v2 的预期：用起来非常简单的东西。

**版本 2.0.0-alpha**

- 30+ 个模板，覆盖常见 JavaScript 框架、CSS 预处理器以及扩展 API。[在 GitHub 上查看](https://github.com/extension-js/examples/tree/main/examples)。
- 支持在 manifest.json 中添加仅对目标浏览器生效的浏览器专属字段，例如 `{chrome:service_worker: "sw.js"}`。
- `extension.config.js` —— 面向需要介入编译过程的高级用例。[查看配置文档](/zh-Hans/docs/features/extension-configuration)。
- 借助 [SWC（Speedy Web Compiler）](https://swc.rs/)实现更快的编译。
- 全新改版的文档站（就是你现在看到的这个！）。
- 全新的模板站（即将上线）。

Alpha 版本聚焦于在我们完成文档站和模板站的同时，修复 bug 并测试现有特性。请关注本博客或在 X 上关注我们以获取后续更新。

下一个版本，我们会把重心放在让所有现有特性达到可文档化、可测试的稳定程度。

感谢阅读。再见。
