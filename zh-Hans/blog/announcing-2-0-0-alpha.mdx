<Update label="v2.0.0-alpha" description="September 9, 2024" tags={["release", "alpha"]}>

![Extension.js 2.0.0-alpha banner](/images/blog/extension-2-0-0-alpha.png)

Extension.js 的目标是让跨浏览器扩展开发变得尽可能简单。

自从[登上 HackerNews](https://hnrankings.info/40210627/) 以来，我们做了大量内部改进，让这个包更小、更快、更对开发者友好。

对我而言，“非常简单”意味着极低的认知负担。它意味着熟悉。开发者熟悉复制/粘贴。作为一名开发者，如果我能够复制/粘贴代码，并立即获得它代表什么的可视化反馈，且无需配置，那就感觉非常简单。即便你一开始不理解这段代码，快速的视觉反馈也能清晰地指示事物**应该**如何工作。这就是 Extension.js v2 的预期目标：一种非常易于上手的工作方式。

**版本 2.0.0-alpha**

- 30+ 个面向流行 JavaScript 框架、CSS 预处理器和扩展 API 的模板。[在 GitHub 上查看](https://github.com/extension-js/examples/tree/main/examples)。
- 支持在 manifest.json 中添加仅对目标浏览器生效的浏览器特定字段，例如 `{chrome:service_worker: "sw.js"}`。
- `extension.config.js` —— 用于需要访问编译过程的高级用例。[查看配置文档](/docs/features/extension-configuration)。
- 得益于 [SWC（Speedy Web Compiler）](https://swc.rs/) 的更快编译速度。
- 全新改版的文档站点（就是这个！）。
- 一个全新的模板站点（即将推出）。

alpha 版本聚焦于修复 bug 并测试所有现有功能，同时我们会完成文档和模板站点。请关注此博客或在 X 上关注我们以获取后续更新。

下一个版本，我们将专注于让所有现有功能足够稳定，从而能够完成文档化和测试。

感谢阅读。回头聊。
