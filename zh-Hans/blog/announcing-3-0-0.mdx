<Update label="v3.0.0" description="2026 年 2 月 18 日" tags={["release", "major"]}>

Extension.js 3.0.0 现已发布。

本次发布聚焦于一件事：帮助团队以更少的配置、更清晰的工作流和更好的默认设置，在 Chrome、Edge 与 Firefox 上交付生产级浏览器扩展。

## v3 带来了什么

**版本 3.0.0**

- 通过统一的命令面，强化了在 Chrome、Edge 和 Firefox 上的跨浏览器工作流。
- 扩展的模板体验，覆盖现代技术栈，包括 TypeScript、React、Vue、Svelte 和 Preact。
- 更完善的开发期更新行为，让热模块替换（HMR）、硬重载与需要重启的边界更加清晰。
- 通过故障排查、安全检查、性能检查、持续集成（CI）模板以及 Playwright 端到端（E2E）测试，提供更可靠的生产信心工作流。
- 通过 `extension.config.js` 以及按浏览器感知的构建输出，让项目配置更可预测。
- 更清爽的文档与首页体验，专为更快的上手和更低的认知负担而设计。

## 首日上手流程更简单

从创建项目到产出生产物，使用同一条路径：

1. `create` 用于脚手架搭建。
2. `dev` 用于快速迭代。
3. `build` 用于生成生产产物。
4. `start` 或 `preview` 用于验证生产行为。

```bash
npx extension@latest create my-extension --template=new-typescript
cd my-extension
pnpm dev
pnpm build
```

## 为团队设计，而不仅是演示

v3 专为维护真实扩展代码库的真实团队设计：

- 类型化模板和以框架为先的基线，减少了配置漂移。
- 按浏览器目标的构建和标志，让兼容性检查更加明确。
- 工作流文档现在直接映射到发布信心相关的任务。
- 标准交付流程现在包含安全与性能指南。

## 这次发布适合谁

- 你正在从自定义扩展构建脚本迁移过来。
- 你正从应用开发转向扩展开发。
- 你正在跨多个仓库与贡献者之间，统一扩展开发体验。

## 升级说明

如果你已经在使用 Extension.js：

- 将依赖包升级到最新版本，
- 优先使用规范的模板别名（`new-react`、`new-typescript`、`new-vue` 等），
- 保持命令流程与 `create` → `dev` → `build` 对齐。
