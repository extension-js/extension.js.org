<Update label="v3.0.0" description="February 18, 2026" tags={["release", "major"]}>

Extension.js 3.0.0 现已发布。

本次发布只聚焦一件事：帮助团队以更少的配置、更清晰的工作流和跨 Chrome、Edge、Firefox 的更优默认值，发布生产级浏览器扩展。

## v3 带来了什么

**版本 3.0.0**

- 通过统一的命令界面，强化了跨 Chrome、Edge 与 Firefox 的工作流。
- 扩展了模板体验，覆盖 TypeScript、React、Vue、Svelte 和 Preact 等现代技术栈。
- 改进开发期更新行为，提供更清晰的热模块替换（HMR）、硬重载与“需要重启”边界。
- 在故障排查、安全检查、性能检查、持续集成（CI）模板以及 Playwright 端到端（E2E）测试方面，提供更可靠的生产信心工作流。
- 通过 `extension.config.js` 与浏览器感知的构建产物，使项目配置更可预测。
- 全新更干净的文档与首页体验，旨在加快上手速度并降低认知负担。

## 第一天的工作流更简单

从项目创建到生产产物，使用一条统一的路径：

1. 使用 `create` 进行脚手架搭建。
2. 使用 `dev` 快速迭代。
3. 使用 `build` 生成生产产物。
4. 使用 `start` 或 `preview` 来验证生产行为。

```bash
npx extension@latest create my-extension --template=new-typescript
cd my-extension
pnpm dev
pnpm build
```

## 不仅适合演示，更适合团队

v3 是为真正维护真实扩展代码库的团队设计的：

- 类型化的模板和以框架为先的基线减少了配置漂移。
- 面向目标浏览器的构建与标志使兼容性检查变得显式。
- 工作流文档现在直接映射到发布信心相关的任务。
- 标准交付流程现在包含安全与性能指引。

## 这个版本适合谁

- 你正在从自定义的扩展构建脚本迁移。
- 你正在从应用开发转向扩展开发。
- 你正在跨多个仓库与贡献者之间统一扩展开发者体验。

## 升级说明

如果你已经在使用 Extension.js：

- 将依赖包更新到最新版本，
- 优先使用规范化的模板别名（`new-react`、`new-typescript`、`new-vue` 等），
- 保持你的命令流程与 `create` → `dev` → `build` 对齐。
