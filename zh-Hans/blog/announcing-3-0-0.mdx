<Update label="v3.0.0" description="February 18, 2026" tags={["release", "major"]}>

Extension.js 3.0.0 现已发布。

这个版本只聚焦一件事：用更少的配置、更清晰的工作流、更好的默认值，帮助团队在 Chrome、Edge 和 Firefox 上交付生产级浏览器扩展。

## v3 带来什么

**版本 3.0.0**

- 用同一组命令面，强化 Chrome、Edge、Firefox 上的跨浏览器工作流。
- 扩展的模板体验覆盖现代技术栈，包括 TypeScript、React、Vue、Svelte 与 Preact。
- 开发期更新行为更清晰：热模块替换（HMR）、硬重载与"需重启"边界更明确。
- 在排错、安全检查、性能检查、持续集成（CI）模板和 Playwright 端到端（E2E）测试方面提供更稳的发布信心工作流。
- 通过 `extension.config.js` 与按浏览器感知的构建产物，让项目配置更可预期。
- 更整洁的文档与首页体验，专为更快上手、更低认知负担而设计。

## 首日工作流更简单

从创建项目到生产产物，使用一条统一路径：

1. `create` 创建脚手架。
2. `dev` 快速迭代。
3. `build` 生成生产产物。
4. `start` 或 `preview` 校验生产行为。

```bash
npx extension@latest create my-extension --template=new-typescript
cd my-extension
pnpm dev
pnpm build
```

## 面向真实团队，而不只是 demo

v3 是为维护真实扩展代码库的真实团队设计的：

- 类型化的模板与"框架优先"的基线减少配置漂移。
- 按浏览器目标的构建与参数让兼容性检查变得显式。
- 工作流文档现在直接对应到发布信心相关的任务。
- 标准交付流程现在涵盖安全与性能方面的指引。

## 这个版本适合谁

- 你正在从自定义的扩展构建脚本迁移过来。
- 你正在从应用开发转向扩展开发。
- 你需要在多个代码库和多位贡献者之间统一扩展开发体验。

## 升级说明

如果你已经在使用 Extension.js：

- 把包升级到最新版本，
- 优先使用规范的模板 slug（`new-react`、`new-typescript`、`new-vue` 等），
- 让命令流程保持 `create` → `dev` → `build`。
